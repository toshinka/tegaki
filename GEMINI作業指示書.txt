## 今回の前提

- 標準描画は PixiJS v8.17.0 + RenderTexture へのライブラスター焼き込み。
- アルバム保管/復帰、PNG エクスポート、DEL 消去とサムネイル整合は良好。
- アニメボタンは表示入口だけ維持。`animationSystem.init()` は現行描画へ副作用が出るため使わない。
- APNG/GIF 自動判定、アニメプレビュー、フレームコピーは後続の動画ツール風マトリクス再設計時に扱う。
- WebGPU / SDF / MSDF / WebGL2 Mesh 経路は凍結継続。
- `dist/` は成果物として必要ない限り差分に残さない。

まず `task-gemini/phase1i.md` の「保守性棚卸し」を実施する。

優先して見るファイル：

- `tegaki_work/core-engine.js`

- `tegaki_work/ui/keyboard-handler.js`
- `tegaki_work/system/layer-transform.js`
- `tegaki_work/system/drawing-clipboard.js`

軽微な修正は実施してよい：

- ファイルヘッダーの責務/依存/イベント/グローバル登録のズレ。
- 古い `ベクターペン`、`phase1a`、明らかに現状と違うコメント。

実装せず止めるもの：

- EventBus payload の大変更。
- 描画パイプライン変更。

これらは `PROGRESS.md` に候補として書き、Codex 判断へ戻す。

### 2. History ステータス確認

確認項目：

- 起動直後。
- 描画後。
- DEL 消去後。
- Undo/Redo 後。
- アルバムロード後。

既に正しいならコード変更しない。軽微な購読漏れや初期表示ズレだけ修正する。

### 3. V キー変形パネルの小型改善

可能なら実装：

- V モード中に `H` で水平反転。
- V モード中に `Shift+H` で垂直反転。
- 回転を 360 度範囲にする。
- 各数値をダブルクリックで直接入力できるようにする。

注意：

- 通常描画モードの `H` / `Shift+H` と衝突させない。
- 変形 bake、履歴、座標整合に深く食い込む場合は実装せず止める。
- 実装した場合は Undo/Redo と描画座標に悪影響がないか確認する。

### 4. レイヤー不透明度の数値ドラッグ

可能なら実装：

- レイヤーパネルの不透明度数値を横ドラッグして 0〜100% 変更。
- 既存の ◀▶ 操作は残す。
- ドラッグ中にサムネ更新を過剰発火させない。

### 5. クリップボード/レイヤーコピー棚卸し

まず棚卸し：

- `Ctrl+C` / `Ctrl+V` の現行割り当て。
- `drawing-clipboard.js` に使える API があるか。
- アクティブレイヤーコピー、新規コピーレイヤー作成が既存 API で可能か。
- クリップボード画像ペーストが既存実装で安全に可能か。

低リスクなら実装：

- `Ctrl+C`: アクティブレイヤーコピー。
- `Ctrl+V`: 新規コピーレイヤー作成。

難しければ実装せず、棚卸し結果と案を `PROGRESS.md` に書いて止める。

## 必ず守ること

- `animationSystem.init()` を有効化しない。
- アニメ本体、APNG/GIF 自動判定、アニメプレビューは実装しない。
- クリッピング、フォルダ、複数レイヤー選択は実装しない。
- 無限キャンバスや V 変形時のキャンバス外保持は実装しない。
- WebGPU / SDF / MSDF / WebGL2 Mesh を復活させない。
- 調査用ログを残さない。残す場合は debug 限定にする。

## ビルドと確認

必須：

```powershell
cd tegaki_work

```

可能ならブラウザで確認：

1. 起動時に致命的エラーがない。

3. `No renderTexture` エラーが出ない。
4. アルバム保管/復帰が壊れていない。
5. PNG エクスポートが壊れていない。
6. V パネル改善を入れた場合、その操作が動く。
7. レイヤー不透明度ドラッグを入れた場合、表示と数値が同期する。
8. History 表示が破綻しない。
