## 今回の前提

- Phase 1f は、描画/消しゴム/バケツ/リサイズをまたいだアンドゥ・リドゥと座標整合が実機確認で良好。
- ペン、消しゴム、サムネイル、基本バケツ、筆圧切替、ステータス表示、アンドゥ/リドゥ、リサイズは壊さない。
- 標準描画は PixiJS v8.17.0 + RenderTexture へのライブラスター焼き込み。
- WebGPU / SDF / MSDF / WebGL2 Mesh 経路は凍結継続。復活させない。
- Canvas2D を本番描画へ混ぜない。ただしエクスポート時の最終抽出・保存形式変換に Canvas を使うのは可。
- 内部 2 倍化、`resolution: 2` は使わない。
- `dist/` は成果物として必要ない限り差分に残さない。
- 出力サイズは作業キャンバスサイズと一致させる。
- 高機能バケツ、範囲選択、アニメ/漫画効果は今回扱わない。

## 今回の主作業

### 1. 保存・出力・レイヤー編集経路の棚卸し

対象候補：

- `tegaki_work/system/export-manager.js`
- `tegaki_work/system/exporters/png-exporter.js`
- `tegaki_work/system/exporters/webp-exporter.js`
- `tegaki_work/system/exporters/psd-exporter.js`
- `tegaki_work/ui/export-popup.js`
- `tegaki_work/ui/album-popup.js`
- `tegaki_work/system/layer-system.js`
- `tegaki_work/ui/layer-panel-renderer.js`
- `tegaki_work/system/data-models.js`
- `tegaki_work/system/history.js`

確認すること：

1. PNG エクスポートが現在の `Container + Sprite(RenderTexture)` レイヤー構造を正しく合成しているか。
2. 背景レイヤー、透明背景、非表示レイヤー、レイヤー不透明度が出力に反映されるか。
3. エクスポート時にカメラ位置・ズーム・UI 表示の影響を受けていないか。
4. 現行の album/save 相当機能があるか。ある場合、何を保存しているか。
5. レイヤー複製/結合/フォルダ/クリッピングの既存実装や残骸があるか。
6. レイヤー操作を history に積む既存 command があるか。

この棚卸し結果を `tegaki_work/PROGRESS.md` の先頭へ追記する。

---

### 2. PNG エクスポート復旧を最優先

受け入れ条件：

- 現在のキャンバスサイズと同じ PNG が出力される。
- 表示中レイヤーだけが出力される。
- 背景表示 ON なら背景込み、背景非表示なら透明背景になる。
- ペン、消しゴム、バケツ、リサイズ後の描画が出力に反映される。
- 倍率指定や高度な書き出し設定は削除または無効化してよい。

---

### 3. 安全に直せる範囲だけ実装

実装してよい候補：

- PNG エクスポート経路の小修正。
- エクスポート UI の不要な倍率指定を隠す、または無効化する小修正。
- レイヤー複製の実装。
- 下レイヤー結合の実装。
- レイヤー操作を `History` に載せる小修正。
- 保存/読み込み形式の調査と、最小実装が安全なら実装。

実装しないで止めるもの：

- 保存形式の大きな設計判断。
- PSD/WEBP/GIF/APNG のまとめて復旧。
- レイヤー階層モデルの大幅変更。
- History command 形式の大幅変更。
- RenderTexture の保持方式変更。
- WebGPU / SDF / MSDF / WebGL2 Mesh の復活。
- 範囲選択、切り取り、自由変形の本格実装。

今回の Gemini 作業では、PNGエクスポート復旧を第一優先にする。レイヤー編集は、複製または下レイヤー結合のうち安全に実装できる方からでよい。

## ビルドと確認

必須：

```powershell
cd tegaki_work
npm.cmd run build
```

可能ならブラウザで確認：

1. 起動時に致命的エラーがない。
2. 通常ペンで描ける。
3. 消しゴムで透明に消せる。
4. 基本バケツが壊れていない。
5. `Ctrl+Z` / `Ctrl+Y` とリサイズが退行していない。
6. PNG エクスポートで現在の描画内容が出力される。
7. 背景表示 ON/OFF と非表示レイヤーが出力に反映される。
8. 実装したレイヤー編集が history とサムネイルに反映される。
9. `TEGAKI_CONFIG.debug === false` の通常状態で Console が過剰ログに埋まらない。
