## CODEX先行調査メモ

確認済み：

- `tegaki_work/config.js` の `debug` が Phase 1c 診断用に `true` のままだった。
- `thumbnail-system.js` の `[ThumbnailSystem] updated` は通常操作で頻発するため debug 条件が必要。
- `layer-system.js` のリサイズ・bake ログは通常確認が済んだため debug 条件でよい。
- `fill-tool.js` の ESM 読込ログは通常時に不要。
- `drawing-engine.js` ヘッダーに `イベント受診` の typo があった。
- `layer-system.js` の `bakeTransform()` は固定サイズ `RenderTexture` に焼き込むため、Vキー変形でキャンバス外へ出た描画が失われる原因候補として濃い。

## 1. 残ログ整理

対象候補：

- `tegaki_work/core-engine.js`
- `tegaki_work/core-initializer.js`
- `tegaki_work/checker-utils.js`
- `tegaki_work/ui/timeline-ui.js`
- `tegaki_work/ui/timeline-thumbnail-utils.js`
- `tegaki_work/ui/status-display-renderer.js`
- `tegaki_work/system/drawing/pressure-handler.js`
- `tegaki_work/system/processing/vector-operations.js`
- `tegaki_work/coordinate-system.js`

方針：

1. `rg -n "console\\.(log|debug|warn|error)" tegaki_work -g "*.js" -g "!dist/**" -g "!node_modules/**"` で確認する。
2. 起動成功やモジュール読込完了だけの `console.log` は削除、または `window.TEGAKI_CONFIG?.debug` / `TEGAKI_UTILS.log` 条件へ移す。
3. `console.error` と、復旧不能・設定不整合を示す `console.warn` は残してよい。
4. `coordinate-system.js` の手動診断関数内ログは、その関数が通常操作で呼ばれないなら残してよい。ただし通常操作で出るなら debug 条件へ移す。

受け入れ条件：

- 通常のペン描画・消しゴム・バケツ・リサイズで Console がログで埋まらない。
- 残したログの理由を `PROGRESS.md` に書く。

対象候補：

- `tegaki_work/core-engine.js`
- `tegaki_work/system/drawing/pointer-handler.js`
- `tegaki_work/system/drawing/drawing-engine.js`
- `tegaki_work/system/drawing/thumbnail-system.js`

- `tegaki_work/system/layer-transform.js`
- `tegaki_work/ui/layer-panel-renderer.js`
- `tegaki_work/ui/timeline-thumbnail-utils.js`

方針：

1. 各ファイル先頭ヘッダーの `依存`、`被依存`、`イベント発火`、`イベント受信`、`グローバル登録` が実装と一致しているか確認する。
2. typo や明らかな不足は修正する。
3. 大規模な依存整理やリネームは今回行わない。必要なら `PROGRESS.md` に「後続候補」として書く。
4. `ThumbnailSystem` と `TimelineThumbnailSystem` は責務が混ざらないように、ヘッダーで用途を明記する。

受け入れ条件：

- 主要対象ファイルのヘッダーが現行実装と矛盾しない。
- 変更したヘッダー一覧を `PROGRESS.md` に書く。

## 3. UI/ESM・window依存の棚卸し

対象候補：

- `tegaki_work/ui/`
- `tegaki_work/core-runtime.js`
- `tegaki_work/core-initializer.js`
- `tegaki_work/core-engine.js`

方針：

1. `rg -n "window\\.|document\\.addEventListener|addEventListener\\(" tegaki_work/ui tegaki_work/core-*.js` で依存を棚卸しする。
2. `window.*` 依存を一括撤廃しない。
3. 明らかに古い読込ログや不要なグローバル別名だけ、小さく安全に整理する。
4. 触ると動作確認が重くなる箇所は、今回は調査記録だけでよい。

受け入れ条件：

- 「今回直した箇所」と「後続へ送る箇所」が分かれている。
- UI 操作の既存経路を壊さない。

## 4. Vキー変形 bake の調査

対象：

- `tegaki_work/system/layer-system.js`
- `tegaki_work/system/layer-transform.js`
- `tegaki_work/system/history.js`

確認ポイント：

1. `LayerSystem.bakeTransform()` が `rt.width` / `rt.height` の固定サイズ `RenderTexture` に焼いていないか。
2. 変形中にキャンバス外へ出たピクセルが、bake 時に `tempRT` へ入らず失われる可能性があるか。
3. `LayerTransform.confirmTransform()` の paths 変形経路と、現在のライブラスター `RenderTexture` 経路が噛み合っているか。
4. undo/redo が RenderTexture の内容を復元できる構造か。

今回の扱い：

- 無限キャンバス化や大規模な bake 設計変更はしない。
- 原因候補と安全な対応案を `PROGRESS.md` に記録する。
- もし数行で明確に直せるガードや debug ログがあれば実装してよいが、描画内容の破壊リスクがある修正は行わない。

受け入れ条件：

- Vキー変形のキャンバス外クリップについて、原因候補・実装する/しない判断・後続案が `PROGRESS.md` にある。
