## 今回の前提

- Phase 1e は、筆圧ペン、消しゴム筆圧、低速描画の途切れ対策、ステータス/UI同期まで完了扱い。
- ペン、消しゴム、サムネイル、基本バケツ、筆圧切替、ステータス表示は壊さない。
- 標準描画は PixiJS v8.17.0 + RenderTexture へのライブラスター焼き込み。
- WebGPU / SDF / MSDF / WebGL2 Mesh 経路は凍結継続。復活させない。
- Canvas2D を本番描画へ混ぜない。
- 内部 2 倍化、`resolution: 2` は使わない。
- `dist/` は成果物として必要ない限り差分に残さない。
- 保存、エクスポート、レイヤー複製/結合/フォルダ、範囲選択、高機能バケツ、アニメ/漫画効果は今回扱わない。

## 報告されている症状

- 時々、アンドゥで描画座標がズレる。
- アンドゥ単体で起きるのか、リサイズ後のアンドゥで起きるのかは未確定。
- Vキーモードでアクティブレイヤーを少し移動すると、描画可能座標がキャンバスと一致して元に戻ることがある。
- Console 更新分では致命的エラーは見えていない。起動ログと popup 表示ログが中心。

## 今回の主作業

### 1. 履歴・リサイズ・座標経路の棚卸し

対象候補：

- `tegaki_work/system/history.js`
- `tegaki_work/ui/keyboard-handler.js`
- `tegaki_work/ui/resize-popup.js`
- `tegaki_work/ui/resize-slider.js`
- `tegaki_work/system/camera-system.js`
- `tegaki_work/coordinate-system.js`
- `tegaki_work/system/layer-system.js`
- `tegaki_work/system/layer-transform.js`
- `tegaki_work/system/drawing/brush-core.js`
- `tegaki_work/system/drawing/fill-tool.js`
- `tegaki_work/system/drawing/stroke-renderer.js`
- `tegaki_work/system/drawing/thumbnail-system.js`

確認すること：

1. 描画 stroke の undo/redo が存在するか。存在しない場合、現状の `History` は何を戻しているか。
2. ペン、消しゴム、バケツ、リサイズ、レイヤー変形がどの形式の history command を push しているか。
3. `resize-popup.js` と `resize-slider.js` のどちらが現行 UI で使われているか。
4. リサイズ undo が旧サイズへ戻すとき、描画レイヤーの `RenderTexture` 内容とサイズも戻しているか。
5. リサイズ時の `alignOptions` が do/undo で逆方向の offset として扱えているか。
6. `camera:resized` 後に `coordinate-system.clearCache()`、レイヤー transform 状態、サムネイル更新が十分に呼ばれているか。
7. Vキー変形モードの enter/exit が、座標ズレを直している副作用を持っていないか。

この棚卸し結果を `tegaki_work/PROGRESS.md` の先頭へ追記する。

---

### 2. 再現手順を試す

最低限、以下を試す。

1. 起動直後 400x400 で線を描く。
2. `Ctrl+Z` / `Ctrl+Y` を数回行う。
3. リサイズする。
4. リサイズ後に線を描く。
5. `Ctrl+Z` / `Ctrl+Y` を数回行う。
6. Vキーでアクティブレイヤーを少し移動し、描画座標が直るか見る。
7. 消しゴム、バケツでも同じ流れを軽く確認する。

再現しない場合でも、どの手順では再現しなかったかを `PROGRESS.md` に記録する。

---

### 3. 安全に直せる範囲だけ実装

実装してよい候補：

- `resize-popup.js` の undo/redo が `RenderTexture` サイズ・内容・表示 sprite を正しく戻すための小修正。
- リサイズ履歴コマンドに、旧キャンバスサイズ、旧レイヤー texture 状態、alignment offset を明確に保持する小修正。
- `camera:resized` 後に coordinate/cache/thumbnail/transform 表示を同期する小修正。
- 履歴適用中に描画・変形・サムネイル更新が不完全にならないようにする小修正。
- `TEGAKI_CONFIG.debug === true` のときだけ、history command 名、canvas size、active layer id、RenderTexture size、layer transform を出す診断ログを追加する。
- 現行で使われていない古い resize 経路が混乱を招く場合は、削除ではなく「非現行」とコメント・PROGRESS に明記する。

実装しないで止めるもの：

- history command 形式の大幅変更。
- RenderTexture の履歴スナップショットを毎回保存する大規模設計。
- レイヤー構造そのものの変更。
- Vキー変形 bake の全面再設計。
- 保存・エクスポート・レイヤー編集機能の追加。
- WebGPU / SDF / MSDF / WebGL2 Mesh の復活。

今回の Gemini 作業では、保存/出力やUI拡張は後回しでよい。アンドゥ座標ズレの原因整理と、小修正で直せる範囲を優先する。

## ビルドと確認

必須：

```powershell
cd tegaki_work
npm.cmd run build
```

可能ならブラウザで確認：

1. 起動時に致命的エラーがない。
2. 通常ペンで描ける。
3. 消しゴムで透明に消せる。
4. 基本バケツが壊れていない。
5. `Ctrl+Z` / `Ctrl+Y` 後に描画座標がズレない。
6. リサイズ後、リサイズ undo/redo 後に描画可能範囲と見た目のキャンバスが一致する。
7. Vキーで少し動かさないと座標が直らない、という状態が解消または原因候補まで絞れている。
8. `Shift+P` / `Shift+E` とステータス表示が壊れていない。
9. `TEGAKI_CONFIG.debug === false` の通常状態で Console が過剰ログに埋まらない。
