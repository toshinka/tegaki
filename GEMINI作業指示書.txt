## 0. 現状整理

オーナー実機確認：

- リサイズセンタリングは良好。
- Space同時押し中のみキャンバス移動も良好。
- 液タブペン描画不可は未解決。
- レイヤーサムネイル黒背景は未解決。
- 新発見：キャンバスをリサイズすると見た目のキャンバスは広がるが、左上 `0,0` から旧 `400x400` 範囲にしか描画できない。

この新発見は重要。現在のリサイズ処理は `config.canvas.width/height`、赤枠、マスク、背景Graphics、checkerを更新しているが、既存レイヤーの `layerData.renderTexture` を新サイズに作り直していない可能性が高い。

ライブラスター描画は `brush-core.js` で `activeLayer.layerData.renderTexture` に直接焼き込む。したがって、そのRenderTextureが400x400のままなら、見た目だけ広がっても400外には描けない。

## 1. リサイズ後の描画領域400固定を最優先で直す

対象：

- `tegaki_work/system/layer-system.js`
- `tegaki_work/system/data-models.js`
- 必要なら `tegaki_work/system/camera-system.js`
- 必要なら animation frame renderTexture 管理部分

疑いが強い箇所：

- `LayerModel.initializeTexture(width, height)` は新規レイヤー作成時だけ呼ばれる。
- `LayerSystem._setupResizeEvents()` は checker と背景Graphicsを更新しているが、既存レイヤーの `renderTexture` / `layerSprite` を新サイズへ更新していない。
- `brush-core.js` は `activeLayer.layerData.renderTexture` へ直接描くため、旧サイズ外の描画が保存されない。

必須調査ログ：

リサイズ前後に各レイヤーで以下を確認する。

```js
console.log('[LayerSystem] resize layer texture', {
    layer: layer.layerData?.name,
    oldCanvas: `${data.oldWidth}x${data.oldHeight}`,
    newCanvas: `${data.width}x${data.height}`,
    rt: layer.layerData?.renderTexture && {
        width: layer.layerData.renderTexture.width,
        height: layer.layerData.renderTexture.height
    },
    spriteTexture: layer.layerData?.layerSprite?.texture && {
        width: layer.layerData.layerSprite.texture.width,
        height: layer.layerData.layerSprite.texture.height
    }
});
```

ログは原因確定後、debug制御に寄せるか削除する。

修正方針：

1. `camera:resized` を受けた時に、全既存レイヤーの RenderTexture を新キャンバスサイズへ作り直す。
2. 旧RenderTextureの内容を新RenderTextureへコピーする。
3. コピー位置はリサイズUIの `alignOptions` に合わせる。中央基準なら `offsetX = (newWidth - oldWidth) / 2`、`offsetY = (newHeight - oldHeight) / 2`。
4. `layer.layerData.renderTexture` を新RenderTextureへ差し替える。
5. `layer.layerData.layerSprite.texture` も新RenderTextureへ差し替える。Sprite自体を作り直す場合は child order を壊さない。
6. 非背景レイヤーの maskTexture / maskSprite があるなら同じサイズへ更新する。
7. 背景レイヤーは backgroundGraphics だけでなく、必要なら background layer の renderTexture / layerSprite も新サイズへ更新する。
8. `frameRenderTextures` やアニメーションフレーム用RenderTextureがある場合も、後で破綻しないよう更新方針を確認する。今回すぐ難しければ `PROGRESS.md` に残す。

実装候補：

`LayerSystem` に次のような専用メソッドを追加する。

```js
resizeLayerTextures(newWidth, newHeight, oldWidth, oldHeight, alignOptions) {
    const offsetX = this._getResizeOffsetX(newWidth, oldWidth, alignOptions?.horizontal);
    const offsetY = this._getResizeOffsetY(newHeight, oldHeight, alignOptions?.vertical);

    for (const layer of this.getLayers()) {
        if (!layer.layerData || layer.layerData.isFolder) continue;
        this._resizeSingleLayerTexture(layer, newWidth, newHeight, offsetX, offsetY);
    }
}
```

`_resizeSingleLayerTexture` の考え方：

```js
const oldRT = layerData.renderTexture;
const oldSprite = layerData.layerSprite;
const newRT = RenderTexture.create({ width: newWidth, height: newHeight, antialias: true });

const tempSprite = new Sprite(oldRT);
tempSprite.position.set(offsetX, offsetY);

renderer.render({
    container: tempSprite,
    target: newRT,
    clear: true,
    clearColor: 0x000000,
    clearAlpha: 0
});

layerData.renderTexture = newRT;
if (oldSprite) {
    oldSprite.texture = newRT;
} else {
    layerData.layerSprite = new Sprite(newRT);
    layer.addChildAt(layerData.layerSprite, 0);
}

oldRT.destroy(true);
tempSprite.destroy({ texture: false, baseTexture: false });
```

注意：

- `oldSprite.destroy()` で newRT まで壊さないようにする。
- 背景GraphicsやmaskSpriteなど、既存child順を壊さない。
- 旧内容を左上基準で維持するか中央基準で維持するかは resize popup の alignOptions と一致させる。
- リサイズ後に `requestThumbnailUpdate()` を全レイヤーへ出す。

受け入れ条件：

- 400x400から横長/縦長へ拡大後、旧400範囲外にも描ける。
- 旧400範囲内の既存描画が消えない。
- 中央/左/右/上/下などのリサイズ基準がある場合、既存描画の位置が意図通り保持される。
- 消しゴムも新しい範囲で透明消去できる。
- サムネイルと書き出しが新サイズを反映する。

## 2. 液タブペン描画不可を再調査する

対象：

- `tegaki_work/system/drawing/pointer-handler.js`
- `tegaki_work/system/drawing/drawing-engine.js`
- `tegaki_work/system/camera-system.js`

現状：

`TegakiConsole.txt` には `[PointerHandler] raw pointerdown Object` しか残っておらず、肝心の `pointerType/button/buttons/pressure` の値がテキストに残っていない。さらに `[DrawingEngine] Pen Down` が見えない。

可能性：

- 液タブが `pointerType: 'pen'` ではなく `mouse` または空文字で来ている。
- `button === 2` / `buttons === 2` 扱いで、PointerHandlerまたはCameraSystemで右クリック扱いになっている。
- CameraSystemが先に `canvasMoveMode` を立て、DrawingEngineがブロックしている。
- pointerdownは来ているが、DrawingEngine側のdown handlerへ到達していない。

必須修正：

raw pointerdownログを、折りたたまれない文字列ログに変更する。

```js
console.log('[PointerHandler] raw pointerdown', JSON.stringify({
    pointerType: e.pointerType,
    button: e.button,
    buttons: e.buttons,
    pressure: e.pressure,
    pointerId: e.pointerId,
    isPrimary: e.isPrimary,
    target: e.target?.tagName,
    id: e.target?.id,
    className: String(e.target?.className || '')
}));
```

DrawingEngine側も、ペン限定ではなく一時的に全pointerdownの入口ログを出す。

```js
console.log('[DrawingEngine] down gate', JSON.stringify({
    pointerType: info.pointerType,
    button: info.button,
    buttons: info.buttons,
    pressure: info.pressure,
    canvasMove: this.cameraSystem?.isCanvasMoveMode?.(),
    vKey: this.layerSystem?.vKeyPressed
}));
```

修正方針：

- 実測値を見るまで「penだけ通す」前提で閉じない。
- 液タブが `mouse + button 2` として来るなら、単純な `pointerType !== 'pen'` 判定では直らない。
- 右クリックキャンバス移動と液タブ接触を区別するため、`buttons`、`pressure`、`pointerId`、発生対象、必要なら最後のUI操作状態を使う。
- 最低限、キャンバス上の通常描画ツール選択中に `button === 2` でも、液タブ実測パターンは描画へ通す。

受け入れ条件：

- 液タブペンでキャンバスに描ける。
- マウス右クリックドラッグのキャンバス移動は壊れない。
- `TegakiConsole.txt` に、液タブ時の pointerType/button/buttons/pressure が読める形で残る。
- 調査用ログは最終的に debug制御または短期ログとして扱う。

## 3. サムネイル黒背景を「結果確認」まで行う

対象：

- `tegaki_work/system/drawing/thumbnail-system.js`
- `tegaki_work/ui/layer-panel-renderer.js`

現状：

GEMINI報告では解消とあるが、実機では未解決。したがって「clearAlpha: 0 と clearRect を入れたから解決」と判断しない。

必須調査：

1. サムネイル生成後の `thumbCanvas` の透明であるべき座標を `getImageData()` で読む。
2. alpha が 0 か 255 かを文字列ログで出す。
3. 生成された dataURL 画像自体が黒いのか、UI背景が黒いのかを切り分ける。

ログ例：

```js
const px = ctx.getImageData(1, 1, 1, 1).data;
console.log('[ThumbnailSystem] sample pixel', JSON.stringify({
    layer: layer.layerData?.name,
    isBackground: !!layer.layerData?.isBackground,
    rgba: Array.from(px),
    thumbW,
    thumbH
}));
```

打開策：

- 通常レイヤーは `layer` Container全体ではなく、`layer.layerData.renderTexture` から直接抽出する経路を検討する。
- `renderer.extract.canvas(renderTexture)` が透明を黒化するなら、`renderer.extract.pixels()` 等で alpha を保持できる経路を検討する。
- 黒色を透明化する雑な処理は禁止。黒ペンで描いた線まで消える。
- alpha が失われている場合だけ、alphaに基づく補正を行う。

受け入れ条件：

- 空の通常レイヤーサムネイルは黒くならない。
- 赤線のみの通常レイヤーは、線だけが表示され、未描画部分はchecker/透明として見える。
- リサイズ後の非正方形キャンバスでも黒背景にならない。
- 背景レイヤー色は現状より悪化しない。
