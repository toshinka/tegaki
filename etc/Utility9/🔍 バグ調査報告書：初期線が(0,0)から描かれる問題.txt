🔍 バグ調査報告書：初期線が(0,0)から描かれる問題
📋 問題の現状

症状: マウス/タッチ開始時に(0,0)座標から描画開始点まで線が引かれる
影響範囲: ペンツール・消しゴムツール両方で発生
再現性: 100%再現（毎回発生）

🔬 根本原因分析
1. PIXI.Graphics の描画仕様による問題
javascript// 現在のコード（問題あり）
path.moveTo(x, y);
path.lineTo(x, y); // この時点で(0,0)からの線が描画される
原因: PIXI.js v7では、Graphicsオブジェクト作成直後の初期位置は(0,0)です。moveTo()を呼んでも内部状態がリセットされず、最初のlineTo()で(0,0)から指定座標への線が描画されてしまいます。
2. lineStyle()のタイミング問題
javascript// 問題のあるシーケンス
const path = new PIXI.Graphics(); // 内部ペン位置: (0,0)
path.lineStyle(size, color, opacity); // lineStyleが有効になる
path.moveTo(x, y); // ペン位置移動（但し描画パスに影響なし）
path.lineTo(x, y); // (0,0) → (x,y) の線が描画される
3. 座標取得システムは正常
getLocalPointerPosition() メソッドの座標取得は正確に動作しており、座標系の問題ではありません。
🧩 技術的詳細
PIXI.js Graphics描画フロー

new PIXI.Graphics() → 内部ペン位置(0,0)、描画パス空
lineStyle() → 線スタイル設定、描画モード開始
moveTo(x, y) → ペン位置移動（描画なし）
lineTo(x, y) → 前のペン位置から線を引く

問題: ステップ4で、前のペン位置(0,0)から指定座標への線が描画される
💡 解決方法の提案
方法1: beginFill/endFill 使用（推奨）
javascriptconst path = new PIXI.Graphics();
path.lineStyle(size, color, opacity);
path.moveTo(x, y);
path.beginFill(color, 0); // 塗りつぶしなしで描画開始
path.drawCircle(x, y, size/2); // 点として描画
path.endFill();
方法2: 描画前のクリア
javascriptconst path = new PIXI.Graphics();
path.clear(); // 内部状態完全リセット
path.lineStyle(size, color, opacity);
path.moveTo(x, y);
// 以降の描画で正常になる
方法3: 複数Graphicsオブジェクト使用
javascript// 点用
const dot = new PIXI.Graphics();
dot.beginFill(color, opacity);
dot.drawCircle(x, y, size/2);
dot.endFill();

// 線用（別途作成）
const line = new PIXI.Graphics();
line.lineStyle(size, color, opacity);
line.moveTo(x, y);
