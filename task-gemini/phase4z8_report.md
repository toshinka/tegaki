# Phase 4z8 — ClipAsset Internal Layer CRUD MVP (GEMINI報告)

## 1. 実施内容

### 🛠️ 内部レイヤーの管理機能 (CRUD) 実装
- **`system/animation/animation-data-model.js`**
    - `TimelineModel` に以下の管理ヘルパーを追加しました：
        - `addClipAssetInternalLayer(assetId, options)`: 新規レイヤー追加。
        - `renameClipAssetInternalLayer(assetId, layerId, name)`: 名前変更。
        - `removeClipAssetInternalLayer(assetId, layerId)`: レイヤー削除。**最後の1枚は削除できないガード**を実装。
        - `toggleClipAssetInternalLayerVisibility(assetId, layerId)`: 可視性（表示/非表示）の切り替え。
- **`ui/animation-table-popup.js`**
    - **Inspector UI の機能拡張**:
        - ヘッダーに `+` ボタンを追加（レイヤー追加）。
        - 各レイヤー行に「👁 (可視性)」「✎ (リネーム)」「× (削除)」ボタンを配置しました。
        - `prompt()` を利用した簡易リネーム機能を実装。
        - レイヤー削除時に、削除対象を選択していた場合は別レイヤーへ選択を自動移動するロジックを導入。
    - **スタイリング**:
        - ホバー時のみ操作ボタンが表示されるモダンなリスト形式を採用。
        - 可視性オフの状態を薄く表示（opacity 0.3）するなど、直感的なフィードバックを追加。

### 🎨 安全性と互換性
- **削除ガード**: アセットから全てのレイヤーが消えることを防ぎ、データ整合性を維持します。
- **Preview への影響回避**: 今回の改修はデータ構造と Inspector UI のみに留めており、キャンバス上のプレビュー合成ロジックには影響を与えていません（将来のマルチレイヤー合成フェーズで対応予定）。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **操作確認**:
    1. インスペクターの `+` で `Layer 2` 等が増えることを確認。
    2. `✎` ボタンで名前が変更され、即座に UI に反映されることを確認。
    3. `👁` ボタンで状態が切り替わり、内部モデルの `visible` が反転することを確認。
    4. `×` ボタンでレイヤーが消えること、および最後の1枚では警告が出て削除されないことを確認。

## 3. 次フェーズへの展望 (以降)

1.  **Virtual Layer Panel**: インスペクターで選択中の内部レイヤーを、メインのレイヤーパネルと同様に直接描画・編集できる接続。
2.  **マルチレイヤー合成 Preview**: アセット内の複数レイヤーを（透明度や合成モードを加味して）1つのテクスチャとして合成し、タイムラインやキャンバスに表示する処理。
3.  **ドラッグ＆ドロップ順序入れ替え**: インスペクター内でのレイヤー順序変更。
