# Phase 4z17 — Selected ClipAsset Internal Layer Mirror (GEMINI報告)

## 1. 実施内容

### 帰 選択中アセットの内部レイヤーミラー実装
- **`ui/layer-panel-renderer.js`**
    - **`createSelectedClipAssetLayerMirror()` の実装**:
        - タイムラインで選択中のクリップ、または Asset Library で選択中のアセットが持つ内部レイヤーを、通常のレイヤーパネル内にリスト表示する機能を実装しました。
        - **表示項目**: 可視性アイコン（👁）、レイヤー名、不透明度、合成モード、Snapshot 状態。
        - タイムラインの Y 軸順（重なり順）を意識しつつ、インスペクターと同様の並び順で表示します。
    - **選択状態の同期**:
        - ミラー内の各行をクリックすることで、アニメーションテーブル側の `selectedInternalLayerId` を更新し、相互にハイライト状態を同期させます。
- **`styles/main.css`**
    - **ミラー専用スタイリング**: 紺色の背景とオレンジ色のアクセント（クリップ選択色）を採用し、通常のレイヤーフォルダと明確に区別しました。
    - コンパクトな 1 行レイアウトにより、限られたパネルスペース内で情報を効率的に表示します。

### 🎨 安全性と干渉防止
- **既存機能への配慮**: ミラー部分には専用の CSS クラスを使用し、通常のレイヤー用のクラス（`.layer-item`）を一切含んでいません。これにより、SortableJS による通常のレイヤー並び替えや D&D ロジックが誤動作するリスクを完全に排除しました。
- **イベント委譲**: クリック操作は `container` への委譲により効率的に処理されます。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **動作確認**:
    1. アニメーションテーブルでクリップを選択すると、レイヤーパネルの CAF ヘッダー直下に「CLIP LAYERS」セクションが出現することを確認。
    2. ミラー内のレイヤーをクリックすると、アニメーションテーブル側のインスペクターでも選択が同期されることを確認。
    3. ミラーが表示されている状態でも、通常のレイヤーの並べ替えやフォルダ操作が正常に行えることを確認。

## 3. 次フェーズへの展望 (以降)

1.  **Virtual Layer Panel (完全統合)**: 現在のミラー表示を拡張し、通常のレイヤーパネルの「中身」そのものを内部レイヤーへ一時的に差し替える、またはミラー側からも表示切り替えやリネームを可能にする。
2.  **アセット内での描画対象切り替え**: ミラーで選択した内部レイヤーを、実際の描画（ペン入れ）ターゲットとして指定可能にする接続。

## 4. Codex確認・補修

- `createSelectedClipAssetLayerMirror()` は通常Layer DOMへ混入せず、専用class / `data-internal-layer-id` / `data-asset-id` のみを使っていることを確認。
- SortableJSは `draggable: '.layer-item'` のままで、内部Layerミラーは並び替え対象外であることを確認。
- 内部Layerの可視判定を `layer.visible !== false` に補修し、`visible` 未定義の既存データを可視扱いに揃えた。
- Codex側でも `npm.cmd run build` 成功を確認。
- Gemini/Codexのビルドで生成された `dist/` 差分は成果物から除外する。
