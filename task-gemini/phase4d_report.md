# Phase 4d — アニメテーブルのセル選択・フレームカーソルMVP (GEMINI報告)

## 1. 実施内容

### 🧭 フレームナビゲーションの実装
- **`system/animation/animation-data-model.js`**
    - `TimelineModel.setCurrentFrame(frameIndex)` を実装し、現在フレームの移動を可能に。
- **`ui/animation-table-popup.js`**
    - フレームヘッダー (`.anim-frame-num`) をクリックすることで、`currentFrame` を移動できるインタラクションを追加。
    - ヘッダー部分に `:hover` スタイルを追加し、クリック可能な領域であることを明示。

### ✨ セル選択と条件付き削除の実装
- **`ui/animation-table-popup.js`**
    - `selectedCelId` を導入し、クリックされたセルを選択状態として保持。
    - 選択されたセルは UI 上で強調表示（オレンジ色、拡大、ボーダー）されるように CSS を追加。
    - **セル削除の変更**: 単純クリックが「選択」になったため、セル削除は `Alt+クリック` または `Shift+クリック` に割り当て。
    - **セル追加の維持**: 空白スロットのクリックは引き続き新規セルの追加＆選択として機能。

### 🎨 ビジュアル・フィードバックの向上
- 現在フレーム列のハイライトと、選択セルの強調表示を共存させ、視覚的に判別しやすい配色に調整。
- アクティブトラック内の選択セルは、さらに判別しやすい色味に微調整。

## 2. 検証結果

### ✅ 動作確認
- **フレーム移動**: ヘッダーの数字をクリックすると、現在フレームのオレンジ色のハイライトが即座に移動することを確認。
- **セル選択**: 既存のセルをクリックすると、そのセルが選択（強調表示）されることを確認。
- **セル追加**: 空のスロットをクリックすると、セルが追加され、同時に選択状態になることを確認。
- **セル削除**: `Alt` または `Shift` を押しながらセルをクリックすると、削除されることを確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **再生連動**: フレーム移動は UI 上のハイライトのみであり、描画レイヤーの切り替えや再生タイマーとの連動は未実装。
- **ドラッグ操作**: セルの移動やフレームのドラッグによる連続移動は次フェーズ以降。
- **保存**: セルの選択状態や配置データは in-memory のみであり、リロードで消失する。

## 3. 次フェーズへの提案 (Phase 4e)

1. **再生機能の最小接続**: 旧 `animation-system.js` の再生タイマーを慎重に再定義し、再生ボタンでフレームが自動進行するようにする。
2. **描画レイヤーとの連動**: 現在フレームにセルがあるレイヤーのみを表示するなどの、基本的な表示制御の試作。
3. **セルの長さ (Duration) 調整**: セルの端をドラッグして長さを変える機能の検討。
