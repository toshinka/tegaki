# Phase 4v — Timeline Clip Move MVP (GEMINI報告)

## 1. 実施内容

### 🏗️ モデル層の移動ロジック実装
- **`system/animation/animation-data-model.js`**
    - `TimelineModel.canMoveClip(clipId, targetLaneId, targetStartFrame)`: 移動の妥当性を検証するガードメソッドを実装。存在確認、レーン種別、範囲、他クリップとの重なりをチェック。
    - `TimelineModel.moveClip(...)`: 実際にクリップを移動。レーンを跨ぐ移動時には、所属配列の付け替えと `sourceLayerId` の更新を行い、タイムラインと実レイヤーの対応関係を維持。

### 🖱️ クリップ移動（D&D）の実装
- **`ui/animation-table-popup.js`**
    - クリップブロック本体をマウスでドラッグして時間軸・レーン間を移動できる機能を実装。
    - **誤操作防止**: 短い距離のドラッグは「選択（Click）」として扱い、一定以上の移動で「移動（Drag）」へ移行するデバウンス処理を導入。
    - **フィードバック**: ドラッグ中はクリップを半透明化し、カーソルを `grabbing` へ変更。
    - **既存機能との共存**: 右端の「リタイミングハンドル」と「移動D&D」の判定を分離し、両方の操作が可能な状態を維持。

### 🎨 ビジュアル・フィードバックの強化
- セルブロックに `cursor: grab` を設定し、操作可能であることを明示。
- ドラッグ開始時に `.moving` クラスを付与し、移動中であることを強調。

## 2. 検証結果

### ✅ 動作確認
- **移動操作**: クリップを掴んで別のフレーム、または別のレーンへ移動できることを確認。
- **重なりガード**: 移動先に他のクリップがある場合、移動がキャンセルされ元の位置に留まることを確認。
- **データの整合性**: 移動後も Snapshot (キャプチャ済みの絵)、`assetId`、`duration` が正しく維持されることを確認。
- **再生連動**: 移動後の位置でプレビュー（パラパラ再生）が正しく行われることを確認。
- **リタイミング継続**: 右端を掴んだ場合は、移動ではなく伸縮が引き続き正常に動作することを確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **Altドラッグコピー**: 今回は「移動」のみであり、コピーを兼ねたドラッグは将来の課題。
- **複数クリップ移動**: 現状は単一クリップのみ対応。

## 3. 次フェーズへの提案 (Phase 4w)

1. **自動キャプチャ (Auto-Key)**: 描画終了時に、選択中のセルへ Snapshot を自動的に書き込む「オートキー」機能の実装。
2. **クリップの複数選択**: 矩形選択や Shift選択による複数クリップの一括操作。
3. **アセットライブラリUI**: `ClipAsset` を一覧表示し、タイムラインへ再配置する UI。
