# Phase 4h — アニメテーブル操作面の安定化MVP (GEMINI報告)

## 1. 実施内容

### 🖱️ パネルのドラッグ移動実装
- **`ui/animation-table-popup.js`**
    - ヘッダー部分 (`.anim-table-header`) をドラッグハンドルとして機能させ、パネルを画面内で自由に移動可能にしました。
    - 画面外への完全な消失を防ぐクランプ処理を実装。
    - ドラッグ中の誤クリック防止フラグを導入。

### 📏 縦スクロール同期とレイアウト改善
- **`ui/animation-table-popup.js`**
    - DOM構造を再編し、左側のトラック一覧と右側のグリッドを同一のスクロールコンテキスト (`.anim-table-viewport`) で管理。
    - CSS `sticky` を活用し、縦スクロール時はヘッダーを、横スクロール時はトラック一覧を固定表示することで、行のズレを物理的に解消しました。
    - 行高を `32px` に完全固定し、左右の整合性を保証。

### ⌨️ キーボードナビゲーション
- **`ui/animation-table-popup.js`**
    - アニメテーブルが表示中かつ入力欄非編集中に、左右キー (`ArrowLeft/ArrowRight`) で現在フレームを移動できる機能を追加。
    - スペースキーによる再生/停止のトグルにも対応（既存ショートカットとの競合に配慮）。

### 📂 フォルダトラックの保護
- フォルダレイヤー由来のトラックを視覚的に区別（背景色の変更、太字）。
- フォルダ行へのセル配置を無効化し、純粋なグループ見出しとしての役割に限定しました。

## 2. 検証結果

### ✅ 動作確認
- **ドラッグ移動**: ヘッダーを掴んでパネルを移動できることを確認。
- **スクロール同期**: 大量のレイヤー（20枚以上）を作成し縦スクロールしても、トラック名とグリッド行が一切ズレないことを確認。
- **キー操作**: 左右キーで再生ヘッドがスムーズに移動することを確認。入力欄（レイヤー名変更など）ではキー操作が優先されることを確認。
- **外クリック制御**: パネル外をクリックしてもテーブルが閉じないことを確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **位置の永続化**: 移動したパネル位置はリロードでリセットされます。
- **フォルダ開閉連動**: レイヤーパネル側でのフォルダ開閉状態は、まだアニメテーブルの行表示/非表示には連動していません。

## 3. 次フェーズへの提案 (Phase 4i)

1. **タイムラインベースの表示制御**: 現在フレームにセルがあるレイヤーのみをキャンバスに表示し、セルがない期間は一時的に非表示（または透明度調整）にする実描画連動の実装。
2. **描画内容の固定 (RenderTexture Snapshot)**: 特定のセルに対して、現在のレイヤー描画内容を「スナップショット」として保持する基礎実験。
