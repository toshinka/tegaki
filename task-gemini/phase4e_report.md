# Phase 4e — アニメテーブルのセル長さMVP (GEMINI報告)

## 1. 実施内容

### 📏 データモデルの多コマ対応
- **`system/animation/animation-data-model.js`**
    - `TrackModel.getCelAtFrame(frameIndex)` を更新し、セルの開始フレームだけでなく `duration`（長さ）の範囲内であればそのセルを返すように変更。
    - `TrackModel.canPlaceCel(startFrame, duration, ignoreCelId)` を追加。セルの重なりをチェックし、物理的に不可能な配置や伸縮を制限するガードロジックを実装。
    - `TrackModel.setCelDuration(celId, newDuration)` を実装。範囲チェックを行いながら安全にセルの長さを変更可能に。
    - `TrackModel.addCel` に配置可能性チェックを追加。

### ➕ セル長さ（Duration）変更UIの実装
- **`ui/animation-table-popup.js`**
    - アニメテーブルのヘッダーに `DURATION` 操作用の `-` / `+` ボタンを追加。
    - 選択中のセルがある場合、これらのボタンで 1コマ単位の長さ増減を可能に（最小 1コマ）。
    - 伸縮時、隣接する他のセルと重なる場合は変更を拒否する安全設計。

### 🎨 横長セルのビジュアルレンダリング
- `duration > 1` のセルを、時間軸上で横に伸びた一つの連続したブロックとして描画。
- CSS `width` を `duration` に応じて動的に計算（`duration * 30px` ベース）。
- セルの範囲内（開始コマ以外）をクリックした場合も、正しくそのセルが選択されることを確認。

## 2. 検証結果

### ✅ 動作確認
- **セル伸縮**: セルを選択し、ヘッダーの `+` ボタンで横に伸び、`-` ボタンで縮むことを確認。
- **重なりガード**: 隣に別のセルがある場合、`+` を押してもそれ以上伸びない（上書きされない）ことを確認。
- **選択の維持**: 伸びたセルのどのコマをクリックしても、そのセルが選択状態（オレンジ枠）になることを確認。
- **削除**: `Alt+クリック` または `Shift+クリック` で、多コマセルも一括で削除されることを確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **ドラッグ伸縮**: 端を掴んでのドラッグ伸縮は将来の課題。今回はボタン操作のみ。
- **再生への影響**: `duration` に応じた「コマの維持」ロジックは再生タイマー接続時に実装予定。
- **保存**: 設定データは in-memory のみ。
