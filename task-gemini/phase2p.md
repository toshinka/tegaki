# Phase 2p — 設定パネルのタブ化とヘルプ・ショートカット統合

> Phase 2o で UI の標準化が完了。
> Phase 2p は、設定パネル (`SettingsPopup`) を拡張し、タブ切り替えによって「アプリ設定」と「ヘルプ・ショートカット一覧」を統合するフェーズ。
> 左サイドバーの資源を節約しつつ、必要な情報に素早くアクセスできるスマートな UI を実現する。

---

## 背景・目的

- 左サイドバーのボタンを増やさず、既存の「設定」導線を活用して情報の集約を図る。
- 今後の拡張性（多機能化）を見据え、ポップアップ内でのタブ切り替え UI パターンを確立する。
- ユーザーがいつでもショートカットキーを確認できる環境を整え、利便性を向上させる。

---

## 実装内容（Gemini主体）

1. **タブ UI の CSS 定義**
   - `main.css` に、ふたば系配色に馴染むタブバー (`.ui-tabs`) とタブボタンのスタイルを追加。
   - 選択中のタブを視覚的に強調するアクティブ状態の定義。

2. **SettingsPopup の構造拡張**
   - `_populateContent()` を修正し、タイトル下にタブボタンエリアを新設。
   - コンテンツエリアを「設定 (`#tab-settings`)」と「ヘルプ (`#tab-help`)」に分割。
   - タブ切り替えのイベント処理を追加（シンプルに `display: block/none` で制御）。

3. **ヘルプ・ショートカットコンテンツの作成**
   - 「ヘルプ」タブ内に、ショートカットキー一覧や基本操作ガイドを、標準化されたリスト形式で流し込む。
   - 内容が増えても共通スクロールバーで快適に閲覧可能にする。

---

## 完了条件

- 設定パネルを開いた際、タブで「設定」と「ショートカット」を切り替えられる。
- 全体のビジュアル（透過・配色）が標準化ルールを維持している。
- 既存の設定機能（筆圧補正等）が正常に動作し続けている。
- `npm.cmd run build` が成功している。

---

## 工程規律・禁止事項の再確認

- **破壊的編集の禁止**: JS ファイルを丸ごと上書きせず、差分単位で編集する。
- **UI 標準の遵守**: `TEGAKI.md` で定めた標準クラス (`.popup-panel--translucent`, `.ui-scrollbar` 等) を必ず使用する。
- **安全な状態遷移**: タブ切り替え時に既存の入力値（スライダーの状態等）がリセットされないよう慎重に実装する。
- **EventBus の整合性**: 設定変更に伴うイベント発火の仕様を変更しない。

---

## Codex 判断へ戻すもの

- 設定項目の大幅な増減や、永続化（保存）ロジックの変更が必要になった場合。
- タブ化によってポップアップのサイズが極端に大きくなり、配置ロジックに影響が出る場合。
