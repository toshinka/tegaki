# Phase 4z — Blank Clip Asset on Cel Create MVP (GEMINI報告)

## 1. 実施内容

### 🆕 空アセットの自動生成と `isBlank` 管理
- **`system/animation/animation-data-model.js`**
    - `DrawingSnapshotModel` に `isBlank` プロパティを追加しました。初期状態（透明）のスナップショットを識別可能にします。
    - `TimelineModel` に `createBlankClipAsset(options)` ヘルパーを追加しました。キャンバスサイズに合わせた透明な Uint8ClampedArray を持つスナップショットと、それに関連付けられたアセットを一括生成します。
    - `makeClipAssetUnique()` メソッドを更新し、独立化（複製）時に `isBlank` 状態が正しく継承されるようにしました。
- **`ui/animation-table-popup.js`**
    - **新規セル作成時の自動割当**: タイムラインの空スロットをクリックしてセルを作成した際、即座にそのセル専用の「空アセット」を生成・割り当てるようにしました。
    - **UI表現の最適化**: `isBlank: true` のアセットを持つセルには、スナップショット存在を示す「白い丸アイコン」を表示しないように変更しました。これにより、中身があるコマとないコマを視覚的に区別できます。
    - **状態遷移**: `AUTO` キャプチャまたは手動 `CAPTURE` 実行時に、描画内容が保存されると自動的に `isBlank: false` へ遷移し、アイコンが表示されるようになります。

### 💎 `UNIQUE` ボタンの洗練
- `UNIQUE` ボタンの有効化ロジックを変更し、「アセットを持っており、かつ他と共有されている（参照数が2以上）」場合のみ押せるようにしました。これにより、不要なアセット複製を防ぎます。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **ワークフロー確認**:
    1. 空スロットをクリック -> セル作成（アイコンなし、アセットID割り当て済み）。
    2. そのまま `EDIT` モードで描画 -> `AUTO` によりキャプチャ -> `isBlank` が `false` になりアイコン出現。
    3. このセルを `COPY` & `PASTE` -> リンク（鎖）アイコン出現（アセット共有）。
    4. 片方で `UNIQUE` 実行 -> 鎖アイコン消滅（独立化成功）。

## 3. 次フェーズへの展望 (Phase 5a / 以降)

1.  **複数選択 (Multi-Selection)**: クリップの矩形選択や Shift+クリックによる複数操作。

4.  **Undo/Redo**: 複雑化するアセット操作（生成・複製・削除）の履歴管理。
