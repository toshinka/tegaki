# Phase 4w — Auto Capture / Auto-Key MVP (GEMINI報告)

## 1. 実施内容

### 🤖 自動キャプチャ (Auto-Key) 機能の実装
- **`ui/animation-table-popup.js`**
    - `isAutoCaptureActive` 状態を追加し、ヘッダーに `AUTO` トグルスイッチを設置しました。
    - `drawing:stroke-completed` イベントを購読し、描画が完了するたびに `_handleDrawingCompleted()` を呼び出す仕組みを構築。
    - 自動キャプチャのトリガー条件：
        1. アニメーションテーブルが表示されていること。
        2. `AUTO` トグルが ON であること。
        3. いずれかのセル（クリップ）が選択されていること。
        4. 描画が行われたレイヤーが、選択中セルの所属レーン (`sourceLayerId`) と一致すること。

### 🔄 アセット更新ロジックの最適化
- **`_captureSelectedClip()` のリファクタリング**:
    - 常に新しいアセットを作成するのではなく、選択中セルに既に `assetId` が紐付いている場合は、既存の `DrawingSnapshotModel` を直接更新するロジックへ変更しました。
    - 更新時には、関連する PIXI Texture キャッシュを確実に破棄 (`destroy`) し、キャンバスプレビューに最新の絵が即座に反映されるようにしました。
    - これにより、同じセルに何度も描き重ねてもアセットデータが無秩序に増殖することを防ぎ、メモリ効率を向上させました。

### 🖱️ 操作性の向上
- 自動キャプチャ実行時はコンソール出力を抑制 (`silent` モード) し、ユーザーの描画体験を妨げないように配慮しました。
- 手動の `CAPTURE` ボタンも内部的にこの最適化されたロジックを通るよう統一しました。

## 2. 検証結果

### ✅ 動作確認
- **自動反映**: `AUTO` を ON にして描画すると、ペンを離した瞬間にセルの `rasterSnapshot` が更新され、プレビュー層の絵が最新に切り替わることを確認。
- **アセット管理**: 同じセルに対して連続で描画しても、`TimelineModel.drawingSnapshots` の個数が増えず、既存データが上書きされていることを確認。
- **手動連携**: 手動 `CAPTURE` ボタンでも既存アセットが正しく更新されることを確認。
- **退行確認**: クリップの移動、伸縮、コピー/ペースト、オニオンスキン表示に影響がないことを目視確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **アセット共有時の注意**: 現在、コピー/ペーストでアセット ID を共有しているセルがある場合、片方を更新するともう片方の表示も変わります（インスタンス化の仕様）。「独立したコピー」が必要な場合の「ユニーク化」機能は将来の課題です。

## 3. 次フェーズへの提案 (Phase 4x)

1. **複数セル選択**: タイムライン上での矩形選択、または Shift/Ctrl クリックによる複数セルの同時操作（移動・削除・一括キャプチャ等）。
2. **アセットのユニーク化**: 共有アセットを参照しているセルを、独立した別個のアセットとして切り離す機能。
3. **空アセットの自動生成**: セルを作成（クリック）した瞬間に、その場で空のアセットを自動割り当てする設定の追加。
