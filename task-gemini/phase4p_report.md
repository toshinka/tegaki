# Phase 4p — ClipAsset 再利用MVP（セルコピー/ペースト） (GEMINI報告)

## 1. 実施内容

### 📋 セルコピー/ペースト機能の実装
- **`ui/animation-table-popup.js`**
    - `COPY` および `PASTE` ボタンをヘッダーUIに追加。
    - `copySelectedCel()`: 選択中のセルの情報を `_copiedCelRef` に保持（assetId, snapshot, duration）。
    - `pasteCopiedCel()`: 保持された情報を元に、現在のアクティブレイヤー（トラック）の現在フレームに新しいセルを作成。
    - 貼り付け時、既存のセルと重なる場合は安全のために処理を中断するガードロジックを実装。

### ♻️ アセットの再利用（インスタンス化）の実現
- **データモデルの活用**: 
    - 貼り付けられた新しいセルは、コピー元と同じ `assetId` を参照します。
    - これにより、同じ描画内容（Snapshot）を異なる時間や異なるレイヤーで使い回す「アセット再利用」の第一歩が成立しました。

### 🛡️ 安全性と整合性
- フォルダトラックへの貼り付けを無効化。
- セル削除（Delete/Backspace）時に、参照先のアセット (`ClipAsset` / `DrawingSnapshot`) を削除しない設計を維持し、他のセルからの参照が壊れないように配慮しました。
- ドラッグ操作中（パネル移動やリタイミング中）の誤クリック判定を改善し、操作の安定性を向上。

## 2. 検証結果

### ✅ 動作確認
- **コピー＆ペースト**: セルを `COPY` し、別のフレームや別のレイヤーを選択して `PASTE` することで、同じ内容のセルが増殖することを確認。
- **重なりガード**: 貼り付け先に既にセルがある場合、コンソールに警告が出て上書きされないことを確認。
- **再生連動**: 貼り付けた複数のセルが、再生中に正しく同じ Snapshot を表示することを確認。
- **退行確認**: 手動キャプチャ、セル伸縮、削除、パネル移動が正常に動作し続けていることを確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **アセット名管理**: 貼り付けられたアセット名はコピー元のままです（将来のアセットライブラリでの課題）。
- **複数セル選択**: コピー/ペーストは現状、単一セルのみ対応です。

## 3. 次フェーズへの提案 (Phase 4q)

1. **オニオンスキン基礎**: `animationPreviewContainer` を活用し、前後フレームの Snapshot を半透明（かつ色味を変えて）表示する機能。
2. **自動キャプチャ (Auto-Key)**: 描画終了時に、選択中のセルへ Snapshot を自動的に書き込むオプションの実装。
3. **セルのドラッグ移動 (Move)**: タイムライン上でのセルのドラッグによる位置変更。
