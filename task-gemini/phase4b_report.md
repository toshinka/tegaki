# Phase 4b — アニメテーブル大改造 同期MVP (GEMINI報告)

## 1. 実施内容

### 🔄 データモデルの同期機能追加
- **`system/animation/animation-data-model.js`**
    - `TimelineModel.syncWithLayers(layers, activeIndex)` を実装。
    - `LayerSystem` から取得したレイヤー一覧を元に、アニメ用トラックを動的に生成・更新するロジックを構築。
    - 既存のトラックが存在する場合は ID を維持して再利用し、名前や順序のみを同期するように設計。
    - `TrackModel` に `active` 属性を追加し、`LayerSystem` のアクティブレイヤー状態をアニメテーブル側へも反映。

### 📡 リアルタイム更新の接続
- **`ui/animation-table-popup.js`**
    - `EventBus` を通じてレイヤー関連の全イベント (`layer:created`, `layer:deleted`, `layer:reordered`, `layer:name-changed` など) を購読。
    - レイヤー構成に変更があった際、デバウンス処理を介してアニメテーブルを自動再描画する `requestUpdate()` を実装。

### 🎨 マトリクスUIの動的レンダリング
- **トラック一覧**: レイヤーパネルと同じ順序（上から下）でトラックを表示。アクティブなトラックは強調表示。
- **タイムライングリッド**: 
    - 当面 24 フレーム固定のグリッドを描画。
    - 背景に CSS `linear-gradient` を使用したガイド線を表示。
    - 現在フレーム（`currentFrame`）を示す列ハイライトおよびヘッダー数字の強調を追加。

## 2. 検証結果

### ✅ 動作確認
- **レイヤー同期**: レイヤーの追加・削除・名前変更・順序入れ替えが、アニメテーブルを開いた状態で即座に反映されることを確認。
- **表示整合性**: 右サイドバーのレイヤーパネルと、新アニメテーブルのトラック一覧が常に一致。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **セルの実体**: 現時点では「グリッド」のみであり、特定のフレームに「絵」を配置する機能（Cel管理）は未実装。
- **再生制御**: 再生ボタンによる `currentFrame` の進行は未接続（次フェーズ以降）。
- **保存形式**: `LayerSystem` のデータ構造をそのまま表示しているため、アニメ固有のタイムラインデータ（どのフレームでどのレイヤーを表示するか等）の保存はまだ行われない。

## 3. 次フェーズへの提案 (Phase 4c)

1. **再生ヘッドの駆動**: `animation-system.js` のタイマーと接続し、再生中に現在フレームのハイライトが動くようにする。
2. **フレーム操作の基礎**: 特定のセルをクリックして「キーフレーム（描画の有効化）」を設定できるUIの試作。
3. **簡易表示切り替え**: 特定フレームでのレイヤー表示/非表示を `CelModel` で管理する第一歩。
