# Phase 2i — ポップアップUI統一 / 透過・移動可能化の棚卸し

> Phase 2h で Vキー変形確定後の Undo / Redo は安定化した。
> Phase 2i は、クイックパネルだけが移動可能な状態を整理し、popup全体の見た目と操作感を現代的な浮遊UIへ寄せるための準備フェーズ。

---

## 背景

オーナーメモでは、以下の要望がまとまっている。

- リサイズポップアップを半透明にして、奥のキャンバス状態を見ながら操作したい。
- 他の popup やサイドバーも最終的には透明度を統一したい。
- 各 popup を移動可能にしたい。クイックパネルだけ動く状態は違和感がある。
- 画像・アニメ出力 popup のプレビューは 200x200 程度を基準にし、画面からはみ出しにくくしたい。

ただし、popup は出力、保存、設定、リサイズなど重要操作を含むため、Phase 2i 初回では棚卸しと小さな見た目調整に留める。

---

## 目的

- 現行 popup の構造、移動可否、閉じる導線、背景色、サイズ制約を確認する。
- クイックパネルのドラッグ実装を、共通化する前に責務として整理する。
- リサイズ popup、出力 popup、設定 popup、アルバム popup のうち、最初に触るべき低リスク対象を決める。
- 透過・移動可能化・プレビューサイズ調整を、一括改修ではなく段階導入できる計画にする。

---

## Gemini に任せてもよい範囲

- 現状棚卸し。
- 関係ファイルと CSS クラスの検索。
- `PROGRESS.md` への整理記録。
- 50行以内の CSS 小調整。

Gemini は独自判断で popup ドラッグ共通ヘルパーを新設しない。複数 popup を同時に移動可能化しない。

---

## Codex 担当候補

- popup ドラッグ共通化の設計。
- 最初の1つの popup の移動可能化。
- リサイズ popup の半透明化と画面内制約。
- 出力 popup のプレビューサイズ上限調整。
- 共通 close / header / drag handle の最小クラス化。

---

## 調査対象候補

- `ui/quick-access-popup.js`

- `ui/export-popup.js`
- `ui/settings-popup.js`

---

## Phase 2i 初回の完了条件

- popup ごとの現状と、最初に触るべき対象が `PROGRESS.md` に整理されている。
- 可能なら、1つの popup だけを低リスクに半透明化または移動可能化している。
- `npm.cmd run build` が成功する。
- Phase 2h の Undo / Redo、Phase 2g の複数選択削除、Phase 2d〜2e の属性 popup / クリッピングを壊していない。

---

## Phase 2i ではやらないこと

- クイックパネル全面刷新。
- トップバー本実装。
- リサイズ増減プレビュー本体。
- アルバムのクリック即ロード廃止。
- 独自D&Dの本実装。
- レイヤー/フォルダ操作の追加。
