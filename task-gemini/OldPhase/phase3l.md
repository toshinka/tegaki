# Phase 3l — メイン/サブカラー導線MVP

## 概要

Phase 3k でスポイト、カラースロット、カラーサークルが現在色へつながった。
次は標準的なお絵かきツールらしい色導線として、メインカラー / サブカラーの小さな重なり表示と `X` キー切替を検討する。

目的は「現在色が何か」だけでなく、「もう一つの待機色へすぐ切り替えられる」状態を作ること。
色管理の本格機能ではなく、MVPとして小さく入れる。

## 優先順

### 1. 現状確認

- [ ] `quick-access-popup.js` の現在色丸、スポイト、カラーサークル、カラースロットの同期経路を確認する。
- [ ] `brushSettings.setColor()` / `getColor()` と `brush:color-changed` を確認する。
- [ ] `keyboard-handler.js` と `config.js` で `KeyX` の既存用途を確認する。

注意:

- `Ctrl+X` はレイヤーカット等の既存用途があるため壊さない。
- `X` 単体だけをメイン/サブカラー切替候補にする。

### 2. UI MVP

- [ ] 現在色丸の近く、またはカラーサークル左下に、重なった小さなメイン/サブカラー表示を追加する。
- [ ] メインカラーは前面、サブカラーは背面に置く。
- [ ] 現在の `qa-current-color-dot` と役割が重複しすぎる場合は、無理に両方を大きくしない。

方針:

- QAPを広げない。
- まずは小さい重なりスウォッチで十分。
- 見た目は標準的なペイントツールの「前景色/背景色」程度を参考にする。

### 3. 状態管理

- [ ] `mainColor` と `subColor` を QAP 側で保持する。
- [ ] 既存の `brushSettings` の現在色は `mainColor` と同期させる。
- [ ] スポイト、カラースロット、カラーサークルで選んだ色はメインカラーへ反映する。
- [ ] `X` キーまたはUIクリックで main/sub を入れ替え、入れ替え後の mainColor を `brushSettings.setColor()` へ流す。

保存:

- MVPでは localStorage 永続化は必須ではない。
- 余裕があれば `tegaki-quick-access-main-sub-colors-v1` のようなキーで保存してもよい。

### 4. キーボード

- [ ] `config.js` に `COLOR_SWAP_MAIN_SUB` のようなアクションを追加する。
- [ ] `keyboard-handler.js` で `X` 単体を受け、QAPまたはEventBusへ `color:swap-main-sub` を送る。
- [ ] 入力欄やポップアップ操作中に誤爆しない既存ガードへ合わせる。

### 5. 後続へ回す項目

- カラースロットへの登録/入れ替え/初期化。
- サブカラーを消しゴムや透明色へ特別接続する仕様。
- 色履歴。
- メイン/サブカラーの初期値プリセット管理。

## 確認事項

- [ ] `npm.cmd run build` が成功する。
- [ ] `X` 単体でメイン/サブカラーが入れ替わる。
- [ ] 入れ替え後の色でペン描画できる。
- [ ] スポイト、カラースロット、カラーサークルで選んだ色がメインカラーへ反映される。
- [ ] `Ctrl+X` など既存ショートカットが壊れない。
- [ ] QAPのサイズが大きく崩れない。

## 禁止事項

- 保存形式、履歴形式、レイヤー構造を変更しない。
- QAP 全体のDOMを丸ごと置換しない。
- 色管理をICCや高度なカラープロファイルへ広げない。
