# Phase 3g 追修正 — Gap Close と塗り残しの切り分け

> Phase 3g の途中実装では `gapClosePixels: 1` の thick wall BFS と、その後の mask dilation が入った。
> ただし実機確認では、線の内側沿いに点線状の塗り残しが残っている。

---

## 現状評価

- 現在の実装は、BFS 中に「周囲 gap px に壁があれば現在ピクセルも壁」と扱っている。
- これは漏れ防止には効くが、塗り領域を線から遠ざけるため、線の下まで塗りが届かない塗り残しが出やすい。
- 後段の dilation は削れた mask を戻そうとしているが、1px ではアンチエイリアス線や太い線の内側までは届きにくい。
- つまり、現状の問題は「gap close 不足」だけではなく、「線の下へ少し潜らせる underpaint / 領域拡張不足」も混ざっている。

---

## 次にやること

### 1. gap close と underpaint を分ける

- `gapClosePixels` は「線の隙間から外へ漏れないための壁補正」として扱う。
- 別途 `underpaintPixels` または内部固定値で「塗り領域を線の下へ少し広げる補正」を分けて考える。
- 1つの dilation で両方を済ませようとしない。

### 2. まずは低リスク固定値で検証する

- `gapClosePixels`: 1 のまま維持。
- underpaint 側: 1px、足りなければ 2px まで。
- UI は増やさない。
- 3px 以上や動的強度は、この Phase では避ける。

### 3. Alpha 閾値を急に上げない

- 「Alpha >= 30%」は約 76/255 で、現行 `FILL_ALPHA_THRESHOLD = 24` よりかなり高い。
- 閾値を上げると薄いアンチエイリアス線を壁と見なさなくなり、漏れが増える可能性がある。
- まずは threshold 変更より、mask 設計の分離を優先する。

### 4. edge strength / 適応型 gap は後続

- エッジ強度判定、1.0-3.0 の動的 gap、2.5px 付近の強い dilation は、現時点では過剰。
- 導入する場合は設定 UI と副作用確認が必要になるため、Phase 3g では扱わない。

---

## 実装候補

推奨は次の順。

1. 参照画像から `wallMask` を作る。
2. `gapClosePixels` が 1 のとき、`wallMask` を 1px 膨張して仮想壁にする。
3. BFS は `wallMask` を越えない範囲だけを `fillMask` にする。
4. 塗り残し対策として、`fillMask` を `underpaintPixels` 分だけ膨張する。
5. 膨張後の mask は、キャンバス外だけ避ける。線画レイヤーの下へ塗る用途では、線の下に少し入ってよい。

注意:

- gap close 用の wall 膨張と、underpaint 用の fill 膨張を同じ変数で混ぜない。
- 表示中レイヤー参照 ON/OFF の両方で確認する。
- 主線の外側へ明らかに漏れる場合は、underpaint を 1px に戻すか無効化して報告する。

## 完了条件

- 添付のような線内側の点線状塗り残しが、現状より明確に減る。
- 小さな隙間から外へ漏れるケースが、Phase 3f より悪化しない。
- 完全に閉じた線画で、塗り結果が不自然に太らない。
- Undo / Redo とサムネイル更新が正常。
- `npm.cmd run build` が成功する。
