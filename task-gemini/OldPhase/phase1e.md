# Phase 1e — 描画の基本品質・筆圧・軽量UI改善

> Phase 1d で「通常操作ログのスリム化・基盤整理・WebGPU/SDF/MSDF凍結方針の確認」まで完了扱い。
> Phase 1e は、制作中に最も目につくペン品質、筆圧、補正、軽量UIを整える。

---

## 作業対象

```text
tegaki_work/
```

`PastFiles/` と `tegaki_phase0/` は参照のみ。通常作業では編集しない。

---

## 前提

- 現在の標準描画は PixiJS v8.17.0 + RenderTexture へのライブラスター焼き込み。
- WebGPU / SDF / MSDF / WebGL2 Mesh 経路は凍結継続。Phase 1e の初手で復活させない。
- Canvas2D を本番描画経路へ混ぜない。
- DPR は 1 倍固定。内部 2 倍化や `resolution: 2` は採用しない。
- ペン・消しゴム・サムネイル・基本バケツは既に動作しているため、大きな描画方式変更で壊さない。
- 保存、エクスポート、レイヤー複製/結合/フォルダ、範囲選択は Phase 1f へ送る。
- 高機能バケツ、エアブラシ、ぼかし、図形、定規は Phase 2 へ送る。
- アニメ、動画、スクリーントーン、集中線などは低優先の将来フェーズへ送る。
- `dist/` は成果物として必要ない限り差分に残さない。

---

## 目的

1. 遅いストロークが丸の連続に見える問題を改善する。
2. 筆圧入力を実描画へ反映し、ON/OFF を切り替えられるようにする。
3. ペン補正設定を実描画へ反映する。
4. 四角ペンを追加する。
5. 設定ポップアップをタブ化し、増える項目を整理する。
6. V キー変形パネルをスリム化し、操作部品以外を掴んで移動できるようにする。
7. 左サイドバーと Q クイックパネルの視覚圧迫を下げる。

## 優先タスク

### 1. 事前棚卸し

対象候補：

- `system/drawing/brush-core.js`
- `system/drawing/stroke-renderer.js`
- `system/drawing/stroke-recorder.js`
- `system/drawing/pressure-handler.js`
- `system/drawing/pointer-handler.js`
- `system/brush-settings.js`
- `system/settings-manager.js`
- `ui/settings-popup.js`
- `ui/quick-access-popup.js`
- `ui/keyboard-handler.js`
- `system/layer-transform.js`

確認すること：

- PointerEvent の `pressure` がどこで受け取られ、どこで捨てられているか。
- `pressure-handler.js` が現在インスタンス化されているか。
- `settings-manager.js` / `settings-popup.js` の筆圧・補正設定が描画へ届いているか。
- `stroke-renderer.js` のライブラスター線分が筆圧・補正を使っているか。
- 筆圧 ON/OFF ショートカットは `Shift+P` を第一候補にする。既存割り当てと衝突が見つかった場合だけ Codex 判断へ戻す。

作業後、棚卸し結果を `tegaki_work/PROGRESS.md` に書く。

### 2. 筆圧入力の反映

方針：

- 既存の PointerEvent `pressure` を優先する。
- マウス入力では固定圧で描ける状態を維持する。
- 筆圧 OFF のときは、従来に近い一定幅で描けるようにする。
- 筆圧 ON のときは、0 荷重付近が完全な 0 で消えないようにする。最初は 0.01 から 0.1 程度の細い点として始まり、リニアに立ち上がる挙動を目指す。
- 筆圧 ON/OFF のショートカットは `Shift+P` とする。
- ステータスバーのツール表示は、現状の `ベクターペン` をやめる。通常時は `ペン（固定幅）`、筆圧 ON 時は `ペン（筆圧ON）` を第一候補にする。
- 余裕があれば、左サイドバーのペンアイコンへ小さい筆圧状態バッジを付ける。ただしペンツール選択状態の背景色と混同しない見た目にする。
- `window.*` の singleton を増やす場合は、既存名を `rg` で確認して重複させない。

注意：

- 筆圧カーブの高度な 2D グラフ編集は後続。
- 既存のストローク履歴形式を壊さない。
- 消しゴムの透明消去を壊さない。
- pressure 処理の責務が不明な場合は、実装を止めて Codex 判断へ戻す。
- `Ctrl+P` はブラウザ印刷と衝突するため使わない。
- 単体 `0` は仮置き候補だったが、正式初期案では使わない。

### 3. 遅いストロークの丸連続問題

症状：

- ゆっくり描いたとき、短い丸キャップ付き線分が重なり、丸い点の連続に見える。
- 半透明時は重なりが濃く見えやすい。

確認すること：

- ライブラスター線分の間隔が広すぎないか。
- 同じ点や極端に近い点の扱いが不自然でないか。
- 透明度 50% 前後で重なりが強調されないか。
- `Graphics.stroke()` の cap/join 指定や、短線分の焼き込み単位が原因になっていないか。

方針：

- まずは小さな補間、線分間隔、cap/join、同一点フィルタの調整で改善する。
- レンダリング方式そのものの置き換えは行わない。
- 半透明ストロークを一時バッファに描いてから合成するような大きい設計変更が必要なら、Codex 判断へ戻す。

受け入れ条件：

- 遅い斜め線が、丸いビーズ状ではなく連続線に見える。
- 透明度 50% のペンでも、点の重なりが過剰に目立たない。
- 速いストロークのレスポンスが悪化しない。

### 4. ペン補正設定の反映

方針：

- `settings-popup.js` / `settings-manager.js` に既にある補正系設定を確認し、既存値を描画側へ流す。
- 既存のデフォルトは弱い補正を維持する。
- 強い補正が常時かかって「後から線が曲がる」感触に戻さない。
- 実装位置は `brush-core.js` または `stroke-renderer.js` の既存責務に合わせる。

注意：

- 設定値を複数箇所へ直書きしない。
- `perfect-freehand` のオプションを変更する場合は、ライブ描画と確定後表示の差を増やさない。

### 5. 四角ペン

方針：

- 可能なら `pen` / `eraser` / `fill` のツール種別を増やすのではなく、ペンの形状設定として `round` / `square` を追加する。
- UI は最小でよい。設定ポップアップまたはクイックパネルから選べる形にする。
- 消しゴム形状と混ぜる場合は、挙動を明記する。

注意：

- ツール mode とブラシ shape を混同しない。
- 四角ペンのために描画パイプラインを大きく分岐させない。

### 6. 設定ポップアップのタブ整理

候補タブ：

- 描画
- 表示
- ショートカット

方針：

- 既存項目を消さず、増える項目を見つけやすく整理する。
- ショートカット変更 UI が重い場合は、まず一覧表示と初期化ボタンの置き場だけでよい。
- タブ切替はシンプルな DOM 状態管理でよい。

### 7. Vキー変形パネルとクイックUI軽量化

Vキー変形パネル：

- 操作部品以外の領域を掴んで移動できるようにする。
- 上部の余白や説明的な領域を減らし、キャンバス上の圧迫を下げる。
- 変形 bake や履歴の設計変更は Phase 1e の必須ではない。

左サイドバー / Q クイックパネル：

- 常時表示 UI の主張を下げる。
- 色プリセット、ペン/消しゴムサイズプリセットを小さく使いやすくする。
- カラーサークルや複数タブの本格拡張は Phase 2 でよい。

## Codex 判断へ戻す項目

以下に当たる場合は、Gemini 側で無理に実装せず `PROGRESS.md` に原因と案を書いて止める。

- RenderTexture 焼き込み方式そのものの変更が必要。
- 半透明ストロークを一時バッファで合成する設計が必要。
- EventBus payload や履歴形式の変更が必要。
- `window.*` singleton を新設しないと進めないが、既存責務との重複が判断できない。
- ショートカットの衝突が解決できない。
- WebGPU / SDF / MSDF / WebGL2 Mesh を復活させたくなる。
- `resolution: 2` や内部 2 倍化で解決したくなる。

---

## 禁止

- WebGPU / SDF / MSDF / WebGL2 Mesh 経路の安易な復活。
- Canvas2D を本番描画へ混入すること。
- 内部 2 倍解像度化、`resolution: 2`。
- `BlendMode` named import。
- `toLocal()` / `toGlobal()` による描画座標変換。
- ペンと消しゴムの別エンジン化。
- 保存・エクスポート・アニメ・動画・漫画効果を Phase 1e に混ぜること。
- `dist/` を不要に差分へ残すこと。
- `PastFiles/` を通常作業で編集すること。
