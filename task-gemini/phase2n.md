# Phase 2n — アルバム popup のビジュアル・配色統一

> Phase 2m で機能面とアイコンのメタファーは整備完了。
> Phase 2n は、アルバム popup の外観（フチ、透過背景、配色、シャドウ等）を、リサイズパネルや画像出力パネルなどの既存 UI 基準に合わせ、アプリ全体としての統一感を出すためのフェーズ。

---

## 背景・目的

- アルバム popup が現状、暫定的なスタイル（不透明な白背景に近いなど）になっている。
- 他のポップアップパネル（リサイズ、画像出力、設定等）は、共通の「ふたば系配色」や「ほどよい透過感」を持っている。
- これらを合わせることで、アルバムを「浮いた機能」から「アプリの標準的な一機能」へとビジュアル的に統合する。

---

## 実装内容（Gemini主体）

1. **基本スタイルの適用**
   - `.popup-panel` クラスの共通定義（`main.css`）を再確認し、インラインスタイルでの上書きを最小限にする。
   - 背景色（透過度含む）、ボーダー（ふたば色）、角丸、ドロップシャドウを統一。

2. **配色の整理**
   - タイトル部分の配色（`--futaba-maroon` 等）やマージンの統一。
   - ツールバーの背景やグループ分けの視認性向上。

3. **ギャラリーエリアの調整**
   - カードが並ぶ背景エリアの透過感や、スクロールバーの感触を他のパネルと合わせる。

---

## 完了条件

- アルバム popup を開いた時の「見た目の質感」が、画像出力パネル等と遜色ない。
- インラインスタイルでガチガチに固めるのではなく、可能な限り `main.css` の共通クラスを活用している。
- `npm.cmd run build` が成功している。

---

## Codex 判断へ戻すもの

- `main.css` の基幹クラス（`.popup-panel` 等）自体を大幅に破壊変更したくなった場合。
- DOM 構造の大幅な変更が必要になった場合。
