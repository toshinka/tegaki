# Phase 4l — アニメテーブルの最終モデル整理と移行計画 (GEMINI報告)

## 1. 現状の「Track = Layer」依存箇所の棚卸し

現在の実装は、既存の `LayerSystem` をアニメーションの土台として再利用するための「暫定接続」状態にあります。以下の箇所が `Track = Layer` 前提になっています。

### 📄 system/animation/animation-data-model.js
- **`CelModel.layerId`**: 各セルがどの実レイヤーに属するかを保持。
- **`TrackModel.layerId`**: トラック1行が実レイヤー1枚に直結。
- **`TimelineModel.syncWithLayers()`**: `LayerSystem` の状態（順序・名前・アクティブ状態）を走査し、`tracks` 配列を破壊的に再構築（またはID維持して更新）する。タイムラインの「正本」が `LayerSystem` にある状態。

### 🎨 ui/animation-table-popup.js
- **`render()` 内の同期**: 描画のたびに `syncWithLayers()` を呼び出し、レイヤーパネルの状態を常にタイムラインへ強制同期している。
- **`_applyVisibilityPreview()`**: レイヤー一覧をループし、対応するトラックのセルの有無で `layer.visible` を直接操作。描画内容（Snapshot）も実レイヤーへ直接書き戻している。
- **`captureSelectedCel()`**: トラックの `layerId` をキーに実レイヤーを探し、キャプチャを実行している。

## 2. 将来モデルの定義（オーナーイメージの言語化）

オーナーの「101匹ワンちゃん」構想や「独立描画コンテナ」を実現するため、以下の4階層にモデルを分離・定義します。

### ① Lane (レーン) — 旧 Track
- **役割**: タイムライン上の水平方向の「行」。
- **性質**: 表示の重なり順（Z-Index）を管理する論理的な階層。
- **脱・レイヤー依存**: 将来的には `Lane.layerId` ではなく、独自の `name` と `order` を持ち、複数の ClipInstance を保持するだけの器となる。

### ② ClipInstance (クリップ・インスタンス) — 旧 Cel
- **役割**: タイムライン上の「時間範囲」に配置される実体。
- **性質**: `startFrame`, `duration` を持ち、特定の **ClipAsset** を参照（インスタンス化）する。
- **重要点**: 1つの Lane 上に、異なる ClipAsset を参照するインスタンスを複数並べられる。

### ③ ClipAsset / ClipDocument (クリップ・アセット/ドキュメント)
- **役割**: 描画内容や設定の「正本」。ライブラリに保存される素材。
- **性質**: 独自の内部レイヤー構造（Internal LayerSystem）、物理演算パラメータ、ボーン設定などを持つ独立した描画単位。
- **再利用性**: 1つの ClipAsset を、異なる Lane や異なる時間に何度も配置できる（コピーではなくインスタンス参照）。

### ④ DrawingSnapshot (ドローイング・スナップショット)
- **役割**: 描画データの最小保存単位。
- **性質**: 現在の `rasterSnapshot` (RenderTexture) に相当。ClipAsset 内の特定の状態や、シンプル描画モード時の内容を保持。

## 3. 段階的移行ロードマップ案

### 案A：アセット・ライブラリ先行導入（ボトムアップ）
1.  **Phase 4m**: `ClipAsset` クラスを新設。現在の `Cel.rasterSnapshot` を `ClipAsset` へ逃がし、`Cel` は ID でアセットを参照する形に変える。
2.  **Phase 4n**: キャンバスに「アニメ専用の描画バッファ」を用意し、プレビュー時に実レイヤーを汚染せずに表示できる仕組みを作る（Pixi Containerの活用）。
3.  **Phase 4o**: `syncWithLayers` を「初回インポート用」へ格下げし、タイムライン独自の `Lane` 管理を開始。

### 案B：仮想レイヤー・コンテナ化（トップダウン / 推奨）
1.  **Phase 4m**: `TrackModel` を `LaneModel` へ改称し、`layerId` を「任意（null可）」にする。
2.  **Phase 4n**: セルをクリックして「内部編集モード」に入ると、メインのレイヤーパネルが「そのセルの内部レイヤー」を表示するように切り替わる（入れ子構造）。
3.  **Phase 4o**: タイムライン自体を一つの「ドキュメント」として保存できるようにし、`LayerSystem` からの完全独立を果たす。

## 5. 結論と次フェーズの提案

**次フェーズ (Phase 4m) の推奨内容:**
「実レイヤーを汚染しないプレビュー表示の試作」
現在は実レイヤーの `visible` やテクスチャを上書きしていますが、これを Pixi.js の Container を用いて「タイムライン専用のプレビューレイヤー（表示用のみ）」を作成し、実レイヤー（編集用）と分離する実験を行うべきです。
これにより、アニメ編集をしながら「下書きレイヤー（タイムライン外）」を常に表示しておく、といった高度な操作が可能になります。
