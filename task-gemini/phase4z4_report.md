# Phase 4z4 — Asset Library Skeleton MVP (GEMINI報告)

## 1. 実施内容

### 🏛️ アセットライブラリの基盤 UI 実装
- **`ui/animation-table-popup.js`**
    - **ライブラリパネルの構築**: タイムライン下部に展開する `anim-asset-library` セクションを新設しました。
    - **ASSETS トグルボタン**: ヘッダー右側にライブラリの表示/非表示を切り替えるボタンを追加しました。
    - **2カラムレイアウト**: 左側に「FOLDERS（フォルダ一覧）」、右側に「ASSETS（アセット一覧）」を配置する標準的なアセット管理の構成を導入しました。

### 📂 データとの連動とレンダリング
- **フォルダ一覧の描画**:
    - `Uncategorized`（未分類）および、前フェーズで基盤を作った `ClipAssetFolderModel` の一覧を生成。
    - クリックによるフォルダ切り替え機能を実装。
- **アセット一覧の描画**:
    - 選択中のフォルダに属するアセットを動的にフィルタリングして表示。
    - 各アセットに **名前・Blank状態（空）・参照数（Refs）** を表示するようにしました。
- **視覚的フィードバック**:
    - タイムライン上で現在選択しているクリップが使用しているアセットには、左端に **青い強調線 (current)** を表示し、ライブラリ内のどこにあるか一目でわかるようにしました。
    - ライブラリ内での個別選択（強調表示）も可能。

### 🎨 スタイリング
- 全体的にダークトーンかつ半透明なデザインを適用し、既存のタイムラインUIと調和させました。
- 縦スクロールに対応し、アセット数が増えてもパネルサイズが崩れないよう調整。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **動作確認**:
    1. `ASSETS` ボタンでパネルが開閉することを確認。
    2. タイムラインでクリップを選択した際、ライブラリ側の該当アセットに `current` 表示が出ることを確認。
    3. `Uncategorized` から各フォルダへの切り替えと、アセットリストの更新を確認。

## 3. 次フェーズへの展望 (以降)

1.  **アセットの再利用 (Assign)**: ライブラリからタイムライン上の空スロットへドラッグ＆ドロップ、または選択して割り当て。
2.  **フォルダ操作**: UI上でのフォルダ新規作成・リネーム・削除。
3.  **アセット整理**: アセットをドラッグしてフォルダへ移動。
4.  **アセットプレビュー**: 選択中のアセットのサムネイル（画像）を表示。
5.  **Clip内部レイヤー**: アセットが「複数のレイヤー」を持てるようにするためのデータ構造拡張。
