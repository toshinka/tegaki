# Phase 4x — Clip Edit Mode MVP (GEMINI報告)

## 1. 実施内容

### ✍️ クリップ編集モード (EDIT Mode) の実装
- **`ui/animation-table-popup.js`**
    - `isClipEditModeActive` 状態を導入し、ヘッダーに `EDIT` トグルを設置しました。
    - **動作仕様**:
        - `EDIT` が ON の間は、アニメーションの合成プレビュー（全セル表示）を一時停止し、現在選択されているセルの「ソース実レイヤー」をキャンバス上に復元して直接描画・編集できるようにしました。
        - これにより、プレビューで動きを確認しながら、特定のセルを修正するワークフローがスムーズになりました。
        - 編集モード終了時、またはフレーム移動時には、自動的に合成プレビュー状態へ安全に復帰します。

### 🤖 AUTO + EDIT の強力な連携
- Phase 4w の自動キャプチャ機能と組み合わせることで、以下のフローが可能になりました。
    1. セルを選択して `EDIT` を ON にする。
    2. 実レイヤー上で自由に描画する。
    3. `AUTO` が ON であれば、ペンを離した瞬間にその描画内容が `ClipAsset` へ自動反映（上書き）される。
    4. `EDIT` を OFF にすれば、即座に最新の絵を含んだ全体合成アニメーションが再生・確認できる。

### 🎨 視覚的フィードバックと UI 制御
- **編集中の明示**: 編集中のセルは水色の発光エフェクト (`.editing`) で強調表示し、パネルヘッダーの色も変化させることで、現在「実レイヤーを直接触っている」状態であることを視覚的に強調しました。
- **排他制御**: 再生中 (`isPlaying`) は強制的に EDIT モードを解除（または開始を抑制）し、プレビューの整合性を優先するようにしました。

## 2. 検証結果

### ✅ 動作確認
- **モード切替**: `EDIT` トグルにより、合成表示と実レイヤー表示が正しく切り替わることを確認。
- **自動保存連携**: `EDIT` モード中に描画した内容が、`AUTO` 設定に従って正しくセル（アセット）へ反映されることを確認。
- **安全な復元**: `EDIT` モードを抜けた際、プレビュー用の表示レイヤー（Container）と実レイヤーの `visible` 状態が矛盾なく復元されることを確認。
- **退行確認**: クリップのドラッグ移動、リタイミング、削除、オニオンスキン等の既存機能が EDIT モードと競合せず正常に動作することを確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **Virtual Layer Panel**: 編集モード中もレイヤーパネルは「全体のレイヤー一覧」を表示し続けています。将来的には、クリップ内部のレイヤーのみを表示する仮想パネル化が望まれます。
- **キーボードショートカット**: `E` キー等での EDIT モードトグルは、既存ショートカットとの競合調査が必要なため今回は見送り、チェックボックス操作に限定しました。

## 3. 次フェーズへの提案 (Phase 4y)

1. **複数セル選択**: タイムライン上での矩形選択、または Shift/Ctrl クリックによる複数セルの同時操作（移動・削除・一括キャプチャ等）。
2. **アセットのユニーク化**: 共有アセットを参照しているセルを、独立した別個のアセットとして切り離す機能。
3. **空アセットの自動生成**: セルを作成（クリック）した瞬間に、その場で空のアセットを自動割り当てする設定。
