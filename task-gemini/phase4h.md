# Phase 4h — アニメテーブル操作面の安定化MVP

## 概要

Phase 4g で、左サイドバーのアニメアイコンから新アニメテーブルを開く導線ができた。
ただし現状は、実描画連動へ進む前にテーブル自体の操作ストレスが残っている。

Phase 4h では、セルに応じたキャンバス表示制御や保存/Exportへ進まず、アニメテーブルを「開いたまま使える」「位置を動かせる」「スクロールしてもトラック名とグリッド行がズレない」状態へ整える。

## 重要な再発防止ルール

過去に `timeline-ui.js` と `animation-table-popup.js` のテンプレート文字列・CSS注入ブロック周辺で `Expected a semicolon` 系のビルドエラーが複数回発生している。

- `innerHTML = \`` や `style.textContent = \`` を編集したら、編集箇所の前後50行を必ず読み返す。
- `timeline-ui.js` は原則触らない。
- `animation-table-popup.js` のテンプレート文字列を大きく組み替えない。
- JS側の大量 inline style は追加しない。見た目はCSSクラスで制御する。
- D&D本格改修、セルドラッグ、レイヤーパネルD&D再設計はこのPhaseに混ぜない。

## 現在の前提

- `AnimationTablePopup` は `popupManager` 管理下で開閉される。
- `ui-panels.js` 側で、外クリック時に `animationTable` は閉じないよう Codex が先に小修正済み。
- 現在のテーブルは、左の `.anim-track-list` と右の `.anim-timeline-grid-container` が別スクロール/別領域になっている。
- フォルダも `LayerSystem` からトラックとして同期されているが、フォルダ開閉状態や子表示の扱いは未整理。

## 目標

- アニメテーブルは、閉じるボタンまたはサイドバーのアニメアイコン/ショートカットで閉じる。パネル外クリックでは閉じない。
- パネル上部ヘッダーをドラッグして、フローティングパネルとして移動できる。
- トラック数が増えて縦スクロールしても、左のトラック名行と右のセル行が同じ縦位置で同期する。
- アクティブレイヤーの上下移動に連動してアクティブトラック表示が更新される現行挙動は維持する。
- 左右キーで現在フレームを1コマ移動できる。ただし入力欄編集中や他のショートカットを壊さない。

## 実装範囲

### 1. 閉じ方の整理

- `animationTable` は外クリックで閉じない。
- 既に Codex が `ui-panels.js` の外クリック除外リストへ `animationTable` を追加済みなので、まず現状を確認する。
- 閉じる操作は以下に限定する。
  - アニメテーブル右上の `×`
  - 左サイドバーのアニメアイコンの再クリック
  - 必要なら既存ショートカットの統一。ただし大きなキーボード設計変更はしない。

### 2. パネルのドラッグ移動

- `.anim-table-header` をドラッグハンドルにする。
- パネルの移動は `left/top` または `transform` のどちらか安全な方法でよい。
- 画面外へ完全に出ないよう、最低限のクランプを入れる。
- ドラッグ中にセルクリック・フレームクリックが誤発火しないようにする。
- 位置保存は今回は必須ではない。必要なら後続に回す。

### 3. 縦スクロール同期

- トラック名とグリッド行が別々にズレないようにする。
- 推奨は、縦スクロールを1つのコンテナへ集約し、左列と右列を同じ行DOMまたは同じスクロールコンテキストで管理すること。
- 大きなDOM再構成が必要な場合は、まず小さく同期イベント方式で止血してよい。
- 行高は `32px` など固定値で統一し、トラック名側とセル側でズレないようにする。

### 4. 左右キーでフレーム移動

- アニメテーブルが表示中の時だけ、左右キーで `currentFrame` を移動する。
- `ArrowLeft`: 1フレーム戻る。
- `ArrowRight`: 1フレーム進む。
- 0未満や最終フレーム超過はクランプする。ループ移動は今回は必須ではない。
- `input`, `textarea`, `select`, `[contenteditable]` などの編集中は処理しない。

### 5. フォルダトラックの扱い

- このPhaseでは、フォルダトラックを削除しない。
- フォルダは「グループ見出し」として残す方針でよい。
- ただしセル追加対象として扱うかはまだ保留。誤操作が多そうなら、フォルダ行ではセル追加を無効化し、見出し風にする。
- フォルダ開閉状態に応じた子トラック表示/非表示は後続でよい。

## 禁止

- セル配置に応じて実レイヤーの visible を切り替えない。
- RenderTexture snapshot、保存/ロード、Export、Albumへ触らない。
- 旧 `animation-system.js` や `animationSystem.init()` を復活させない。
- レイヤーパネルD&Dの根本改修を混ぜない。
- セルドラッグ移動、セル端ドラッグ伸縮、範囲選択へ踏み込まない。
- `timeline-ui.js` を廃止・大改造しない。
