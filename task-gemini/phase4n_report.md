# Phase 4n — Frame Composite Preview 正本化MVP (GEMINI報告)

## 1. 実施内容

### 🎞️ Frame Composite Preview (全体合成プレビュー) の正本化
- **`ui/animation-table-popup.js`**
    - `_applyVisibilityPreview()` の設計を「選択セル単独の確認」から「現在フレームに存在する全セルの合成表示」へ完全に切り替えました（Phase 4n A案を採用）。
    - プレビュー有効時、タイムラインで管理されている実レイヤーをすべて非表示 (`visible = false`) にし、代わりに `animationPreviewContainer` に全 Snapshot を表示順（下から上）に重ねて描画する仕組みを正本としました。

### 🛡️ 実レイヤー非破壊の徹底
- プレビュー中に実レイヤーの RenderTexture (`restoreLayerRasterSnapshot`) を一切触らない設計を維持。
- `layerData.visible` (永続データ) は変更せず、Pixi オブジェクトの `visible` のみを一時的に操作することで、パネルを閉じた際やプレビューOFF時の完全な復元を保証。

### 📦 テクスチャキャッシュの安全な管理
- Codex による補修を引き継ぎ、`WeakMap` (`_snapshotTextureCache`) を用いて Pixi Texture をキャッシュ。
- `CelModel` や `Snapshot` オブジェクト本体に Pixi 固有のオブジェクトを混ぜないことで、将来の保存/シリアライズにおける安全性を確保しました。

### 🖱️ 選択状態とプレビューの分離
- セルをクリックして選択状態を切り替えても、プレビュー表示は「現在フレームの全セル合成」のまま維持されるようにしました。
- これにより、再生中やフレーム移動中に「全体の動き」を確認しながら、特定のセルを編集・キャプチャするフローが安定しました。

## 2. 検証結果

### ✅ 動作確認
- **合成表示**: 複数のレイヤーに Snapshot をキャプチャし、同じフレームに並べた際、それらが正しく重なって表示されることを確認。
- **Snapshotなしセルの扱い (A案)**: 指示に基づき、Snapshot がないセルはプレビュー層に表示されず、実レイヤーも非表示となる（＝キャプチャ済みの絵だけがパラパラ見える）挙動を確認。
- **操作の退行**: セル追加、削除、長さ変更、キャプチャ操作が正常に動作し、即座に合成表示へ反映されることを確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **オニオンスキン**: 合成表示の背後に薄く前後フレームを表示する機能は次フェーズ以降。
- **編集中のリアルタイム反映**: プレビューONの状態で絵を描いても、`CAPTURE` を押すまでは合成表示へ反映されません（実レイヤーが隠れているため）。「描きながらリアルタイムにアニメーションが見える」状態への進化は今後の課題です。

## 3. 次フェーズへの提案 (Phase 4o)

1. **オニオンスキン基礎**: `animationPreviewContainer` を活用し、前後フレームの Snapshot を半透明（かつ色味を変えて）表示する機能の追加。
2. **自動キャプチャ (Auto-Key)**: 描画終了時に、選択中のセルへ Snapshot を自動的に書き込むオプションの実装。
3. **セル移動 (Drag & Drop)**: タイムライン上でのセルの位置変更。
