# Phase 3j — QAP色UI整理とカラーサークル受け皿

## 概要

Phase 3i でスポイト MVP は実用状態になった。
次は色操作を広げる前に、QAP の色まわりを小さく整理し、後続のカラーサークルやカラースロット管理を入れやすい受け皿を作る。

今回の主目的は「今のQAPを壊さず、色操作の視認性と拡張余地を増やす」こと。
カラーサークル本体を無理に完成させるフェーズではない。

## 優先順

### 1. 現状確認と退行防止

- [ ] `quick-access-popup.js` の COLOR / TOOL / SIZE / OPACITY の DOM と CSS を確認する。
- [ ] `brush:color-changed`、`brush:mode-changed`、`tool:changed` の QAP 同期経路を確認する。
- [ ] スポイト、ペン、消しゴム、バケツ、消しバケツ、投げ縄塗りの切替が維持されることを確認する。

注意:

- Phase 3i 追補修で、スポイトは現在色へ反映し、スポイト横の丸に現在色を表示する状態になっている。
- スポイトで拾った色は現時点ではカラースロットへ自動保存しない。保存・入れ替え・初期化は後続の明示操作として扱う。

### 2. QAP 色ヘッダーの軽整理

- [ ] COLOR 見出し横の「カラースロット切替」「スポイト」「現在色丸」の並びを整える。
- [ ] スポイト選択時に TOOL 内の既存ツールボタンの active 枠が残る場合は、低リスクで外せるか確認する。
- [ ] スポイトのミニボタンに active 表示を出せるなら実装する。

方針:

- 無理に大きく直さない。
- TOOL 内の active 枠が残る問題は、簡単に直せるなら直す。難しければ大規模QAP整理時の残件としてメモする。
- 左サイドバーとQAPの表示を完全一致させるために、ツール管理全体へ大きく踏み込まない。

### 3. QAP スリム化の小実装

- [ ] SIZE / OPACITY セクションの上下余白を少し詰める。
- [ ] スライダー本体の高さやボタンサイズを、操作性を落とさない範囲で少し小型化する。
- [ ] TOOL ボタンとプリセットスロットの横幅を少し詰められるか確認する。
- [ ] COLOR スロットの小型化は、現在色丸とスポイトボタンの視認性を見ながら調整する。

方針:

- まず余白を詰める。いきなり全面再設計しない。
- `COLOR` / `TOOL` などの見出し削除は今回は原則しない。面積が足りない場合の後続候補にする。
- スマホより PC / 液タブでの視認性を優先する。

### 4. カラーサークル受け皿の設計だけ作る

- [ ] QAP 内に折りたたみ式カラーサークルを置く場合の表示位置を決める。
- [ ] カラースロットとタブ切替にする案、折りたたみで同時表示する案を比較する。
- [ ] 本命案を `NOTES.md` に短く残す。

候補:

- 第一候補は折りたたみ式。カラースロットとカラーサークルを同時に使えるため。
- QUICK 見出し付近、または COLOR ヘッダー付近に開閉アイコンを置く。
- 開閉アイコンは将来的に虹色リングが理想だが、当面は単色アイコンでもよい。

### 5. 後続へ回す項目

- カラースロットへのスポイト色登録ボタン。
- カラースロットの入れ替え、現在色の登録、初期値へ戻す操作。
- カラーサークル本体の高度な色空間、履歴色、HSV/RGB数値入力。
- QAP 全体のタブ化や大規模レイアウト変更。

## 確認事項

- [ ] `npm.cmd run build` が成功する。
- [ ] QAP を開いた状態で、現在色丸が色変更・スポイト取得に追従する。
- [ ] スポイト選択中に描画や消去が発生しない。
- [ ] ペン、消しゴム、バケツ、消しバケツ、投げ縄塗りへ戻した時に通常操作できる。
- [ ] QAP の縦幅が増えすぎない。

## 禁止事項

- WebGPU / SDF / MSDF / JFA 経路へ触れない。
- 保存形式、履歴形式、レイヤー構造を変更しない。
- カラーサークル本体を無理に完成させない。
- QAP のDOM全体を丸ごと置換しない。差分編集で進める。
