# Phase 2m — アルバム popup の誤ロード防止 / 選択・ロード導線整理

> Phase 2l で透明市松表示は完了。
> Phase 2m は、アルバムカードを触っただけで即ロードされる驚きをなくし、選択と明示ロードの導線へ整理するための小フェーズ。

---

## 背景

オーナーから以下の追加メモがある。

- アルバムは、触っただけでいきなりロードされると驚く。
- カードクリックは選択にしたい。
- 1つだけ選択されている時のみ、ロードボタンで復帰できるようにしたい。
- 既存の選択モードボタンは不要になる可能性がある。
- 現在の状態を保存、ロードなどは SVG アイコン化候補がある。
  - 保存候補: `square-library`
  - ロード候補: `monitor-up`

既存のアルバムは Phase 1l〜1m で HTML保存/読み込み、カード選択、一括削除/一括書き出しの土台がある。
今回は保存形式や IndexedDB / localStorage の正本変更ではなく、誤ロードしにくい操作導線へ寄せる。

---

## 目的

- アルバムカードのクリック動作を棚卸しする。
- 即ロードされる経路と、選択だけの経路を明確に分ける。
- 「選択中1件だけロード可能」という UI 条件を整理する。
- 既存の一括削除、一括書き出し、HTML保存/読み込みを壊さない。
- 大きなアルバム popup 再設計や SortableJS の根本刷新は行わない。

---

## Gemini に任せる範囲

Gemini は初回、棚卸しと記録を優先する。

- 関係ファイルの読み取り。
- `rg` による album / snapshot / selected / load / restore / import / export 経路の検索。
- `PROGRESS.md` への調査結果追記。
- 低リスクな CSS やアイコン候補の整理。

Gemini は初回で、カードクリック挙動や保存形式を勝手に変更しない。
実装が必要な場合は、変更対象関数と変更方針を `PROGRESS.md` に整理し、Codex / オーナー判断へ戻す。

## 調査観点

1. **カードクリック**
   - 現在カードクリックで即ロードしている関数名と呼び出し元。
   - 選択モード時と通常時の分岐。
   - ダブルクリックやボタン操作が既にあるか。

2. **選択状態**
   - 選択中カードの ID 管理方法。
   - 複数選択時の一括削除 / 一括書き出しの経路。
   - popup を閉じた時の選択解除処理。

3. **ロード導線**
   - 1件選択時だけロードボタンを有効化できるか。
   - 0件 / 複数件選択時のボタン disabled 表示。
   - ロード実行時に確認ダイアログが必要かどうか。

4. **アイコン候補**
   - `square-library` を「現在の状態を保存」に使えるか。
   - `monitor-up` を「選択中アルバムをロード」に使えるか。
   - 既存 `ui-icons.js` に近いアイコンがあるか。

5. **触らないもの**
   - localStorage key `tegaki_album`。
   - アルバムHTML保存/読み込み形式。
   - SortableJS のドラッグ感触。
   - Windows風ドラッグ追従の独自実装。
