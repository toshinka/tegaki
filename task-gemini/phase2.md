# Phase 2 — レイヤー/UI編集基盤

> Phase 1m で描画、履歴、保存、アルバムHTML入出力の基礎復旧は完了扱い。
> Phase 2 は、レイヤーパネルとレイヤー操作を「AIが読みやすく、ユーザーが多数レイヤーでも扱いやすい」形へ整える。

---

## 作業対象

```text
tegaki_work/
```

`PastFiles/` と `tegaki_phase0/` は参照のみ。通常作業では編集しない。
`dist/` は成果物として必要ない限り差分に残さない。

---

## 前提

- 標準描画は PixiJS v8.17.0 + RenderTexture へのライブラスター焼き込み。
- Phase 2 では描画エンジン、履歴、保存形式、アルバム形式を不用意に変えない。
- WebGPU / SDF / MSDF / WebGL2 Mesh 経路は凍結継続。
- `animationSystem.init()` は有効化しない。
- アニメ、物理演算、動画ツール風タイムラインは Phase 4 以降。
- 範囲選択、定規、エアブラシ、囲い塗りなどの描画補助は Phase 3。

---

## Phase 2 の目的

1. レイヤーパネルを多数レイヤーでも扱いやすい密度へ近づける。
2. レイヤー複製、結合、クリッピング、フォルダの仕様と実装順を整理する。
3. 複数レイヤー選択と一括操作の土台を作る。
4. レイヤーUIのアイコン、閉じる/削除ボタン、透明度操作を共通設計に寄せる。
5. AIが読みやすい責務境界を維持し、巨大な一括改修を避ける。

## 初回でやること

### 1. 棚卸し

必ず確認する：

- `tegaki_work/system/layer-system.js`
- `tegaki_work/ui/layer-panel-renderer.js`
- `tegaki_work/system/layer-transform.js`
- `tegaki_work/system/drawing/thumbnail-system.js`
- `tegaki_work/ui/ui-icons.js`
- `tegaki_work/styles/main.css`
- `tegaki_work/ARCHITECTURE.md`

確認観点：

- レイヤー作成、削除、複製、結合、アクティブ切替の責務。
- レイヤーパネル描画とイベント処理の責務。
- クリッピング/フォルダ/複数選択を入れる場合の自然な配置。
- 既存アイコン/CSSを再利用できるか。
- すでにある未接続機能や二重実装がないか。

### 2. 最初の小実装候補

Geminiが担当しやすい候補：

- レイヤーパネルの表示密度調整。
- 削除/閉じるボタンの見た目統一の残り確認。
- レイヤー透明度UIの軽い改善。
- レイヤー複製/結合ボタンの配置整理。
- 複数レイヤー選択の見た目だけの土台。

Codex判断へ戻す候補：

- クリッピングのデータ構造。
- レイヤーフォルダの内部表現。
- 複数レイヤーの一括移動/一括結合の仕様。
- レイヤーパネルを ToonSquid 風に大きく再設計する判断。

---

## ドラッグ感触について

アルバムの並べ替えは Phase 1m で動作完了。
ただし、Windows のファイル移動のような「薄いカードがホバリングして等速追従する」感触は SortableJS 調整だけでは未達。

Phase 2 でレイヤーパネルのドラッグも触る場合：

- まずは既存 SortableJS を維持する。
- どうしても感触を追い込む場合は、独自ドラッグUIとして Codex判断へ戻す。
- レイヤー順序と内部配列の同期を壊さないことを優先する。

---

## 完了条件

- `npm.cmd run build` が成功する。
- Console に致命的エラーがない。
- レイヤー作成/削除/複製/結合/サムネイル/アクティブ切替の既存挙動を壊さない。
- 変更内容が `PROGRESS.md` に記録される。
- 大きな仕様判断が必要なものは、実装せず Codex/オーナー判断へ戻す。

---

## Codex判断へ戻す条件

- レイヤーの内部データ構造変更が必要になる。
- クリッピングやフォルダが保存形式・履歴形式に影響する。
- 複数レイヤー操作が `layer-system.js` と `layer-panel-renderer.js` の責務をまたいで複雑化する。
- SortableJS の限界を超えて独自ドラッグ実装が必要になる。
- ToonSquid 風UIへ大きく寄せるため、現行UIの配置を大幅変更したくなる。
