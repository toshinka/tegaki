# Phase 4i — アニメテーブル実描画プレビューMVP

## 概要

Phase 4h で、アニメテーブルは開いたまま操作でき、ドラッグ移動・縦スクロール同期・左右キーでのフレーム移動まで安定した。
Phase 4i では、現在フレームに配置されたセルに応じて、キャンバス上の実レイヤー表示を一時的に切り替える。

目的は「テーブル上のセル配置を再生/フレーム移動した時、キャンバスでもどのレイヤーが見えるか分かる」状態にすること。
ただし、このPhaseでは描画内容のスナップショット化、保存/ロード、Export は扱わない。

## 重要な再発防止ルール

- `animationSystem.init()` を復活させない。
- 旧 `animation-system.js` のフレーム保存/再生には接続しない。
- RenderTexture snapshot、保存/ロード、Export、Album はこのPhaseで触らない。
- 実レイヤーの本来の表示状態を破壊しない。プレビュー終了時に必ず元へ戻す。
- 背景レイヤーとフォルダは特殊扱いする。通常レイヤーと同じ visible 切替に巻き込まない。
- `animation-table-popup.js` のテンプレート文字列やCSS注入ブロックを編集したら、前後50行を読み返してからビルドする。
- Spaceキーショートカットは追加しない。Spaceは既存のパン操作と競合しやすい。

## 現在の前提

- `TimelineModel` は `tracks` と `playback.currentFrame` を持つ。
- `TrackModel.getCelAtFrame(frameIndex)` で、そのフレームにセルがあるか判定できる。
- `TrackModel.layerId` は `LayerSystem` の `layerData.id` と同期している。
- フォルダトラックは Phase 4h でセル配置対象から外れている。
- アニメテーブルを閉じる時は `AnimationTablePopup.hide()` が呼ばれる。

## 目標

- アニメテーブル表示中、現在フレームにセルがある通常レイヤーだけをキャンバス上で表示する。
- セルがない通常レイヤーは、プレビュー中だけ非表示にする。
- 背景レイヤーは原則そのまま表示する。
- フォルダは表示制御対象にしない。フォルダ配下の通常レイヤー単位で制御する。
- アニメテーブルを閉じる、またはプレビュー解除した時、元のレイヤー visible 状態へ戻る。
- 再生ヘッドの移動、Play再生、左右キー移動、フレームヘッダークリックに合わせて表示が更新される。

## 実装範囲

### 1. プレビュー状態の導入

- `AnimationTablePopup` に、プレビュー適用中かどうかを持たせる。
- 初回適用時に、各レイヤーの元の `visible` 状態を `layerId -> visible` で保存する。
- `hide()` / `stop()` のうち、少なくとも `hide()` では必ず元の visible を復元する。
- 可能なら、プレビュー切替の小さなトグルをヘッダーに置く。ただしUIが重くなる場合は「テーブル表示中は自動プレビュー」でよい。

### 2. 現在フレームに応じた表示制御

- `this.model.playback.currentFrame` を基準にする。
- 各通常レイヤーについて、対応する track に `getCelAtFrame(currentFrame)` があれば表示、なければ非表示。
- フォルダや背景は対象外。
- レイヤーの `layer.visible` と `layer.layerData.visible` の扱いに注意する。
  - 元状態復元のため、恒久設定としての `layerData.visible` を上書きしない方が安全。
  - 可能ならプレビュー表示は Pixi の `layer.visible` だけに限定し、目アイコンの状態は変えない。

### 3. 更新タイミング

- `render()` の最後、またはフレーム変更処理後にプレビュー表示を適用する。
- Play再生中も現在フレームが進むたびに表示が更新される。
- レイヤー追加/削除/並び替え時も、次回 `render()` / `requestUpdate()` で再同期する。

### 4. 復元

- `hide()` で必ず元の表示状態へ戻す。
- プレビュー適用中に元のレイヤーが削除されていてもエラーにしない。
- 復元後は `layer:panel-update-requested` を発行して、レイヤーパネルの目アイコンと状態がズレないようにする。

## 禁止

- RenderTexture の内容をコピー・固定・差し替えしない。
- セルごとに描画内容を保存しない。
- プロジェクト保存形式を変更しない。
- Export/Album へ接続しない。
- 旧タイムラインや旧 `animation-system.js` へ接続しない。
- セルドラッグ移動やリタイミングハンドルへ踏み込まない。
- レイヤーパネルD&Dを触らない。
