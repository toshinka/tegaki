# Phase 4z9 — ClipAsset Internal Layer Order MVP (GEMINI報告)

## 1. 実施内容

### ↕️ 内部レイヤーの順序入れ替え機能の実装
- **`system/animation/animation-data-model.js`**
    - `TimelineModel` に `moveClipAssetInternalLayer(assetId, layerId, direction)` を実装しました。
    - `direction` 引数により、配列内でのインデックスを増減させ、レイヤーの前後関係を操作します。
- **`ui/animation-table-popup.js`**
    - **Inspector UI の機能拡張**:
        - 各レイヤー行のアクションエリアに `▲` (前面/上へ) と `▼` (背面/下へ) ボタンを追加しました。
        - **境界制御**: 配列の先頭にあるレイヤーは `▲` が、末尾にあるレイヤーは `▼` がそれぞれ `disabled` (無効) になるよう制御しました。
        - **選択維持**: 順序変更後も移動したレイヤーが選択状態を維持するようにしました。
    - **スタイリング**:
        - 既存のアクションボタンと調和するよう、小さなフォントサイズ (9px) で配置しました。

### 🎨 順序の解釈と将来像
- 現状、Inspector上では「配列の先頭 (index 0) が一番上」に見えるように配置しています。
- これは一般的なペイントソフトのレイヤーパネルの慣習（上が前面）に合わせたものです。
- 将来のマルチレイヤー合成時には、この配列を逆順（末尾から描画開始）で処理することで、自然な重なりを実現する計画です。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **操作確認**:
    1. 内部レイヤーを複数作成し、中間のレイヤーで `▲` / `▼` をクリックして位置が変わることを確認。
    2. 先頭レイヤーの `▲` が押せないこと、末尾レイヤーの `▼` が押せないことを確認。
    3. 移動後もハイライト（選択状態）が追従することを確認。
