# Phase 2f — レイヤー複数選択の土台 / 選択操作整理

> Phase 2e で通常レイヤー同士のクリッピング本体を接続した。
> Phase 2f は、複数レイヤー操作へ進む前に「選択状態」と「名前変更導線」を整理する。

---

## 目的

- レイヤーパネルで複数レイヤーを選択できる土台を作る。
- 将来の一括結合、一括クリッピング、まとめて移動へ進める状態にする。
- ただし Phase 2f 初回では、一括結合や一括移動の本実装は行わない。

---

## 現状の注意点

- 現在は単一 active レイヤー中心の設計。
- `Shift+クリック` は名前変更に使っているが、オーナーメモでは将来的に `Shift+クリック` を複数選択へ使いたい。
- フォルダ D&D は SortableJS 継ぎ足しの暫定実装として凍結済み。Phase 2f では深追いしない。
- クリッピング本体は Phase 2e で通常レイヤー同士のみ実装済み。フォルダ合成結果、逆クリッピング、複数一括クリッピングは後続。

---

## 推奨する操作方針

- 通常クリック: 単一 active 化し、複数選択を解除。
- Ctrl/Cmd + クリック: 個別選択トグル。
- Shift + クリック: 最後に選択した行から範囲選択。
- 名前変更: `F2`、または小さな名前編集ボタン/メニューへ逃がす。`Shift+クリック` 依存は撤去候補。

---

## Codex 担当候補

1. `LayerSystem` に `selectedLayerIds` または同等の選択状態を追加する。
2. active レイヤーと selected レイヤーを区別する。
3. `layer-panel-renderer.js` に複数選択の見た目を追加する。
4. Ctrl/Cmd/Shift クリックの入力整理を行う。
5. 名前変更導線を `F2` などへ移す。

---

## Gemini に任せてもよい範囲

- 現状棚卸し。
- `PROGRESS.md` への整理記録。
- CSS の選択色や枠線など、50行以内の小さな見た目調整。
- Codex が指定した局所的なヘッダー同期。

Gemini は独自判断で `LayerSystem` の選択状態本体、一括結合、一括移動、D&D 再設計を実装しない。

---

## Phase 2f 初回の完了条件

- 単一 active と複数 selected の状態が混線しない。
- Ctrl/Cmd クリックで個別選択できる。
- Shift クリックで範囲選択できる。
- 通常クリックで単一 active に戻せる。
- 名前変更の導線が `Shift+クリック` 以外へ移り、複数選択と競合しない。
- `npm.cmd run build` が成功する。

---

## Phase 2f ではやらないこと

- 複数レイヤーの一括結合。
- 複数レイヤーの一括クリッピング。
- 複数レイヤーのまとめてドラッグ移動。
- SortableJS / 独自 D&D の再設計。
- フォルダ合成結果を使ったクリッピング。
