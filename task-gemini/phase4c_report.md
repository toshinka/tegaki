# Phase 4c — アニメテーブルのセル配置MVP (GEMINI報告)

## 1. 実施内容

### 🧠 データモデルの拡張 (In-Memory)
- **`system/animation/animation-data-model.js`**
    - `TrackModel` にセル管理用のヘルパーメソッドを追加：
        - `getCelAtFrame(frameIndex)`: 指定フレームのセルを取得。
        - `addCel(options)`: 新規セルを作成して追加。
        - `removeCelAtFrame(frameIndex)`: 指定フレームのセルを削除。
        - `toggleCelAtFrame(frameIndex)`: セルの有無を反転（クリック操作用）。
    - レイヤー同期 (`syncWithLayers`) 時も、`layerId` が一致する限り既存のトラックとセルを維持する仕組みを継続。

### 🖱️ UI インタラクションの実装
- **`ui/animation-table-popup.js`**
    - グリッドのスロット (`.anim-cell-slot`) に `data-track-id` と `data-frame-index` を付与。
    - タイムライングリッド全体にクリックイベントリスナーを設置し、デリゲート方式でクリックされたセルの `toggleCelAtFrame` を実行。
    - セルが存在する場合、スロット内に `.anim-cel-block` 要素をレンダリングし、視覚的に判別可能に。
    - アクティブなトラック上のセルは、より強調された色（オレンジ）で表示されるよう CSS を調整。

### 🎨 スタイル定義
- セル表示用の CSS クラスを追加：
    - `.anim-cel-block`: セルを表現する丸角の矩形。
    - `:hover` 時のフィードバックを追加し、クリック可能な領域であることを明示。
    - アクティブトラック内での配色変更。

## 2. 検証結果

### ✅ 動作確認
- **セル配置**: アニメテーブル上の任意の空きスロットをクリックするとセルが配置され、再度クリックすると消えることを確認。
- **レイヤー同期の維持**: レイヤーの名前を変更したり、順序を入れ替えても、配置したセルが消えずに正しいレイヤー（トラック）に紐付いたまま維持されることを確認。
- **レイヤー削除**: レイヤーを削除すると、対応するトラックとその上のセルが適切に消滅することを確認。
- **退行確認**: 通常描画、QAP、レイヤーパネルの操作に影響がないことを目視確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **セルの属性**: セルはまだ「存在するかどうか」のフラグのみであり、`duration`（長さ）の変更や、特定の描画内容（スナップショット）との紐付けは未実装。
- **ドラッグ操作**: セルのドラッグ移動や範囲選択は、本フェーズのスコープ外として未実装。
- **再生連動**: セルがある場所で再生が止まったり、表示が切り替わったりする機能は次フェーズ以降。

## 3. 次フェーズへの提案 (Phase 4d)

1. **再生ヘッドの自動駆動**: `animation-system.js` と接続し、再生中に現在フレームが自動更新されるようにする。
2. **セルの長さ (Duration) の導入**: セルを右にドラッグして長さを伸ばせる UI の検討。
3. **簡易な非表示化連動**: 現在フレームにセルがない場合、そのレイヤーを一時的に非表示にするなどの表示制御の試作。
