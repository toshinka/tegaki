# Phase 4n 以降メモ — アニメプレビュー範囲と編集対象の切り分け

> Phase 4n 以降でアニメテーブル、Lane、ClipInstance、ClipAsset、プレビュー、オニオンスキン、Solo/Mute、Export を触る場合は、実装前にこのメモを読むこと。

## 前提

Phase 4m で、キャプチャ済みセルの表示は実レイヤー直接復元ではなく、アニメプレビュー専用 Container へ出す方向になった。
次に Lane / ClipInstance / ClipAsset 化へ進む前に、プレビューが「何を表示するのか」を整理しておく。

## 推奨する基本方針

通常の再生・フレーム移動時の **PREVIEW は、そのフレームに存在する全クリップの合成結果** を表示する。

理由:

- ユーザーが「このフレームがどう見えるか」を確認するには、タイムライン上の全レーン合成が自然。
- 動画/アニメ制作ツールの再生ビューとしても、最終出力に近い合成表示が基本。
- 後で Export する時の考え方と一致しやすい。

一方で、**セル/クリップを編集する時は、そのセル/クリップ内部だけを編集対象にする**。

理由:

- クリックしたセルの中に独立レイヤーがある、という長期方針と合う。
- 「犬クリップを編集している間は犬の内部レイヤーを見る」「全体プレビューでは複数の犬や背景を合成して見る」という切り分けができる。
- そのレーンだけ再生、選択クリップだけ再生、ソロ/ミュートのUIを後から追加しやすい。

## 用語整理

- `Frame Composite Preview`
  - 現在フレームに存在する全レーン/全クリップの合成表示。
  - 再生ボタン、左右キー、フレームヘッダークリック時の標準表示。

- `Clip Edit View`
  - 選択セル/クリップの内部だけを編集する表示。
  - 将来、メインレイヤーパネルがそのクリップ内部レイヤーへ切り替わる候補。

- `Lane Solo Preview`
  - 指定レーンだけを再生・確認する表示。
  - 後続機能。今は実装しない。

- `Clip Solo Preview`
  - 指定クリップだけを再生・確認する表示。
  - 後続機能。今は実装しない。

## セルフォルダ案について

「そのフレームにあるものはレイヤーパネルでも全部表示され、セルフォルダのように扱う」案は便利そうだが、初期実装では危険。

理由:

- メインレイヤーパネルが通常ドキュメントのレイヤーなのか、現在フレーム合成の仮想レイヤーなのか曖昧になる。
- `Track = Layer` から抜けたい目的と逆方向に戻りやすい。
- レイヤーパネルD&Dやフォルダ開閉バグと絡みやすい。

採用するなら後で `Virtual Layer Panel` として明示的に設計する。
Phase 4n 直後では、レイヤーパネルは通常編集用のままにし、アニメ側はプレビューContainerで合成する方が安全。

## 次に実装するなら

1. まず `Frame Composite Preview` を正本にする。
   - 現在フレームの全Snapshotセルを、レーン順にプレビューContainerへ積む。
   - 将来のExportもこの合成順に寄せる。

2. その後に `Clip Edit View` を作る。
   - セルをダブルクリック、または Edit ボタンで内部編集へ入る。
   - この時だけ、レイヤーパネルをクリップ内部レイヤーへ切り替える候補を検討する。

3. さらに後で Solo / Mute を足す。
   - レーン単位の再生確認。
   - クリップ単位の再生確認。
   - オニオンスキン対象の限定。

## 現時点の判断

- 再生時: フレーム上の全クリップ合成を表示する。
- フレームヘッダークリック時: フレーム上の全クリップ合成を表示する。
- セルクリック時: そのセル/クリップを選択し、編集確認用に選択セル単独表示へ切り替える。
- セル編集モードに入った時: そのセル/クリップ内部だけを編集対象にする。
- レーン単独再生・セル単独再生は後続の Solo/Mute 機能として扱う。

## Phase 4n補足

Phase 4n 実機確認で、選択セルまで常に全体合成のままだと「犬」「猫」「花」などが混じって編集対象を見失いやすいことが分かった。

そのため、以下へ補正する。

- 再生中は `Frame Composite Preview`。
- フレームヘッダークリック/左右キー移動は選択解除して `Frame Composite Preview`。
- セルクリック中は `Clip Solo Preview` に近い暫定表示として、選択セルだけを表示する。
- Snapshot未取得の選択セルは、編集対象が見えるように対応する実レイヤーだけを暫定表示してよい。
- 正式な Solo/Mute UI は後続で設計する。

## Phase 4t補足

オニオンスキン実装後の実機確認で、タイムスケール上の Frame 番号を見ている時は、そのフレームに存在する複数クリップの合成結果を作業ビューでも把握したい、という要求が再確認された。

現状の実装では、プレビューContainer上では以下の切り分けになっている。

- フレームヘッダークリック/左右キー移動/再生時: `Frame Composite Preview` として、そのフレームの全Snapshotセルを合成表示する。
- セルクリック時: 編集対象を見失わないため、選択セル単独表示へ切り替える。

ただし、メインのレイヤーパネルはまだ通常ドキュメントの実レイヤーを表示しており、「Frame 1 にあるクリップ群をセルフォルダとしてレイヤーパネルへ展開する」機構は存在しない。

これは `Track = Layer` 暫定接続から、`Lane / ClipInstance / ClipAsset / Clip内部レイヤー` へ移行した後に扱うべき課題とする。

将来方針:

- タイムスケールの Frame 番号をクリックした時は、そのフレームに存在する各 ClipInstance を `Virtual Layer Panel` 上でフォルダ分け表示する候補を検討する。
- 個別クリップをクリックした時は、その ClipAsset 内部のレイヤー構成だけを編集対象として表示する。
- この切り替えはレイヤーパネルD&D、フォルダ開閉、Solo/Mute、オニオンスキン対象範囲と絡むため、レーン整備前には深追いしない。
