# Phase 4z21 — CAF Operation Authority Boundary (GEMINI報告)

## 1. 実施内容

### 🛡️ 操作権限の境界（Boundary）の定義と実装
- **設計方針の確立**: 「CAF/クリップの構造操作はアニメーションテーブルが正本、レイヤーパネルは状態の反映と内部編集に専念する」という権限境界を明確に定義しました。
- **`ui/animation-table-popup.js`**
    - **`_requestLayerPanelSync()` ヘルパーの実装**: レイヤーパネル側への更新通知 (`layer:panel-update-requested`) とインジケーター更新を一括で行うヘルパーを追加しました。
    - **同期ロジックの徹底**:
        - クリップの作成、削除、移動、伸縮。
        - アセットのユニーク化、キャプチャ。
        - アセットフォルダの作成、リネーム、移動。
        - 内部レイヤーの追加、削除、順序変更。
        - これら全ての構造変更操作の後に `_requestLayerPanelSync()` を呼び出すようにし、レイヤーパネルとの不整合を解消しました。
    - **パフォーマンス最適化**: `render()` メソッドの末尾でも同期を行いますが、再生中 (`isPlaying`) は通知をスキップすることで、再生パフォーマンスへの影響を最小限に抑えました。
    - **イベント通知の補強**: 再生中やキーボードナビゲーションによるフレーム移動時、`animation:frame-changed` イベントを確実に発火するようにし、レイヤーパネル側のフレームインジケーターや CAF ヘッダーがリアルタイムに追従するようにしました。

### 🎨 用語と UI の整合
- **用語の統一**: レイヤーパネル側の CAF ヘッダーやミラーにおいて、「CAF」「CLIP LAYERS」といったアニメーションテーブル側と共通の用語が使用されていることを確認・維持しました。
- **権限の分離**: レイヤーパネル側には CAF 自体の追加・削除・フォルダ移動といった「構造操作」の UI が存在しない（読み取りと内部編集のみである）ことを確認しました。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **動作確認**:
    1. タイムラインでクリップを作成・削除した際、レイヤーパネル上部の CAF リストが即座に更新されることを確認。
    2. アセットを別フォルダへ移動した際、パネル上のグループ表示が正しく切り替わることを確認。
    3. 再生中もレイヤーパネルの赤線（フレームインジケーター）が滑らかに動き、停止後には CAF ヘッダーが最新状態に同期することを確認。
    4. 内部レイヤーの可視性や名前を変更した際、インスペクターとミラーの両方が常に一致することを確認。

## 3. 次フェーズへの展望 (以降)

1.  **Virtual Layer Panel (完全統合)**: 現在の「ミラー表示」をさらに進め、選択中の内部レイヤーをメインのレイヤーパネルの「実体」として扱い、直接ペン入れ・編集を可能にする接続。
2.  **アセットサムネイルの高度化**: キャプチャ時や編集時にサムネイルを自動更新し、レイヤーパネル上での視認性をさらに向上させる。
3.  **複数選択への対応**: タイムラインでの複数クリップ選択を、レイヤーパネル側のハイライト表示にも反映させる。
