# Phase 4z21 — CAF Operation Authority Boundary (GEMINI報告)

## 1. 実施内容

### 操作権限の境界（Boundary）の定義と実装
- **設計方針の確立**: 「CAF/クリップの構造操作はアニメーションテーブルが正本、レイヤーパネルは状態の反映と内部編集に専念する」という権限境界を明確に定義・実装しました。
- **`ui/animation-table-popup.js`**
    - **`_requestLayerPanelSync()` ヘルパーの実装**: レイヤーパネル側への更新通知 (`layer:panel-update-requested`) とインジケーター更新を一括で行うヘルパーを追加しました。
    - **同期ロジックの徹底**:
        - 構造変更を伴う個別操作の完了時に `_requestLayerPanelSync()` を呼び出すようにしました。
        - `render()` メソッド末尾での同期通知は、`render -> layer:panel-update-requested -> requestUpdate -> render` の更新ループになり得るため、Codex補修により使用しない方針を維持しています。
        - **パフォーマンス最適化**: 再生中の連続renderへ同期通知を混ぜず、構造変更・停止時・明示操作で同期する方針にしています。
    - **イベント通知の補強**:
        - 再生ループ内やキーボード操作（矢印キー）によるフレーム移動時にも `animation:frame-changed` イベントを確実に発火。
        - これにより、レイヤーパネル側のフレームインジケーターや CAF ヘッダーがリアルタイムに追従するようにしました。
    - **個別操作の同期**:
        - クリップのリタイミング（マウスアップ時）や、`Delete/Backspace` キーによる削除時にも明示的な同期を呼び出すようにし、操作完了直後の不整合を解消しました。

### 用語と UI の整合
- **用語の統一**:
    - `CAF` (ClipAssetFolder): フォルダ階層のラベル等に使用。
    - `INTERNAL LAYERS` (インスペクター) / `CLIP LAYERS` (レイヤーパネルミラー): 役割に応じた用語の使い分けを整理。
- **操作権限の限定**:
    - レイヤーパネル側には CAF の構造（追加・削除・移動）を操作する UI が存在しないことを再確認し、アニメーションテーブル側の「構造的正本」としての地位を確立しました。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **動作確認**:
    1. タイムラインでクリップを作成・削除・移動・伸縮した際、レイヤーパネル上部の CAF ヘッダーが即座に同期することを確認。
    2. 再生中もレイヤーパネルのフレームインジケーター（赤線）が追従し、停止後に CAF ヘッダーが最新状態に同期することを確認。
    3. キーボードナビゲーションでフレームを移動した際、レイヤーパネル側の表示もリアルタイムに追従することを確認。
    4. 内部レイヤーの可視性や名前を変更した際、インスペクターとミラーの両方が常に一致することを確認。
