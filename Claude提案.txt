ゼロ荷重付近の挙動：perfect-freehandで何が起きるか
perfect-freehandは筆圧からストロークの輪郭ポリゴンを生成します。
javascriptgetStroke(points, {
  size: 16,        // 基本サイズ
  thinning: 0.9,   // 1.0に近いほど低筆圧で極細になる
  simulatePressure: false
})
thinning: 1.0にすると、筆圧≈0のとき幅≈0のポリゴンが生成されます。これをWebGL（PixiJS）で描画すると…
筆圧 1.0  → 幅16px のポリゴン → しっかり描画
筆圧 0.3  → 幅5px のポリゴン → 細く描画  
筆圧 0.05 → 幅0.8px のポリゴン → ここがポイント
サブピクセル幅のポリゴンはWebGLのアンチエイリアスで自然に薄く表示されます。 つまり「幅が細い＝アルファが下がる」という現象がGPU側で自動的に起きます。

ただし「薄くなる」の表現には2種類ある
アプローチ特徴向いてる表現幅で細らせる（perfect-freehand）線が細くなる、点に近づく筆・鉛筆的不透明度で薄らせる幅はそのまま、色が薄くなる水彩・エアブラシ的両方同時より自然な筆感高品質ブラシ
鉛筆的なリアリティを出すなら両方を組み合わせるのがモダンな実装です：
javascript// 筆圧から幅と不透明度を両方計算
const pressure = event.pressure  // 0.0 〜 1.0

// perfect-freehandへの入力（幅制御）
points.push([x, y, pressure])

// PixiJSへの描画時（不透明度も下げる）
graphics.alpha = Math.max(0.1, pressure ** 0.5)
// ↑ 平方根で低筆圧でも急に消えすぎない調整

実用上の注意点
ゼロ近辺で幅が0に近づくと、PointerEventがpressure=0を送ってくる誤検知（ペンを浮かせる直前）でゴミ点が出ることがあります。現実的な対処は：
javascript// 最小筆圧をクランプして完全ゼロを避ける
const pressure = Math.max(event.pressure, 0.02)

// または一定以下は描画しない
if (event.pressure < 0.01) return
このあたりの調整がペン実装で一番手間のかかる部分で、Klecks等のOSS実装を参考にするのが有効です
