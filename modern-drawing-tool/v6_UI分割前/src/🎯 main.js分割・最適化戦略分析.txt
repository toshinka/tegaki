## 📊 現状問題分析

### 現在のmain.js構造問題点

**問題点**:
- **Import過多**: 10個以上のクラス直接import
- **依存関係複雑**: 各Enhancerが相互依存
- **冗長性**: 類似処理の重複
- **将来性**: chroma-js, lodash-es追加で更に増大

## 🎯 推奨解決策: **Facade Pattern + コア分離**

### **戦略A: OGLCore分離 + Enhanced Facade**

```javascript
// === NEW: OGLCore.js (完全放置部) ===
export class OGLCore {
    constructor(canvas) {
        this.canvas = canvas;
        this.setupOGLBase();
        this.setupEventBus();
    }
    
    setupOGLBase() {
        this.renderer = new Renderer({
            canvas: this.canvas,
            alpha: true,
            antialias: true,
            powerPreference: 'high-performance'
        });
        this.gl = this.renderer.gl;
        this.camera = new Camera(this.gl);
        this.scene = new Transform();
        this.setupCamera();
    }
    
    // コア描画処理（変更頻度低）
    render() {
        this.renderer.render({ scene: this.scene, camera: this.camera });
    }
    
    resizeCanvas(width, height) {
        this.canvas.width = width;
        this.canvas.height = height;
        this.renderer.setSize(width, height);
        this.setupCamera();
        this.render();
    }
}

// === NEW: OGLEnhancedFacade.js (すべてのEnhancer統合) ===
import { OGLQualityEnhancer } from './quality/OGLQualityEnhancer.js';
import { OGLInteractionEnhancer } from './OGLInteractionEnhancer.js';
import { OGLProEnhancer } from './OGLProEnhancer.js';
import { OGLSettingsManager } from './OGLSettingsManager.js';
import { UIController } from './UIController.js';

export class OGLEnhancedFacade {
    constructor(oglCore) {
        this.core = oglCore;
        this.initializeEnhancers();
        this.setupIntegration();
    }
    
    initializeEnhancers() {
        // 順序重要: 設定 → 品質 → インタラクション → プロ → UI
        this.settingsManager = new OGLSettingsManager(this.core);
        this.qualityEnhancer = new OGLQualityEnhancer(this.core);
        this.interactionEnhancer = new OGLInteractionEnhancer(this.core);
        this.proEnhancer = new OGLProEnhancer(this.core);
        this.uiController = new UIController(this.core);
    }
    
    // Facade pattern: 外部からの統一アクセス
    setPenSize(size) {
        this.settingsManager.set('drawing.penSize', size);
        this.qualityEnhancer.updatePenSize(size);
    }
    
    setOpacity(opacity) {
        this.settingsManager.set('drawing.opacity', opacity);
        this.qualityEnhancer.updateOpacity(opacity);
    }
}

// === SIMPLIFIED: main.js (最小構成) ===
import { OGLCore } from './OGLCore.js';

class OGLUnifiedEngine {

    // 描画処理のみに集中

        this.strokes = [];
```

## 🎯 戦略C: **Module Federation + 動的Import**

```javascript
// === MODERN: OGLModuleLoader.js (動的読み込み) ===
export class OGLModuleLoader {
    constructor() {
        this.loadedModules = new Map();
        this.loadingPromises = new Map();
    }
    
    async loadEnhancer(name) {
        if (this.loadedModules.has(name)) {
            return this.loadedModules.get(name);
        }
        
        if (this.loadingPromises.has(name)) {
            return this.loadingPromises.get(name);
        }
        
        const loadPromise = this.dynamicImport(name);
        this.loadingPromises.set(name, loadPromise);
        
        try {
            const module = await loadPromise;
            this.loadedModules.set(name, module);
            this.loadingPromises.delete(name);
            return module;
        } catch (error) {
            this.loadingPromises.delete(name);
            console.warn(`Failed to load ${name}:`, error);
            return null;
        }
    }
    
    dynamicImport(name) {
        const moduleMap = {
            quality: () => import('./quality/OGLQualityEnhancer.js'),
            interaction: () => import('./OGLInteractionEnhancer.js'),
            pro: () => import('./OGLProEnhancer.js'),
            settings: () => import('./OGLSettingsManager.js'),
            ui: () => import('./UIController.js')
        };
        
        return moduleMap[name]?.() || Promise.reject(`Unknown module: ${name}`);
    }
}

// === ASYNC: main.js (動的読み込み版) ===
import { OGLCore } from './OGLCore.js';
import { OGLModuleLoader } from './OGLModuleLoader.js';

class OGLUnifiedEngine {
    constructor(canvas) {
        this.core = new OGLCore(canvas);
        this.loader = new OGLModuleLoader();
        this.enhancers = {};
        
        this.initializeAsync();
    }
    
    async initializeAsync() {
        // 必須モジュールのみ最初に読み込み
        const coreModules = ['settings', 'quality'];
        
        for (const moduleName of coreModules) {
            const module = await this.loader.loadEnhancer(moduleName);
            if (module) {
                this.enhancers[moduleName] = new module.default(this.core);
            }
        }
        
        // オプションモジュールは遅延読み込み
        this.loadOptionalModules();
        
        this.setupDrawingSystem();
        this.startRenderLoop();
    }
    
    async loadOptionalModules() {
        const optionalModules = ['interaction', 'pro', 'ui'];
        
        optionalModules.forEach(async (moduleName) => {
            const module = await this.loader.loadEnhancer(moduleName);
            if (module) {
                this.enhancers[moduleName] = new module.default(this.core);
            }
        });
    }
}
```

## 🎯 推奨: **戦略A (Facade Pattern + コア分離)**

### 理由
1. **責任分離明確**: コア機能と拡張機能の明確分離
2. **保守性最高**: 各部分が独立してテスト・修正可能
3. **将来性抜群**: 新機能追加時もFacade内で完結
4. **import最小**: main.jsは2つのimportのみ
5. **段階移行**: 現在のコードから段階的移行可能

### 実装順序
1. **Phase 1**: OGLCore.jsにコア機能分離
2. **Phase 2**: OGLEnhancedFacade.js作成・全Enhancer統合
3. **Phase 3**: main.js簡素化
4. **Phase 4**: 冗長性削除・最適化

## 🚨 冗長化修正だけでは不十分な理由

現在のコードを見ると、**構造的問題**が根本原因:

- **責任範囲不明**: main.jsが描画・設定・UI・エンハンサー全管理
- **密結合**: 各Enhancerがmain.jsに直接依存
- **import爆発**: 新機能追加のたびにimport増加

**冗長性修正のみでは30%改善程度**。根本解決には**アーキテクチャ再設計**が必要。
