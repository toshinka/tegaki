## バケツツール：囲い塗り

### 要望

線で囲った領域の中をバケツでクリックしたとき、内側だけが塗られるようにしたい。

例：

```text
○ を描く
内側でバケツを使う
● になる
```

### 後続実装時に決めること

- 参照対象を現在レイヤーのみにするか、表示中レイヤー合成にするか。
- 半透明線やアンチエイリアスの薄い縁を境界として扱うしきい値。
- 小さな隙間を閉じる gap close の有無と強さ。
- 塗りを同一レイヤーへ入れるか、線画の下の別レイヤーへ入れるか。

## Vキー変形パネルの回転スライダー改善案 (Phase 1j 以降)

- **現状の問題**: 
  - -360°〜360° の固定範囲であり、回転を繰り返すと端に到達して止まってしまう。
  - 数値入力は可能だが、スライダーでの連続的な回転操作がしにくい。
- **改善案**:
  - **無限ループスライダー**: 1周（360°）を超えても累積角度としてカウントし、スライダーのつまみ位置は周期的に戻るが内部数値は増え続ける仕組み。
  - **累積角度の保持**: 720° や -1080° などの値を許容し、アニメーション等への応用を容易にする。
  - **リセットボタン**: クリック一つで 0° に戻せるボタンをスライダー横に配置。

## UIアイコン・小ボタン共通化

- close / trash / duplicate / download / upload など、複数の popup やカードで再利用されるアイコンは単一ソース化する。
- 短期的には `DOMBuilder.ICONS` を参照し、増えすぎる場合は `ui/ui-icons.js` へ移す。
- 見た目は `main.css` の共通クラスに寄せ、個別ファイルの inline style を増やさない。
- レイヤー削除は small、popup/Qパネル閉じるは medium など、用途差はサイズ修飾子で扱う。

## アルバム管理拡張案

- **ドラッグ並べ替え**: アルバムカードの順番をドラッグで変更し、保存順を維持する。
- **複数選択**: まずは `Ctrl+クリック` で選択トグル。通常クリック復元の既存操作を壊さない。
- **一括操作**: 一括削除、一括エクスポート、移動は選択状態の土台が安定してから扱う。
- **選択操作の受け入れ方針**: `Ctrl/Cmd+クリック` は個別トグル、`Shift+クリック` は前回選択から範囲選択。選択中カードのゴミ箱は複数削除に使えると自然。
- **HTML保存/読込**: thumbnail だけでは復元できないため、`projectData`、canvas情報、レイヤー情報、順番、作成日時、アプリバージョンを含める必要がある。ファイルサイズと旧データ互換性を要検討。
- **保存正本**: 現行 `AlbumPopup` は localStorage、`VirtualAlbum` は IndexedDB の別系統。どちらを正本にするかは設計判断が必要。

## ログ整理・保守方針

- 恒久ログ: 起動時の重要設定、重大エラー、復旧不能な状態。
- debugログ: `TEGAKI_CONFIG.debug` が true のときだけ出す診断。
- 削除ログ: ペン移動やサムネイル生成など高頻度で、確認完了後は不要なもの。

## アルバム管理基盤拡張 (Phase 1l 設計メモ)

### HTML保存/読込の設計

- **コンセプト**: プロジェクト全体（全スナップショット + メタデータ）を一つのポータブルな HTML ファイルとして保存する。
- **HTMLに含めるべき情報**:
  - `projectData`: 全レイヤーの構造、画像、設定を保持する JSON。
  - `thumbnails`: ギャラリー表示用の軽量画像データ。
  - アプリケーション本体（ビューワーモード）を含めることで、ファイル単体での閲覧を可能にする案もあり。
- **なぜ両方が必要か**:
  - プレビュー（サムネイル）のみでは再編集が不可能。
  - プロジェクトデータのみでは、エクスプローラー等での一覧性が損なわれる。
- **ファイルサイズ問題**:
  - レイヤー画像が多いと数十MB〜数百MBに達する可能性がある。
  - 必要に応じて画像の圧縮（JPEG/WEBP）や、履歴の除外を検討する。
- **保存データの役割分担**:
  - **localStorage**: 編集中の自動保存、数件の最新スナップショット（即時アクセス用）。
  - **IndexedDB**: 数十〜数百件のローカルアーカイブ。大容量データの永続化。
  - **HTMLファイル**: 外部へのバックアップ、ポータブルな受け渡し用。
- **互換性**:
  - 旧形式（画像のみのスナップショット）も破損させずに「閲覧専用」として維持する。
