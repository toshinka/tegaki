# オーナーの実装したいことメモ

> ブレスト用。順番は固定しない。
> 実装順は、AI が壊しにくいこと、ファイル構造上の積み上げ効率、現在フェーズとの相性を優先する。
> 達成済み・現フェーズから外れたものは、Codex が整理してよい。

---

## 次フェーズ相談で優先的に見たい候補

### アニメテーブル / 動画ツール風UI

- 動画ツール風なアニメ化機能はモチベーションが高い。必要な足回り整備が済んだら少し優先したい。
- ただし、全体的な実装をしにくくなるような無理な繰り上げまではしない。
- アニメを Photoshop 的なものから、トラックやキーフレームを持つ動画ツール的なものへ変更したい。表示やデザインは ToonSquid を参考にする。
- トラックをクリックするとグラフが出て、イージング管理できるようなイメージ。
- アニメ用のタイムテーブルは別タブ・別ウィンドウ化したい。
- アニメ系操作は `Alt+` を主体、またはキー操作切り替えモードとし、キーボードだけでもなるべく完結する作りにしたい。
- 液タブで絵を描きながら、別モニター側でタイムテーブルやイージングを操作する使い方も想定する。
- 映像編集重視モードと描画モードの入れ替えを瞬時にできると便利かもしれない。

### 独自D&D / レイヤー・アルバム移動

- アルバム移動やレイヤーパネル移動は、Windows のように半透明でスムースに動かしたい。
- 現状の SortableJS 調整だけでは限界があり、独自実装または新ライブラリ検討が必要そう。
- レイヤーパネルのスムースな移動は `PastFiles/OldFiles/v8.13_anime14` 辺りでも出来ていた。参考にできないか確認したい。
- 複数選択での移動は、選択の仕方による位置入れ替えが煩雑になりそうなので保留寄り。
- レイヤーはサムネイル左右フリックで削除できるようにしたい。
- `Shift` 押しながらレイヤーパネルを押すと複数選択されるようにしたい。
- 複数選択したレイヤーを、一括結合、一括クリッピング、まとめて移動できるようにしたい。

### トップバー / キーボードなし操作

- トップバーは縦画面資産のためにもなるべく避けたかったが、キーボードがない人向けに必要そう。
- 画面基準値にするホームアイコン、Undo、Redo、回転、上下左右反転、レイヤーパネル ON/OFF を置きたい。
- 不要な人のために環境設定で OFF にできるようにする。
- アイコンはなるべく右上に固める。
- キャンバス表示の左右反転・上下反転時は上にメッセージを出す。左右反転のままずっと描いてしまう事故を避けたい。

### QAP / ツール別クイックパネル

- Q で出るクイックパネルをさらに改修したい。
- クイックパネルは横幅を今の 60% くらいに近づける目安でスリム化する。アイコンも小型化・密度調整する。
- ペンや色のパネルを司るQAPは、選択ツールをアクティブにするとフリップして矩形・円・折れ線・メッシュ変形などのパネルに変わる案がある。
- 選択ツール中はペンで絵を描けないので、QAPの表示資産をツール別に使い回す。
- ツールによってQAPが入れ替われば、サイドバーに大量のアイコンを並べる必要が減るかもしれない。
- QAP内アイコン候補は、ペン / 消しゴム / バケツ / エアブラシ / 図形 / 選択。スポイトはカラーサークル周りに置く。
- ペン / 消しゴム / バケツ / スポイトはショートカット使用が中心なので、将来的にサイドバーから消す判断も検討。ただし現時点では保留。
- ステータス側で `Tool: ✒ ペン ■ 5px 100%` のように、SVG・色・サイズ・透明度を左上に表示できると、サイドバーに頼らなくても視覚処理しやすいかもしれない。設定で表示 ON/OFF 可能にする。
- ショートカットでQAPを展開できないペン操作中心の人向けに、設定でQAP開閉用の小さな糊代を出したい。
- QAPを閉じるだけではなく、小さく畳むモードも欲しい。設定次第で `X` ボタンを小型化アイコンへ置き換える案。

## 描画・編集ツール

### 描画補助

- 四角ペンを実装したい。
- ペン補正を設定できるようにしたい。最初はスライダーでよいが、最終的には筆圧カーブなどを 2D グラフ化したい。
- 筆圧での透明度設定と手ブレ補正スライダーを、環境設定またはQAP拡張ポップアップから設定できるようにしたい。
- カスタムペン作成を設定で行い、登録してQAPなどに入れ替え可能にしたい。優先度低め。
- ペンの傾き検知。
- 通常ペンは WebGL2 で磨く。WebGPU は先にエアブラシ・変形系で試し、成功したらペン系にも拡張する。
- キャンバスサイズを最大にしてカメラも引き気味に縮小して描くと線がジャギーになった。ペン精度やアンチエイリアスがカメラ拡縮の影響を受けるのか調査したい。

### 図形・選択・テキスト

- 範囲選択、切り取り、貼り付けなどの編集ツールを実装したい。
- 直線、円、矩形ツールを実装したい。直線は `Shift` 押しながら引ける形も検討する。
- 定規、自在定規ツールを実装したい。
- 優先度は高くないが文字入力モードを実装したい。漫画セリフでよく使われるフリーフォントを導入できるなら検討する。
- 数字入力系は数値入力用ポップアップを出せるとよいかもしれない。優先度はかなり低め。

### 漫画向け特殊ツール

- スクリーントーンを実装したい。網点、グラデーション、縦線ストライプなど。
- 集中線、ウニフラッシュツールを実装したい。
- 漫画のコマで使える自動マスキングを検討したい。四角などの外側がマスクされ、図形を変形するとマスク形状も追従するイメージ。
- 特別な判定の線に追従するマスク塗料、または判定用クリッピングレイヤーのような仕組みを検討する。
- アニメツール的には、目や口の開閉にも応用できるかもしれない。

### 色・スポイト

- スーパースポイト。ブラウザ外からもカラーピックできるとよい。
- カラーサークルとカラーセットは、`-` ボタンや `<` ボタンなどで折り畳めるようにしたい。
- デフォルトに戻すボタン、現状のカラーセット保存、カラーセット呼び出しを入れたい。
- カスタムカラーセットやペン設定は、環境設定内の保存ボタンなどで HTML に記憶されるとよい。
- アルバムの JSON にも保存時に記録され、ロードやインポート時に反映されるとよい。

## レイヤー / フォルダ / 変形

### レイヤーパネル

- レイヤーパネルは、多数レイヤー向けに今よりスリムにしたい。
- 表示情報は、目、クリップ状態、レイヤー名、透明度、サムネイルを中心にする。
- 編集操作は、レイヤー行内に詰め込むより、アクティブレイヤーに反応する操作バーや補助パネルへ集約する案がよい。
- ToonSquid の配置は参考になる。共通アイコンが上部ではなく右端に縦並びで、アクセスしやすい印象。
- 透明度は大きめのスライダーや透明度プリセットを出す方式も検討したい。ただし透明度操作の頻度はそこまで高くないかもしれない。
- レイヤーパネルと右サイドバーが、縦スライダーが入るとはいえ離れすぎている。枠線が消えない良い位置にする。
- レイヤーパネルやアルバムで項目が増えた時に出る縦スライダー、各種調整の横スライダーのデザインを統一したい。
- スライダーは共通コンポーネント参照がよいか、規格だけ決めて各場所で独立配置がよいか検討する。

### フォルダ / クリッピング / 変形

- クリッピングは、同じ階層のレイヤーかフォルダに効く形がよさそう。影響を与えたいレイヤーは同階層フォルダにまとめる設計も検討する。
- レイヤーフォルダは実装済みの土台があるが、保存形式・履歴・複数操作との整合を詰めたい。
- アクティブフォルダでVキーのレイヤー編集モードを使うと、フォルダ内の全てを変形・移動できるようにしたい。
- 今後実装予定の選択範囲での変形や移動、物理演算アニメでも同様にフォルダ単位操作を扱いたい。
- フォルダには絵は描けないが、フォルダ内のまとめて変形はできる形にする。
- Vキーによるアクティブレイヤー編集の回転で、楕円軌道設定もできるようにしたい。縦・横・角度を数値とスライダーで調整する案。
- アクティブレイヤー編集パネルに初期化/リセットボタンを置きたい。色々動かした後にパネルを開く前の状態に戻すのが大変なため。
- Redo が安定しない気がする。アクティブレイヤーで回転や縮小で遊んだ時、元に戻らないことがあった。
- `Shift+DEL` でアクティブレイヤー/フォルダ削除。全削除は設定内にボタンを入れる。

## UI / ポップアップ / ステータス

### UI透明化・表示

- 左サイドバーを半透明にし、キャンバスが重なっても裏が見えるようにして視覚圧迫を減らしたい。
- ステータスバーはデフォルトでは目立たせず、下側に数字中心で控えめに表示したい。
- トップバーやサイドバーは、設定や矢印アイコンでスライドして隠せるようにしたい。
- 他のポップアップ背景やサイドバーも、最終的には透明度を統一したい。
- ツールは下記カラーで構成する。スライダーが灰色になったり、安易に黒や白を使わないように注意する。

```text
maroon: '#800000'      アクティブペンカラー（デフォルト）
lightMaroon: '#9c3836'
medium: '#b8706b'
lightMedium: '#d4a8a1' パネル背景
cream: '#f0e0d6'       キャンバス背景（レイヤー0デフォルト）
background: '#ffffee'  ページ背景
```

### ポップアップ

- リサイズポップアップ背景を半透明にし、奥のキャンバス状態を見ながら操作できるようにしたい。最初は透明度 50% くらいで試す。
- 各ポップアップを移動可能にしたい。QAPだけ動く状態は少し違和感がある。
- アルバムポップアップにレイヤーパネルがかかるので、QAPと同じ階層にする。
- アルバムポップアップも他のポップアップ同様にドラッグ移動可能にする。
- 画像・アニメ出力ポップアップのプレビュー画面は、200 x 200 程度を基準にして画面からはみ出ないようにしたい。アスペクト比で多少増減してよい。
- 設定内の項目が増えそうなので、設定 popup 内をタブで整理したい。
- 画像の出力 popup の中にも入力タブを入れたい。
- 入力タブでは、ファイル選択とクリップボードからの貼り付けを選べるようにする。
- 現在のキャンバスに合わせてアスペクト比を保持して縮小するかのチェックボックスも入れる。
- 入力/出力アイコンは、写真アイコンに下矢印だけではなく、矢印なしまたは上下矢印ありの候補も検討する。
- 閉じるボタンがCSSで共通化されていると思ったが、されていない可能性がある。調査し、各ポップアップで共通化する。

### アイコン

- 使用SVGは `lucide-static` / https://lucide.dev/icons/
- エアスプレーSVGは、缶の部分が圧縮しているせいか太く見える。一段階線を細くしたい。
- エアスプレーの透明化で色表現を反転するか別の中間色カラーにしたかったが、Maroonで塗りつぶされている。改善したい。
- エアスプレーやバケツの消しモードは、背景と枠色だけでなく、消しゴムの小アイコンを右上に重ねる案もある。
- ペンと消しゴムも筆圧時は圧を表すアイコンを表示する案。

## リサイズ / キャンバス / 性能

### リサイズUI

- リサイズ画面の幅/高さ矢印は、`↔` / `↕` 選択時は均等増減、`←` / `→` / `↑` / `↓` 選択時は指定方向へ増減するようにしたい。
- 増減部分はキャンバス側にプレビュー表示したい。半透明または着色で増減範囲を示す案。
- このプレビューを見るためにも、リサイズポップアップは半透明がよさそう。

### キャンバスサイズ / 無限キャンバス

- キャンバスサイズ変更の片辺最大は 2500px を第一候補にする。根拠は漫画原稿サイズで 1700 x 2400 くらいが一つの目安であるため。
- 3000px 案は性能/履歴メモリの安全確認後の保留候補。
- キャンバス操作での拡縮倍率ももう少し広げる。
- キャンバス外に描画物を移動して確定するとキャンバス外の絵が切れる問題は、将来的に無限キャンバスまたは外部領域保持で解消したい。
- 物理演算アニメ化では、キャンバス外領域保持が必須になりそう。

### 性能方針

- オーナー環境は RTX4070 / メインメモリ 64GB。
- この性能を越えるような過剰高性能前提にはしない。
- 逆に下限を大きく取りすぎて性能を下げる実装にもせず、モダンでそこそこリッチなツールを目指す。足切り上等。
- タブレットやスマホ環境はオーナー側で確認できないため、ピンチ操作などのハンドジェスチャー対応は慎重に進める。

## 物理演算 / WebGPU / 長期構想

- 物理演算で XY 移動/回転させたい。Z 軸的な拡縮も考えたい。
- Live2D 的な制御も将来の参考にする。
- 物理演算アニメ導入時の WebGPU 利用は、描画レイヤー全体のWebGPU化ではなく、まず物理計算結果だけを座標・メッシュ・変形パラメータとして返す方式が安全そう。
- WebGL2 継続で問題なし。WebGPU化を急ぐ必要はない。
- ただし将来のため、`CPUPhysicsEngine` / `WebGPUPhysicsEngine` のような差し込み口を作る案はあり。
- 基幹データはGPUリソース直結ではなく、`object.meshData`、`object.bones`、`object.physicsParams`、`object.transform` のように API 非依存で持つのが安全。
- 物理演算対象まで Pixi Container / RenderTexture / window 参照に密結合させないことが重要。
- window グローバル登録が多いので、将来 WebGL2 / WebGPU / CPU 版を差し替える時に依存が見えにくくなる懸念がある。CoreEngine 経由の依存注入へ少しずつ寄せるとよい。

## ロングパス / 最後の方でよいもの

- 個人的に水彩とぼかしはあまり使わないので、最後の仕上げ辺りでよい。アニメ化のほうが優先度高い。
- 多言語化のために、アイコン説明は別ファイルへ一元化した方が翻訳を一括で作りやすいかもしれない。実装段階で検討する。

## 完了済み・現フェーズでは保留したもの

### 描画・ツール

- ペンの遅いストロークが丸の集合に見える問題は Phase 1e で改善済み。
- 筆圧入力、筆圧 ON/OFF ショートカットは Phase 1e で実装済み。
- 囲い塗り / 投げ縄塗り / 高機能バケツ基盤は Phase 3c〜3h で実装・整理済み。
- エアブラシ、透明エアブラシ、ぼかしツールは Phase 3a で実装済み。
- スポイト、カラーサークル、メイン/サブカラー、`X` 入れ替えは Phase 3i〜3l で実装済み。
- Undo/Redo での黒色化疑いは Phase 3m で止血修正済み。オーナー実機確認では現時点で再現なし。再発時に再調査する。

### レイヤー・変形

- V パネルの水平/垂直反転、`H` / `Shift+H`、回転 360 度化、数値ダブルクリック入力は Phase 1i で実装済み。
- レイヤー透明度の数値ドラッグは Phase 1i で実装済み。
- クリップボード画像貼り付け、アクティブレイヤーコピー/新規コピーレイヤー作成は Phase 1i で土台実装済み。
- レイヤーフォルダ、薄型レイヤー行、右サイドバー型レイヤーパネル、合成モードUI、クリッピングUI、フォルダ保存復元は Phase 2 系で土台実装済み。
- レイヤーD&Dの短期止血は実施済み。ただし快適な旧型風D&Dは未解決で、独自D&D候補として保留。

### UI・QAP・アルバム

- レイヤー削除ボタンや popup 閉じるボタンの共通化は Phase 1k で実装済み。ただし閉じるボタン共通化の実態は再調査候補。
- 背景レイヤー非表示時の透明市松表示は Phase 2l で実装済み。色は `cream + background`、カメラ拡縮時もマス目サイズ一定。
- QAPのツール別サイズ/不透明度プリセット、COLOR 1〜5、カラーサークル受け皿、スポイト、メイン/サブカラーは Phase 3b〜3l で実装済み。
- QAP未表示時に `[` / `]` を押すと未初期化スロットで例外になる問題は Phase 3m 追加修正で解消済み。
- アルバムはクリック即ロードを避け、選択と明示ロード導線へ整理済み。最初から選択モードに近い運用になっている。
- アルバムのドラッグ並べ替え、HTMLエクスポート/インポート、一括削除/一括書き出しの土台は Phase 1l〜1m で実装済み。
- PNG エクスポート、プロジェクト保存/読み込み、アルバムHTML保存/読み込みは Phase 1g〜1m で復旧済み。
