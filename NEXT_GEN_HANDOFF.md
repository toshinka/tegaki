# 次世代 AI (Gemini 3.5 / Antigravity 2.0) への引き継ぎ書

## 1. 現在の到達地点 (2026-05-19 時点)

Phase 2m/2n/2o/2p を一気に駆け抜け、Tegaki アプリの UI 基盤は非常に強固で洗練された状態にあります。

- **UI デザインシステム**: `main.css` へのスタイル集約が完了。`.popup-panel--translucent`（透過・ぼかし）や `.ui-scrollbar`（目に優しいスライダー）といった共通クラスを指定するだけで、高品質な UI が構築可能です。
- **アルバム機能**: 「本」のメタファー（保存＝`book-plus`, ロード＝`book-open-check`）に基づき、トグル選択による誤ロード防止が完璧に実装されています。
- **設定・ヘルプ統合**: 設定パネルがタブ化され、サイドバー資源を消費せずにショートカット一覧を確認できるようになりました。

## 2. 実装上の「お約束」と規律 (AI への助言)

次世代 AI モデルが作業する際は、以下の「Tegaki 独自の規律」を継承してください。

1.  **インラインスタイルの禁止**: `element.style.xxx` を JS で書くのは避けてください。必ず `main.css` に定義し、`classList.add` で制御してください。
2.  **DOMBuilder の活用**: ポップアップの新設時は `ui/dom-builder.js` の `createCloseButton` 等を活用し、UI の一貫性を守ってください。
3.  **初期化の堅牢性**: `SettingsPopup` で発生した「外枠だけある場合の初期化ミス」を教訓に、要素の存在確認を常に行ってからイベントを紐付けてください。
4.  **根幹ロジックの保護**: 描画エンジン (`core-engine.js`) やレイヤーシステムは非常にデリケートです。UI 修正のついでにこれらのロジックをリファクタリングするのは、オーナーの許可なく行わないでください。

## 3. 次フェーズへの提案：Phase 2q — ステータスバーの現代化

次は、画面下部に停滞している「ステータスバー」の刷新を推奨します。

- **目的**: 座標、キャンバスサイズ、History カウントなどの情報を、今の透過デザインに合わせて整理・視覚化する。
- **AI へのメリット**: これも独立した UI 層であるため、安全に能力を発揮できます。
- **具体的なアイデア**:
    - `.popup-panel--translucent` と同等の質感へ変更。
    - 座標表示のリアルタイムレスポンス向上。
    - History 数をただの数字ではなく、小さな「履歴スタック」のようなアイコンと組み合わせる。
