# ⚠️ 総合AIコーディング規約 v2.4 憲章統合検査特化版

## 🎯 基本方針（憲章統合）
- **再発明回避**: 既存優良ライブラリ最大限活用（Bezier.js, twgl.js, OGL等は完成品扱い）
- **ライブラリ統合責務**: エンジンは統合役、アルゴリズム再実装禁止
- **ユーザー期待値最優先**: 技術完璧性より体験重視
- **🆕 描画モード完全分離**: ベクター/ラスター描画の同時実行完全禁止
- **Phase1前提**: HTML単一ファイル + CDN読み込み運用

## 📂 命名・構造規則

### ❌ 全面禁止命名
| 禁止 | 理由 | 代替 |
|------|------|------|
| *Manager.js | 責務曖昧 | *Store.js, *Controller.js, *Actions.js |
| *Handler.js | 処理不明 | *Processor.js, *Validator.js |
| *Helper.js | 目的不明 | *Utils.js, *Calculator.js |
| *Service.js | DI以外禁止 | 具体的な役割名 |

### 🧭 強制配置・憲章準拠構造
```
features/[機能名]/
├── [機能名]Store.js     # 状態管理層
├── [機能名]Actions.js   # 操作実行層
└── [機能名]Controller.js # 制御統括層(必要時)

engine/modes/            # 🆕 描画モード分離（憲章重要項目）
├── DrawingModeController.js    # モード切り替え制御（競合回避）
├── VectorModeEngine.js         # Bezier.js専用エンジン
├── RasterModeEngine.js         # Canvas2D専用エンジン
└── InteractiveModeEngine.js    # Konva.js専用エンジン（選択・移動のみ）
```

## 💎 SOLID原則（憲章統合）

### 実装前チェック（憲章項目統合）
- [ ] **この処理は既存ライブラリで解決可能か？**
- [ ] **独自実装する合理的理由はあるか？**
- [ ] **ライブラリの結果を信頼しているか？**
- [ ] **過度な制御・カスタマイズをしていないか？**
- [ ] **ユーザー期待値と一致しているか？**
- [ ] **複数描画エンジンが同時動作していないか？**
- [ ] **座標系・イベント処理の競合は発生しないか？**

**S: 単一責任** - 1クラス1責務のみ
**O: 開放閉鎖** - 拡張で対応、修正は最小限
**L: 置換可能** - 同インターフェースは完全置換可能
**I: インターフェース分離** - 不要メソッド実装を強制しない  
**D: 依存性逆転** - 抽象に依存、具体実装への直接依存禁止

## 🎨 描画モード分離（憲章重要項目）

### 描画モード完全分離ルール
```javascript
// 同時実行完全禁止（憲章違反）
enum DrawingMode {
    VECTOR = 'vector',    // Bezier.js専用（ペン描画）
    RASTER = 'raster',    // Canvas2D専用（ブラシ描画）
    INTERACT = 'interact' // Konva.js専用（選択・移動・変形）
}

// ツール→モード対応（厳格）
const TOOL_MODE_MAPPING = {
    'bezier-pen': DrawingMode.VECTOR,      // Bezier.js
    'canvas-brush': DrawingMode.RASTER,    // Canvas2D
    'selection': DrawingMode.INTERACT,     // Konva.js
    'move': DrawingMode.INTERACT,          // Konva.js
    'transform': DrawingMode.INTERACT      // Konva.js
};
```

### 検査項目
- [ ] モード切り替え時の現在モード完全停止
- [ ] 単一モードのみ動作確認
- [ ] Konva.jsペン描画使用なし
- [ ] 座標系統一（UnifiedCoordinateSystem）
- [ ] イベント処理競合なし

- UI内状態保持（Store集約必須）
- 1ファイル複数責務混在

**D: 中間層肥大化** (ISP違反)
- AppControllerパススルー関数量産
- 曖昧中間層への機能集約

### 🧹 中重要: 実装品質
**E: 実装方式不統一** (OCP違反)
- 非同期処理記述不統一
- 同処理の異なる実装

**F: Phase1制約違反**
- ES6 Modules使用（import/export禁止）
- npm パッケージ使用（CDN必須）
