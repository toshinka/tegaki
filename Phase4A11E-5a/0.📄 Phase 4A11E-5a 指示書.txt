🎯 改修目的
ユーザーがHTML上のカラーパレットをクリックした際に、ToolStore.mainColor を更新する処理を追加する

WebGLRenderer 内で mainColor が null になっている現象を解消する

📁 対象ファイル
vite-project/index.html

vite-project/src/features/tools/ToolStore.js

vite-project/src/engine/WebGLRenderer.js
※ToolActions.js に機能がない場合は、ToolStoreへの直接アクセスで可とする

🔨 作業内容
① index.html の末尾（</body>直前）に以下のスクリプトを追加：
js
コピーする
編集する
import { toolStore } from './src/features/tools/ToolStore.js';

document.querySelectorAll('.color-btn').forEach(btn => {
  btn.addEventListener('click', () => {
    const color = btn.dataset.color;
    toolStore.setState({ mainColor: color }); // 🎯 ToolActionsがあればそちらを使用可
    document.querySelectorAll('.color-btn').forEach(b => b.classList.remove('active'));
    btn.classList.add('active');
  });
});
② WebGLRenderer.js の該当描画関数にフォールバック処理を追加（例）：
js
コピーする
編集する
import { toolStore } from '../features/tools/ToolStore.js'; // 必要に応じて

const state = toolStore.getState();
const color = state.mainColor || '#800000'; // fallback
drawCircle(x, y, size, color);
③ ToolStore.js に変更不要（現状問題なし）
