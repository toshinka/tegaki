## 3. 非破壊的変形ルール（アニメ制作対応）

### 3.1 ベクター保持原則
- **ラスター変換禁止**: 回転・拡縮でもベクターデータ保持
- **変形行列適用**: PixiJS Transform行列による非破壊変形
- **品質劣化防止**: アニメ制作時の品質劣化完全回避

### 3.2 変形システム構成
```javascript
// transform/vector-transform.js
class VectorTransform {
    applyTransform(layer, transform) {
        // ✅ ベクターデータに変形行列適用（非破壊）
        layer.transform.setFromMatrix(transform);
        // ✅ 履歴に変形行列記録（完全復元可能）
        this.memoryManager.record('transform', layer.id, transform);
    }
}
```

### 3.3 アニメ制作対応
- **キーフレーム品質**: フレーム間でのベクター品質保持
- **変形履歴**: アンドゥによる完全復元

## 5. コードスタイル規約（Phase対応強化版）

### 5.1 命名規則（Manager階層対応）
- **Manager命名**: `{Domain}Manager`（例：`UIManager`, `ToolManager`）
- **専門ファイル命名**: `{function}-{type}.js`（例：`pen-tool.js`, `popup-manager.js`）
- **分業識別**: ファイル先頭にAI作業識別コメント

```javascript
// 🎯 AI_WORK_SCOPE: ペンツール専用ファイル（他ツール依存なし）
// 🎯 DEPENDENCIES: app-core.js, managers/tool-manager.js のみ
// 🎯 SPLIT_READY: 200行超過時 pen-settings.js 分割
```

### 5.2 分業開発用コメント規約
```javascript
// 🎯 AI作業識別（必須）
// 🎯 AI_WORK_SCOPE: 作業対象の明確化
// 🎯 DEPENDENCIES: 必要な依存ファイル列挙
// 🎯 SPLIT_READY: 分割予告・分割先ファイル名

// 📋 システム管理（任意）  
// 📋 SPLIT_PLAN: 分割計画詳細
// 📋 V8_MIGRATION: v8移行対応予定

```

### 5.3 Manager階層命名規約
- **統括Manager**: `{Domain}Manager`（UI・Tool・Canvas・Memory・Settings）
- **専門Manager**: `{Function}Manager`（Popup・Slider・Layer・Transform）
- **統合ファイル**: `{domain}-{function}.js`（layer-transform.js）

- **内部プロパティアクセス**: PixiJS内部プロパティ直接操作禁止

- **責務侵害**: Managerの責務領域を他ファイルで実装禁止
- **循環依存**: Manager ↔ 専門ファイル間の循環依存禁止
- **統括重複**: 複数Managerでの同一統括機能実装禁止

### 11.3 非破壊品質固有禁止事項
- **ラスター変換**: ベクターからラスターへの破壊的変換禁止
- **品質劣化**: 回転・拡縮での品質劣化を招く実装禁止  
- **履歴破綻**: 非破壊履歴システムを破綻させる実装禁止
- **ベクター破壊**: ベクターデータの破壊的操作禁止

### 11.4 Phase移行期禁止事項
- **段階無視**: Phase順序を無視した実装禁止
- **v8早期導入**: Phase1-3でのv8機能使用禁止
