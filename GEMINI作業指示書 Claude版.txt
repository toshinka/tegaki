【修正1】サムネイル黒背景問題（根本修正）
ファイル: system/drawing/thumbnail-system.js
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 原因
PixiJS v8 の renderer.render() は clearColor / clearAlpha を
個別パラメータとして受け付けない。現在のコードではこれらが無視され、
clear: true が「黒でクリア」として動作している。

■ 修正箇所
generateLayerThumbnail() 内の renderer.render() を以下に変更する。

【変更前】
this.app.renderer.render({
    container: layer,
    target: renderTexture,
    clear: true,
    clearColor: clearColor,
    clearAlpha: clearAlpha
});

【変更後】
// 背景レイヤーは背景色でクリア、通常レイヤーは透明でクリア
const clearValue = layer.layerData?.isBackground
    ? (() => {
        const c = layer.layerData.backgroundColor || 0xf0e0d6;
        const r = ((c >> 16) & 0xff) / 255;
        const g = ((c >> 8)  & 0xff) / 255;
        const b = ( c        & 0xff) / 255;
        return [r, g, b, 1.0];
    })()
    : [0, 0, 0, 0];  // 透明でクリア

this.app.renderer.render({
    container: layer,
    target: renderTexture,
    clear: clearValue   // PixiJS v8 の正式形式: RGBA配列 [0〜1]
});

■ 確認観点
- 新規レイヤー（何も描いていない）のサムネイルが黒でなく透明（チェッカー等）で表示される
- 背景レイヤーのサムネイルに正しい背景色が表示される
- TegakiConsole の sample pixel の rgba[3] が 0 になっている（透明レイヤーの場合）

【修正2】リサイズ時に背景が黒くなる問題
ファイル: system/layer-system.js
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 原因①
resizeLayerTextures() が背景レイヤーも _resizeSingleLayerTexture() に渡している。
背景レイヤーは backgroundGraphics（PixiJS Graphics）で色を管理しており、
layerSprite の renderTexture は空テクスチャ。
これを黒クリアして上書きするため、背景が黒になる。

■ 修正①
resizeLayerTextures() のループ条件に isBackground を追加する。

【変更前】
for (const layer of this.getLayers()) {
    if (!layer.layerData || layer.layerData.isFolder) continue;
    this._resizeSingleLayerTexture(layer, newWidth, newHeight, offsetX, offsetY);
}

【変更後】
for (const layer of this.getLayers()) {
    // 背景レイヤーとフォルダはスキップ（背景は別途 backgroundGraphics で処理済み）
    if (!layer.layerData || layer.layerData.isFolder || layer.layerData.isBackground) continue;
    this._resizeSingleLayerTexture(layer, newWidth, newHeight, offsetX, offsetY);
}

■ 原因②
_resizeSingleLayerTexture() 内の clearColor / clearAlpha も PixiJS v8 では無効。
旧内容のコピー時に黒クリアが発生する。

■ 修正②
_resizeSingleLayerTexture() の renderer.render() を以下に変更する。

【変更前】
this.app.renderer.render({
    container: tempSprite,
    target: newRT,
    clear: true,
    clearColor: 0x000000,
    clearAlpha: 0
});

【変更後】
this.app.renderer.render({
    container: tempSprite,
    target: newRT,
    clear: [0, 0, 0, 0]   // PixiJS v8 形式: 透明でクリアしてから旧内容を描画
});

■ 確認観点
- キャンバスを 400x400 → 600x600 にリサイズしても背景色（Bizarre #f0e0d6）が維持される
- 通常レイヤーの既存描画が新サイズでも保持される
- 新しく増えた領域が透明になっている（黒でない）

【調査3】液タブペン入力がキャンバスに届かない問題
ファイル: core-engine.js, system/drawing/pointer-handler.js, styles/main.css
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 現状
マウスの pointerdown は Console に出るが、ペンは何も出ない。
JavaScript レベル以前（DOM / ブラウザ / OS）でイベントが止まっている可能性大。

■ 調査ステップ1: ドキュメントレベルの診断ログ追加
core-engine.js の initialize() 末尾（全初期化完了後）に以下を追加する。

// [一時診断] ペン入力のドキュメントレベル到達確認
// 確認完了後は削除すること
document.addEventListener('pointerdown', (e) => {
    console.log('[DOCUMENT CAPTURE] pointerdown', JSON.stringify({
        pointerType: e.pointerType,
        button: e.button,
        buttons: e.buttons,
        pressure: e.pressure,
        target: e.target?.tagName,
        targetId: e.target?.id,
        targetClass: String(e.target?.className || '').substring(0, 40)
    }));
}, { capture: true });  // capture: true で最優先で受け取る

■ 調査ステップ2: touch-action の補強
styles/main.css の .canvas-area に touch-action: none を追加する。

【変更前】
.canvas-area { 
    background: var(--futaba-background); 
    position: relative; 
    overflow: hidden; 
    display: flex; 
    justify-content: center; 
    align-items: center; 
}

【変更後】
.canvas-area { 
    background: var(--futaba-background); 
    position: relative; 
    overflow: hidden; 
    display: flex; 
    justify-content: center; 
    align-items: center;
    touch-action: none;          /* 追加: スタイラス入力をブラウザがキャプチャしないよう */
    -ms-touch-action: none;      /* 追加 */
}

■ 調査ステップ3: PixiJS canvas への明示的スタイル強制
core-engine.js の initialize() 内、drawingCanvasContainer.appendChild(this.app.canvas) の直後に追加する。

// PixiJS が canvas スタイルを上書きする場合に備えて明示設定
if (this.app.canvas) {
    this.app.canvas.style.touchAction = 'none';
    this.app.canvas.style.msTouchAction = 'none';
    this.app.canvas.style.pointerEvents = 'auto';
}

■ 調査後の TegakiConsole 記録と報告
修正後にブラウザをリロードし、液タブペンでキャンバスに触れて以下を確認する。

確認ポイント:
1. [DOCUMENT CAPTURE] pointerdown が出るか？
   → 出ない場合: ブラウザ/OSレベルの問題（Chrome の Wacom タブレット設定を確認）
   → 出る場合: target を確認（どの DOM 要素を叩いているか）

2. target が canvas でない場合:
   → その要素に pointer-events や z-index の問題がある
   → 報告書に target: XXX を記載すること

3. target が canvas の場合でも [PointerHandler] が出ない場合:
   → PixiJS の EventSystem が干渉している可能性
   → pointer-handler.js の attach() を capture: true に変更して再試行

■ 一時ログの削除条件
調査完了・原因特定後は [DOCUMENT CAPTURE] ログを削除すること。
