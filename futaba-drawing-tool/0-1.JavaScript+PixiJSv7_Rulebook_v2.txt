# JavaScript + PixiJS v7 Rulebook v2（UI分割対応・責務分離強化版）

## 1. 基本理念
- 安定動作・可読性・拡張性を優先し、将来的なTypeScript + Vite + ES6移行を意識する。
- 現在はJavaScript+fetch API分割でありグローバル変数形式で実装する。
- ファイル間の依存は明確化し、循環参照を作らない。
- **責務分離の強化**: 単一ファイルへの機能集中を避け、適切な粒度でファイル分割する。
- **UI分割設計**: UIシステムを複数の専門ファイルに分離し、メンテナンス性を向上させる。
- PixiJS v7の公式API仕様に沿い、非推奨機能は使用しない。
- PixiJS v7の範囲で可能な限りモダンな実装を行い、ブラウザ互換は考慮せず最新Chrome APIを優先的に活用する。
- ES2023構文使用OK（可読性のためなら）
- OffscreenCanvasや最新Canvas APIも使用OK
- 初期開発段階ではPC＋液タブ操作を優先し、タッチ操作は未実装とする。将来的に追加予定の箇所はコメントアウトで残し、後の拡張を容易にする
- 責務を明確に分離し、保守しやすいコードにしてください。DRY原則とSOLID原則に基づいて、コンポーネントを設計し、各ファイルが単一の責任を持つように整理し、重複コードを排除する事。ボイラープレートが多すぎないか、適切なのかも確認。
- **ファイルサイズ制限**: 単一ファイルは800行以下を目安とし、それを超える場合は責務に応じて分割する。

## 2. ファイル構成ルール（UI分割対応拡張版）
拡張された11ファイル構成（責務分離強化版）：

### 2.1 コア（必須）ファイル群
1. **app-core.js**  
   - PixiJSアプリ生成・描画ループ管理・リソースロード。
   - Application / Stage / Renderer 初期化はここに集約。

2. **drawing-tools.js**  
   - 各描画ツールや機能（ペン、消しゴム、ベクターペンなど）をクラス化。
   - PixiJS Graphics / Mesh / Container の使用はツール内に閉じる。

3. **history-manager.js**
   - アンドゥ・リドゥ・履歴管理システム。
   - 操作履歴の記録と復元機能を提供。

4. **config.js** 🆕
   - アプリケーション全体の設定値を定数として管理。
   - デフォルト値、制限値、UI設定値を統一管理。

### 2.2 UI システム（分割強化版）
5. **ui-manager.js**（縮小版）
   - **責務**: UI全体の統合制御のみ（800行以下）
   - 各専門UIマネージャーの初期化・連携調整
   - **削除された責務**: プリセット管理、パフォーマンス監視、詳細UIロジック

6. **ui/components.js** ✅（既存・軽微修正のみ）
   - 基本UIコンポーネント（Slider, Popup, StatusBar, PresetDisplay）
   - 独立性の高いUI要素

7. **ui/preset-manager.js** 🆕
   - プリセット管理専門システム
   - プリセット選択、リセット、履歴記録、プレビュー生成

8. **ui/performance-monitor.js** 🆕
   - パフォーマンス監視専門システム
   - FPS、メモリ、GPU使用率の監視・表示

9. **ui/ui-events.js** 🆕
   - UIイベント処理専門システム
   - キーボードショートカット、マウス操作の統合管理

### 2.3 拡張機能ファイル群
10. **settings-manager.js**
    - アプリ全体の動作設定を一元管理。
    - 高DPI設定、ユーザー設定保存。

11. **transform-manager.js**（新規）
    - キャンバス操作とオブジェクト変形の一元管理。
    - キャンバス全体の移動・回転・反転機能。
    - アクティブレイヤー内コンテンツの移動機能。

### 2.4 将来実装予定
12. **layer-manager.js**（将来実装）
    - レイヤー機能の管理（追加、削除、並び替え）。
    - レイヤー表示・非表示、透明度制御。
    - アクティブレイヤーの管理。

### 2.5 ファイル間依存関係（UI分割対応）
```
config.js (設定値統一)
├─ app-core.js
│  ├─ drawing-tools.js
│  └─ history-manager.js
├─ settings-manager.js
├─ transform-manager.js
├─ ui-manager.js (統合制御のみ)

- 下位から上位への依存のみ

- ui/ 配下は ui-manager.js からのみアクセス
```

## 3. UI分割設計原則

### 3.1 UI責務分離の指針
```javascript
// 分割基準
1. 機能単位での分離
   - プリセット管理 → ui/preset-manager.js
   - パフォーマンス監視 → ui/performance-monitor.js
   - イベント処理 → ui/ui-events.js

2. サイズ制限による分離
   - 単一ファイル800行超過時は自動分割対象
   
3. 独立性による分離
   - 他システムとの依存が少ない機能は分離

4. テスタビリティによる分離
   - 単独でテスト可能な機能は分離
```

### 3.2 分割後のファイルサイズ目標
| ファイル名 | 目標サイズ | 主な責務 |
|-----------|-----------|----------|
| ui-manager.js | 300-500行 | 統合制御のみ |
| ui/preset-manager.js | 200-400行 | プリセット専門 |
| ui/performance-monitor.js | 100-200行 | パフォーマンス専門 |
| ui/ui-events.js | 200-300行 | イベント専門 |
| ui/components.js | 400-600行 | 基本UI要素 |

### 3.3 独立性の高いコンポーネント（ui/配下）
```javascript
// 高独立性コンポーネント
class SliderController {
    // スライダー操作のみ・他システム依存最小
}

class PopupManager {
    // ポップアップ表示管理・完全独立
}

class StatusBarManager {
    // ステータス表示・他システムへの影響なし
}

class PresetDisplayManager {
    // プリセット表示・データ受信のみ
}

// 専門システム
class PresetManager {
    // プリセット管理・ロジック集約
}

class PerformanceMonitorSystem {
    // パフォーマンス監視・完全独立
}

class UIEventSystem {
    // UIイベント・入力処理統合
}
```

## 4. コードスタイル規約（UI分割対応）

### 4.1 命名規則
- **変数 / 関数**: `camelCase`（例：`drawLine`, `pointerPosition`）
- **定数**: `SCREAMING_SNAKE_CASE`（例：`CANVAS_WIDTH`, `BG_COLOR`）
- **クラス / コンストラクタ関数**: `PascalCase`（例：`VectorPenTool`, `PresetManager`）
- **ファイル名**: `kebab-case`（例：`preset-manager.js`, `ui-events.js`）
- **UIイベント名**: 定数化（例：`UI_EVENTS.SLIDER_CHANGE`）
- **UI設定定数**: `UI_CONFIG`に集約（例：`UI_CONFIG.POPUP_FADE_TIME`）
- **アプリ設定定数**: `CONFIG`に集約（例：`CONFIG.DEFAULT_BRUSH_SIZE`）
- **PixiJSオブジェクト変数**: 役割接尾辞（`backgroundLayer`, `drawingContainer`）

### 4.2 UI関連関数名の原則
- **コンポーネント操作**: `update{Component}`, `show{Component}`, `hide{Component}`
- **イベント処理**: `handle{Event}`, `on{Event}`
- **状態管理**: `get{State}`, `set{State}`, `toggle{State}`
- **管理クラスの操作**: `{action}{Target}`（例：`selectPreset`, `resetActivePreset`）

### 4.3 ファイル分割時の命名規則
```javascript
// 管理クラス命名
{Domain}Manager    // 例: PresetManager, LayerManager
{Domain}System     // 例: PerformanceMonitorSystem, UIEventSystem
{Domain}Controller // 例: SliderController, TransformController

// ファイル命名
{domain}-manager.js    // 例: preset-manager.js
{domain}-system.js     // 例: ui-events.js
{domain}-controller.js // 例: transform-controller.js
```

```javascript
// 依存関係チェック強化
function checkUISystemDependencies() {

        'CONFIG', // config.js
        'PresetManager', // ui/preset-manager.js  
        'PerformanceMonitorSystem', // ui/performance-monitor.js
        'UIEventSystem' // ui/ui-events.js

    }
}
```

## 13. Phase2-4対応指針

### 13.1 Phase2: デフォルト値変更対応
```javascript
// config.js での統一管理
export const CONFIG = {
    DEFAULT_BRUSH_SIZE: 4,        // 16 → 4
    DEFAULT_OPACITY: 1.0,         // 0.85 → 1.0 (100%)
    MAX_BRUSH_SIZE: 500,          // 100 → 500
    SIZE_PRESETS: [1, 2, 4, 8, 16, 32]
};

// ui/preset-manager.js での適用
class PresetManager {
    createDefaultPresets() {
        return CONFIG.SIZE_PRESETS.map(size => ({
            size: size,
            opacity: CONFIG.DEFAULT_OPACITY // 100%統一
        }));
    }
    
    // プレビュー制限（外枠を超えない）
    calculatePreviewSize(actualSize) {
        if (actualSize <= 32) {
            // 線形スケール
            return Math.min(CONFIG.PREVIEW_MAX_SIZE, 
                           actualSize / 32 * CONFIG.PREVIEW_MAX_SIZE);
        } else {
            // 対数スケール（大サイズ対応）
            const logScale = Math.log(actualSize / 32) / 
                           Math.log(CONFIG.MAX_BRUSH_SIZE / 32);
            return Math.min(CONFIG.PREVIEW_MAX_SIZE, 
                          CONFIG.PREVIEW_MAX_SIZE * (0.7 + logScale * 0.3));
        }
    }
}
```
