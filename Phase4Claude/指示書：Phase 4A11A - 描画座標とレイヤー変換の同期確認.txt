📜 指示書：Phase 4A11A - 描画座標とレイヤー変換の同期確認（core-engine.js用）
🎯 目的
modelMatrix を使ってレイヤーを移動した後でも、ユーザーがマウスで描こうとした位置に正しく描画が行われるように、描画座標とレイヤー変換の同期を実現します。

現在の core-engine.js には transformWorldToLocal() の定義がありますが、描画処理に適用されておらず、ズレの原因になっています。

🛠 やるべきこと
✅ 1. drawToLayerTexture() または描画に関わる関数内で、transformWorldToLocal() を 必ず使ってください
js
コピーする
編集する
const local = transformWorldToLocal(worldX, worldY, layer.modelMatrix);
✅ 2. ローカル座標（local.x, local.y）を使って描画処理を行ってください
これは必須です。変換せずに worldX, worldY のまま描画をすると、レイヤーを移動したときに描画結果がズレて見えます。

js
コピーする
編集する
drawToLayerCanvasAt(local.x, local.y);  // ← これは仮の例です。既存の描画関数に置き換えてください
🧪 3. 座標変換ログを追加してください（デバッグ用）
js
コピーする
編集する
console.log(`[描画位置] World(${worldX}, ${worldY}) → Local(${local.x}, ${local.y})`);
✅ 守るべき条件
transformWorldToLocal() は既に定義されているものを使ってください（再定義不要）

layer.modelMatrix が未定義または壊れている場合は、描画をスキップしてください

既存の描画処理（ペンツールなど）にローカル座標を適用するよう改修してください

他の機能（保存、UI、イベントなど）を壊さず、座標変換のみに集中してください

✅ 成功条件
レイヤーを移動しても、描いた線がマウスカーソルと一致した位置に現れること

console.log で出力されるローカル座標が、実際の描画位置と一致していること

transformWorldToLocal() の座標変換が毎回使われていること

✍️ 補足
今回添付した Phase4A11A-0/core-engine.js をベースに、必要最小限の修正だけで実現してください。

drawToLayerTexture() もしくは描画位置を受け取る関数が複数ある場合は、全てに transformWorldToLocal() を適用してください。
