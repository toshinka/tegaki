# アニメエフェクト特化 WebUI 動画編集ツール 実装計画 (v2)

アニメ動画に特化し、透明背景の素材を重ねて光の効果（ディフュージョン等）を時間変化（キーフレーム・イージング）で制御できる、ブラウザ完結の非破壊動画編集ツールの実装計画です。

ユーザー様からのフィードバックを反映し、対応フォーマットの追加、将来的な3D表現（深度・光源）を見据えた基盤設計、およびAI開発支援を意識したコーディング規約を追加しました。

---

## 1. 目標・コンセプト
*   **非破壊編集 (Non-Linear Editing)**: オリジナル画像を保持したままメタデータで編集を管理し、リアルタイムにレンダリングします。
*   **初心者向けシンプルUI**: iPadの有名動画編集アプリ（CapCutやLumaFusion等）のような、直感的で分かりやすいシンプルなUIを目指します。色やレイアウトは後から容易に変更できるよう、CSS変数等で柔軟な構造にします。
*   **AIフレンドリーなコードベース**: 将来的に別のAIエージェントがコードを編集・拡張しやすいよう、明確な命名規則と、ファイル冒頭の「依存関係・用途を示すヘッダーコメント」を徹底します。
*   **3D対応を見据えたレンダリング基盤**: 将来のアップデート（3D空間へのレイヤー配置、光源の移動による影・ハイライトの表現等）に備え、最初からWebGPUの3D座標系（Z軸/深度）を意識した基礎設計を行います。

---

## 2. 提案アーキテクチャ・技術スタック

1.  **レンダリングエンジン: WebGPU**
    *   **処理**: 2Dの動画編集でありながら、内部的には3D空間のポリゴン（Quad）に画像を貼り付けて描画する方式（コンポジター）を採用します。これにより、将来的な「立体的な絵の配置（Z値の操作）」や「動く光源（Lightingシェーダー）」の追加がシームレスに行えます。
    *   **エフェクト**: WGSLシェーダーを用いて、ディフュージョン、ブラー、ブレンドモード（加算・オーバーレイ等）を高速処理します。
2.  **メディアデコード / エンコード (GIF / APNG / WEBP 対応)**
    *   **入力**: MP4/WebMに加えて、アニメーションで多用される **APNG, GIF, アニメーションWEBP** のインポートに対応します（ブラウザのImageDecoder APIや各種パーサーライブラリを活用）。
    *   **出力**: 上記フォーマットでのエクスポート機能もサポートします。
3.  **UIフレームワーク: Vite + React + TypeScript**
    *   UIの柔軟性を担保しつつ、高速な開発サイクルを実現します。
4.  **状態管理 (State Management): Zustand**
    *   タイムラインデータの一元管理とプロジェクトファイルの保存・復元に用います。

---

## 3. コーディング規約 (AIフレンドリー)

他のAIが後からコードを修正する際の「勘違い・ミス」を防ぐため、以下のルールを徹底します。

*   **ヘッダーコメントの義務化**: すべての主要ファイルの先頭に、以下のフォーマットでコメントを記述します。
    ```typescript
    /**
     * @file {ファイル名}
     * @description {このファイルが何をするか、どういう役割か}
     * @relatedFiles {連携する他のファイルやコンポーネント (例: Timeline.tsx, useTrackState.ts)}
     * @mainFunctions {主要な関数やエクスポートしているクラス (例: renderFrame(), applyShader())}
     * @state {参照している、あるいは変更しているグローバル状態 (例: timelineState.tracks)}
     */
    ```
*   **明確な命名規則**: `handleUpdate` のような曖昧な名前ではなく、`updateTrackOpacityKeyframe` のような、何をしているか一目でわかる命名を採用します。

## 5. 初期フェーズの開発ステップ (MVP構築)
※音声の対応は後回しとし、まずは映像処理に特化します。

1.  **環境構築**: 指定ディレクトリに Vite + React 環境を作成。
2.  **UIの土台作り**: プレビュー（Canvas）、タイムライン（簡易版）、インスペクター（プロパティ調整）のレイアウト構築。
3.  **メディア読み込み基盤**: APNG / GIF / WEBP / 通常画像の読み込みとフレーム分割の検証。
4.  **WebGPUレンダリング基盤**: 3D空間（Z軸）を意識したQuad（四角形ポリゴン）への画像のテクスチャマッピングと描画。
5.  **基本エフェクトの実装**: 拡縮、回転、透明度、および「ブラー」または「加算合成」のテスト実装。
