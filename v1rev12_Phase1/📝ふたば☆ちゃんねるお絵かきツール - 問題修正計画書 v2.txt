# ふたば☆ちゃんねるお絵かきツール - 問題修正計画書 v2 (パワーアップ版)

## 📋 発生している問題の分析 (更新)

### 🚨 **問題1**: ペンツールでアンドゥ後再描画時のPixiJSエラー


**原因分析** (更新):
- PixiJS Spriteオブジェクトが null/undefined (textureソース無効時)
- アンドゥ処理後の描画復元時にGraphicsオブジェクトの参照不正、texture劣化による解像度損失
- 履歴管理とPixiJS描画オブジェクト間の同期不整合、GPUガベージコレクションの影響

### 🚨 **問題2**: 消しゴムツールへの切り替えが不可能（解決済）
**原因分析** (変更なし):
- `tool-manager.js`でツール切り替えは正常動作
- `ui-manager.js`の`handleGeneralToolButtonClick`でeraser-toolは処理されている
- しかし実際のツール状態変更が反映されていない可能性

### 🚨 **問題3**: ペンツールポップアップが表示されない (Phase1後も残存)
**原因分析** (更新):
- `ui-manager.js`からペンツール専用機能が`PenToolUI`に移譲済みだが、初期化タイミングの競合 (DOMロード後でない)
- `popup-manager.js`はPenToolUI配下で動作するが、イベントリスナー重複/バブリング失敗
- ペンツールボタンクリック処理の重複・競合、ui-manager.jsの残存コードが干渉

### 🚨 **新規問題4**: アンドゥ後の画像劣化
**原因分析**:
- Textureスナップショットの再レンダリング時、canvas抽出の解像度損失 (anti-aliasing不整合)
- restore時のspriteスケーリングエラー

### **Phase 1: 緊急修正（エラー解決優先） (拡張: タイムライン詳細化)**

#### **Task 1.1**: PixiJS Spriteエラーの修正 (更新: nullチェック強化)
- **対象ファイル**: `app-core.js`, `history-manager.js`
- **修正内容**:
  - `captureDrawingState()`でtexture.validate()追加、null時はfallback空texture作成
  - `restoreDrawingState()`でif (!texture.baseTexture) { console.warn('Invalid texture'); return new PIXI.Sprite(PIXI.Texture.EMPTY); }
  - Graphics/Spriteオブジェクト生成時のエラーハンドリング追加 (try-catch)
- **SOLID原則適用**: 単一責任原則 - 描画復元ロジックの分離
- **タイムライン**: Step 1内で優先

#### **Task 1.2**: 消しゴムツール切り替え修正 （解決済）
- **対象ファイル**: `tool-manager.js`, `eraser-tool.js`
- **修正内容**:
  - ツールアクティベーション処理のデバッグ
  - イベントリスナー設定の確認・修正
  - ツール状態とUI状態の同期確保
- **DRY原則適用**: ツール切り替えロジックの共通化

#### **Task 1.3**: ペンツールポップアップ表示修正 (更新: イベントデバッグ追加)
- **対象ファイル**: `pen-tool-ui.js`, `popup-manager.js`, `main.js`
- **修正内容**:
  - PenToolUI初期化をDOMContentLoaded後に移動
  - PopupManager.show()にdisplay: 'block'強制、イベントリスナー重複除去 (removeEventListener)
  - ペンツールボタンクリック処理の統合、console.logでトリガー確認
- **タイムライン**: Step 3内で拡張 (イベントバブリングテスト追加)

#### **Task 1.4**: アンドゥ画像劣化修正 (新規)
- **対象ファイル**: `history-manager.js`, `app-core.js`
- **修正内容**:
  - captureState()でrenderer.resolution = 2; (高解像度キャプチャ)
  - restore時、sprite.smoothing = false; で劣化防止
  - Texture.from()前にcanvas.toDataURL()で検証
