# JavaScript + PixiJS v8 Rulebook v1）

## 1. 基本理念

### 1.1 核心方針
- **安定動作・可読性・拡張性を優先**
- **PixiJSエコシステム活用**により、車輪の再発明を避け、標準実装を採用する
- **統合システム設計**により、座標変換・履歴管理・デバッグの一元化を推進する
- **段階的品質向上**と既存システムとの互換性維持を両立する
- **AI協働最適化**を考慮した標準パターンの採用を推奨する

### 1.2 実装原則
- 現在はJavaScript+fetch API分割でありグローバル変数形式で実装する
- ファイル間の依存は明確化し、循環参照を作らない
- **SOLID・DRY原則準拠**: 単一責任・責務分離・重複コード排除を徹底する
- **統合システム設計**: 座標変換・履歴管理・監視機能の統合による効率化
- **モジュール分割設計**: 機能ごとの専門ファイル分離による保守性向上
- PixiJS v8の公式API仕様に沿い、非推奨機能は使用しない
- ES2023構文使用OK、OffscreenCanvasや最新Canvas APIも使用OK
- 最新Chrome+デスクトップPC環境を想定、後方互換は意識しない
- **ファイルサイズ制限**: 単一ファイルは800行以下を目安とし、それを超える場合は責務に応じて分割する
- 初期開発段階ではPC＋液タブ操作を優先し、タッチ操作は未実装とする。将来的に追加予定の箇所はコメントアウトで残し、後の拡張を容易にする
- Rulebookは恒久的な指針・規約であり、改修の成果ではなく運用ルールを記載すべき
  ・Rulebook：指針策定・構造ガイドライン・品質基準・移行戦略
  ・計画書：具体的手順・ファイル別詳細・実装ガイド・テスト手順

## 2. ファイル構成ルール（UI分割対応版）
**責務レベルでの2段階分割**を採用した8ファイル構成：

### 2.1 コア（必須）ファイル群
1. **app-core.js**（旧 core-engine.js）  
   - PixiJSアプリ生成・描画ループ管理・リソースロード。
   - Application / Stage / Renderer 初期化はここに集約。

2. **drawing-tools.js**（旧 tools-features.js）  
   - 各描画ツールや機能（ペン、消しゴム、ベクターペンなど）をクラス化。
   - PixiJS Graphics / Mesh / Container の使用はツール内に閉じる。

3. **ui-manager.js**（統括管理専用に縮小）  
   - UIの統括管理・メイン制御に集約
   - `UIManager`クラス（メイン統括管理）
   - `SettingsPanelManager`クラス（UIManagerと密結合のため残存）

4. **ui/components.js**（新規作成）
   - 独立性の高いUIコンポーネント群を分離
   - `SliderController`クラス（スライダー操作）
   - `PopupManager`クラス（ポップアップ管理）
   - `StatusBarManager`クラス（ステータス表示）
   - `PresetDisplayManager`クラス（プリセット表示）

5. **history-manager.js**
   - アンドゥ・リドゥ・履歴管理システム。
   - 操作履歴の記録と復元機能を提供。

### 2.2 拡張機能ファイル群
6. **settings-manager.js**
   - アプリ全体の動作設定を一元管理。
   - 高DPI設定、ショートカット機能、ユーザー設定保存。

7. **transform-manager.js**
   - キャンバス操作とオブジェクト変形の一元管理。
   - キャンバス全体の移動・回転・反転機能。
   - アクティブレイヤー内コンテンツの移動機能。

8. **layer-manager.js**
   - レイヤー機能の管理（追加、削除、並び替え）。
   - レイヤー表示・非表示、透明度制御。
   - アクティブレイヤーの管理。

### 2.3 ファイル間依存関係（UI分割対応）
```
app-core.js
├─ drawing-tools.js
├─ ui-manager.js
│  └─ ui/components.js ★依存追加
├─ history-manager.js
├─ settings-manager.js
├─ transform-manager.js
└─ layer-manager.js

ui/components.js
├─ UI_CONFIG（UI設定定数）
├─ UI_EVENTS（UIイベント定数）
└─ 4つの独立UIクラス

settings-manager.js
└─ app-core.js (設定適用)

transform-manager.js
├─ app-core.js (キャンバスアクセス)
└─ layer-manager.js (アクティブレイヤー取得)

layer-manager.js
├─ app-core.js (レイヤーコンテナアクセス)
└─ history-manager.js (レイヤー操作履歴)
```

### 2.4 UI分割の具体的効果
**分割前（ui-manager.js）**: 約1800行
- UIManager + 5つのクラス混在
- 複数の責務が同一ファイルに混在

**分割後（ui-manager.js + ui/components.js）**: 
- **ui-manager.js**: 約700行（メイン統括管理のみ）
- **ui/components.js**: 約600行（独立UIコンポーネント群）
- **ファイルサイズ約60%削減効果**

## 3. UIコンポーネント分離ルール
### 3.1 独立性の高いコンポーネント（ui/components.js移行対象）
```javascript
// 移行対象クラスの特徴
class SliderController {
    // スライダー操作のみに特化・他システムとの依存最小
}

class PopupManager {
    // ポップアップ表示管理のみ・独立性が高い
}

class StatusBarManager {
    // ステータス表示のみ・他システムへの影響なし
}

class PresetDisplayManager {
    // プリセット表示管理・ツールシステム依存だが分離可能
}
```

### 3.2 統括管理残存クラス（ui-manager.js残存）
```javascript
// 残存対象クラスの特徴
class UIManager {
    // メイン統括管理・全体調整・中央集権的処理
}

class SettingsPanelManager {
    // UIManagerと密結合・分離困難・設定パネル専用管理
}
```

## 4. コードスタイル規約（UI分割対応）
### 命名規則
- **変数 / 関数**: `camelCase`（例：`drawLine`, `pointerPosition`）
- **定数**: `SCREAMING_SNAKE_CASE`（例：`CANVAS_WIDTH`, `BG_COLOR`）
- **クラス / コンストラクタ関数**: `PascalCase`（例：`VectorPenTool`, `UIManager`）
- **UIイベント名**: 定数化（例：`UI_EVENTS.SLIDER_CHANGE`）
- **UI設定定数**: `UI_CONFIG`に集約（例：`UI_CONFIG.POPUP_FADE_TIME`）
- **PixiJSオブジェクト変数**: 役割接尾辞（`backgroundLayer`, `drawingContainer`）

### UI関連関数名の原則
- **コンポーネント操作**: `update{Component}`, `show{Component}`, `hide{Component}`
- **イベント処理**: `handle{Event}`, `on{Event}`
- **状態管理**: `get{State}`, `set{State}`, `toggle{State}`

## 7. イベント・入力ルール（UI分割対応）
- **描画系**: 常にPixi座標系を使用（`event.data.getLocalPosition(layer)`）
- **UI系**: DOM座標系とPixi座標系の適切な変換
- **イベント分離**: UIイベントは`stopPropagation()`で描画層と分離
- **イベント種別**: `pointerdown / pointermove / pointerup`で統一
- **UI専用イベント**: `UI_EVENTS`定数で一元管理

---

## 8. 状態管理（UI分割対応）
- **描画状態**: `app-core`の`state`に集約
- **UI状態**: 各UIコンポーネント内で自己管理
- **設定状態**: `settings-manager`で一元管理
- **変形状態**: `transform-manager`で管理
- **レイヤー状態**: `layer-manager`で管理
- **履歴管理**: 全操作種別を統合記録（UI操作も含む）

## 9. 各ファイルの詳細責務（UI分割対応版）

### 9.1 ui-manager.js（縮小・統括専用）
**主責務**: UI全体の統括管理
- **全体調整**: 各UIコンポーネント間の連携制御
- **メイン制御**: アプリケーション全体のUI制御
- **設定パネル**: SettingsPanelManagerクラス（密結合のため残存）
- **イベント統括**: UI全体のイベントフロー管理

### 9.2 ui/components.js（新規作成）
**主責務**: 独立UIコンポーネント群
- **SliderController**: スライダー操作とイベント処理
- **PopupManager**: ポップアップの表示・非表示・アニメーション
- **StatusBarManager**: ステータス情報の表示更新
- **PresetDisplayManager**: プリセットの表示・選択管理
- **UI_CONFIG**: UI専用設定定数の定義
- **UI_EVENTS**: UI専用イベント定数の定義

### 9.3 settings-manager.js
**主責務**: アプリケーション設定の一元管理
- **高DPI設定**: resolution値の動的切り替え
- **ショートカットキー**: キーボード入力監視とアクション実行
- **ユーザー設定**: LocalStorage連携（将来実装）
- **設定UI**: 設定変更用のダイアログ管理（ui/components.js連携）
- **設定履歴**: 設定変更も履歴管理対象とする

### 9.4 transform-manager.js
**主責務**: 変形操作の一元管理
- **キャンバス変形**: 全体移動・回転・反転・ズーム
- **レイヤーコンテンツ変形**: アクティブレイヤー内オブジェクトの移動
- **座標変換**: 変形状態を考慮した座標計算ユーティリティ
- **変形履歴**: transform操作も履歴管理対象

### 9.5 layer-manager.js
**主責務**: レイヤー機能の完全実装
- **レイヤー管理**: 作成・削除・複製・並び替え・名前変更
- **レイヤー状態**: 表示・非表示・透明度・ロック状態管理
- **アクティブレイヤー**: 描画対象レイヤーの管理
- **レイヤー履歴**: レイヤー操作も履歴管理対象

UI分割を考慮した推奨初期化順序：

1. **app-core.js**: PixiJSアプリケーション基盤
2. **settings-manager.js**: 設定適用（高DPI等）

7. **ui/components.js**: UIコンポーネント初期化 ★追加
8. **ui-manager.js**: UI統合・最終調整

### HTML読み込み順序
```html

<!-- その後UI統括管理を読み込み -->
<script src="ui-manager.js"></script>
```

## 11. 禁止事項

### 11.1 基本禁止事項（継続）
- DOM直接操作（Pixi外の`querySelector`等）
- Pixi内部プロパティへの直接アクセス
- 複数責務を1関数にまとめる
- **単一ファイル800行超過**（分割必須）

### 11.2 統合システム固有禁止事項
- **統合システム迂回**: TransformManager・HistoryManager・DebugManagerを経由しない直接操作
- **分散状態管理**: 統合システムと並行した独自状態管理の新規作成
- **履歴システム重複**: 独自履歴機能の新規実装（LogHistoryManager使用必須）
- **座標変換分散**: 統合システム外での座標・変形処理の実装
- **監視機能重複**: 独自デバッグ・監視機能の新規実装

### 11.3 Phase移行期禁止事項
- **段階的移行無視**: 一度に複数システム統合変更禁止
- **互換性破壊**: 既存API削除による破壊的変更禁止
- **性能劣化放置**: 統合前後の性能測定・改善義務
- **エラーハンドリング省略**: 統合システム障害対応必須
