📌 目的：描画・移動・入力の三者で、Y軸と座標の一貫性を保つ
🎯 背景
Phase4A11Bでは modelMatrix の導入と、transformWorldToLocal による描画座標の補正が導入された。しかし：

WebGLの表示に使われる projectionMatrix や modelMatrix のY軸と、

マウス入力 (getCanvasCoordinates) や描画ロジックのY軸が、

統一されておらず、描画・移動・表示が一致していない。

🛠 修正タスク一覧（以下すべて満たすこと）
✅ 1. projectionMatrix のY軸を Canvas 準拠にする
js
コピーする
編集する
// webgl-engine.js にて
// Y軸の下方向が正の方向（Canvasと一致）
mat4.ortho(this.projectionMatrix, 0, this.superWidth, this.superHeight, 0, -1, 1);
※既に正しい方向なら変更不要

✅ 2. transformWorldToLocal() にY軸反転を入れない
js
コピーする
編集する
return {
  x: localPos[0],
  y: localPos[1]
};
✅ 3. getCanvasCoordinates() の出力と描画座標がズレないように、canvasとglの解像度を統一
js
コピーする
編集する
const rect = canvas.getBoundingClientRect();
const scaleX = canvas.width / rect.width;
const scaleY = canvas.height / rect.height;
const x = (e.clientX - rect.left) * scaleX;
const y = (e.clientY - rect.top) * scaleY;
確認点：

canvas.width は gl.viewport() と一致しているか？

canvas.height は WebGL用描画領域の高さと一致しているか？

✅ 4. ペン描画・消しゴム・バケツツール等の全座標は transformWorldToLocal() のみ通す
例：

js
コピーする
編集する
const { x: localX, y: localY } = transformWorldToLocal(worldX, worldY, activeLayer.modelMatrix);
ctx.lineTo(localX, localY);
→ ctx.lineTo(worldX, worldY) のような未変換描画を排除

✅ 5. レイヤー移動時、transform後の modelMatrix を必ず saveState() に保存・復元
modelMatrix が復元されても、描画時にその行列を反映していないと「移動してないように見える」ことがある

すべての描画時に modelMatrix を transformWorldToLocal() に渡して描くこと

✅ 6. （任意補正）レイヤー移動時の dx/dy に devicePixelRatio を加味（速度補正）

編集する
const ratio = window.devicePixelRatio || 1;
const dx = (coords.x - this.transformStartX) * ratio;

💬 備考
transformWorldToLocal() は 描画と入力を繋ぐ中核です。絶対にすべての描画関数で使うこと。

projectionMatrix を変えると見た目に直結するため、そこがY軸反転していないか常に確認。

modelMatrix の保存と復元が動いていても、表示と一致しなければ意味がないので、座標系一貫性がすべてのカギ。
