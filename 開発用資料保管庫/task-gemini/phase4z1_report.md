# Phase 4z1 — Playback Scope: All / Active Lane MVP (GEMINI報告)

## 1. 実施内容

### 🎞️ 再生スコープ (ALL / LANE) の実装
- **`ui/animation-table-popup.js`**
    - `playbackScope` 状態（'all' または 'activeLane'）を導入しました。
    - **動作仕様**:
        - `ALL`: 全てのレーンのクリップを合成してプレビュー・再生します。
        - `LANE`: 選択中のクリップがあるレーン、またはアクティブなレイヤーに対応するレーンのみをプレビュー・再生します。
    - **プレビュー合成のフィルタリング**:
        - `_renderFrameComposite` および `_renderOnionFrame` に `laneFilter` オプションを追加し、指定されたレーン以外の描画をスキップするようにしました。
    - **再生時の挙動**:
        - `play()` 開始時に `playbackScope` が 'activeLane' の場合、その時点でのアクティブレーンを `activePlaybackLaneId` として固定します。再生中に他のセルをクリックしても、プレビュー対象がブレないようにしました。
        - `stop()` 時に固定を解除します。

### 🎨 UI/UX の統合
- ヘッダー中央に `SCOPE: ALL | LANE` ボタンを新設しました。
- **視覚的フィードバック**:
    - 現在選択されているスコープを強調表示（太字・背景色変更）するようにしました。
    - `LANE` モード時はオニオンスキンもそのレーンのみに限定されるため、複雑なアニメーションの特定パートに集中して作業しやすくなりました。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **動作確認**:
    1. `ALL` モードで全レーンの重なりを確認。
    2. `LANE` モードに切り替え、特定のレーンのみが表示されることを確認。
    3. `LANE` モードで再生を開始し、再生中に対象が固定されることを確認。
    4. 停止後に他レーンを選択し、プレビュー対象が切り替わることを確認。
