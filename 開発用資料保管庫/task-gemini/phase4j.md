# Phase 4j — アニメセル編集ミニ実用化MVP

## 概要

Phase 4i で、セルの有無に応じてキャンバス上のレイヤー表示を一時的に切り替える実描画プレビューが成立した。
ただし、まだセルの作成・削除・長さ変更が試作UI寄りで、実際にパラパラ確認するには操作が弱い。

Phase 4j では、RenderTexture snapshot や保存/Exportへ進む前に、セル編集の最小操作を使いやすくする。
目的は「同じレイヤーの表示タイミングを作る」作業を、ダイアログや上部 +/- ボタンに頼りすぎず、タイムライン上で完結しやすくすること。

## 重要な再発防止ルール

- RenderTexture snapshot、保存/ロード、Export、Album はこのPhaseで触らない。
- 旧 `animation-system.js` と `animationSystem.init()` は復活させない。
- 実レイヤーの `layerData.visible` は恒久的に書き換えない。
- レイヤーパネルD&Dの根本改修を混ぜない。
- `animation-table-popup.js` のテンプレート文字列やCSS注入ブロックを編集したら、前後50行を読み返してからビルドする。
- Spaceキーショートカットは追加しない。

## 現在の前提

- セルは `startFrame` と `duration` を持つ。
- セル追加、選択、Alt/Shiftクリック削除、ヘッダーの DURATION +/- は実装済み。
- 同一トラック内のセル重なりは `TrackModel.canPlaceCel()` / `setCelDuration()` で防止できる。
- フォルダトラックはセル配置対象外。
- 実描画プレビューは、セル有無による表示ON/OFFとして成立済み。

## 目標

- セルの右端をドラッグして duration を変更できる。
- セルを選択した状態で `Delete` / `Backspace` キーで削除できる。
- 空スロットのクリック追加、既存セルクリック選択、Alt/Shiftクリック削除は維持する。
- セル編集中も実描画プレビューが現在フレームに追従する。
- フォルダ行ではセル編集できない。

## 実装範囲

### 1. リタイミングハンドル

- セルブロック右端に小さなハンドルを表示する。
- ハンドルを左右ドラッグして `duration` を変更する。
- 1フレーム未満にはしない。
- タイムライン末尾を超えない。
- 同一トラック内の別セルと重なる方向へは伸ばさない。
- まずは右端伸縮だけでよい。左端伸縮やセル移動は後続。

### 2. キー削除

- アニメテーブル表示中、選択セルがある場合だけ `Delete` / `Backspace` で削除する。
- 入力欄編集中は処理しない。
- 削除後は `selectedCelId = null` にする。
- 削除後に `render()` し、プレビューも更新する。

### 3. 選択表示

- 選択セルの見た目は現状を維持してよい。
- 右端ハンドルは選択セルで少し分かりやすくしてよい。
- 大きなデザイン変更は不要。

## 禁止

- セルのドラッグ移動はしない。
- 複数セル選択、範囲選択、コピー/ペーストはしない。
- RenderTexture snapshot を作らない。
- 保存形式を変更しない。
- Export/Albumへ接続しない。
- レイヤーパネルD&DやフォルダD&Dを触らない。
