# Phase 4g — 新アニメテーブルの正式入口化MVP (GEMINI報告)

## 1. 実施内容

### 🚪 サイドバー導線の切り替え
- **`ui/ui-panels.js`**
    - 左サイドバーの `gif-animation-tool` クリックハンドラを更新。
    - 旧 `ui:toggle-timeline` イベントの emit を廃止し、新アニメテーブルポップアップ (`animationTable`) のトグ表示へ切り替え。
    - 新アニメテーブルを開く際、旧タイムラインが表示されている場合は自動的に閉じるように調整。

### 🏷️ UI表示の調整
- **ツール表示名の更新**:
    - 左下のステータスパネル等に表示されるツール名を「GIFアニメーション」から「アニメテーブル」へ変更。
- **サイドバー状態の連動**:
    - 新アニメテーブルの表示・非表示に合わせて、サイドバーのアニメアイコンの `active`（選択中）状態が切り替わるように連動ロジックを実装。
    - テーブルを閉じた際は、現在選択されているブラシ（ペン等）の表示状態へ安全に復帰。

## 2. 検証結果

### ✅ 動作確認
- **サイドバー操作**: 左サイドバーのアニメアイコンをクリックすると、新アニメテーブルが正常に開閉することを確認。
- **旧タイムラインとの共存**: 
    - 旧タイムライン内の `USE NEW TABLE` ボタンから新テーブルへ移行できる導線が維持されていることを確認。
    - サイドバーから新テーブルを開いた際、旧タイムラインが重ならないよう自動で閉じることを確認。
- **表示名**: ツール選択時に「アニメテーブル」と正しく表示されることを確認。
- **退行確認**: 他のツール（ペン、消しゴム、バケツ等）の切り替えや、設定・アルバム等のポップアップ開閉に影響がないことを目視確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **キーボードショートカット**: `keyboard-handler.js` 内の `GIF_TOGGLE_TIMELINE` は、本フェーズのスコープ（サイドバー導線の正式化）に基づき、あえて変更せず維持しています（旧タイムラインを開く挙動のまま）。必要に応じて次フェーズ以降で統一を検討します。
- **描画連動**: 再生ヘッドとレイヤー表示の連動は次フェーズの課題です。

## 3. 次フェーズへの提案 (Phase 4h)

1. **タイムラインベースの表示制御**: 現在フレームにセルがあるレイヤーのみをキャンバスに表示し、セルがない期間は一時的に非表示（または透明度調整）にする実描画連動の実装。
2. **ショートカットの統一**: アニメーション関連のキーボードショートカットを新アニメテーブルへ完全に紐付け。
