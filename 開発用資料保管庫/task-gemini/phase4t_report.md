# Phase 4t — Animation Onion Skin MVP (GEMINI報告)

## 1. 実施内容

### 🧅 オニオンスキン機能の実装
- **`ui/animation-table-popup.js`**
    - タイムラインプレビューにおいて、現在フレームの前後（-1, +1）を半透明で重ねて表示する「オニオンスキン」機能を実装しました。
    - **色分け表示**: 
        - 前フレーム：青系 (`0x4f8cff`) / 不透明度 30%
        - 次フレーム：赤系 (`0xff8c42`) / 不透明度 20%
    - **表示順序**: 前フレーム -> 次フレーム -> 現在フレーム の順で描画（現在フレームが最前面）。

### 🎨 プレビュー描画の構造化（リファクタリング）
- プレビューロジックを責務ごとに分割し、メンテナンス性を向上させました。
    - `_renderFrameComposite`: 指定フレームの全セルを合成して描画。
    - `_renderOnionSkins`: 前後のオニオンスキンを適切なオプション（alpha, tint）で描画。
    - `_renderCelPreview`: 指定されたセルを描画（alpha/tint オプション対応、実レイヤー fallback 制御）。

### 🔘 Onion Skin 切り替えUI
- アニメテーブルのヘッダーに `ONION` チェックボックスを追加。
- `PREVIEW` が有効な場合のみ動作し、再生中 (`isPlaying`) は自動的に非表示になるよう制御。

### 🖱️ 選択状態との連動
- **非選択時**: タイムライン全体の合成（Frame Composite）に対してオニオンスキンを適用。
- **セル選択中**: 選択されているトラックの前後のみを表示。作業中のレイヤーに集中できる設計にしました。

## 2. 検証結果

### ✅ 動作確認
- **表示連動**: `ONION` を ON にすると、現在の絵の背後に前後のコマが薄く（かつ色付きで）見えることを確認。
- **再生中制御**: 再生ボタンを押して自動進行している間は、オニオンスキンが非表示になり、本来のアニメーション確認を妨げないことを確認。
- **不透明度の継承**: 実レイヤー側の不透明度設定を維持したまま、オニオンスキンの alpha 乗算が正しく機能することを確認。
- **退行確認**: 保存（Album）、緊急復帰（Hospital）、および通常の描画・レイヤー操作に影響がないことを確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **範囲設定**: 現状は前後 1 コマ固定です。
- **カスタムカラー**: プレビュー色は定数固定であり、設定 UI は未実装です。

## 3. 次フェーズへの提案 (Phase 4u)

1. **自動キャプチャ (Auto-Key)**: 描画終了（`drawing:finished` 等）を検知し、選択中のセルへ Snapshot を自動的に書き込む「オートキー」機能の実装。
2. **セルのドラッグ移動 (Move)**: タイムライン上でのセルのドラッグによる位置変更。
3. **空アセットの自動生成**: セルを作成した際、空の `ClipAsset` を自動的に割り当てる機能。
