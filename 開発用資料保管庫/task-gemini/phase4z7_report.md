# Phase 4z7 — ClipAsset Internal Layer Inspector Skeleton (GEMINI報告)

## 1. 実施内容

### 🔍 内部レイヤー Inspector の実装
- **`ui/animation-table-popup.js`**
    - **`selectedInternalLayerId` の導入**: Asset Library 内での内部レイヤー選択状態を保持するようにしました。
    - **`_renderInternalLayerInspector()` の実装**: 選択中のアセットが持つ内部レイヤーをリスト表示するインスペクターを実装しました。
        - **表示項目**: レイヤー名、タイプ（raster等）、表示状態、不透明度、合成モード、スナップショット状態（Blank/Snapshot/None）。
        - **自動補完**: 内部レイヤーが空のアセットを参照した場合、その場で初期レイヤーを構成して表示する安全策を導入しました。
    - **Asset Library のレイアウト拡張**: 
        - これまでの「FOLDERS」「ASSETS」の右側に「INTERNAL LAYERS」セクションを追加し、3カラム構成の管理画面としました。
        - タイムラインでの選択中クリップが持つアセットが、自動的にインスペクターの表示対象となります（ライブラリ内での個別選択も可能）。

### 🎨 スタイリングと操作
- **Inspector UI**: ダークトーンのリスト形式を採用し、選択中のレイヤーをオレンジ色の左線で強調表示するようにしました。
- **イベント委譲**: ライブラリパネル全体でのクリックイベント委譲により、フォルダ・アセット・内部レイヤーの切り替えをスムーズに行えるようにしました。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **データ整合性**:
    1. クリップを選択 -> 自動的に右側の Inspector に内部レイヤー（Layer 1等）が表示されることを確認。
    2. ライブラリ内の別アセットをクリック -> インスペクターの内容が切り替わることを確認。
    3. 内部レイヤー行をクリック -> 選択ハイライトが移動することを確認。

## 3. 次フェーズへの展望 (以降)

1.  **内部レイヤー操作 (CRUD)**: インスペクター上でのレイヤー追加、削除、リネーム、表示切り替え（目玉アイコン）の実装。
2.  **Virtual Layer Panel**: ここで選択した内部レイヤーを、メインのレイヤーパネルと同様の操作感で描画・編集可能にする仕組み。
3.  **合成プレビューの高度化**: アセット内の複数レイヤーを正しく合成してサムネイルやタイムラインプレビューに反映させる処理。
