# Phase 3k — カラーサークルMVP

## 概要

Phase 3j で QAP の色ヘッダーとカラーサークル受け皿が整った。
次は、折りたたみ表示の中に最小限のカラーサークルを実装し、現在色を直接選べる導線を作る。

目的は「色を拾うスポイト」と「色を選ぶカラーサークル」を QAP 内で一通り使える状態にすること。
高度な色管理やスロット編集は後続へ回す。

## 優先順

### 1. 現状確認

- [ ] `quick-access-popup.js` の `qa-color-circle-container`、現在色丸、スポイト、カラースロット周辺を確認する。
- [ ] `brushSettings.setColor()` と `brush:color-changed` の同期経路を確認する。
- [ ] QAP の開閉、カラーサークルトグル、スポイト、カラースロット切替が退行しないことを確認する。

### 2. カラーサークルMVP UI

- [ ] `qa-color-circle-container` 内に小さめのカラー選択UIを置く。
- [ ] 直径は現在のカラースロット4個分以下を目安にし、横幅いっぱいに広げない。
- [ ] クリック/ドラッグで色を選び、現在色へ反映する。
- [ ] 現在色丸が即更新されるようにする。

方針:

- 最初は Canvas 2D または CSS/DOM でよい。重いライブラリは入れない。
- 色相リング + 内側の明度/彩度まで一気に作るのが重ければ、まずは円形 hue/saturation 相当の簡易実装でよい。
- 厳密な色空間やICC対応は不要。

### 3. 操作感

- [ ] ペン/マウスの pointerdown / pointermove で連続選択できる。
- [ ] pointerup / pointercancel でドラッグを終了する。
- [ ] カラーサークル操作中に QAP ドラッグや外側クリック閉じと競合しない。
- [ ] 透明背景やキャンバス描画には触れない。

### 4. カラースロットとの関係

- [ ] カラーサークルで選んだ色は現在色へ反映する。
- [ ] カラースロットへ自動保存しない。
- [ ] 後続で「現在色をスロットへ登録」「スロット入れ替え」「初期値へ戻す」を追加できるよう、処理の責務を分ける。

### 5. 後続へ回す項目

- 色相リング + 三角/四角SVエリアの本格実装。
- RGB/HSV数値入力。
- 履歴色。
- カラースロット編集、登録、入れ替え、ロック。
- パレット幅のさらなる横詰め。

## 確認事項

- [ ] `npm.cmd run build` が成功する。
- [ ] カラーサークルを開閉できる。
- [ ] カラーサークルで選んだ色が現在色丸へ反映される。
- [ ] その色でペン描画できる。
- [ ] スポイトで拾った色、カラースロット選択、カラーサークル選択が互いに破綻しない。
- [ ] QAP の縦幅が極端に増えない。

## 禁止事項

- WebGPU / SDF / MSDF / JFA 経路へ触れない。
- 保存形式、履歴形式、レイヤー構造を変更しない。
- 外部カラーピッカーライブラリを追加しない。
- QAP 全体のDOMを丸ごと置換しない。差分編集で進める。
