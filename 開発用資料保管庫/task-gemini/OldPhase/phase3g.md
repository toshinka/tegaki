# Phase 3g — 隙間閉じ Gap Close の検証実装

> Phase 3f で「表示中レイヤー参照バケツ」が実用化された。
> 次は通常バケツの弱点である、小さな線の隙間から塗りが漏れる問題を、まず 1-2px 程度の低リスク検証として扱う。

---

## 目的

- 小さな線の隙間で flood fill が漏れるケースを減らす。
- 表示中レイヤー参照バケツと編集レイヤーのみバケツの両方を壊さない。
- まずは UI を増やさず、固定値または内部設定で効果と副作用を確認する。

---

## 前提

- `fill-tool.js` には `settings.gapClosePixels` と `_applyGapCloseToMask()` の足場がある。
- 現状の `_applyGapCloseToMask()` はプレースホルダで、隙間閉じ本実装はまだ入っていない。
- クリスタ系の高品質 gap close や JFA/SDF 距離場方式は後続研究候補。現行方針では WebGPU / SDF / MSDF 系は凍結中なので復活させない。

---

## 実装方針

### 1. 小半径だけ検証する

- MVP は `gapClosePixels = 1` または `2` に限定する。
- 大半径やスライダー UI は実装しない。
- クリックごとの CPU 処理が重くならないか確認する。

### 2. 方式は小さく選ぶ

- まずは既存 mask に対する Dilation -> Erosion、または同等の低リスクなマスク補正を検証する。
- 主線外へにじむ、副作用で塗り領域が太る、アンチエイリアス境界が汚れる場合は無理に完成させない。
- 実装が広く波及する場合は Codex 判断へ戻す。

### 3. UI とツール数は増やさない

- クイックパネルに新しいスライダーやアイコンを追加しない。
- 消しバケツ、G キー循環、環境設定での並べ替えはこの Phase では扱わない。

---

## Gemini に任せる範囲

- `fill-tool.js` 内の小さな gap close 検証実装。
- `settings.gapClosePixels` の固定値または内部値による確認。
- `npm.cmd run build` の実行。
- `PROGRESS.md` への効果、副作用、継続可否の記録。

Gemini が独自判断で以下をしないこと。

- WebGPU / SDF / MSDF / JFA 経路の復活。
- gap close の UI 追加。
- 消しバケツの実装。
- G キー循環の実装。
- 保存形式、履歴形式、レイヤー構造の変更。

---

## 確認条件

- 1-2px 程度の隙間で、漏れが減るケースが確認できる。
- 完全に閉じた線画で、従来より塗り結果が悪化しない。
- 主線の外側へ明らかににじむ副作用がない、または副作用として記録されている。
- 表示中レイヤー参照 ON/OFF の両方で破綻しない。
- Undo / Redo とサムネイル更新が正常。
- `npm.cmd run build` が成功する。
