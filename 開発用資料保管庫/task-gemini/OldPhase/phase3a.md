# Phase 3a — 描画補助・ブラシ拡張（エアブラシとぼかしブラシ）の実装

> Phase 2 の UI 標準化とバグ補修が完了し、本アプリの基本構造が強固になりました。
> Phase 3a では、表現力の更なる向上を目指し、描画ブラシの拡張として「エアブラシ（Airbrush）」および「ぼかしブラシ（Blur brush）」を実装します。
> この作業はアルゴリズムと描画パイプラインの設計が重いため、外部AI（GPT-5.5等）へ設計・実装を委託し、ローカル（Gemini/Antigravity）で結合・検証を行います。

---

## 背景・目的

- **表現力の向上**: 通常のペンと消しゴムに加え、イラスト制作に必須の「エアブラシ（柔らかい描画）」と「ぼかし（ピクセルの平滑化）」を導入します。
- **RenderTexture 焼き込みとの親和性**: Tegakiのラスター焼き込み方式（各レイヤーが持つ `RenderTexture` への逐次焼き込み）と適合する描画アルゴリズムを構築します。
- **外部AIの活用**: アルゴリズムが重い本フェーズの設計・実装案を GPT-5.5/Claude に委託し、ローカルのリソースとトークンを節約します。

---

## 実装内容（外部AI委託 ＆ ローカル結合）

### 1. 外部AI（GPT-5.5）への委託内容 (`task-gpt/` に仕様書を配置)
- **エアブラシ（Airbrush）アルゴリズム設計**:
  - `BrushSettings` に新しいモード `'airbrush'` を追加。
  - `StrokeRenderer` で、Perfect-Freehandのポリゴンではなく、設定サイズに基づいた「柔らかい円形ブラシグラフィックス（またはソフトなドロップテクスチャ）」をストロークパス上に適度な間隔で連続スタンプする描画経路の実装。
  - 筆圧に応じてサイズだけでなく不透明度やスタンプ密度が連動する挙動のサポート。
- **ぼかしブラシ（Blur Brush）アルゴリズム設計**:
  - `BrushSettings` に新しいモード `'blur'` を追加。
  - PixiJS v8 の `RenderTexture` を活用し、描画位置周辺のピクセルデータを一時的にサンプリングして `BlurFilter` を適用し、ブラシ形状のマスクを用いて対象レイヤーに焼き直す処理の実装。
- **設定 UI と EventBus の拡張**:
  - `brush-settings.js` の設定拡張、および設定パネルへのブラシ選択UIと個別プロパティの追加。

### 2. ローカルAI（Gemini/Antigravity）の役割
- 外部AIの成果物を受け取り、`tegaki_work/` の対象ファイルへ安全にマージ。
- ビルド検証（`npm run build`）および動作確認。
- 既存の Undo/Redo 履歴（`RasterSnapshot` 方式）との整合性確認。

---

## 完了条件

- 設定パネル等から「エアブラシ」「ぼかし」を選択可能になり、それぞれ正常にキャンバス上に描画できる。
- エアブラシは柔らかいエッジのストロークを生成できる。
- ぼかしは描画済みの線を擦ることで柔らかく平滑化（ぼかし）できる。
- Undo/Redo 時にもエアブラシ・ぼかしの描画結果が正しく復元される。
- `npm run build` が成功している。
