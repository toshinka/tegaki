# Phase 4m — 実レイヤー非破壊のアニメプレビュー層MVP (GEMINI報告)

## 1. 実施内容

### 🖼️ アニメーション専用プレビュー層の導入
- **`ui/animation-table-popup.js`**
    - `animationPreviewContainer` (PIXI.Container) を新設。
    - `LayerSystem` の `canvasContainer` 内、通常レイヤー群よりも前面に配置しました。
    - これにより、プレビュー時に実レイヤーの内容を書き換える（汚染する）ことなく、Snapshot を表示できるようになりました。

### 🔄 非破壊プレビューロジックの実装
- **`_applyVisibilityPreview()` の刷新**:
    - Snapshot を持つセルの場合：Snapshot から PIXI Texture を生成し、プレビュー層に Sprite として追加。同時に、対応する実レイヤーを一時的に非表示（`visible = false`）に設定。
    - Snapshot を持たないセルの場合：従来通り実レイヤーの表示状態をトグル（TODO: 将来のアセット化に合わせ、これもコンテナ表示へ移行予定）。
- **`_restoreVisibility()` の刷新**:
    - プレビュー層の子要素（Sprite）をすべて削除。
    - 全実レイヤーの `visible` を本来の状態 (`layerData.visible`) に復帰。
    - 実レイヤーへの直接的な Snapshot 復元（`restoreLayerRasterSnapshot`）を完全に廃止しました。

### ⚡ パフォーマンスとリソース管理
- 生成した PIXI Texture を `CelModel.rasterSnapshot._pixiTexture` にキャッシュする仕組みを導入し、再生中のチラつき防止と CPU 負荷軽減を図りました。
- セルの再キャプチャ時には、古いテクスチャを明示的に `destroy` するように配慮しました。

## 2. 検証結果

### ✅ 動作確認
- **実レイヤーの安全性**: プレビュー中にどれだけ Snapshot セルを表示・再生しても、PREVIEW OFF またはパネルを閉じれば、元のレイヤー描画内容（描線など）が完全に維持されていることを確認。
- **再生ヘッド連動**: 再生中、Snapshot があるフレームではプレビュー層の Sprite が切り替わって表示され、アニメーションが成立することを確認。
- **退行確認**:
    - 通常のレイヤーパネル操作（追加・削除・名前変更）に影響なし。
    - セルの伸縮・選択・削除も正常に機能。
    - 背景レイヤーやフォルダが不意に消える現象もなし。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **テクスチャの一括解放**: アニメテーブル自体を破棄する際の全キャッシュテクスチャ解放処理は、今後のリソース管理フェーズで実装予定。
- **非Snapshotセルの扱い**: Snapshot がないセル（空のセル）は、まだ実レイヤーの `visible` 制御に頼っています。これも将来は「空のアセット」を表示する形に統一すべきです。

## 3. 次フェーズへの提案 (Phase 4n)

1. **セルのドラッグ移動 (Move/Copy)**: 操作性の強化。
