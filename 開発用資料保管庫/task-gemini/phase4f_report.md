# Phase 4f — アニメテーブル内だけの再生ヘッドMVP (GEMINI報告)

## 1. 実施内容

### 🔄 データモデルの拡張 (Playback)
- **`system/animation/animation-data-model.js`**
    - `TimelineModel.advanceFrame()` を実装。
    - 現在フレームを 1コマ進めるロジックを構築。
    - 最終フレーム到達時に `playback.loop` 設定に従い、最初に戻るか停止するかを制御。

### ▶️ 再生タイマーと制御ロジック
- **`ui/animation-table-popup.js`**
    - `isPlaying` と `_playTimer` 状態を追加。
    - `play()` / `stop()` / `togglePlayback()` メソッドを実装。
    - `TimelineModel.fps` (初期値12fps) に基づいた `setInterval` による自動進行を実現。
    - パネルを閉じる (`hide`) 際に自動で再生を停止する安全設計。

### 🖱️ 再生コントロールUI
- **ヘッダーボタン**:
    - アニメテーブルヘッダーの左側に「再生/停止」トグルボタンを追加。
    - 再生中はアイコンが「■ (Stop)」に変化し、オレンジ色で強調。
    - 停止中はアイコンが「▶ (Play)」を表示。

### 🎨 ビジュアル・フィードバック
- 再生中、現在フレーム（再生ヘッド）のオレンジ色のハイライトがリアルタイムで移動。
- CSS によるボタン状態の視覚化（`.playing` クラス）。

## 2. 検証結果

### ✅ 動作確認
- **自動進行**: 再生ボタンを押すと、12fps の速度で現在フレームのハイライトが右へ移動することを確認。
- **ループ再生**: 24フレーム目に到達すると、自動的に 0フレーム目に戻り再生が継続されることを確認。
- **停止**: 再度ボタンを押すか、パネルを閉じると進行が即座に止まることを確認。
- **テンプレート整合性**: 過去の不具合を教訓に、HTML/CSS のテンプレート文字列が正しく閉じられていることを前後50行の目視とビルドで確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **描画連動**: 再生ヘッドは動くが、まだキャンバス上のレイヤー表示/非表示（セルの有無による制御）には連動していない。これは次フェーズの課題。
- **FPS変更**: 内部データとしては 12fps だが、UI 上での変更機能は未実装。

## 3. 次フェーズへの提案 (Phase 4g)

1. **タイムラインベースの表示制御**: 現在フレームにセルがあるレイヤーのみを表示し、セルがない期間は一時的に非表示にする連動機能の実装。
2. **描画内容の固定 (RenderTexture Snapshot)**: 特定のセルに対して、現在のレイヤー描画内容を「スナップショット」として保持する基礎実験。
