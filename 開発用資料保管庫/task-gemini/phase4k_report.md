# Phase 4k — セル別RenderTexture Snapshot手動キャプチャMVP (GEMINI報告)

## 1. 実施内容

### 📸 セル別 Snapshot 保持機能の追加
- **`system/animation/animation-data-model.js`**
    - `CelModel` に `rasterSnapshot` プロパティを追加し、描画内容（RenderTexture Snapshot）を紐付け可能にしました。
- **`ui/animation-table-popup.js`**
    - `captureSelectedCel()` メソッドを実装。選択中のセルに対して、現在の実レイヤーの描画内容を `LayerSystem.createLayerRasterSnapshot` を用いて手動でキャプチャし、メモリ上に保持する仕組みを構築しました。

### 🔄 タイムライン連動プレビューの高度化
- **`ui/animation-table-popup.js`**
    - `_applySnapshotPreview()` を実装。`PREVIEW` が ON の間、タイムライン上の各セルの有無（Visibility）だけでなく、保持されている `rasterSnapshot` を実レイヤーへ一時的に適用（`restoreLayerRasterSnapshot`）するように強化しました。
    - これにより、同一レイヤーであってもフレーム（セル）ごとに異なる絵を「切り替えて表示」することが可能になりました。

### 🛡️ 安全なバックアップと復旧
- プレビュー適用前に、現在のキャンバスの状態を `_backupSnapshots` (Map) に保存し、アニメテーブルを閉じる際やプレビューを OFF にした際に、確実に「キャプチャ前/プレビュー前」の状態へ復元する安全ロジックを実装しました。

### 🎨 視覚的フィードバック
- **CAPTUREボタン**: ヘッダーに専用ボタンを設置。
- **Snapshotインジケータ**: 描画内容が保持されているセルには、ブロック内に小さな白丸アイコンを表示し、未キャプチャのセルと区別可能にしました。

## 2. 検証結果

### ✅ 動作確認
- **手動キャプチャ**: セルを選択して `CAPTURE` を押すと、その時のレイヤーの絵がセルに記録されることを確認。
- **パラパラ再生**: 異なる絵をキャプチャした複数のセルを再生・移動すると、キャンバス上の絵が即座に切り替わる（アニメーションする）ことを確認。
- **確実な復元**: プレビューを終了すると、キャプチャ操作に関わらず、元のレイヤー描画内容へ正しく戻ることを確認。
- **退行確認**: 背景、フォルダ、通常のレイヤー操作、ビルド等に悪影響がないことを確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **自動キャプチャ**: 今回は「手動ボタン」のみです。描画した瞬間に自動でセルへ反映する機能は将来の課題です。
- **永続化**: スナップショットデータは in-memory のみであり、プロジェクトの保存・ロードにはまだ対応していません。
