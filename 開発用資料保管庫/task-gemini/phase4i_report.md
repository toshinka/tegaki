# Phase 4i — アニメテーブル実描画プレビューMVP (GEMINI報告)

## 1. 実施内容

### 👁️ 実描画プレビュー（Visibility連動）の実装
- **`ui/animation-table-popup.js`**
    - タイムラインの現在フレームに応じたレイヤー表示制御ロジックを実装。
    - 各通常レイヤーに対し、現在フレームに「セル」が配置されている場合のみキャンバスに表示（`layer.visible = true`）し、セルがない場合は一時的に非表示にする仕組みを構築。
    - 背景レイヤーおよびフォルダ自体は制御対象外とし、原則として常に表示（または元の設定を維持）するように調整。

### 🔄 表示状態の動的同期と復元
- **`ui/animation-table-popup.js`**
    - 再生ヘッドの移動、Play再生、左右キーによるフレーム移動に連動して即座にプレビュー表示が更新されるよう `render()` 内に統合。
    - パネルを閉じる (`hide()`) 際、またはプレビュー機能をオフにした際に、各レイヤーの本来の表示状態 (`layerData.visible`) を自動的に復元する安全機能を実装。
    - 復元後は `layer:panel-update-requested` を発行し、レイヤーパネルの目アイコンとの整合性を保証。

### 🔘 プレビュー切替UI
- アニメテーブルのヘッダー左側に `PREVIEW` チェックボックスを追加。
- ユーザーが任意にタイムライン連動プレビューを ON/OFF できるようにしました。

## 2. 検証結果

### ✅ 動作確認
- **表示連動**: タイムライン上にセルを置いた場所でのみレイヤーがキャンバスに表示され、セルがない場所では消えることを確認。
- **再生連動**: 再生ボタンを押してヘッドが動く間、セル配置に合わせてレイヤーがパラパラと切り替わって表示される（アニメーションプレビュー）ことを確認。
- **復元**: アニメテーブルを閉じると、プレビュー前のレイヤーパネルでの表示/非表示状態が正しくキャンバスに復元されることを確認。
- **退行確認**: 背景レイヤーがプレビュー中に消えないこと、フォルダ操作に支障がないことを目視確認。
- **ビルド**: `npm.cmd run build` 成功。

### ⚠️ 未着手・制限事項
- **内容の固定**: 本フェーズはあくまで「表示/非表示」の連動のみであり、同じレイヤーでフレームごとに異なる絵を描き分ける（RenderTextureのスナップショット化）機能は次フェーズ以降。
- **目アイコン連動**: プレビュー中はキャンバス上の表示が変わるが、レイヤーパネルの目アイコンは「本来の設定」を表示し続ける仕様（混乱防止のため）。

## 3. 次フェーズへの提案 (Phase 4j)

1. **描画内容の固定 (RenderTexture Snapshot)**: 特定のセルに対して、現在のレイヤー描画内容を「スナップショット」として保持し、別のフレームでは別の絵を描けるようにする実体化の開始。
2. **オニオンスキン基礎**: 前後フレームのセルを薄く表示する機能の検討。
