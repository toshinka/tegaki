# Phase 4z15 — Layer Panel CAF Readonly Header (GEMINI報告)

## 1. 実施内容

### 📺 レイヤーパネルへの CAF 情報統合
- **`ui/layer-panel-renderer.js`**
    - **`createCafReadonlyHeader()` の実装**:
        - `window.PopupManager` 経由でアニメーションモデルの `getFrameAssetTree()` を呼び出し、現在フレームで使用されているアセット情報を取得します。
        - 取得した情報を元に、CAF（フォルダ名）、アセット数、およびアセット名のサマリーを表示する読み取り専用のヘッダー要素を生成します。
    - **`render()` への統合**:
        - レイヤーリストの描画開始時に、上記のヘッダーを最上部へ挿入するようにしました。
        - これにより、通常のレイヤー構成を確認しながら、「今この瞬間にアニメーションとして動いている素材」の背景情報を一目で把握できるようになりました。

### 🎨 スタイリングと干渉防止
- **`styles/main.css`**
    - `.caf-readonly-header` クラスを定義しました。
    - 通常のレイヤーフォルダ（薄桃色）と明確に区別するため、**濃い紺色（#2c3e50）の背景と青色のアクセント線**を採用しました。
    - `pointer-events: none` および `user-select: none` を設定し、SortableJS によるレイヤー並び替えやクリック操作を一切妨げない「完全な読み取り専用領域」として実装しました。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **動作確認**:
    1. アニメーションテーブルでクリップを配置し、レイヤーパネルを開く。
    2. パネル上部に「FRAME ASSETS」という紺色のセクションが出現し、CAF名や素材名が表示されることを確認。
    3. タイムラインのフレームを移動すると、パネルのヘッダー内容が即座に同期して更新されることを確認。
    4. ヘッダーの上をドラッグしてもレイヤーの並び替えが開始されず、通常のレイヤー操作に支障がないことを確認。

## 3. 次フェーズへの展望 (以降)

1.  **アセットのクリック選択**: 読み取り専用ではなく、ヘッダー内のアセット名をクリックすることで、タイムライン上の該当クリップを選択状態にするインタラクション。
2.  **Virtual Layer Panel**: このヘッダー領域をさらに拡張（または通常レイヤーと置換）し、アセット内部のレイヤー構造を直接編集できるようにする。
3.  **アセットサムネイルの統合**: ヘッダー内にも、現在のアセット内容を示す小さなサムネイルを表示する。

## 4. Codex確認・補修

- `window.PopupManager.get('animationTable')` の参照経路は `core-engine.js` のグローバル登録と一致していることを確認。
- CAFヘッダーは `.layer-item` / `.folder-item` / `data-layer-index` / `data-is-folder` を使っていないことを確認。
- SortableJSの対象からCAFヘッダーをより明確に除外するため、`initializeSortable()` に `draggable: '.layer-item'` を追加。
- Geminiのビルドで生成された `dist/` 差分は成果物から除外する。
