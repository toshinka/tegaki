# Phase 4z2 — Lane Playback Include MVP (GEMINI報告)

## 1. 実施内容

### 🎭 SETスコープと複数Lane選択の実装
- **`ui/animation-table-popup.js`**
    - `playbackScope` に `'includedLanes'` (SET) モードを追加しました。
    - `includedLaneIds` (Set) 状態を導入し、ユーザーが個別にプレビュー対象に含めるレーンを選択できるようにしました。
    - **複数Laneフィルタリング**:
        - `_getPreviewLaneFilterIds()` を新設。ALLならnull、LANEなら単一IDのSet、SETなら選択済みのSetを返すように統一しました。
        - `_applyVisibilityPreview`, `_renderFrameComposite`, `_renderOnionFrame` を更新し、Setに含まれる全てのレーンを描画対象とするよう拡張しました。
    - **再生時の固定ロジック**:
        - `play()` 開始時に、その時点の `playbackScope` に基づく全ての対象IDを `activePlaybackLaneIds` として固定するようにしました。これにより、再生中に選択を変えてもプレビューが乱れません。

### 🎨 UI/UX の統合
- **SCOPEヘッダー**: `ALL | LANE | SET` の3ボタン構成にアップデート。
- **レーンリスト**: 各レーン名の左側に `+` (未選択) または `✓` (選択済み) ボタンを配置しました。
    - クリックでSETスコープへの追加/除外を切り替えられます。
    - フォルダトラックには表示しないよう制御。
- **視覚的フィードバック**:
    - `SET` モード時は、チェックを入れたレーンのみがキャンバスに合成され、オニオンスキンもそれらのレーンに限定されます。

## 2. 検証結果

- **ビルド確認**: `npm.cmd run build` 成功。
- **動作確認**:
    1. 複数レーンに `✓` を入れ、`SET` モードに切り替え。
    2. 選択したレーンのみが重なって表示されることを確認。
    3. 再生を開始し、選択中の複数レーンが同時に動くことを確認。
    4. 再生中に他レーンの `✓` を変えても、再生終了までプレビュー対象が固定されていることを確認。
