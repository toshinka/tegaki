## 1. 描画品質・描画補助・漫画機能（改修計画 1 に対応）

### 描画補助・品質
*   **非破壊変形（劣化しない変形）の導入**: 
    *   Vキー変形時、ドラッグ中は `position/scale/rotation` パラメータのみを変更してプレビューし（劣化ゼロ）、確定時（VキーOFF）に元のキャッシュテクスチャから1回だけ新規 RenderTexture へ焼き込むことで累積劣化を防ぐ。
    *   XY移動のみの場合は、確定時に座標値を四捨五入して整数（`Math.round`）に吸着させ、補間を発生させず**完全劣化ゼロ移動**を実現する。
*   **逆クリッピング（反転クリップ）の実装**:
    *   通常クリッピング（下のレイヤーの不透明部分にのみ表示）に加え、下のレイヤーの「透明な部分（描かれていない領域）」にのみ描画を表示する「逆クリッピング（マスク反転）」機能を実装。
    *   レイヤーパネルのクリップボタンを押すたびに、`クリップなし ➡️ 通常クリップ ➡️ 逆クリップ` の順で循環切り替えする。
    *   逆クリップ選択時は、クリップボタンのアイコン色が反転（または禁止・反転用のマーク）に変化し、視覚的に区別できるようにする。
*   **手ブレ補正の本格接続**: 設定スライダー（`smoothing` 0.0〜1.0）を描画側（PointerHandler等）へ動的に反映する。
*   **キャンバスサイズ制限緩和**: 片辺最大サイズを `2500px`（漫画原稿サイズ目安 1700x2400）へ引き上げる。リサイズ時のメモリ解放（RenderTexture の `.destroy(true)`）の安全性チェックを徹底。
*   **図形描画ツール**: 直線、円、矩形。ドラッグ中に仮プレビューし、pointerupで確定。`Shift` 押下で45度スナップ、正円・正方形スナップ。

### 漫画向け特殊ツール
*   **スクリーントーン**: 網点、グラデーション、縦線ストライプなど。選択範囲内やブラシにトーンパターンを乗せる。
*   **集中線・ウニフラッシュ**: ある中心点に向かって描画線が引き寄せられる、または中心点を基準に回転する「集中線定規（仮想定規）」。
*   **自動マスキング（コマ枠用）**: コマ枠の形状を定義し、その外側を自動でマスク（描画不可）する。

### 外部連携（PSD・コピペ）
*   **アクティブCAFのPSD出力**: 他ツールとの互換性を保つため、全体ではなく「アクティブなCAF（動画用コンテナフォルダ）」のみをPSD形式でエクスポートする。
*   **PSDインポート＆自動リサイズ**: PSDを読み込む際、キャンバスより大きい画像はアスペクト比を維持してキャンバス最大幅に収まるよう自動縮小リサイズする。
*   **クリップボード画像ペースト**: 外部（エクスプローラーやブラウザ）からコピーされた画像を Ctrl+V でアクティブレイヤーにペーストする。この際、アスペクト比を保持してキャンバスサイズへ自動縮小リサイズを行う。

## 2. UI操作性・キャンバスリサイズ・カラー管理（改修計画 2 に対応）

### UI・ポップアップ
*   **トップバーの追加**: ホーム（画面基準値）、Undo、Redo、回転、反転、レイヤーON/OFFアイコンを配置。設定でOFFにできるようにする。
*   **キャンバス反転警告**: 左右反転・上下反転時に「左右反転中」などのメッセージを画面上に常時薄く表示し、反転したまま清書してしまう事故を防ぐ。
*   **QAP（クイックアクセス）のスリム化＆ツール別フリップ**: 横幅を60%程度にスリム化。選択ツールアクティブ時は自動で選択専用オプションパネルへ切り替える。小さく折り畳むモードも追加。
*   **レイヤーパネルON/OFFボタン**: 右サイドバーのレイヤー「＋」アイコンの上に、少し隙間を空けてレイヤーパネルON/OFFアイコンを追加。
*   **デザイン統一**: スライダー、スクロールバー、各種ポップアップ背景の透過度を統一（Maroon/Creamテーマ基準）。OS標準のUI部品を廃止しCSSクラスで統一。
*   **タイムライン縦線強調**: 設定FPSに合わせてタイムラインの縦線を少し太くし、1秒ごとの区切りを分かりやすくする（例: 12FPSなら12フレームと13フレームの間の縦線を強調表示）。

### カラー管理・配色ジェネレーター
*   **配色パターンジェネレーター**: 
    *   選択中の「アクティブカラー（ベース色）」に基づいて、調和する配色パターン（補色、アナログ、トライアド、スプリットコンプリメンタリー、モノクロマティック等）を自動生成。
    *   自動生成された6色が、パレットの指定した1行（6スロット）にリアルタイムで自動マッピングされて置き換わるロジック。
*   **カラースロットの任意入れ替え**: パレット内のカラースロットを、ドラッグ＆ドロップなどでユーザーが任意に入れ替え（並べ替え）できるようにする。
*   **基本カラーセット呼び出し**: デフォルトの基本カラーセット（プリセット）をワンクリックでいつでも呼び出し可能にする。

### リサイズUI
*   **方向指定リサイズ**: 幅/高さ矢印で、均等増減（`↔` / `↕`）と指定方向への増減（`←` / `→` / `↑` / `↓`）を切り替え可能にする。
*   **半透明プレビュー**: リサイズポップアップ自体を半透明（50%程度）にし、キャンバス上に増減範囲を色付きプレビュー表示する。

### アルバムUI
*   **保存成功トースト**: Ctrl+S でアルバムへ保存した際に、「保存しました」メッセージを表示する。
*   **アルバムの一括選択**: アルバムポップアップに一括選択ボタンと、選択解除ボタン（`list-restart`）を追加。
*   **アイコンの適正化**: アルバム内のアイコン（現在 Maroon 一色になっているもの）を他のUI同様の中間色（Maroon/Creamテーマ）に揃える。アイコン候補：`square-library` (保存), `monitor-up` (ロード)。

### レイヤーサムネイル表示
*   **市松模様背景の表示制限**: 
    *   常時表示すると見えづらく目障りになるため、クリスタ等の仕様に準じ、「背景レイヤーを非表示（透明）にした際のみ」連動して市松模様背景が表示される仕組み、もしくは設定でのON/OFF選択式にする。

### 性能向上
*   **履歴限界の改善**: ヒストリー（Undo/Redo）が500を超えたあたりから動作が重くなる問題を改善（配列操作の最適化、テクスチャ破棄処理の見直し）。

## 3. アニメーション・物理演算・メッシュ変形（改修計画 3 に対応）

### データモデルとUI
*   **Lane / Clip / ClipAsset 方式への移行**: アニメーションデータモデルを `Track = Layer` 暫定モデルから、タイムライン上の「Lane」、配置アセット「Clip」、再利用素材「ClipAsset」へと移行。
    *   **CAFデータモデル**: 画像の物理演算設定、レイヤーカード構造、タイムテーブル情報（変形など）を含む。
    *   **レイヤーパネル・カード/PSD**: 画像情報のみを保持し、タイムテーブル変形には関わらない。
*   **イージング管理とグラフUI**: キーフレームの加減速（イージング）を、グラフ上でカーブ（ベジェ曲線）を操作して管理できる簡易UIを構築。
*   **キー操作のCAF連携**: セル選択・編集時のコピー（Ctrl+C）、ペースト（Ctrl+V）、削除（Delete/Backspace）のキー操作が、通常LayerSystemと混線しないように連携強化。
*   **GIFアニメ出力とループプレビュー**: タイムラインのセル画像をGIFアニメとして出力する機能。出力ポップアップ内でアニメーションがループ再生されるプレビュー画面（200x200程度、アスペクト比保持）も実装する。

### 物理演算・変形
*   **物理演算アニメーション**: 髪・服・アクセサリーなどの遅れ追従（バネ、揺れ）をパラメータ調整で適用可能にする。
    *   物理計算結果だけを座標・メッシュ・変形パラメータとして返すAPI非依存なモデル設計（GPU直結にしない）。
*   **Perform（実演記録）**: タイムライン再生中に Vキー（変形）で対象レイヤーを動すと、その軌跡が自動的にキーフレームとしてリアルタイム記録される。
*   **Live2D風ワープデフォーマ**: イラストの一部を格子状のメッシュ変形で滑らかにモーフィング（目パチ・口パクなど）させる仕組み。

## 4. ストレージ管理・外部フォルダ保存（改修計画 5 に対応）

### 保存方式の使い分け（ハイブリッド保存）
*   **ブラウザ保存 (IndexedDB / 高速・一時保存)**:
    *   `Ctrl+S` やワンクリック保存でブラウザ内に IndexedDB で瞬時に一時保存。
*   **外部フォルダ同期保存 (File System Access API / 大容量・永続保存)**:
    *   `Ctrl+Shift+S` や同期保存ボタンで、ユーザーがOS上で指定した任意のローカル同期フォルダに、JSON やフレーム画像ファイルとして自動同期書き出しを行う（ブラウザの容量制限なし）。
*   **プロジェクトの相互移動**: ブラウザ保存データをローカルフォルダへエクスポート、またはその逆のインポートをアルバムUIから簡単に行えるようにする。

### 容量可視化とリフレッシュ
*   **ブラウザ容量表示**: `navigator.storage.estimate()` を使用し、ブラウザストレージの現在の使用率（例: 「使用量: 64MB / 1.2GB (5.2%)」）をアルバムポップアップ等に表示。容量が 80% を超えたら警告。
*   **ストレージのリフレッシュ（容量解放）**: 一定期間（例: 2週間）以上経過した古い一時保存プロジェクトや、Undo用の古いテクスチャキャッシュを一括クリアして空き容量を確保するボタンをアルバムUIに設置。

## 5. AI動画・ロトスコープ支援（改修計画 4 に対応）

*   **お手本動画の流し込み**: MP4動画やGIF画像を読み込み、最初の24フレームをお手本テンプレートとしてAI参考レーンに流し込む機能。
*   **AI Adapterサーバー**: Node.js等で構築し、画像の送信、LLMによる指示文の英語化、ComfyUIへのキューイング、ffmpegによる動画分解などを一括管理。
*   **始点・終点アセット指定**: タイムライン上のセルを「始点」「終点」に設定し、その間をAIで補間させるためのUI。
*   **AI参考レーンの自動配置**: AIが生成した動画を1フレームずつ分解し、不透明度30%前後の編集ロックされた「AI参考レーン」に自動流し込みする。その上で手描きトレース（ロトスコープ）を可能にする。

## 【保留 / 長期構想】（高難易度・依存関係が複雑なため安全確認後に実施）

*   **スタンプエンジン化（Task 9）**: 描画エンジンを完全に「等間隔ブラシスタンプ ＋ スクラッチバッファ蓄積合成」に変更するタスク。描画の根本であるため、保存環境が安定した後に慎重に着手する。
*   **WebGPU Computeへの移行**: スプラインサンプリングとスタンプBlit合成を WebGPU 計算シェーダー（Compute Shader）へ移植する構想。
*   **Live2D風ワープデフォーマの完全実装**: 制御点変形・メッシュ変形は PixiJS のシーングラフへの影響が大きく、データシリアライズが複雑なため、アニメーション基盤が完全に整った後の長期構想とする。
*   **無限キャンバス / 外部領域保持**: キャンバス外に描画したアセットを保持したままにする構想。メモリ負荷が高いため、他の最適化が完了した後の保留案件。
