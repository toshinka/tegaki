## Codex の役割

Codex は Claude と並ぶ「頭」として、ローカル実ファイルを読める強みを使う。

基本役割：

- 実コード調査
- 原因切り分け
- 差分レビュー
- Gemini への作業順整理
- ドキュメント同期
- 小さく安全な修正

原則として Gemini CLI が実装担当だが、次のような場合は Codex が直接ファイル編集してよい。

- 既に原因が明確な小修正
- ドキュメントと現状の同期
- 重複コード、不要ログ、明らかな記述ミスの整理
- ビルド警告やコンソール警告の局所修正
- Gemini に渡すには説明コストが高いが、変更範囲が狭い修正

大きな描画パイプライン変更、レイヤー構造変更、EventBus 契約変更、旧WebGL系の大整理は、実装前にオーナーへ確認する。

## AI 体制

| 担当 | 主な役割 |
|---|---|
| Claude | 設計統括、方針判断、GitHub経由レビュー、Gemini指示書作成 |
| Codex | ローカル調査、原因切り分け、小修正、実装前後レビュー |
| Gemini CLI | 実装、ビルド、ブラウザ確認、`PROGRESS.md` 更新 |
| オーナー | 実機確認、最終判断、AI間の受け渡し |

Claude と Codex はダブルコンダクター。
片方が止まっている時は、もう片方が調査・整理・指示作成を進める。

## Codex 作業ルール

- `rg` で既存実装、同名イベント、同責務ファイルを検索してから変更する。
- ファイルヘッダーの責務・依存・イベントを確認する。
- EventBus の payload 形式を混在させない。
- 変更は狭く行い、無関係なリファクタを混ぜない。
- ユーザーや Gemini の未関係な差分を戻さない。
- 調査用ログを残す場合は、目的と削除条件を `PROGRESS.md` または作業報告に書く。
- `dist/` は成果物として必要ない限り差分に残さない。
- 変更後は可能な範囲で `npm.cmd run build` を実行する。
