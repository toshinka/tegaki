# TEGAKI ペン品質向上・タブ設定・カメラ挙動改善 設計書

本文書は、ペンの描画品質、手ブレ補正や筆圧カーブ等の設定接続、リサイズ後の描画範囲、および液タブペンにおけるカメラのブロック挙動改善に関する設計と実装指示をまとめたものです。

---

## 1. ペン・消しゴム実装の現状評価と基本方針

### 採用されている基本パイプライン
```
PointerEvent (筆圧・座標)
  ↓
perfect-freehand (輪郭ポリゴン生成)
  ↓
PixiJS v8 Graphics.poly() (ポリゴン塗りつぶし)
  ↓
RenderTexture 焼き込み (ラスター確定)
```
この「輪郭ポリゴン方式」自体は正しく機能しており、全面作り直しは行いません。ただし、入力の安定化、リアルタイム消去経路、液タブペンのガード条件の整備を通じてパイプラインの不整合を修正します。

## 2. 実装タスク

### タスク1：手ブレ補正（LazyBrush）と筆圧カーブの設定接続
設定ポップアップに「ペン」タブを新設し、設定値を描画ロジックへ動的に反映します。

#### A. `LazyBrush` クラスの追加 ([pointer-handler.js](file:///d:/GitHub/tegaki/tegaki_work/system/drawing/pointer-handler.js))
Lerp方式（指数移動平均）の手ブレ補正クラスを `PointerHandler` の前に定義し、線補正値 `smoothing` (0.0〜1.0) を `LazyBrush.radius = smoothing * 16` (0〜16px) として動的反映します。
*   `smoothing 0.0` → radius 0 （即時追従・補正なし）
*   `smoothing 0.5` → radius 8 （デフォルト・従来のlazyRadius=8相当）
*   `smoothing 1.0` → radius 16（最大補正）

#### B. `normalizeEvent()` での筆圧補正と筆圧カーブ適用
*   **筆圧補正**: `TegakiSettingsManager.get('pressureCorrection')`（デフォルト 1.0）を生の筆圧に乗算し `[0.0, 1.0]` にクランプする。
*   **筆圧カーブ**: `TegakiSettingsManager.get('pressureCurve')` を適用する。
    *   `ease-in`（軽め）：`p * p`（弱押しで細くなる）
    *   `ease-out`（重め）：`1 - (1 - p)**2`（しっかり押さないと太くならない）
    *   `linear`（通常）：そのままの補正圧

### タスク2：リサイズ後の RenderTexture 拡張不全バグ修正
キャンバスリサイズ時（400x400から拡張後等）に、旧サイズ外に絵が描けなくなる現象を解消するため、全描画レイヤーの `RenderTexture` を新サイズへリビルド・内容コピーします。

#### [LayerSystem.resizeLayerTextures()](file:///d:/GitHub/tegaki/tegaki_work/system/layer-system.js) の動作
1.  リサイズイベント受信時、背景レイヤーとフォルダを除くすべての描画レイヤーを対象とする。
2.  古い RenderTexture を破棄する前に、新しいサイズの RenderTexture を作成する。
3.  古い RenderTexture の内容を新しい RenderTexture へコピー描画（指定の offset / 左上等のアラインメントを適用）する。
4.  `layerData.renderTexture` および `layer.layerSprite.texture` を差し替え、元のテクスチャを `.destroy(true)` で破棄する。

### タスク3：液タブペンの接触によるカメラ（キャンバス移動）ブロックバグ修正
液タブでペンを接触した際、Spaceキー押下状態や右ボタン判定の混線により、カメラが「キャンバス移動モード（`canvasMoveMode = true`）」のまま引っかかり、描画がロックされるバグを修正しました。

#### 修正ポイント ([camera-system.js](file:///d:/GitHub/tegaki/tegaki_work/system/camera-system.js))
1.  **`pointerdown` リスナー**: 液タブペンからの入力（`e.pointerType === 'pen'`）はカメラ移動のトリガーから即時早期リターン（`return`）させて除外する。
2.  **`pointerup` リスナー**: 液タブペンからの `pointerup` 受信時は、何らかの理由でドラッグが開始されていた場合でも確実に `_stopDragging()` を呼び出して `canvasMoveMode = false` にクリアする。

---

### タスク4：サムネイル黒背景・余白のチェッカーパターン化
*   [thumbnail-system.js](file:///d:/GitHub/tegaki/tegaki_work/system/drawing/thumbnail-system.js) でサムネイル生成時に 2Dコンテキストの `clearRect(0, 0, width, height)` を確実に行い、余白を透明化する。
*   レイヤーパネルのサムネイルラッパー背景に、CSSクラスを介してふたばカラーのチェッカーパターン背景を表示する。

## 3. 将来的なペン描画品質向上ロードマップ（スタンプエンジン）

現行の「輪郭ポリゴン方式」の構造的限界（半透明時の三角分割ノイズ、ストローク確定時の見た目のズレ）を抜本解決するための将来計画です。

*   **Step 1: 入力安定化 (Lazy Nezumi方式)** → 【完了】上記タスク1で実装済み。
*   **Step 2: スタンプ描画エンジンへの置換** (Catmull-Rom スプライン補間 ＋ 等間隔ブラシスタンプ ＋ スクラッチバッファ蓄積合成)
    *   1ストロークを一時的なスクラッチ RenderTexture に `normal` ブレンドで等間隔スタンプ描画し、描画完了時（`pointerup`）にレイヤーへ `opacity` と指定の `blendMode` で焼き込むことで、半透明オーバーラップノイズと確定前後の形状変化を根絶する。
*   **Step 3: WebGPU Computeへの移行**
    *   スプラインサンプリングとスタンプBlit合成処理を WebGPU の Compute Shader へ移植し、120Hzペン入力時にも遅延ゼロの描画を実現する。
