# 4. ストレージ管理・外部フォルダ保存計画

ブラウザ内保存（IndexedDB）の容量制限を打破し、ローカルのファイル管理とシームレスに同期するための「ハイブリッド保存方式」と、ブラウザの容量監視に関する改修計画です。

## 各タスクの詳細仕様・実装ステップ

### Task 1: ブラウザ保存 (IndexedDB / 一時高速保存) の実装
*   **内容**: `Ctrl+S` ショートカットや、画面上の「ブラウザに保存」ボタンを押した際、ファイルダイアログを挟まずに IndexedDB へ直接一時保存（キャッシュ）する。
*   **仕様**: 保存処理がミリ秒単位で完了するため、描画中のクイック保存に適している。

### Task 2: 外部フォルダ同期保存 (File System Access API)
*   **内容**: ブラウザの容量制限（クォータ）を回避するため、ローカルの任意のディレクトリと同期する保存方式。
*   **仕様**:
    *   初回保存時に File System Access API（`showDirectoryPicker`）を使用し、ユーザーにPC上の保存先フォルダ（例: `MyProjects/`）を選択させる。
    *   選択されたフォルダ構造内に、プロジェクトデータ（JSON）やフレーム画像（PNG）をファイルとして直接同期書き出しする。
    *   `Ctrl+Shift+S` などで自動的に差分同期をかける。容量限界がなくなり、他ツール（Photoshop等）への持ち出しもエクスプローラーから直接行える。

### Task 3: プロジェクトの相互移動 (ブラウザ ↔ ローカル)
*   **内容**: ブラウザ内 IndexedDB に溜まった一時データを、後から指定のローカル同期フォルダへ書き出してバックアップ（エクスポート）する、またはその逆のインポートをアルバムUIから簡単に行えるようにする。

### Task 4: ストレージ容量の可視化インジケーター
*   **内容**: 現在のブラウザ内の空き容量を表示するインジケーター。
*   **仕様**:
    *   `navigator.storage.estimate()` API を用いて、本アプリが使用しているIndexedDBのバイト数と、ブラウザから割り当てられている最大上限値を動的取得。
    *   アルバムポップアップ上部に「ストレージ使用状況: 5.2% (64MB / 1.2GB 使用中)」のようにゲージ付きで表示。80%を超えた場合は警告（赤色表示）を行う。

### Task 5: ストレージ容量の解放（リフレッシュ）機能
*   **内容**: IndexedDB の空き容量が逼迫した際、またはユーザーが手動で整理したい際に、不要な一時データをクリーンアップする。
*   **仕様**:
    *   一定期間（例: 2週間）以上更新されていない古い一時保存プロジェクトや、Undo用の古いテクスチャキャッシュを一括クリアして空き容量を確保する「容量解放」ボタンをアルバムUIに設置。
    *   削除前に「削除対象の一時プロジェクト一覧」をダイアログ表示し、必要な作品はローカルへ退避できるように保護する。
