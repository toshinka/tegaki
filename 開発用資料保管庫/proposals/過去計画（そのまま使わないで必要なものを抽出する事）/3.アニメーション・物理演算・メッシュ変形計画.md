## 各タスクの詳細仕様・実装ステップ

### Task 1: GIFアニメ出力とループプレビュー画面
*   **内容**: 制作したアニメーションをプレビュー・出力する機能。
*   **仕様**:
    *   **ループプレビュー**: 出力ダイアログ内に、指定範囲のアニメーションが自動ループ再生される 200x200 程度（アスペクト比保持）のプレビュー領域を実装する。
    *   **GIF書き出し**: `gifshot` 等のライブラリを用いて、タイムラインの各フレームの描画内容を束ねてGIFファイルとしてエクスポートする。

### Task 2: Lane / Clip / ClipAsset 方式への移行
*   **内容**: `Track = Layer` 暫定モデルから、タイムライン上の「Lane（行）」と、そこに配置される「Clip（アセットの配置インスタンス）」、および再利用可能な「ClipAsset（アセット）」へとモデルを高度化する。
*   **アセットモデルの設計境界**:
    *   **CAF (Clip Asset Frame) データモデル**: 画像の物理演算設定、レイヤーカード構造、タイムテーブル情報（変形キーフレームなど）を包括。
    *   **画像アセット (PSD互換) 層**: 純粋な画像情報（ピクセル・レイヤー階層構造）のみを保持し、タイムテーブル上での変形や物理演算処理には直接関わらない。これにより、PSD書き出し時の外部ソフトとの完全な画像互換性を担保する。

### Task 3: 物理演算アニメーション（XY移動・回転・Z軸拡縮）
*   **内容**: 髪・服・アクセサリーの遅れ追従（揺れ）を、パラメータ調整で半自動生成する機能。
*   **仕様**:
    *   物理計算結果だけを座標・メッシュ・変形パラメータとして返すAPI非依存なモデル設計（GPU直結にしない）。

### Task 4: Perform（実演記録）機能のMVP実装
*   **内容**: タイムライン再生中に、ユーザーが Vキー（変形）でキャンバス上のレイヤーをドラッグ移動させると、その軌跡がリアルタイムに変形キーフレームとして自動記録される機能。

### Task 5: Live2D風ワープデフォーマ（メッシュ変形）
*   **内容**: イラストの一部を格子状のメッシュ変形で滑らかにモーフィング（目パチ・口パクなど）させる仕組み。

### Task 6: イージング（Easing）管理とグラフUI
*   **内容**: グラフ上でイージングカーブを直感的に操作できる簡易UIを構築。

### Task 7: コピー・ペースト用ショートカット＆アイコン追加
*   **内容**: アニメセル・キーフレームのコピー＆ペーストを最適化。
*   **仕様**: コピー用（`clipboard-pen`）、ペースト用（`clipboard-list`）などのアイコンを追加。
