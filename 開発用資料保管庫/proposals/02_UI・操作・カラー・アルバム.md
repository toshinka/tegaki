# UI・操作・カラー・アルバム計画

更新日: 2026-06-19

## 直近候補

### Layer Panel上部Frame表示の整理 `候補`

- アニメテーブルを開く前は `NO FRAME` パネルを非表示にする。
- アニメ利用時はCAF枠と同程度の幅へ縮め、右サイドバーへ寄せる。
- CAFの目アイコンと右余白を内側へ詰める。
- 左余白はCAF内部の階層線に必要なため、左右非対称でよい。
- 表示責務はLayer Panel内の分かりやすい入口へ集約する。

### オプション式トップバー `候補`

- ホーム、Undo、Redo、キャンバス回転、表示反転、Layer Panel表示切替。
- 設定で非表示にできること。
- キャンバスを主役にする方針を崩さず、大型常設UIにしない。

### キーボードなし操作とQAP再編 `候補`

- QAPをスリム化し、ツール別オプションへ切り替える。
- COLOR見出しや余白を削り、カラーサークル・ツールスロット用の空間を確保する。
- 小型折りたたみモードを検討する。

## カラー管理

### 配色ジェネレーター `候補`

- 補色、アナログ、トライアド、スプリット補色、モノクロを生成する。
- 生成色を指定パレット行へ適用する。
- 基本カラーセットを復元できるようにする。

### カラースロット並べ替え `候補`

- Pointer Eventsで任意入れ替え。
- 順序を設定保存する。
- Layer Panel共通D&Dを直接流用せず、操作契約とanimationだけ参考にする。

### リアルパレット `保留`

- カラーサークル横の小型混色・色置き場。
- 保存形式と混色操作を先に設計する。

## リサイズとUI品質

### 方向指定リサイズ `候補`

- 均等増減と上下左右への片方向増減。
- キャンバス上に変更範囲を半透明表示する。
- resize do/undo/redoの既存整合を壊さない。

### UI統一 `継続`

- 共通ボタン、スライダー、スクロールバー、透過パネルを使用する。
- JSの大量inline styleを増やさない。
- OS標準部品を置き換える場合はアクセシビリティを維持する。

## Album

### 現在までに完了

- IndexedDBベースの通常Album保存。
- カード選択、一括操作の基盤、並べ替え。
- HTML保存・読込と旧JSON互換の基盤。
- 新規保存データの通常Layer/Folder復元修正。

### 残候補

- Ctrl+S保存成功トースト。
- 全選択・選択解除の明示ボタン。
- アイコン配色の最終統一。
- Windows風D&DをAlbumにも適用するかは、Layer Panelの実機評価後に判断する。

## 性能

### History長期使用時の遅延 `候補`

- 500件前後での配列操作、snapshot保持量、RenderTexture破棄を計測する。
- 単純に上限を増減せず、メモリ使用量と復元時間を基準にする。
