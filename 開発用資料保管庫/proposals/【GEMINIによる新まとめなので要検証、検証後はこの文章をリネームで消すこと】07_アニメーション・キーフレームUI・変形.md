# アニメーション・キーフレームUIおよび変形機能に関する調査報告と提案

## 1. 調査目的と背景
本稿は、`tegaki`における「演算によるX/Y移動・回転・拡縮（トランスフォーム）」アニメーション機能の実装に向けたUIデザインと、将来的な「モーフィング（メッシュ変形）」や「ボーン（リギング）」拡張を見据えたデータ構造の設計方針をまとめたものである。
参考ツールとして、Live2D、Procreate Dreams、ToonSquidのUI仕様を分析し、それぞれの良し悪しを評価した上で、`tegaki`に最適なスタイルを提案する。

---

## 2. 既存ツールのUI・仕様分析

### 2.1 Live2D Cubism
キャラクターアニメーションに特化したデファクトスタンダード。
*   **UIスタイル**: 
    *   **ドープシート（Dopesheet）**と**グラフエディタ（Graph Editor）**を切り替えて使用する。
    *   パラメータ（角度X、角度Yなど）ごとにタイムラインが分かれており、グラフエディタでは縦軸が値、横軸が時間を示す。
    *   複数パラメータの波形を同時に表示し、色分けされたベジェ曲線のハンドルを直接操作してイージングを調整する。
*   **良い点**: 複雑な連動や微細なタイミング（イージング）の調整において、グラフエディタの精度と視認性が非常に高い。
*   **懸念点**: UIが専門的でパネルが多く、初心者には学習コストが高い。レイヤーとアニメーションパラメータが分離しているため、直感的な操作には慣れが必要。

### 2.2 Procreate Dreams
iPad向けの次世代アニメーションアプリ。直感的な操作感が特徴。
*   **UIスタイル**:
    *   **Perform（パフォーム）モード**: 再生しながらキャンバス上のオブジェクトを直接ドラッグして動かすと、その動きが自動的にキーフレームとして記録される。
    *   **Keyframe（キーフレーム）モード**: タイムライン上で対象のクリップ（CAFに相当）をタップし、アクションメニューから「移動」や「フィルタ」を追加。クリップの下にキーフレームトラックが追加される。
*   **良い点**: 「手で動かす」アプローチが極めて直感的で、モダンかつ敷居が低い。タッチインターフェースに最適化されている。
*   **懸念点**: グラフエディタのような数値的な微調整や、複雑なイージング曲線の精密なコントロールが難しく、プロユースの細かな調整には限界がある。

### 2.3 ToonSquid
高機能かつモダンなUIで高い評価を得ているiPad向けアニメーションツール。
*   **UIスタイル**:
    *   タイムラインのレイヤーを展開すると、そのレイヤーが持つ「位置」「回転」「スケール」などのプロパティがツリー状に表示される（After Effectsに近いスタイル）。
    *   **Keyframing Mode**: プロパティごとにキーフレームのひし形アイコンが打たれる。
    *   **イージング曲線（Easing Curves）**: キーフレームを選択すると、その区間のイージング設定（リニア、イーズイン・アウトなどのプリセットやカスタムベジェ）を設定できるポップアップ/インスペクタが表示される。
*   **良い点**: 伝統的なタイムライン（レイヤーベース）とキーフレームアニメーションが見事に融合している。画面がすっきりしており、必要なときだけプロパティトラックを展開できるため、UIがモダンで洗練されている。
*   **懸念点**: 項目が増えるとタイムラインの縦幅を大きく消費する。

## 3. 各ツールの評価と`tegaki`へのUI提案

### 推奨UIスタイル：ToonSquidベース ＋ インライングラフエディタ
ユーザーからの提案「CAFを長くした後でクリックしてアクティブにし、アイコンを押すとイージング波形が現れ、色分けされた線でパラメータ調整を行う」というアイデアは、非常にモダンでUXに優れています。
これを実現するために、以下のUIスタイルを提案します。

1.  **プロパティの展開（アコーディオンUI）**
    *   長いCAF（またはアニメーションクリップ）を選択状態にし、下部（あるいはCAFブロック内）のトグルボタン（▼）を押す。
    *   CAFブロックの下に「トランスフォーム（X, Y, 回転, 拡縮）」のサブトラックがインラインで展開される。
2.  **インライン・イージング波形（Live2DとToonSquidの融合）**
    *   各プロパティ（X: <span style="color:red">赤</span>, Y: <span style="color:green">緑</span>, 回転: <span style="color:blue">青</span>, 拡縮: <span style="color:orange">黄</span>）のグラフが、サブトラック上のタイムラインに直接重ねて表示される。
    *   別ウィンドウのグラフエディタを開くのではなく、タイムラインのトラック自体がグラフの背景となり、波形を直接ドラッグしてイージングを調整できる。
    *   これにより、ToonSquidのすっきりした階層構造と、Live2Dの視覚的なグラフ編集の良さを両立できる。
3.  **オンキャンバス操作との連動**
    *   該当フレームに再生ヘッドがある状態でキャンバス上の画像をドラッグすると、自動的にその時間へキーフレームが打たれ（または値が更新され）、波形が変化する。

## 4. 将来の拡張（モーフィング・ボーン）を見据えたアーキテクチャ設計

将来的にモーフィング（メッシュ変形）やボーン（リギング）を導入するためには、**「画像データ（ピクセル）」と「変形・演算データ」を明確に分離し、階層化できる構造**にしておくことが必須です。

### 4.1 データ構造の分離とノードツリー
現在CAFが持っているのは「描画の集まり（Stroke / Pixel）」ですが、これを一つの「レンダリング要素（Drawable）」として扱い、それを包む「変形ノード（Transform Node）」を導入します。

```json
// 将来的なアニメーション・レイヤー構造のイメージ
{
  "id": "layer_01",
  "name": "Arm",
  "transform": { // ベースとなる移動・回転・拡縮
    "x": 100, "y": 150, "rotation": 45, "scaleX": 1, "scaleY": 1
  },
  "keyframes": { // アニメーションするプロパティ群
    "transform.rotation": [
      { "time": 0, "value": 0, "easing": "ease-out" },
      { "time": 30, "value": 90, "easing": "linear" }
    ]
  },
  "content": {
    "type": "image",
    "caf_id": "caf_xyz" // 実際の描画データへの参照
  },
  // --- 将来の拡張用 ---
  "mesh": { ... }, // モーフィング用の頂点データ
  "bones": [ ... ] // ボーン階層
}
```

### 4.2 ボーンとモーフィングに繋がる仕組み
1.  **アフィン変換の独立性**:
    すべてのレイヤーやボーンは「ローカル座標系」を持ち、親から子へ変換行列（Matrix）が掛け合わされる構造（シーングラフ構造）にします。今の段階から、画像の描画時に `ctx.transform()` や PixiJS の `Container` を用いて、座標変換を分離して描画するパイプラインを確立しておきます。
2.  **プロパティの抽象化**:
    キーフレームで動かせるプロパティをハードコードせず、「対象のパス（例: `transform.x`, `bone[2].rotation`, `mesh.vertex[5].y`）」に対して値を設定する構造（Property Track機構）にしておきます。これにより、将来ボーンやメッシュの頂点データが増えても、同じアニメーションエンジン・同じグラフエディタUIで動かすことができます。
3.  **レンダリングパイプラインの準備**:
    （※本Phaseではまだ不要ですが）モーフィングを行うには、最終的に画像を三角形ポリゴン（メッシュ）に分割し、頂点を移動させてテクスチャを歪ませる処理（WebGL/WebGPU等）が必要になります。現在のCanvas2D/PixiJSベースの実装において、画像（テクスチャ）と変換行列を切り離して管理しておくことが、スムーズな移行への絶対条件となります。

## 5. 結論と次のステップ
まずは「ToonSquid」のような**展開式プロパティトラック**をベースとしつつ、ユーザー案である**インライン波形グラフ（パラメータごとの色分け）**を取り入れたモダンなUIを目指すのが、`tegaki`にとって最も魅力的なスタイルと言えます。

**実装のフェーズ分け提案:**
*   **Phase 1**: アニメーションエンジン（Property Track機構）の基盤作成と、単一レイヤー全体の単純なX/Y/回転/拡縮のキーフレーム補間処理の実装。
*   **Phase 2**: タイムラインUIの拡張（CAFクリップ下のトグル展開、インラインでの波形・キーフレーム表示）。
*   **Phase 3**: グラフ上での直接的なベジェハンドル操作（イージング調整機能）の実装。
*   **将来Phase**: WebGL/PixiJSのMesh機能を用いたメッシュ分割とボーン・モーフィングへの適用。
