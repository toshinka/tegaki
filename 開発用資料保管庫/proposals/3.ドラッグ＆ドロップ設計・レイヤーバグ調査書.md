# TEGAKI レイヤーパネルD&D設計 ＆ バグ修正報告書

本文書は、レイヤーパネルおよびアルバムパネルのドラッグ＆ドロップ（D&D）操作改善の設計提案と、SortableJSの使用に伴って発生していたバグの修正内容をまとめたものです。

---

## 1. 現状のバグ修正報告（止血パッチ）

SortableJS のフラットリスト前提の設計と、PixiJS の内部配列の直接操作に起因する重大なバグを修正しました。

### Bug 1：レイヤー並び替え時のシーングラフ・内部配列の破壊
*   **箇所**: `system/layer-system.js` — `reorderLayers()`
*   **原因**: `getLayers()` が返すシーングラフ配列 `currentFrameContainer._children` を直接 `splice()` して書き換えたため、PixiJS 内部の状態マップと不整合が起き、その後のレイヤー追加・順序変更操作が不全になっていた。
*   **対策**: `splice()` による直接配列書き換えを廃止。PixiJS の正規 API である `removeChildAt(fromIndex)` と `addChildAt(layer, toIndex)` のみを使用するように修正した。

### Bug 2：非表示（折りたたまれた）レイヤーがある場合のインデックス誤認
*   **箇所**: `ui/layer-panel-renderer.js` — `initializeSortable()` の `onEnd` コールバック
*   **原因**: SortableJS が返す `evt.newIndex` は「DOM 上で見えているアイテムだけ」を数えたインデックスであるのに対し、`layers.length` は非表示のインデックスも含んでいたため、順序の逆算時にインデックスがズレて誤ったレイヤーが操作されていた。
*   **対策**: `evt.newIndex` をそのまま計算に使わず、DOM 上のドロップ先要素の `data-layer-index` 属性を直接読み取ることで、折りたたまれた非表示レイヤーがあっても常に正しい絶対インデックスが取得できるようにした。

### Bug 3：閉じているフォルダへのレイヤー追加時の非表示バグ
*   **箇所**: `system/layer-system.js` — `addLayerToFolder()`
*   **原因**: 閉じているフォルダにレイヤーを追加した際、フォルダが閉じたまま（`folderExpanded = false`）だったため、追加したレイヤーがパネル上で非表示となって消えたように見えていた。
*   **対策**: レイヤーの追加時に、受け入れ先のフォルダを強制的に展開状態（`folderExpanded = true`）にし、トグル状態の同期イベントを発火するようにした。

### Bug 4：アクティブ表示枠のゴースト・残存
*   **箇所**: `ui/layer-panel-renderer.js` — `layer:activated` ハンドラ
*   **原因**: インデックスの更新が非同期で反映される前に、`_applyActiveLayerState` が即時実行されてしまい、ズレたインデックスに対応する古いDOMカードに選択枠が残ってゴースト表示になっていた。
*   **対策**: `_applyActiveLayerState` の即時高速更新処理を廃止し、`requestUpdate()` のデバウンス（16ms後）を介して安全に全体をリレンダーして更新するようにした。

## 2. 将来的な D&D 操作性改善提案（Pointer Events 自前実装）

SortableJS の調整には構造上の限界（並び替えが急すぎる、ツリー操作ができない等）があるため、将来的には液タブのペン入力とも親和性の高い **Pointer Events による完全自前実装**への移行を計画しています。

### 目標とする挙動
1.  **ドラッグ開始**: 要素の複製（ゴースト要素）を生成し、半透明かつ軽い傾き（`rotate(2deg)` 等）をかけてカーソルへ遅延なく追従させる。
2.  **ドラッグ中**:
    *   ホバー中の他要素をハイライトする。
    *   フォルダ上に 500ms 滞留した場合は、フォルダを自動的に展開する。
    *   ドロップ先の判定において、フォルダの「内側（収納）」に入るのか「隣（順序入替）」に並ぶのかを、ドラッグ位置の高さ（Y座標の比率）から判定する。
3.  **ドロップ**: ドロップ先に要素を移動し、GSAPアニメーション等で滑らかに吸着させる。
4.  **キャンセル**: `Escape` キー押下や範囲外ドロップ時、ゴーストを元の位置へ滑らかに戻す。

### 実装構成案
*   **`system/drawing/drag-drop-manager.js`** (新規): D&D全体のライフサイクル、ゴースト移動、ドロップインテントの判定を管理。
*   **`ui/layer-panel-renderer.js`** / **`ui/album-popup.js`**: マネージャーと連携し、順序入替やフォルダ収納のコマンドを実行。
*   *HTML5 Drag and Drop API は、液タブのペン入力（`pointerType: 'pen'`）と根本的に非互換であるため、採用を禁止する。*
