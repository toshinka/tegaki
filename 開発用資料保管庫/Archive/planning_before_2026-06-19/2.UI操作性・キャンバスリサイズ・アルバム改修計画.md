## 各タスクの詳細仕様・実装ステップ

### Task 1: 配色パターンジェネレーター（トライアド等配色ロジック）
*   **内容**: 選択中の「アクティブカラー（ベース色）」に基づいて、色彩調和の取れた自動配色パターンを生成するロジックを実装。
*   **仕様**:
    *   ベースカラーから、補色（Complementary）、アナログ（Analogous）、トライアド（Triad / 120度角）、スプリットコンプリメンタリー（Split-Complementary）などの配色パターンを計算。
    *   生成された6つのカラーを、パレット（カラーセット）の指定した1行（6スロット分）に自動で流し込んで置き換える。ベースカラーの変更にリアルタイムで追従するオプションを実装。
    *   **カラープリセット**: デフォルトの基本カラーセット（プリセット）をワンクリックでいつでも呼び出せるリセット/ロード機能を実装。

### Task 2: カラースロットのドラッグ＆ドロップ任意入れ替え
*   **内容**: パレット内の各カラースロット（色枠）をドラッグして、ユーザーが自由に並び替え（入れ替え）できる機能。
*   **仕様**: pointerイベントによる軽量な入れ替え処理を `QuickAccessPopup` に実装し、順序データをローカルストレージへ自動保存する。

### Task 3: トップバーの追加＆ON/OFF環境設定
*   **内容**: 画面上部にホーム、Undo、Redo、回転、反転、レイヤーON/OFFアイコンを配置したトップバーを実装。左右/上下反転時は、警告メッセージ（「左右反転中」など）を常時薄く表示し、事故を防ぐ。

### Task 4: 方向指定＆プレビュー対応のキャンバスリサイズUI
*   **内容**: 均等増減（`↔` / `↕`）と指定方向への増減（`←` / `→` / `↑` / `↓`）を切り替え。ポップアップを半透明にし、キャンバス上に増減範囲をプレビュー表示。

### Task 5: 設定FPSに応じたタイムライン縦線の強調表示
*   **内容**: 12FPSなら12フレームと13フレームの間、24FPSなら24と25フレームの間など、1秒ごとの区切り線の幅（太さ）を強調表示してタイミングを分かりやすくする。

### Task 6: アルバムUIの強化
*   **仕様**: `Ctrl+S` 保存成功時の「保存しました」トースト表示、一括選択・選択解除（`list-restart`）ボタンの追加、アイコンのMaroon/Creamテーマカラー統一。

### Task 7: レイヤーサムネイル市松模様背景の表示条件限定化
*   **内容**: 常時サムネイルに市松模様が表示されていると見づらいため、表示条件を制限する。
*   **仕様**: 「背景レイヤーを非表示（透明）にした際のみ」に連動してサムネイル背景に市松模様を表示、または設定での常時OFF切り替えに対応させる（クリスタル等のペイントソフトに準拠）。

### Task 8: スライダー・スクロールバー・UI透明度の統一
*   **内容**: アプリ全体の透過度とカラースキーム（Maroon/Cream）を統一。CSS（`.ui-slider`, `.ui-scrollbar`）を適用する。

### Task 9: ヒストリーが500を超えた際の挙動遅延の改善
*   **内容**: Undo/Redo の履歴数増加による配列操作の最適化、古いテクスチャの明示的破棄（`.destroy(true)`）の徹底。
