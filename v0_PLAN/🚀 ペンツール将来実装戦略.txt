📊 WebGL / WebGPU 系 ベクターペン ライブラリ比較
1. pixi-graphics-smooth（PixiJS拡張）

概要: PixiJS の Graphics を滑らかに描画する拡張。太線やアンチエイリアスに強い。

対応環境: WebGL（PixiJS v7互換）

評価:

描画品質 ⭐⭐⭐⭐

パフォーマンス ⭐⭐⭐（ストロークが多いと重い）

開発継続性 ⭐⭐⭐（更新少なめ）

PixiJS互換性 ⭐⭐⭐⭐⭐

総合: 16 / 20

2. Paper.js

概要: ベクターパス編集の老舗ライブラリ。キャンバス描画がメインだが WebGL 統合も可能。Undo/Redoや曲線編集が強力。

対応環境: Canvas2D が基本（WebGL/WebGPUは非対応だが統合実例は多数）

評価:

描画品質 ⭐⭐⭐⭐⭐（ベジエ曲線処理は最強）

パフォーマンス ⭐⭐（Canvasベースなので大量描画は重い）

開発継続性 ⭐⭐⭐（メンテは細々続いている）

PixiJS互換性 ⭐⭐（統合は工夫が必要）

総合: 13 / 20

3. Two.js

概要: ベクターレンダリングを抽象化し、Canvas/SVG/WebGL に切替可能。シンプルなAPI。

対応環境: Canvas / SVG / WebGL

評価:

描画品質 ⭐⭐⭐（基本図形中心、滑らかなストロークは工夫要）

パフォーマンス ⭐⭐⭐⭐（軽量設計）

開発継続性 ⭐⭐（更新は緩やか）

PixiJS互換性 ⭐⭐（直接統合は難しい）

総合: 11 / 20

4. HieroGL / regl-line2 / gl-lines 系

概要: WebGL向けのラインレンダリング専用ライブラリ群。太線やジョイントを GPU で処理。

対応環境: WebGL

評価:

描画品質 ⭐⭐⭐⭐（GPUでアンチエイリアス処理可）

パフォーマンス ⭐⭐⭐⭐⭐（大量ラインに最適）

開発継続性 ⭐⭐（研究/個人レベルが多い）

PixiJS互換性 ⭐⭐（PixiJSとは別系統）

総合: 13 / 20

5. Pathfinder / Pathfinder2（Mozilla発 WebGPU 実装）

概要: WebGPUベースのベクターレンダラー。文字やパスをGPUで直接ラスタライズ。

対応環境: WebGPU（モダン環境限定）

評価:

描画品質 ⭐⭐⭐⭐⭐（GPU曲線レンダリング最高峰）

パフォーマンス ⭐⭐⭐⭐⭐（WebGPUの恩恵で非常に速い）

開発継続性 ⭐⭐（Mozillaで開発止まりぎみ）

PixiJS互換性 ⭐（直接統合不可、独立ランタイム）

総合: 13 / 20

6. NVIDIA NV_path_rendering (WebGL extension)

概要: GPUハードウェアでベクターを描く WebGL 拡張。超高品質の曲線。

対応環境: 特定GPU + WebGL拡張サポート時のみ

評価:

描画品質 ⭐⭐⭐⭐⭐

パフォーマンス ⭐⭐⭐⭐（環境依存）

開発継続性 ⭐⭐（標準化されず、普及も低い）

PixiJS互換性 ⭐（Pixiで使うのは困難）

総合: 12 / 20

✅ 総評

PixiJSをベースに開発するなら → pixi-graphics-smooth 一択

編集性（ベジエ・Undo/Redo）重視なら → Paper.js のロジックを参考に、描画は PixiJS に任せる

本気でGPUベクター描画を狙うなら → Pathfinder（WebGPU） を研究用に導入

大量の線を高速に引くなら → regl-line2 / gl-lines 系をPixi外で統合

👉 実用路線なら「pixi-graphics-smooth + パス管理」が現実解。
👉 将来の布石なら「Paper.jsの編集性 + WebGPUベースのPathfinder描画」という二層構造が理想です。


PixiJS v8

将来的に「pixi-graphics-smooth不要」な高品質ベクター描画ができるようになる

ただし今の v7 ベース環境を v8 にすぐ移すのはリスクあり（依存崩壊）

Pathfinder / Pathfinder2

今のファイル群とは完全に別系統 → 新しい描画モジュールとして追加されるイメージ

PixiJSに直接統合するより、「Pathfinderで描いたものをテクスチャに転送してPixiシーンに載せる」方が現実的
