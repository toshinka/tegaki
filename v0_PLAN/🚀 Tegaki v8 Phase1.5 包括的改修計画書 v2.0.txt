# 🚀 Tegaki v8 Phase1.5 包括的改修計画書

## 📊 現状分析・問題点の特定

### 🔥 重大な問題（即座に修正が必要）

1. **EraserTool構文エラー** 
   - `eraser-tool.js:572` で構文エラー発生
   - 日本語コメントが不正な位置に混入
   - `window.Tegaki.EraserTool is not a constructor` エラー

2. **ToolManager準備状態問題**
   - `ToolManager still not ready after verification`
   - `isReady()` メソッドの実装不備

3. **座標表示オフセット問題**
   - 座標(0,0)が(110,95)に表示される座標変換の不具合

4. **Manager依存注入の不整合**
   - `RecordManager not ready - call setManagers() first`
   - `this.toolManager.setActiveTool is not a function`

### ⚠️ 中程度の問題

5. **ペン継続描画問題**
   - ペンを離しても描画が続く不具合
   - `onPointerUp` 処理の不備

6. **消しゴム機能完全停止**
   - EraserToolの読み込み失敗により消しゴムが使用不可

### 🔧 軽微な問題

7. **キャンバス枠の復活**
   - 削除したはずのキャンバス枠が表示される

## 🎯 改修方針・アーキテクチャ設計

### 🏗️ Manager初期化フロー（推奨順序）

```
1. ConfigManager.configure() 
2. CanvasManager.attach() → CanvasManager.init() 
3. EventBus.attach/init()
4. NavigationManager.attach/init()
5. CoordinateManager.attach/init()
6. RecordManager.attach/init() 
7. ToolManager.attach/init()
8. UI/Sidebar起動（EventBus 'app:ready'待機）
9. ShortcutManager.attach/init()
```

### 📦 依存関係マッピング

#### Manager間依存関係
```
CanvasManager (基幹) 
├── CoordinateManager (Canvas参照必須)
├── ToolManager (Canvas/Draw Container必須)
├── RecordManager (独立、但しCanvas状態参照)
├── NavigationManager (Canvas変形処理)
└── EventBus (全Manager共通通信層)
```

#### Tool依存関係
```
AbstractTool (基底)
├── PenTool
│   ├── CanvasManager (DrawContainer取得)
│   ├── CoordinateManager (座標変換)
│   └── RecordManager (描画記録)
└── EraserTool 
    ├── CanvasManager (DrawContainer取得)
    ├── CoordinateManager (座標変換)
    └── RecordManager (消去記録)
```

### 🔄 データフロー（描画プロセス）

```
1. DOM Event → TegakiApplication.onPointerXxx()
2. TegakiApplication → CoordinateManager.clientToWorld()
3. TegakiApplication → ToolManager.currentTool.onPointerXxx()
4. Tool → CanvasManager.getDrawContainer()
5. Tool → PIXI.Graphics操作 (v8 API)
6. Tool → RecordManager.addStroke() (描画完了時)
```

## 🛠️ 具体的改修計画

### Phase 1: 緊急修正（即座に実行）

#### 1.1 EraserTool構文修正
- **対象**: `tools/eraser-tool.js` 
- **問題**: 日本語コメント構文エラー
- **対策**: ファイル全体を新規作成・完全修正版に置換

#### 1.2 ToolManager準備状態修正
- **対象**: `managers/tool-manager.js`
- **問題**: `isReady()` の実装不備
- **対策**: 正しい `isReady()` 実装・エラーハンドリング改善

#### 1.3 座標変換修正
- **対象**: `utils/coordinate-manager.js`
- **問題**: DPR・Container変形の重複適用
- **対策**: 座標変換ロジックの見直し・検証機能追加

### Phase 2: Manager統合修正（依存関係修正）

#### 2.1 依存注入統一化
- **対象**: 全Manager・Tool
- **問題**: Manager注入方法の不整合
- **対策**: 
  ```javascript
  // 統一形式
  setManagersObject(managers) // Object形式受け取り
  configure(config)          // 設定注入
  attach(context)           // Pixi参照注入
  init()                   // Promise<void> 初期化
  isReady()               // boolean 準備完了確認
  dispose()               // 解放処理
  ```

#### 2.2 初期化順序の厳格化
- **対象**: `js/app-core.js`, `js/tegaki-application.js`
- **対策**: Promiseチェーン・await/async による順次初期化

### Phase 3: 描画処理最適化

#### 3.1 PixiJS v8 API完全準拠
- **対象**: 全描画処理
- **問題**: v7とv8のAPI混在
- **対策**: 
  ```javascript
  // v8新API使用
  graphics.circle(x, y, radius).fill({color: 0xff0000});
  graphics.moveTo(x, y).lineTo(x2, y2).stroke({width: 2, color: 0x000000});
  ```

#### 3.2 座標系統一
- **対象**: 全座標処理
- **対策**: `CoordinateManager` 経由のみ・直接座標計算禁止
