# 🎨 ふたば☆ちゃんねる風お絵描きツール Phase1.4 改修計画書

---

## 🚨 緊急課題と根本原因

### 診断結果（GROK4分析ベース）
1. **タブレットペン描画不可**: `pen-tool.js` の `isActive` が常に `false`
2. **筆圧値非取得**: `event.pressure` ではなく `event.data?.originalEvent?.pressure` が必要
3. **イベント処理競合**: DOM直付けとPixiJS両方のイベントリスナー
4. **UI管理不備**: popup-manager.js と slider-manager.js が未実装

### 技術的問題点
- PenTool のイベントリスナーがDOM直付け、PixiJS側と別経路
- main.js でペンツール選択時に `isActive = true` 設定なし
- 座標変換が `getCanvasCoordinates()` 経由で不統一

## 🎯 Phase A: 緊急修正（2-3日）

### A1. タブレットペン即効修正

**対象ファイル**: `tools/pen-tool.js`
**参考ファイル**: 診断書の修正内容

**修正内容**:
1. `activate()` メソッドで `this.isActive = true` を確実に設定
2. `extractPointerInfo()` で筆圧値参照を `event.data?.originalEvent?.pressure` 優先に変更
3. `setupEventListeners()` をPixiJS app.view経由に統一

**対象ファイル**: `js/main.js`
**参考ファイル**: なし（単純なフラグ設定）

**修正内容**:
1. `selectPenTool()` メソッドで取得したPenToolの `isActive = true` を強制設定

## 💡 Claude向け実装ガイド

### 推奨実装アプローチ
1. **Phase A優先**: 即効性のある修正から開始
2. **既存コード活用**: HTMLの構造やクラス名をそのまま利用
3. **エラーハンドリング**: try-catch でフォールバック実装
4. **DRY原則**: 共通処理は関数化して重複排除

### ファイル間依存関係
```
index.html (スクリプト読み込み順序重要)
├── libs/pixi-extensions.js
├── js/app-core.js
├── js/main.js
├── managers/ui-manager.js
├── managers/popup-manager.js (新規)
├── managers/slider-manager.js (新規)
└── tools/pen-tool.js
```

### 実装時の注意点
- **Pure JavaScript**: ESM/import禁止、グローバル変数使用
- **既存HTML活用**: IDやクラス名は変更せずそのまま使用
- **段階的実装**: Phase A → B → C の順序で実装
- **後方互換性**: 既存機能を破壊しない修正
