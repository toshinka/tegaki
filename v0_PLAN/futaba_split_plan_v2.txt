## 🔄 レイヤー機能配置の変更理由

### **現行問題**
- LayerManager が main.html にあるため、レイヤーUI改修時に2ファイル跨ぎ
- UI操作（ドラッグ、表示切替）なのに描画基盤側に配置
- AI改修効率が悪い（レイヤー関連改修の95%がUI絡み）

### **改訂後の利点**
- ✅ **改修効率**: レイヤー関連はtools.js完結
- ✅ **機能的近接**: InterfaceManagerと密結合
- ✅ **AI処理負荷軽減**: 1ファイルで改修完結
- ✅ **拡張性**: 将来的なレイヤー機能（ブレンドモード等）追加が容易

## 🌉 接続インターフェース設計

### **LayerBridge パターン**
main.html と tools.js 間の疎結合を実現

```javascript
// === futaba_main.html ===
class LayerBridge {
    constructor(drawingEngine) {
        this.engine = drawingEngine;
        this.manager = null;
    }
    
    // tools.js から呼び出し
    connectManager(layerManager) {
        this.manager = layerManager;
        log('🌉 LayerManager connected to DrawingEngine');
    }
    
    // PIXI操作のプロキシメソッド
    addContainer(container) {
        this.engine.containers.world.addChild(container);
    }
    
    removeContainer(container) {
        this.engine.containers.world.removeChild(container);
    }
    
    reorderContainer(container, index) {
        this.engine.containers.world.removeChild(container);
        this.engine.containers.world.addChildAt(container, index);
    }
}

class DrawingEngine {
    constructor() {
        // ...
        this.layerBridge = new LayerBridge(this);
    }
}
```

```javascript
// === futaba_tools.js ===
class LayerManager {
    constructor(layerBridge) {
        this.bridge = layerBridge;
        this.layers = new Map();
        // ...
    }
    
    createLayer(name) {
        const container = new PIXI.Container();
        // ...
        this.bridge.addContainer(container); // 疎結合でPIXI操作
    }
    
    reorderLayers(fromIndex, toIndex) {
        // ...
        this.bridge.reorderContainer(container, toIndex);
    }
}
```

## 📋 詳細クラス配置

### **futaba_main.html** 
| クラス | 責務 | 主要メソッド |
|--------|------|-------------|
| AppBootstrap | 初期化・ライブラリ管理 | `init()`, `checkLibraries()` |
| AppController | アプリ統合制御・入力処理 | `initialize()`, `setupInputHandlers()` |
| DrawingEngine | PIXI管理・描画コア・パス作成 | `createPath()`, `extendPath()`, `resize()` |
| LayerBridge | レイヤー接続インターフェース | `connectManager()`, `addContainer()` |
| DomPositionManager | DOM位置制御・ペン対応 | `startPanning()`, `updatePosition()` |
| SystemMonitor | パフォーマンス監視 | `trackFPS()`, `updateStats()` |

### **futaba_tools.js**
| クラス | 責務 | 主要メソッド |
|--------|------|-------------|
| DrawingTools | 描画ツール・ブラシ管理 | `startDrawing()`, `setBrushSize()` |
| InterfaceManager | UI管理・ポップアップ・スライダー | `togglePopup()`, `updateSliders()` |
| LayerManager | レイヤーシステム・ドラッグ機能 | `createLayer()`, `reorderLayers()`, `toggleVisibility()` |

## ⚡ 初期化フロー（改訂版）

```javascript
// 1. main.html - 基盤システム起動
AppBootstrap.init()
→ AppController.initialize()
→ DrawingEngine.initialize()
→ LayerBridge 準備完了

// 2. tools.js - ツールシステム起動

→ UI初期化完了

// 3. 相互接続
→ AppController.setTools(tools, ui, layerManager)
→ システム統合完了
```
