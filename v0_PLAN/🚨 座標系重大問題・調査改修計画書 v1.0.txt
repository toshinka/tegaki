# 🚨 座標系重大問題・調査改修計画書 v1.0

## 📋 問題現象・根本原因分析

### 🚨 重大問題症状
```
✏️ 描画開始: pen at (0, 0)  ← **座標が常に0,0になる**
🔧 描画フォールバック: continue - AppCore経由
🔧 描画フォールバック: stop - AppCore経由
```

### 🔍 根本原因分析

#### **問題1: 座標変換チェーンの多重化**
```
実際の流れ（問題のある状態）:
マウスクリック → BoundaryManager → CoordinateManager(変換1) → 
AppCore.handleBoundaryCrossIn → CoordinateManager(変換2) → 
ToolManager.startDrawing(0,0) → PenTool.handleBoundaryCrossIn(変換3)
```

#### **問題2: ToolManagerの描画フォールバック暴走**
- 現在のToolManagerは `registeredTools` が空のため、常に `handleDrawingFallback` に逸れる
- ペンツール・消しゴムツールが未登録のため、すべて(0,0)座標のフォールバック描画になる

#### **問題3: AppCoreの境界処理で座標再変換**
```javascript
// AppCore.handlePixiBoundaryCross() で不要な再変換
const pixiCoords = this.coordinateManager.canvasToPixi(
    data.position.x,  // <- 既にキャンバス座標なのに
    data.position.y,  // <- さらにPixi座標に変換
    this.app
);
```

## 🎯 段階的改修戦略

### **Phase1: 座標系整理・重複除去**

#### **STEP1: 座標変換フロー正規化**
**対象ファイル**: `js/app-core.js`  
**作業時間**: 30分  
**参考ファイル**: `js/utils/coordinate-manager.js`, `managers/boundary-manager.js`

**問題箇所**:
```javascript
// js/app-core.js 349行目付近（修正対象）
handlePixiBoundaryCross(data) {
    // ❌ 不要な座標変換（data.positionは既にキャンバス座標）
    const pixiCoords = this.coordinateManager.canvasToPixi(
        data.position.x, data.position.y, this.app
    );
}
```

**修正内容**:
```javascript
handlePixiBoundaryCross(data) {
    // ✅ BoundaryManagerから来る座標は既にキャンバス座標なのでそのまま使用
    const canvasCoords = data.position; // 座標変換不要
    
    // アクティブツールに直接渡す
    const currentTool = this.toolManager?.getCurrentTool();
    const currentToolInstance = this.toolManager?.registeredTools.get(currentTool);
    
    if (currentToolInstance?.handleBoundaryCrossIn) {
        currentToolInstance.handleBoundaryCrossIn(
            canvasCoords.x, canvasCoords.y, data
        );
    }
}
```

#### **STEP2: ToolManager登録システム修正**
**対象ファイル**: `managers/tool-manager.js`  
**作業時間**: 45分  
**参考ファイル**: `tools/pen-tool.js`, `tools/eraser-tool.js`

**問題箇所**: ToolManagerの初期化でツールが実際に登録されていない

**修正内容**:
```javascript
// ToolManager.registerBasicTools() の改良
registerBasicTools() {
    // 実際のツールクラスを検索・登録
    if (window.PenTool) {
        const penTool = new window.PenTool(this);
        penTool.initialize();
        this.registeredTools.set('pen', penTool);
    }
    
    if (window.EraserTool) {
        const eraserTool = new window.EraserTool(this);
        eraserTool.initialize();
        this.registeredTools.set('eraser', eraserTool);
    }
}
```

#### **STEP3: 境界越え座標正規化**
**対象ファイル**: `tools/pen-tool.js`, `tools/eraser-tool.js`  
**作業時間**: 20分  
**参考ファイル**: `managers/boundary-manager.js`

**確認事項**: BoundaryManagerが渡す座標は**キャンバス座標**で統一されている

### **Phase2: 描画システム統合修正**

#### **STEP4: AppCore描画イベント統合修正**
**対象ファイル**: `js/app-core.js`  
**作業時間**: 30分  

**問題箇所**: handlePointerDown等で座標抽出・ToolManager連携

**修正内容**:
```javascript
handlePointerDown(event) {
    const coords = this.coordinateManager.extractPointerCoordinates(
        event, 
        this.app.view.getBoundingClientRect()
    );
    
    // ✅ キャンバス座標をそのまま渡す
    this.toolManager.startDrawing(coords.canvas.x, coords.canvas.y, coords.pressure);
}
```
