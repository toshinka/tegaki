## 📋 変更概要

統一計画書に従い、以下の統一システムを追加し、既存の重複関数を統合しました：

1. **ConfigManager**: 設定値一元化
2. **ErrorManager**: エラー処理統一  
3. **StateManager**: 状態管理統一
4. **EventBus**: 循環依存排除・イベント駆動アーキテクチャ

### 🏗️ ConfigManager（新規）

**責務**: 全設定値の一元管理・統一アクセス

```javascript
class ConfigManager {
    // 統一設定値（DRY原則準拠）
    static defaultConfig = {
        canvas: { width: 400, height: 400, backgroundColor: 0xf0e0d6, ... },
        drawing: { 
            pen: { defaultSize: 16.0, minSize: 0.1, maxSize: 100.0, ... },
            eraser: { defaultSize: 16.0, opacity: 1.0, ... }
        },
        ui: { slider: { updateThrottle: 16, ... }, popup: { ... } },
        performance: { targetFPS: 60, maxInitTime: 5000, ... },
        error: { displayDuration: 10000, maxRetries: 3, ... },
        colors: { futabaMaroon: 0x800000, ... },
        v8Migration: { enabled: false, useWebGPU: false, ... }
    }
    
    // 主要メソッド
    static get(path)                    // ドット記法で設定取得
    static set(path, value)             // 設定値変更
    static getCanvasConfig()            // キャンバス設定取得
    static getDrawingConfig(tool)       // 描画ツール設定取得
    static getColors()                  // ふたばカラーパレット取得
    static validate(path, value)        // 設定値妥当性確認
    static getDebugInfo()               // デバッグ情報取得
}
```
