原因は Viewport.Viewport is not a constructor です。

これが意味すること：

pixi-viewport の読み込み形式の問題

<script src="https://cdn.jsdelivr.net/npm/pixi-viewport/dist/viewport.min.js"></script> を使った場合、グローバルオブジェクトは Viewport であり、Viewport.Viewport ではなく new Viewport(...) で作る必要があります。

ES Module のように import { Viewport } from 'pixi-viewport' ではなく、ブラウザ用の UMD バンドルを <script> で読み込む場合の仕様です。

コード上の誤り

現状のコードでは new Viewport.Viewport(...) としているため、存在しないコンストラクタを呼んで TypeError になっています。

改修計画

目的: pixi-viewport をブラウザ読み込みで正しく初期化し、キャンバス移動/拡縮/回転が動作するようにする。

Viewport の初期化修正

// 修正前
const viewport = new Viewport.Viewport({ ... });

// 修正後（UMD版の場合）
const viewport = new Viewport({ ... });


Viewport がグローバル変数として UMD で提供されているため、.Viewport は不要。

そのほか依存関係確認

PIXI.js v8 はすでに読み込まれていることを確認済みなので問題なし。

viewport に対する drag/pinch/wheel/decelerate の呼び出しもそのままでOK。

テストポイント

ブラウザでページを開き、console に TypeError が出ないこと

Space+ドラッグ / Shift+Space+ドラッグ / ホイール / H / Home が期待通り動作すること

背景＋描画レイヤーが一括で移動・回転・拡縮されること

後続対応

将来的に ES Modules 版に切り替える場合は、import { Viewport } from 'pixi-viewport' を使えるように変更する。

既存の <script> 方式で十分なら、上記修正だけで問題は解消。

要するに、修正は new Viewport.Viewport(...) → new Viewport(...) だけで済みます。



☑サイドバーアイコンの色がふたばカラーでない事について

1. 原因

lucide-static の SVG は内部で stroke="currentColor" ではなく灰色など固定値になっている場合がある

PixiJS の PIXI.Sprite.from() で SVG を読み込むと、SVG 内の色情報がそのまま反映されるため、後から tint で変更しても期待通りの色にならないことがある

2. 改修方針

方針 A：SVG 内の stroke を書き換えてから PixiJS で読み込む

function loadIconSVG(url, color="#800000"){
    return fetch(url)
        .then(resp => resp.text())
        .then(svgText => {
            // stroke と fill を指定色に書き換え
            svgText = svgText.replace(/stroke=".*?"/g, `stroke="${color}"`);
            svgText = svgText.replace(/fill=".*?"/g, `fill="${color}"`);
            // Blob に変換して URL を生成
            const blob = new Blob([svgText], { type: "image/svg+xml" });
            const blobURL = URL.createObjectURL(blob);
            return PIXI.Sprite.from(blobURL);
        });
}

// 使用例
loadIconSVG('icons/pencil-line.svg').then(sprite => {
    sprite.x = 10;
    sprite.y = 10;
    app.stage.addChild(sprite);
});


ポイント

fetch で SVG を取得して文字列操作

stroke と fill を #800000 に統一

Blob URL を生成して PixiJS の Sprite に変換

方針 B：PixiJS の tint で色を変更する（SVG が currentColor 使用の場合のみ）

const sprite = PIXI.Sprite.from('icons/pencil-line.svg');
sprite.tint = 0x800000; // 16進数カラー


ただし、SVG 内で固定色が設定されている場合は効かない

3. 改修計画

SVG アイコンを一括で色変更するラッパ関数を作る

新規作成アイコンも同じ関数で読み込む

既存 toolbar アイコン読み込み箇所を置き換え

PIXI.Sprite.from(url) → loadIconSVG(url, "#800000")

確認ポイント

灰色で出ていたアイコンがすべて #800000 に統一されること

PixiJS 上での描画やスケーリングに問題がないこと

💡 補足

ランタイムで色を変えたい場合は方針 A が確実

事前に SVG を編集して stroke/fill を currentColor にしておけば tint で簡単に変えられる
