vector_drawing_tool_v8rev5.html 改修計画（PIXI 仕様にリサイズ反映）
1. 改修の目的

描画範囲の自動管理
現状の自作バックエンドでは、backendWidth / backendHeight の変更が Canvas や描画処理に正しく反映されない場合がある。
→ PIXI の Container + Graphics を使うことで、透明矩形を置くだけで描画範囲を自動管理可能。

スケール変更の統一化
現状では backendScale を掛ける処理を自前で管理しているが、PIXI を使うことで scale プロパティで一括管理できる。

将来的な拡張の容易化
マスク、スクロール、回転など、PIXI の標準機能をそのまま使えるため、追加機能の実装コストを下げられる。

2. 改修方針

透明矩形（透明 Graphics）を常に配置

const backendContainer = new PIXI.Container();
const backendRect = new PIXI.Graphics();
backendRect.beginFill(0x000000, 0); // 完全透明
backendRect.drawRect(0, 0, backendWidth, backendHeight);
backendRect.endFill();
backendContainer.addChild(backendRect);
app.stage.addChild(backendContainer);


これにより PIXI が描画範囲を認識する

子要素（描画オブジェクト）の表示/スケーリングが容易になる

リサイズ時の反映

function resizeBackend(newWidth, newHeight) {
    backendWidth = newWidth * backendScale;
    backendHeight = newHeight * backendScale;
    backendRect.clear();
    backendRect.beginFill(0x000000, 0);
    backendRect.drawRect(0, 0, backendWidth, backendHeight);
    backendRect.endFill();
}


透明矩形のサイズを更新するだけで、描画領域が自動調整される

backendContainer.scale.set(...) で全体スケールも統一可能

既存自作描画処理の置換

backendContext.drawXXX 系の自作描画処理は PIXI Graphics / Sprite で置換

最小限のラッパー関数を作り、従来の描画 API 呼び出しを PIXI にマッピングする

3. 期待される効果

リサイズやスケール変更が正確にバックエンドに反映される

子オブジェクトの描画が透明矩形に収まることを自動保証

将来のカメラ、スクロール、回転、マスク機能への対応が容易

自作 Canvas 管理コードを減らし、保守性が向上

4. 注意点

PIXI を導入するため、vector_drawing_tool_v8rev5.html 内の自作 Canvas 初期化コードは置換

描画速度はほぼ同等だが、大量オブジェクト描画時は PIXI の最適化（cacheAsBitmap など）を考慮

既存スケール・座標計算は PIXI のスケールと座標に置き換え

💡 ポイントまとめ：

透明矩形 = 描画範囲の基準

Container + scale = バックエンド全体のスケール管理

既存描画関数は PIXI ラッパーに置換


※下記はサンプルコードです

<!DOCTYPE html>
<html lang="ja">
<head>
<meta charset="UTF-8">
<title>Vector Drawing Tool v8rev5 - PIXI互換ラッパー版</title>
<style>
  body { margin: 0; overflow: hidden; }
  canvas { display: block; }
</style>
</head>
<body>

<script src="https://cdnjs.cloudflare.com/ajax/libs/pixi.js/7.2.4/pixi.min.js"></script>
<script>
/* ===== グローバル変数 ===== */
let backendScale = 5;      // 好みの倍率
let backendWidth = 400 * backendScale;
let backendHeight = 400 * backendScale;

let app, backendContainer, backendRect;

/* ===== 初期化 ===== */
function init() {
    app = new PIXI.Application({
        width: window.innerWidth,
        height: window.innerHeight,
        backgroundColor: 0xffffff,
    });
    document.body.appendChild(app.view);

    // バックエンド用コンテナ
    backendContainer = new PIXI.Container();
    app.stage.addChild(backendContainer);

    // 透明矩形（描画範囲）
    backendRect = new PIXI.Graphics();
    drawBackendRect(backendWidth, backendHeight);
    backendContainer.addChild(backendRect);

    // リサイズイベント
    window.addEventListener('resize', onResize);

    // サンプル描画（既存 API 風）
    SampleAPI.drawCircle(50, 50, 25, 0xff0000);
    SampleAPI.drawRect(100, 100, 60, 40, 0x0000ff);
}

/* ===== バックエンド矩形描画 ===== */
function drawBackendRect(width, height) {
    backendRect.clear();
    backendRect.beginFill(0x000000, 0); // 完全透明
    backendRect.drawRect(0, 0, width, height);
    backendRect.endFill();
}

/* ===== バックエンドリサイズ ===== */
function resizeBackend(newWidth, newHeight) {
    backendWidth = newWidth * backendScale;
    backendHeight = newHeight * backendScale;
    drawBackendRect(backendWidth, backendHeight);

    // 全体スケール調整（既存描画 API の互換のため）
    backendContainer.scale.set(1 / backendScale);
}

/* ===== ウィンドウリサイズ時 ===== */
function onResize() {
    app.renderer.resize(window.innerWidth, window.innerHeight);
    // 必要に応じて backendWidth / backendHeight を更新
    // resizeBackend(window.innerWidth, window.innerHeight);
}

/* ===== 既存描画 API 風ラッパー ===== */
const SampleAPI = {
    drawCircle: function(x, y, radius, color) {
        const g = new PIXI.Graphics();
        g.beginFill(color);
        g.drawCircle(x * backendScale, y * backendScale, radius * backendScale);
        g.endFill();
        backendContainer.addChild(g);
    },
    drawRect: function(x, y, width, height, color) {
        const g = new PIXI.Graphics();
        g.beginFill(color);
        g.drawRect(x * backendScale, y * backendScale, width * backendScale, height * backendScale);
        g.endFill();
        backendContainer.addChild(g);
    },
    clear: function() {
        backendContainer.removeChildren(1); // 透明矩形は残す
    }
};

/* ===== 初期化実行 ===== */
init();
</script>
</body>
</html>
🔹 改修ポイントと説明（改修担当向け）
透明矩形を常に置く

描画範囲を明示的に確保。PIXI 側でスケールや座標管理がしやすくなる。

backendScale による統一スケール

従来自作バックエンドの「ピクセル単位」感覚を維持。

既存コードの座標・サイズをそのまま使える。

互換ラッパー SampleAPI

drawCircle, drawRect, clear を用意。

従来自作描画関数をほぼそのまま置き換え可能。

PIXI Graphics を内部的に生成し、backendContainer に追加。

backendContainer.scale

拡大縮小やリサイズに合わせた描画調整を統一的に実施。

リサイズ対応

ウィンドウリサイズ時に PIXI レンダラーを更新。

必要に応じてバックエンド描画領域もリサイズ可能。

💡 この形にすると、従来の自作描画コードをほぼ変えずに PIXI に置き換えられるため、既存ロジックはそのまま再利用、拡張や高速化も容易です。
