🛠 改修計画

下地キャンバスを動的にリサイズ

resizeCanvas(width, height) 時に
offscreenCanvas.width = width * 3;
offscreenCanvas.height = height * 3;
と再確保する。

これにより「カメラの 3倍×3倍の透明下地」が常に確保され、リサイズ後もズレが発生しない。

カメラ座標の導入

cameraX, cameraY を基準に
mainCanvas.drawImage(offscreenCanvas, cameraX, cameraY, width, height, 0, 0, width, height);
とすることで「下地に描いたものをカメラで切り取って表示」できる。

これで無限キャンバスに近い仕組みが可能になる。

消しゴム修正

offscreenCtx.globalCompositeOperation = "destination-out" のあと必ず
mainCanvas に drawImage をかけ直す。

さらに履歴 (history) にも「消去ストローク」を push する必要がある。

リサイズ時の背景色

mainCanvas の背景色（f0e0d6）を再塗り直す処理を
resizeCanvas に追加する。

👉 まとめると、
「非破壊記録・透明下地キャンバス」はコード上でも正しく機能しているけれど、 動的リサイズとカメラ座標管理が未完成 なので、今後の拡張（無限キャンバス・レイヤー移動）を見据えて修正が必要、という状態です。


🎯 改修目的

下地キャンバス (offscreenCanvas) を カメラ範囲の3倍×3倍 に動的確保する。

ユーザーのリサイズ操作に応じて再計算・再描画を行い、見た目のズレや背景色の残存を防ぐ。

将来的なカメラ移動にも備え、カメラ座標を基準に mainCanvas に切り出す方式に改修する。

🛠 改修計画
1. resizeCanvas の修正

現在：mainCanvas.width/height しか更新していない。

改修：offscreenCanvas も「カメラ範囲の3倍」に合わせてリサイズ。

function resizeCanvas(width, height) {
    mainCanvas.width = width;
    mainCanvas.height = height;

    // 背景塗り直し（f0e0d6）
    const ctx = mainCanvas.getContext("2d");
    ctx.fillStyle = "#f0e0d6";
    ctx.fillRect(0, 0, width, height);

    // 下地キャンバスを動的再確保（3倍）
    offscreenCanvas.width = width * 3;
    offscreenCanvas.height = height * 3;
}

2. 描画ループの修正

現在：drawImage(offscreenCanvas, 0, 0, width, height, 0, 0, width, height);

改修：カメラ座標 (cameraX, cameraY) を考慮。

function render() {
    ctx.fillStyle = "#f0e0d6";
    ctx.fillRect(0, 0, mainCanvas.width, mainCanvas.height);

    ctx.drawImage(
        offscreenCanvas,
        cameraX, cameraY, mainCanvas.width, mainCanvas.height, // 切り出し
        0, 0, mainCanvas.width, mainCanvas.height             // mainCanvasに表示
    );
}


👉 こうすることで、カメラ移動や拡張が自然に実装できる。

3. ストローク記録時の座標変換

mousemove などで取得する x, y は mainCanvas 基準。

改修：オフスクリーン記録時に カメラオフセットを加算する。

offscreenCtx.lineTo(x + cameraX, y + cameraY);


これで「offscreenCanvas に常にワールド座標で記録」される。

4. 履歴管理の維持

history.push(stroke) はそのまま活かせる。

ただし stroke 内の座標も cameraX, cameraY 加算済みで保存する必要がある。

✅ 期待される改善効果

キャンバスを小さくリサイズしても 見た目の背景と下地が一致する。

大きくしても下地が拡張されるので 端ギリギリまで描画しても破綻しない。

将来の 無限キャンバス / カメラ移動 / レイヤー移動 に備えられる。

この改修を入れると、「ユーザーが意識するのはカメラ座標だけ」で済みます。
👉 下地キャンバスは常に3倍確保 → カメラで切り出し → 背景色を mainCanvas で描画。


🔧 参考用コード（差し込み用）
<script>
const mainCanvas = document.getElementById("canvas");
const ctx = mainCanvas.getContext("2d");

// 下地キャンバス（透明）
const offscreenCanvas = document.createElement("canvas");
const offscreenCtx = offscreenCanvas.getContext("2d");

// カメラ座標（将来の移動も考慮）
let cameraX = 0;
let cameraY = 0;

// ================================
// 📌 キャンバスリサイズ処理
// ================================
function resizeCanvas(width, height) {
    // 表示用キャンバス
    mainCanvas.width = width;
    mainCanvas.height = height;

    // 背景色を塗り直す
    ctx.fillStyle = "#f0e0d6";
    ctx.fillRect(0, 0, width, height);

    // 下地キャンバスをカメラ範囲の3倍で確保
    offscreenCanvas.width = width * 3;
    offscreenCanvas.height = height * 3;

    // 中央にカメラを置く（カメラ移動を考慮）
    cameraX = width;
    cameraY = height;
}

// ================================
// 📌 描画ループ（mainCanvasへ切り出し）
// ================================
function render() {
    // 背景塗り直し
    ctx.fillStyle = "#f0e0d6";
    ctx.fillRect(0, 0, mainCanvas.width, mainCanvas.height);

    // 下地キャンバスの一部を mainCanvas へ表示
    ctx.drawImage(
        offscreenCanvas,
        cameraX, cameraY, mainCanvas.width, mainCanvas.height, // 切り出し元
        0, 0, mainCanvas.width, mainCanvas.height              // 表示先
    );
}

// ================================
// 📌 ペン描画処理（例）
// ================================
let drawing = false;

mainCanvas.addEventListener("mousedown", (e) => {
    drawing = true;
    const x = e.offsetX + cameraX;
    const y = e.offsetY + cameraY;
    offscreenCtx.beginPath();
    offscreenCtx.moveTo(x, y);
});

mainCanvas.addEventListener("mousemove", (e) => {
    if (!drawing) return;
    const x = e.offsetX + cameraX;
    const y = e.offsetY + cameraY;
    offscreenCtx.lineTo(x, y);
    offscreenCtx.stroke();
    render();
});

mainCanvas.addEventListener("mouseup", () => {
    drawing = false;
});

// ================================
// 📌 初期化
// ================================
resizeCanvas(400, 400);
render();
</script>

✅ このコードのポイント

下地キャンバスは常に3倍サイズ

resizeCanvas 時に自動で再確保。

cameraX, cameraY で「どこを切り出すか」を決定。

描画は常にオフセット加算

x + cameraX, y + cameraY を記録。

将来カメラ移動しても記録がズレない。

背景色は mainCanvas 側で管理

下地キャンバスは常に透明。

表示時に f0e0d6 を塗ってから drawImage。
