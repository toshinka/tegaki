# Tegaki v8_train_rev7 改修計画書

## 現状把握
- `tegaki-v8-train-main.html` は **全機能を内包した肥大化ファイル**  
  - CSS（`tegaki-styles.html` 相当）  
  - Core処理（`tegaki-a2-core.html` 相当）  
  - UI/ポップアップ（`tegaki-b1-ui.html` 相当）  
  - Status表示（`tegaki-b2-status.html` 相当）  
- 他の分割車両（`a1〜b2`）と役割が重複しており、**単体で動作可能** な状態。  
- ペンポップアップ・キャンバスリサイズなどの挙動は **main 内の実装に依存** しており、割り当てが未整理。

---

## 改修目的
1. `tegaki-v8-train-main.html` の肥大化を解消  
2. UI 機能（ペンポップアップ等）を **`b1-ui.html` に集約**  
3. ステータス表示を **`b2-status.html` に集約**  
4. 共通スタイルを **`tegaki-styles.html`** へ一元化  
5. `tegaki-v8-train-main.html` は「車両連結テスト用の制御ファイル」として軽量化

---

## 改修ステップ

### Step 1. 機能抽出
- main 内の以下の処理を確認し抽出：
  - **ペンポップアップUI関連**
  - **キャンバスリサイズ処理**
  - **ステータス更新処理**

### Step 2. 車両への移管
- ペンポップアップ処理 → `tegaki-b1-ui.html`  
- キャンバスリサイズ処理 → `tegaki-a2-core.html`（描画系に近い）  
- ステータス更新処理 → `tegaki-b2-status.html`

### Step 3. スタイル整理
- main 内の `<style>` を削除し、`tegaki-styles.html` へ移管。  
- 各車両ではスタイルを読み込むだけに統一。

### Step 4. main 軽量化
- `tegaki-v8-train-main.html` から実装コードを削除し、以下の責務に限定：
  - 各車両（a1〜b2）の読み込み  
  - 初期化処理の呼び出し  
  - デバッグ時の統合確認

### Step 5. 動作検証
- ペンポップアップの挙動が **移管後も現状と同等** であることを確認  
- キャンバスリサイズが **main 依存から切り離されて正常動作** することを確認  
- ステータス表示が **b2-status 側で管理可能** であることを確認  

---

## リスクと対策
- **リスク**：main 側の修正でポップアップやリサイズ挙動が変化  
  → **対策**：必ず機能を移管したうえで main から削除、二重管理を防ぐ。  
- **リスク**：スタイルが分散し UI 崩壊  
  → **対策**：全スタイルを `tegaki-styles.html` へ統合。

---

## 改修完了後の姿
- `tegaki-v8-train-main.html`  
  → 車両ロード & 全体制御のみ（軽量化）  
- `tegaki-a1-header.html`  
  → メタ情報、ヘッダー管理  
- `tegaki-a2-core.html`  
  → キャンバス初期化、リサイズ処理  
- `tegaki-a3-integration.html`  
  → イベント連結処理  
- `tegaki-b1-ui.html`  
  → ペンポップアップ、UI要素  
- `tegaki-b2-status.html`  
  → ステータス表示・更新処理  
- `tegaki-styles.html`  
  → 全スタイルを集約
