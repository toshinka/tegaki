# Rulebook Scope

## 目的
全体の指針・原則・品質基準を定める。

## 含める内容
- 開発の理念・原則
- 構造ガイドライン
- 品質基準
- 移行戦略の方向性

## 含めない内容
- 実装コード
- 手順
- テスト方法

## 相関関係
- Symbol Dictionary → 用語参照
- Migration Plan → 指針を参照


# Symbol Dictionary Scope

## 目的
共通言語の辞書化、表記ゆれ防止、設計で使う用語の統一。

## 含める内容
- 専門用語の定義
- アイコンや識別子の意味
- 用語間の関係
- 命名規則に沿った用例

## 含めない内容
- 手順
- コード
- テスト方法

## 相関関係
- Rulebook → 命名規則の方針を参照
- Migration Plan → 用語の正確性を保証


# Migration Plan Scope

## 目的
実務の具体的な進め方を定め、工程を管理する。

## 含める内容
- フェーズ分け（Phase1, Phase2 …）
- ファイルごとの対応方針
- 実装ガイド（具体的手順）
- テスト方法・チェックリスト
- 依存関係や優先度

## 含めない内容
- 開発理念（Rulebook に委任）
- 用語定義（Dictionary に委任）

## 相関関係
- Rulebook → 方針を参照
- Symbol Dictionary → 用語を参照

# Document Relations

- Rulebook → Symbol Dictionary (用語参照)
- Rulebook → Migration Plan (方針に従う)
- Migration Plan → Symbol Dictionary (用語統一)
