改修計画書：vector_drawing_tool_v8rev6.html ベースのワールドビュー操作導入
目的

従来の実装では、

ペン描画は安定している

画面外の描画も保持できる

Undo/Redo も正しく機能

という長所があるが、ユーザーが自由に 画面全体を移動・拡縮・回転して絵を扱うことができない。

本改修では、「ビュー全体を操作できる仕組み」（以下、ワールドビュー操作）を追加し、
ユーザーがまるで「カメラを動かす」ように自由に描画内容を扱えるようにする。

改修方針

カメラ用コンテナ（worldContainer）を導入

全レイヤーや描画オブジェクトはこの worldContainer に格納

ユーザーが操作するのはキャンバスそのものではなく worldContainer

→ これにより「全部まとめて動く」挙動を実現

操作割り当て

移動

Space + ドラッグ

Space + 矢印キー

拡縮・回転

Shift + Space + ドラッグ

Shift + 矢印キー

マウス操作

ホイール → 拡縮

Shift + ホイール → 回転

反転

H → 左右反転

Shift + H → 上下反転

※ドラッグ操作は「縦なら拡縮・横なら回転」とし、混在しない仕様にする。

Undo/Redo との整合性

描画オブジェクトの座標は常に worldContainer 内のローカル座標で記録

ビュー操作は worldContainer に対する transform のみで、描画履歴には影響しない

サンプルコード（簡易）
<script src="https://pixijs.download/release/pixi.min.js"></script>
<script>
const app = new PIXI.Application({ background: '#ffffff', resizeTo: window });
document.body.appendChild(app.view);

// ワールドビュー用コンテナ
const worldContainer = new PIXI.Container();
app.stage.addChild(worldContainer);

// 仮の描画オブジェクト（ここに線やレイヤーが入る）
const graphics = new PIXI.Graphics();
graphics.lineStyle(4, 0x800000);
graphics.moveTo(0, 0).lineTo(200, 100).lineTo(300, 50);
worldContainer.addChild(graphics);

// カメラ操作変数
let isDragging = false;
let lastPos = { x: 0, y: 0 };

app.view.addEventListener('mousedown', e => {
  if (e.shiftKey || e.code === 'Space') {
    isDragging = true;
    lastPos = { x: e.clientX, y: e.clientY };
  }
});

app.view.addEventListener('mousemove', e => {
  if (isDragging) {
    const dx = e.clientX - lastPos.x;
    const dy = e.clientY - lastPos.y;

    if (e.shiftKey) {
      // Shift + ドラッグ → 縦なら拡縮、横なら回転
      if (Math.abs(dy) > Math.abs(dx)) {
        worldContainer.scale.x += dy * 0.01;
        worldContainer.scale.y += dy * 0.01;
      } else {
        worldContainer.rotation += dx * 0.01;
      }
    } else {
      // Space + ドラッグ → 移動
      worldContainer.x += dx;
      worldContainer.y += dy;
    }

    lastPos = { x: e.clientX, y: e.clientY };
  }
});

app.view.addEventListener('mouseup', () => { isDragging = false; });
app.view.addEventListener('wheel', e => {
  if (e.shiftKey) {
    // Shift + ホイール → 回転
    worldContainer.rotation += e.deltaY * 0.001;
  } else {
    // ホイール → 拡縮
    const zoom = 1 - e.deltaY * 0.001;
    worldContainer.scale.x *= zoom;
    worldContainer.scale.y *= zoom;
  }
});

// H キー反転
window.addEventListener('keydown', e => {
  if (e.key === 'h') {
    if (e.shiftKey) {
      worldContainer.scale.y *= -1; // 上下反転
    } else {
      worldContainer.scale.x *= -1; // 左右反転
    }
  }
});
</script>

まとめ

これまでの「部分的な移動」ではなく、全描画オブジェクトを包む worldContainer を操作する方式に一本化する

これにより「カメラが動いたときと同じ挙動」を直感的に得られる

Undo/Redo への影響を避け、描画履歴は純粋に線データのみ保持する

👉 これを改修AIに渡せば、「ワールドビュー操作」という言葉で混乱せず進められると思います。
