📑 不具合報告書 & 解消方法
1. 発生状況

環境:

PixiJS v7

JavaScript（fetch API分割, node_modules直読み構成）

外部ライブラリ: @pixi/ui, @pixi/layers, @pixi/gif, GSAP, Lodash, Hammer.js

エラーメッセージ（Console 出力）:

pixi-extensions.js:338 ⚠️ Lodash 未検出 - 基本JavaScript使用
pixi-extensions.js:65 ⚠️ 警告: ["Lodash 未検出 - 基本JavaScript使用"]
pixi-extensions.js:690 ❌ lodash機能: 無効


他のライブラリ（ui, layers, gif, gsap, hammer 等）は正常検出。

Lodash のみ「未検出」と判定されている。

2. 原因分析

pixi-extensions.js 内の integrateUtilityLibraries() にて Lodash を次のように検出している
:

if (window._) {
    // Lodash 検出成功
} else {
    console.warn("⚠️ Lodash 未検出 - 基本JavaScript使用");
}


しかし、index.html 内に Lodash を読み込む <script> 記述が無い
。

結果として window._ が存在せず、「未検出」扱いになっている。

3. 解消方法
方法A（推奨: 現行構成のまま解消）

node_modules 直読み構成の場合
index.html に UMD 版 Lodash を追加する。

<!-- Pixi や他拡張の前に記述 -->
<script src="node_modules/lodash/lodash.min.js"></script>
<script src="pixi-extensions.js"></script>


→ Lodash 読み込み時に自動で window._ が定義され、PixiExtensionsManager が正常に検出可能となる。

方法B（将来 Vite/ESM 移行時）

ESM import 構成の場合
pixi-extensions.js 冒頭で Lodash を import し、手動で window._ に代入する。

import _ from "lodash";
window._ = _;  // PixiExtensionsManager が window._ を参照するため必要

方法C（軽量化を優先する場合）

Lodash の利用を最小限に絞るなら、cloneDeep 等の必要関数だけ個別 import する。

ただし pixi-extensions.js 側が window._ を前提にしているため、shim として必要関数を登録する必要あり。

import cloneDeep from "lodash/cloneDeep";
window._ = { cloneDeep };

4. 対応方針

現行フェーズ（PixiJS v7 + fetch API分割 + node_modules直読み）では 方法A を採用。

将来の Vite/ESM 移行時に 方法B に切り替えることで、軽量化やツリーシェイキングの恩恵を受ける。

✅ 結論:
Lodash は Vite/ESM 専用ライブラリではなく、UMD版を <script> で読み込めば現状環境でも問題なく利用可能。
window._ の存在が必須のため、index.html に追加することでエラーは解消する。

この形でまとめておけば、将来の v8化や Vite移行のロードマップにも流用できます。
