**現状分析**: main.htmlは約800行で、完全なTransform中心設計が完成済み  
**新戦略**: 既存のクラス構造を活かしつつ、機能的にJavaScript分割でスリム化

## 🎯 Phase1 分離対象（Sidebar-Tools.js）

### 分離するクラス・機能
```javascript
// main.html → Sidebar-Tools.js
- UnifiedUIController（UI管理統合版）
- DrawingTools（Transform対応版） 
- 関連するユーティリティ関数
- ツールボタンのイベントハンドリング
- ポップアップ管理
- スライダー制御
- リサイズ機能
```

### main.htmlに残すコア機能
```javascript
// main.htmlに残すコア部分
- APP_CONFIG（設定定数）
- TransformSystem（座標変換統一管理）
- TransformHistoryManager（軽量化された履歴管理）
- LayerManager（Transform対応版）
- HammerJSPositionManager（Transform統合版）
- DrawingEngine（Transform統合版）
- ShortcutManager（キーボード制御）
- SystemMonitor（パフォーマンス監視）
- AppController（メインアプリケーション制御）
- アプリケーション起動部分
```

## 🔧 分離手法：JavaScript外部化

### 1. 外部JSファイル読み込み方式
```html
<!-- main.html内 -->
<script src="Sidebar-Tools.js"></script>
<script>
    // 既存のコア処理
</script>
```

### 2. グローバル名前空間設計
```javascript
// Sidebar-Tools.js
window.FutabaSidebarTools = {
    UnifiedUIController: UnifiedUIController,
    DrawingTools: DrawingTools,
    initialize: function(appController) {
        // 初期化処理
    }
};
```

### 3. 相互通信API
```javascript
// main.html側でのSidebar-Tools.js利用
const tools = new window.FutabaSidebarTools.DrawingTools(
    this.engine, 
    this.layerManager, 
    this.transformSystem
);
const ui = new window.FutabaSidebarTools.UnifiedUIController(
    tools, 
    this.engine, 
    this.layerManager, 
    this.transformSystem
);
```

## 🚨 重要制約・原則

### ❌ やってはいけないこと
- **HTML構造の移動**: DOM要素・CSS定義はmain.htmlに残す
- **依存関係の破綻**: TransformSystemなどのコア機能への直接依存は禁止
- **二重実装**: 機能の重複実装は避ける
- **グローバル汚染**: 適切な名前空間を使用

### ⭕ 推奨する手法
- **インターフェース設計**: clear API boundaries
- **依存注入**: コンストラクタでの依存性注入
- **エラー伝播**: main.htmlのエラーハンドラーに統一
- **デバッグ情報**: console出力の統一

class AppController {
    constructor() {

        this.engine = new DrawingEngine(this.transformSystem);
        
        // Sidebar-Tools.jsから取得

            this.tools, this.engine, this.layerManager, this.transformSystem
