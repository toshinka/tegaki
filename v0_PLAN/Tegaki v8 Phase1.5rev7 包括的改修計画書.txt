# Tegaki v8 Phase1.5rev7 包括的改修計画書

## 📋 現状問題分析

### 🚨 致命的エラー
1. **ConfigManager.configure() メソッド未実装**
   - AppCore が無条件で `configManager.configure()` を呼び出し
   - ConfigManager クラスにメソッドが存在しない

2. **循環参照によるログ出力エラー**
   - `JSON.stringify` で PixiJS 内部オブジェクトを丸ごと出力
   - 結果として Manager 初期化が失敗

3. **Canvas 初期化の重複実行**
   - Canvas 作成処理が複数回実行される
   - 無駄な処理とログの肥大化

### 🐛 機能不具合
4. **座標オフセット問題**
   - x0y0 が x110y95 で表示される座標ずれ
   - CoordinateManager の変換処理に問題

5. **消しゴム機能の無効化**
   - EraserTool が正常に動作しない

6. **描画継続バグ**
   - ペンを離しても描き続ける、または描画が消える

### 🗂️ コード肥大化問題
7. **過度な冗長ログ**
   - Manager ごとに「開始」「method確認」「注入確認」「完了」の多段階ログ
   - デバッグ用途を超えた冗長性

8. **責任分離の破綻**
   - AppCore が初期化・検証・デバッグをすべて担当
   - Manager 間の依存関係が複雑化

## 🛠️ 架空メソッド撲滅案

### 存在しないメソッド一覧
- `ConfigManager.configure()` ❌
- `RecordManager.configure()` ❌
- サイドバー UI の export/import 不整合 ❌

### 対策
1. **存在チェック付き呼び出し**
   ```javascript
   if (typeof manager.configure === 'function') {
       await manager.configure(config);
   }
   ```

2. **必須ライフサイクルメソッド実装確認**
   - すべての Manager: configure/attach/init/isReady/dispose
   - すべての Tool: setManagersObject/onPointer*/forceEndDrawing/destroy/getState

## 📝 改修実施計画

### Step 1: 致命的エラー修正
**優先度: 🔥 最高**

#### 1.1 AppCore.js の修正
- `initializeOtherManagers()` に存在チェック追加
- `verifyManagerReady()` の JSON.stringify 削除
- Manager 情報出力を最小限に制限（className, version, isReady のみ）

#### 1.2 ConfigManager.js の修正
- 空の `configure()` メソッド追加
- または AppCore 側での条件分岐実装

#### 1.3 RecordManager.js の修正
- 空の `configure()` メソッド追加

### Step 2: ログ出力の簡素化
**優先度: 🔥 高**

#### 2.1 各 Manager のログ削減
- 「開始」「完了」の2段階のみ残す
- 中間プロセスログ削除
- PixiJS 内部構造出力禁止

#### 2.2 二重初期化防止
- AppCore に `isInitialized` フラグ追加
- 各 Manager に `_status` 内部状態管理追加

### Step 3: 座標系修正
**優先度: 🔥 高**

#### 3.1 CoordinateManager の座標変換修正
- オフセット計算の見直し
- DPR 重複適用防止
- Container 階層変形の正確な処理

#### 3.2 描画ツールの座標処理統一
- すべての座標変換を CoordinateManager 経由に統一
- 直接座標参照の禁止

### Step 4: UI 修正
**優先度: 🟡 中**

#### 4.1 icons.js の export/import 統一
- export 形式の確認と修正
- UI 側との参照整合性確保

#### 4.2 サイドバー表示修復
- EventBus 'app:ready' イベント待機
- アイコン描画処理の復旧

### Step 5: ツール機能修正
**優先度: 🟡 中**

#### 5.1 EraserTool の修復
- Manager 注入 API の修正
- v8 新 API への準拠

#### 5.2 PenTool の描画継続バグ修正
- forceEndDrawing() の確実な実行
- イベントハンドラの適切な終了処理

## 🚫 今後の肥大化防止ルール

### コーディング原則
1. **Manager は「開始」「完了」以外の冗長ログ禁止**
2. **PixiJS 内部構造をログに含めない**
3. **configure() 必須か否かを Manager 定義側に明記**
4. **AppCore は「初期化制御と依存解決」のみに責任を持つ**
5. **架空メソッド呼び出し前に @provides を必ず確認**

### 監査ルール
1. **依存関係の循環チェック**
2. **未実装メソッド呼び出しの検出**
3. **責任分離原則の遵守確認**
4. **DRY・SOLID 原則違反の検出**
