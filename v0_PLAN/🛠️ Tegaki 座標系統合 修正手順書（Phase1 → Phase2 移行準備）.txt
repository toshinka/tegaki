# 🛠️ Tegaki 座標系統合 修正手順書（Phase1 → Phase2 移行準備）

## 📌 問題分析

### 現在のコンソール警告
- `coordinates.js:13-311` - 削除対象ファイルからの警告
- `boundary-manager.js:131` - canvasElement未提供エラー
- 座標統合の不完全な移行

### 根本原因
1. **AppCore の初期化順序問題**: BoundaryManager に canvasElement が渡されていない
2. **CoordinateManager の統合未完了**: 旧 coordinates.js への依存が残存
3. **Manager間の依存関係の不整合**: 初期化時の引数不足

## 🔧 修正方針

### Phase 1: AppCore 初期化順序修正
- `initialize()` を `async initialize()` に変更
- Manager初期化の引数を統一
- canvasElement と CoordinateManager の適切な受け渡し

### Phase 2: BoundaryManager 引数修正
- `initialize(canvasElement, coordinateManager)` の引数を必須化
- AppCore → CanvasManager → BoundaryManager の順序で初期化
- 引数の妥当性確認を強化

### Phase 3: 座標系統合完了
- coordinates.js の使用箇所を CoordinateManager に置換
- 重複する座標処理の統一
- Phase2 移行準備の完了確認

## 🎯 修正対象ファイル

1. **app-core.js**: 初期化順序とManager引数修正
2. **boundary-manager.js**: 引数必須化と初期化処理修正  
3. **tool-manager.js**: CoordinateManager統合強化
4. **canvas-manager.js**: 座標統合処理確認
