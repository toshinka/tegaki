① Consoleエラー: tool-manager.js:11 Uncaught SyntaxError: Illegal return statement
原因

return 文が グローバルスコープ または class定義の外 に書かれている可能性が高いです。
JavaScriptでは 関数またはメソッド内以外で return は使えません。

例（エラーになる書き方）:

// tool-manager.js
return;   // ← これは関数外なのでSyntaxError

対策

return を削除するか、正しい関数スコープに移動してください。
もし「初期化が失敗したら処理を止めたい」意図なら、throw を使うのが正解です。

修正例:

// NG: グローバル return
return;

// OK: 初期化失敗なら例外
throw new Error("ToolManager initialization failed");

② 描画すると (0,0) から直線が伸びる
原因

ペンツールの mousedown / pointerdown の初期化処理で 前回座標を未設定のまま描画開始 しているため。
その結果、最初の点が (0,0) で初期化されてしまい、最初の線が「(0,0) → ペン位置」に伸びてしまいます。

対策

ペンを押した瞬間に 現在の座標を「開始点」として記録 し、描画処理はその後の mousemove / pointermove から開始するように修正します。

修正例（pen-tool.js 内）:

class PenTool {
    constructor(canvas) {
        this.canvas = canvas;
        this.isDrawing = false;
        this.lastX = null;
        this.lastY = null;
    }

    onPointerDown(event) {
        this.isDrawing = true;
        const pos = this.getPointerPosition(event);
        this.lastX = pos.x;
        this.lastY = pos.y;
    }

    onPointerMove(event) {
        if (!this.isDrawing) return;

        const pos = this.getPointerPosition(event);
        if (this.lastX !== null && this.lastY !== null) {
            this.drawLine(this.lastX, this.lastY, pos.x, pos.y);
        }
        this.lastX = pos.x;
        this.lastY = pos.y;
    }

    onPointerUp() {
        this.isDrawing = false;
        this.lastX = null;
        this.lastY = null;
    }
}


これにより、描画開始時の 「直線が(0,0)から出る」現象 がなくなります。
