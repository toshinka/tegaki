🎨 Tegaki Layer & Timeline 計画書
【目的】

@pixi/layers を導入してマルチレイヤー描画を可能にする

Adobe Fresco風のレイヤーパネルUI を DOM ベースで実装

カット（フレーム）管理 を導入し、将来的に GIF/動画タイムライン に拡張

機能実装の進行を Phaseごとに分割 し、段階的に完成度を高める

【全体構造】

PixiJS: 描画専用（PIXI.display.Layer によるレイヤー管理）

DOM: レイヤーパネル・サイドバー・タイムラインUIを担当

双方向同期：DOM操作 → Pixiに反映、Pixi状態 → DOM更新

【Phase分割】
Phase 1: 基本レイヤー管理

レイヤー構造

worldStage 内に PIXI.display.Layer を複数保持

activeLayer のみ描画対象にする

レイヤーパネルUI（DOM）

上下に伸びるリスト形式でレイヤーを表示

各項目に以下の操作を実装

👁 表示/非表示（チェックボックス → layer.visible）

📝 名前編集

🗑 削除

新規レイヤー追加（➕ ボタン）

Phase 2: レイヤー操作強化

操作追加

ドラッグ＆ドロップで順序入れ替え（Sortable.js を利用）

透明度スライダー → layer.alpha に反映

ブレンドモード選択（NORMAL, MULTIPLY, SCREEN 等）

フォルダ管理

レイヤーをフォルダ（グループ）に入れ子化可能

フォルダの展開/折り畳みUI

フォルダごと非表示切替

Phase 3: レイヤーパネルUI改善

表示演出

Adobe Fresco風に 中央からスライドイン（CSS transition）

レイヤー追加時にフェードインアニメーション

サイドバーアイコン

lucide-static のSVGをCDNで取得し、縦に配置

➕ 新規レイヤー

🗂 新規フォルダ

👁 表示切替

🗑 削除

ドラッグ操作感

ハンドルアイコン [≡] を掴んで並び替え可能

Phase 4: カット（フレーム）管理

カット構造

1カット = 1フレーム（cutContainer）

各カットは独自のレイヤーセットを保持

カット操作UI

「CUT1 ⇄ CUT2」切り替えボタン

カット追加/削除

カット複製

同期

現在のカット変更時に対応するレイヤーセットを読み込み

Phase 5: タイムライン拡張（GIF/動画向け）

UI

横軸 = レイヤー（トラック）

縦軸 = カット（フレーム）

各セル = そのカットにおけるレイヤー状態

機能

カットごとにレイヤーON/OFFを設定可能

再生/停止ボタンでプレビュー

GIF/MP4書き出し準備（将来拡張）

【実装ポイント】

DOMとPixiの同期

DOMの順序変更イベント → PixiのworldStage.children更新

DOMのチェックボックス → layer.visible に反映

DOMのスライダー → layer.alpha に反映

ドラッグ＆ドロップ

Sortable.js を利用してDOMリストを制御

並び順が変わったらPixiのzIndexを再設定

アニメーション

レイヤーパネル表示/非表示 → CSS transition

タイムラインスクロール → CSS overflow + スクロールバー

【検証項目】

[Phase1] 新規レイヤーを追加し描画できる

[Phase2] レイヤーをドラッグで順序変更できる

[Phase2] 透明度やブレンドモードが正しく反映される

[Phase3] レイヤーパネルが中央からアニメーションで出現

[Phase4] CUT1 ⇄ CUT2 の切替でレイヤーセットが入れ替わる

[Phase5] タイムラインで再生するとアニメのプレビューが動く
