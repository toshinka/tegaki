検査内容
1. 発生箇所

pixi-extensions.js の PixiExtensionsManager.attemptUILibraryDetection 内で失敗。

PIXI.ui (または PIXI.UI) が存在しないために例外を投げている。

2. 原因の可能性

index.html で pixi-ui.js を読み込んでいない

現状、pixi-extensions.js の初期化時点で PIXI.ui が未定義。

node_modules 配下を直接参照していないので検出できない。

読み込み順序の問題

pixi-extensions.js が pixi-ui.js より先に読み込まれている場合、統合処理で PIXI.ui が未定義になる。

名前空間の揺れ

pixi-ui.js は PIXI.ui ではなく PIXI.UI にエクスポートされるケースがある。

どちらにも対応できていないと検出エラーが出る。

確認結果

ライブラリ自体 (pixi-ui.js) はプロジェクト内に存在。

しかし index.html に読み込み指定が無いため pixi-extensions.js からは見えない。

CDN不足ではなく「読み込み漏れ or 順序違い」が原因。

修正案
1. index.html にローカル参照を追加
<script src="node_modules/@pixi/ui/dist/pixi-ui.js"></script>
<script src="libs/pixi-extensions.js"></script>


⚠️ 順序は必ず pixi-ui.js → pixi-extensions.js の順。

2. pixi-extensions.js 側の存在チェック改善
attemptUILibraryDetection() {
    if (typeof PIXI === "undefined") {
        throw new Error("PIXI が読み込まれていません");
    }
    if (!PIXI.ui && !PIXI.UI) {
        throw new Error("@pixi/ui が見つかりません (PIXI.ui / PIXI.UI 未定義)");
    }
    return PIXI.ui || PIXI.UI;
}


これで pixi-ui.js が PIXI.UI を使う場合でも対応可能。

報告まとめ（ローカルファイル前提版）

直接原因: index.html で node_modules/@pixi/ui/dist/pixi-ui.js を読み込んでいない、または読み込み順序が逆。

副次要因: pixi-extensions.js が PIXI.ui のみに依存しており、PIXI.UI などの名前空間違いに未対応。

推奨修正:

index.html にローカル参照の <script> を追加（pixi-ui.js を先に読み込む）。

pixi-extensions.js 側で PIXI.ui / PIXI.UI 両対応にする。
