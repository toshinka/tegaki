🔍 検証結果と原因
1. coordinates.js 関連

ログに大量の「削除予定」「使用禁止」「移行済み」などが出ているのは 意図的に仕込まれた削除カウントダウン警告。

実際に coordinates.js 内で console.warn や setTimeout によるアラートを仕込んであるため、残っている限り出続ける。

原因は「まだ一部のモジュールが window.CoordinateUtils 等を参照している」ため、完全削除できず警告を出している状態。

→ Phase2 で消して良いが、現時点では参照箇所が残っているため「削除するとクラッシュ」します。

2. BoundaryManager の警告
boundary-manager.js:131 ⚠️ canvasElement未提供 - 基本機能のみ初期化
boundary-manager.js:149 ⚠️ canvasElement未設定のため拡張ヒットエリア作成をスキップ


これは BoundaryManager.initialize() に canvasElement が渡されていないため。

初期化自体はできるが、ヒットエリア拡張など「座標系依存の機能」がスキップされている。

原因は AppCore → CanvasManager → BoundaryManager の初期化順序/引数渡し不備。

→ main.js の initializeManagersWithCoordinateIntegration() 内で BoundaryManager.initialize(canvasElement, coordinateManager) を正しく呼ぶ必要があります。

✅ 改修作業書（修正版）
Step 1. coordinates.js の削除準備

全コードで window.CoordinateUtils, window.ColorUtils, window.MathUtils を検索

以下に置換する

CoordinateUtils.distance() → CoordinateManager.calculateDistance()

CoordinateUtils.calculateAngle() → CoordinateManager.calculateAngle()

ColorUtils.getFutabaColor() → ConfigManager.getColors().futabaMaroon

MathUtils.xxx() → window._（lodash）または標準 Math.xxx()

置換が完了したら coordinates.js を 読み込みから外す

Step 2. BoundaryManager の初期化修正

main.js の初期化シーケンスを確認

今は BoundaryManager.initialize() が 引数無しで呼ばれている

修正案：

// main.js 内 initializeManagersWithCoordinateIntegration
const canvasElement = document.getElementById("drawingCanvas");
const coordinateManager = new CoordinateManager();

window.boundaryManager.initialize(canvasElement, coordinateManager);


BoundaryManager.initialize() のシグネチャを統一：

initialize(canvasElement, coordinateManager) {
    if (!canvasElement) {
        console.warn("⚠️ canvasElement未提供 - 基本機能のみ初期化");
        return;
    }
    this.canvasElement = canvasElement;
    this.coordinateManager = coordinateManager;
    // …座標系依存の初期化処理
}

coordinates.js 警告が 完全に出なくなることを確認

BoundaryManager の警告 (canvasElement未提供) が出なくなることを確認

ペンツールで 描画が正常に行え、座標・筆圧が反映されることを確認

📌 結論

今出ている coordinates.js のログは「削除直前の警告」であり、参照残りの証拠。

本質的なエラーは BoundaryManager に canvasElement が渡っていないこと。

解決には「CoordinateManager完全統合」と「BoundaryManager初期化の引数統一」が必要です。


🔍 問題の確認

initializeManagersWithCoordinateIntegration() では、BoundaryManagerの初期化がこうなっています：

if (this.canvasManager?.coordinateManager) {
    this.boundaryManager = new window.BoundaryManager(this.canvasManager.coordinateManager, boundaryOptions);
} else {
    this.boundaryManager = new window.BoundaryManager(boundaryOptions);
}


→ canvasElement を渡していないため、BoundaryManager内部で ⚠️ canvasElement未提供 が出ているのです。
