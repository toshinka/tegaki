## 📋 問題分析レポート

### 🚨 Critical Issues (重要な問題)

#### 1. StateManager API不整合
```javascript
// 問題: tool-manager.js:65行目
this.stateManager.updateSystemState('toolManager', 'initialized', {...});
```
**原因**: StateManagerに`updateSystemState`メソッドが存在しない  
**影響**: ToolManager初期化失敗 → ペン描画不可  
**対象ファイル**: `js/utils/state-manager.js`, `managers/tool-manager.js`

#### 2. ToolManager初期化順序問題
```javascript
// 問題箇所: tool-manager.js validateAndSetupUnifiedSystems()
this.stateManager.updateSystemState // ← 存在しないメソッド呼び出し
```
**原因**: StateManagerのAPI設計と実装の不一致  
**影響**: ツール管理システム全体の初期化失敗

#### 3. Boundary Manager連携問題
```javascript
// 境界管理と座標管理の分離に伴う参照エラーの可能性
```

## 📝 改修実装計画

### 1. StateManager API拡張

#### 追加メソッド仕様
```javascript
/**
 * システム状態更新（ToolManager向け）
 * @param {string} system - システム名
 * @param {string} state - 状態名
 * @param {Object} data - 状態データ
 */
updateSystemState(system, state, data = {})

/**
 * システム状態取得
 * @param {string} system - システム名
 * @returns {Object} システム状態
 */
getSystemState(system)

/**
 * 全システム状態取得
 * @returns {Object} 全システム状態
 */
getAllSystemStates()
```

#### 実装方針
- 既存`componentStates`と類似の`systemStates`Map追加
- EventBus連携維持
- エラーハンドリング統一

### 2. ToolManager初期化修正

#### 修正箇所
```javascript
// Before (エラー)
this.stateManager.updateSystemState('toolManager', 'initialized', {...});

// After (修正)
if (this.stateManager && this.stateManager.updateSystemState) {
    this.stateManager.updateSystemState('toolManager', 'initialized', {...});
}
```

#### 追加改善
- **初期化順序の最適化**
- **依存関係チェック強化**
- **フォールバック機能拡充**

### 3. エラーハンドリング改善

#### 統一エラー処理
```javascript
// 統一システム依存エラー
if (!this.configManager) {
    throw new UnifiedSystemError('ConfigManager required');
}

// API互換性エラー
if (!this.stateManager.updateSystemState) {
    console.warn('StateManager API version mismatch - using fallback');
    this.useStateManagerFallback();
}
```
