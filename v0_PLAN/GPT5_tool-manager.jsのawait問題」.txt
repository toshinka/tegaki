🛑 現状のエラー
tool-manager.js:132 Uncaught SyntaxError: await is only valid in async functions and the top level bodies of modules

原因

await が 普通の function の中に書かれている。

Pure JS 環境なので「トップレベル await」は使えない。

つまり async 付きの関数でしか await が許されない。

✅ 修正方針

ToolManager の initialize() を async function に変更する。
呼び出し側（app-core.js → main.js）も await できるようにする。

修正例
tool-manager.js
// 修正前（エラーになる）
ToolManager.prototype.initialize = function(coordinateManager) {
  this.coordinateManager = coordinateManager;
  await this.loadDefaultTools();  // ← asyncじゃない関数内でawaitしている
};

// 修正後
ToolManager.prototype.initialize = async function(coordinateManager) {
  this.coordinateManager = coordinateManager;
  await this.loadDefaultTools();  // OK: async関数内なのでawait可能
};

app-core.js

initializeManagers() 側も await に対応する。

// 修正前
function initializeManagers() {
  window.toolManager = new ToolManager();
  toolManager.initialize(window.coordinateManager); // ← async対応できてない
}

// 修正後
async function initializeManagers() {
  window.toolManager = new ToolManager();
  await toolManager.initialize(window.coordinateManager); // async対応
}

🚨 注意点

initializeManagers() が async になると、呼び出し元（main.js）も await initializeManagers() にする必要があります。

Pure JS（グローバル）なので「asyncの伝播」を途中で止めるとまた同じエラーが出ます。
