1. 座標系とキャンバス移動の現状問題
現象

coordinates-changed イベントが大量発生 → console 警告

座標取得は常に 0.0

描画キャンバスは Camera は動かないが、レイヤーが動いている状態

描画位置が左に 150px、上に 50px ずれている

原因推定

座標系の二重管理

現状は「World 座標を直接レイヤーに適用」している可能性。

Camera 座標（PositionManager）を使ってスクリーン変換をしていない → World 座標とスクリーン座標が同期していない。

描画フローが world → layer 直置きになり、Camera 移動と独立してしまっている。

coordinates-changed が大量発生

PositionManager で座標変更時に dispatchEvent({ type: "coordinates-changed", payload: world }) を毎フレーム送っている可能性。

payload がスクリーン変換されておらず、UI が参照できないので警告が出ている。

ズレ量が固定値

左 150px / 上 50px のズレは、キャンバスオフセット・Container の初期 transform の未考慮や canvas.offsetLeft/Top の未反映が原因。

2. レイヤー挙動の問題
現象

レイヤー+を押すと 2枚同時作成

レイヤーIDに 1757122410106 のような変な値が混入

原因推定

createLayer 呼び出しが重複

UI ボタンの click イベントで LayerManager.createLayer() が 複数回登録されている。

以前の registerLayerButton が二重に呼ばれている可能性。

ID 生成ロジック

現状は Date.now() などで ID を生成している場合、短時間で複数作成されると想定外の大きな数字が混ざる。

これにより UI 側に不自然なレイヤーIDが出る。

3. 改修案（優先度順）
A. 座標系とキャンバス移動フロー修正

World → Camera → Screen の順に座標変換

// PositionManager
getScreenCoordinates(worldX, worldY) {
  const cam = this.camera;
  return {
    x: (worldX - cam.x) * cam.scale,
    y: (worldY - cam.y) * cam.scale
  };
}

// 描画フロー
const world = { x: strokeX, y: strokeY };
const screen = positionManager.getScreenCoordinates(world.x, world.y);
drawingEngine.drawAt(screen.x, screen.y);


Camera 移動を反映

レイヤー Container は動かさず、Camera コンテナを移動

世界座標は固定 → 描画判定も World 座標で行う

これによりズレと 0.0 座標が解消される

coordinates-changed イベント

スクリーン座標を添えて dispatch

連打防止のため、座標が変わった時のみ送信

B. レイヤー作成修正

重複登録の排除

// UIManager
if (!this.layerButtonRegistered) {
  button.addEventListener("click", () => layerManager.createLayer());
  this.layerButtonRegistered = true;
}


ID 生成の安定化

// LayerManager
createLayer(name = "Layer") {
  const id = `layer-${this.layers.length + 1}`; // シンプル連番
  const layer = new PIXI.Container();
  layer.id = id;
  layer.name = name;
  this.layers.push(layer);
  this.activeLayer = layer;
  this.dispatchEvent({ type: "layer-created", payload: { id, name } });
}


これにより奇妙な ID の混入や二枚作成は防げます

C. 次の優先改修

レイヤー入れ替えドラッグは次のステップ

ペン描画はすでに動作しているので、World→Screen 座標変換を統合すればズレは解消

Camera 移動と描画位置の同期が取れれば、全体の座標系が安定

まとめ（優先度）

座標系修正

World→Camera→Screen を統一

Camera コンテナを移動、レイヤーは固定

coordinates-changed イベントの最適化

変化時のみ dispatch

payload は軽量データのみ

レイヤー作成挙動修正

二重登録防止

ID を連番など安定した方法に変更

次フェーズ

レイヤードラッグ入れ替え

UI 側座標判定

💡 この改修を行えば：

ペン描画の位置ズレ解消

座標取得が正しく 0.0 にならない

Camera 移動が正しく反映

レイヤー作成の重複・ID異常が解消

1. PositionManager 修正版

// position-manager.js
export class PositionManager extends EventTarget {
  constructor() {
    super();
    this.camera = { x: 0, y: 0, scale: 1 };
  }

  // カメラ移動
  moveCamera(dx, dy) {
    this.camera.x += dx;
    this.camera.y += dy;
    this.dispatchCoordinatesChanged();
  }

  // World座標 → Screen座標
  worldToScreen(worldX, worldY) {
    const cam = this.camera;
    return {
      x: (worldX - cam.x) * cam.scale,
      y: (worldY - cam.y) * cam.scale
    };
  }

  // Screen座標 → World座標
  screenToWorld(screenX, screenY) {
    const cam = this.camera;
    return {
      x: screenX / cam.scale + cam.x,
      y: screenY / cam.scale + cam.y
    };
  }

  // 座標変更イベント（変化時のみ送信）
  dispatchCoordinatesChanged() {
    const payload = {
      camera: { ...this.camera }
    };
    this.dispatchEvent({ type: "coordinates-changed", payload });
  }
}

2. LayerManager 修正版

// layer-manager.js
export class LayerManager extends EventTarget {
  constructor() {
    super();
    this.layers = [];
    this.activeLayer = null;
    this.nextLayerId = 1; // 安定した連番
  }

  init() {
    this.createBackgroundLayer();
  }

  createBackgroundLayer() {
    const bgLayer = this.createLayer("Background");
    this.activeLayer = bgLayer;
  }

  createLayer(name = "Layer") {
    const layer = new PIXI.Container();
    layer.name = name;
    layer.id = `layer-${this.nextLayerId++}`; // 安定ID

    this.layers.push(layer);
    this.activeLayer = layer;

    // イベントは軽量化したデータのみ
    this.dispatchEvent({
      type: "layer-created",
      payload: { id: layer.id, name: layer.name, index: this.layers.length - 1 }
    });

    return layer;
  }

  setActiveLayer(layerId) {
    const layer = this.layers.find(l => l.id === layerId);
    if (layer) this.activeLayer = layer;
  }
}

3. UIManager 修正版（レイヤー+ ボタン）

// ui-manager.js
export class UIManager extends EventTarget {
  constructor(layerManager) {
    super();
    this.layerManager = layerManager;
    this.layerButtonRegistered = false;
  }

  registerLayerButton(button) {
    if (this.layerButtonRegistered) return;
    this.layerButtonRegistered = true;

    button.addEventListener("click", () => {
      const layer = this.layerManager.createLayer("UI Layer");
      this.dispatchEvent({
        type: "ui-layer-button-clicked",
        payload: { id: layer.id, name: layer.name }
      });
    });
  }
}


4. DrawingEngine 修正版（World→Screen 座標使用）

// drawing-engine.js
export class DrawingEngine extends EventTarget {
  constructor(positionManager) {
    super();
    this.positionManager = positionManager;
    this.currentStroke = null;
  }

  startStroke(worldX, worldY, color, size) {
    const screen = this.positionManager.worldToScreen(worldX, worldY);

    this.currentStroke = {
      points: [{ x: screen.x, y: screen.y }],
      color,
      size
    };

    this.dispatchEvent({
      type: "stroke-started",
      payload: {
        color,
        size,
        start: { x: worldX, y: worldY }
      }
    });
  }

  addPoint(worldX, worldY) {
    if (!this.currentStroke) return;
    const screen = this.positionManager.worldToScreen(worldX, worldY);
    this.currentStroke.points.push({ x: screen.x, y: screen.y });
  }
}

レイヤー作成の安定化

createLayer は 連番ID

UI ボタンの click は二重登録を防止

イベント payload の軽量化

Pixi オブジェクトを含まない

JSON.stringify 可能なデータのみ送信
