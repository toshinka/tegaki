# Tegaki StarDivide 辞典（理想形）

## MainController (主星)
- registerSatellite(id, instance)
- notify(event: { type, payload })
- getState() : 現在の状態を返す（座標・レイヤーなど）
- dispatch(action) : Actionを各Managerに配信

### 通信フロー
- Main ⇄ LayerManager
- Main ⇄ DrawingEngine
- Main ⇄ PositionManager
- Main ⇄ UIManager
- Main ⇄ HistoryService

---

## DrawingEngine
- startStroke(point, toolId)
- continueStroke(point)
- endStroke(point)
- clearLayer(layerId)

### 通信
- 受信: toolId は必ず ToolManager 経由で Main から受け取る
- 送信: notify("drawing.strokeStart"/"drawing.strokeEnd")

---

## LayerManager
- createLayer(name)
- deleteLayer(id)
- selectLayer(id)
- reorderLayer(id, position)
- getActiveLayer()

### 通信
- notify("layers.updated")

---

## PositionManager
- toWorld(point) : 画面座標 → ワールド座標
- toScreen(point) : ワールド座標 → 画面座標
- pan(dx, dy)
- zoom(scale)

### 通信
- notify("position.changed")

---

## UIManager
- showPopup(message)
- updateToolbar(state)
- bindShortcuts()

### 通信
- listen("history.changed")
- listen("layers.updated")

---

## HistoryService
- record(action)
- undo()
- redo()

### 通信
- notify("history.changed")

---

## ToolManager（仮想）
- selectTool(toolId)
- getCurrentTool()
- getToolConfig(toolId)

---

# 修正計画（rev6 → 理想形）

## 1. 架空メソッドの削除
- mainAPI.getSatellite('drawingEngine') → 廃止
- manager.request(...) 呼び出し → 廃止
- PositionManager は toWorld(), toScreen() に統一

## 2. DrawingEngine の責務修正
- 現状: currentTool を保持して判定している
- 修正: toolId は必ず ToolManager → Main → DrawingEngine のフローで渡す

## 3. Notification の統一
- 現状: notify("drawing.ready")
- 修正: notify({ type: "drawing.ready", payload: {} })
- 全モジュールで domain.action 形式を強制

## 4. UIManager の不要メソッド整理
- updateHistoryState() → 使用箇所なし、削除または統合

## 5. フローフィックス
- 座標変換: DrawingEngine 内で request() を直叩きしている
    → 修正: MainController 経由で PositionManager に依頼
- 記録フロー: Stroke 終了時に HistoryService.record(action) を呼ぶよう統一

## 6. ヘッダールール追記
- 各ファイル先頭に @notes:
  - 責務の範囲
  - 通信対象
  - 提供メソッド（@provides）
