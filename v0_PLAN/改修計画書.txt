改修計画書（テキスト）
【目的】
- カメラ窓をリサイズ可能にする
- ワールドは無限に描けるようにする
- マスクは「表示枠」だけに適用、描画レイヤー自体は自由に描画可能とする

【改修手順】
1. CameraContainer構造を見直す
   - app.stage
     ├─ cameraContainer (マスク有り、サイズ変更対応)
     │   └─ worldContainer (実際のキャンバス・描画レイヤー)
     └─ uiContainer (UI系、マスク無し)

2. マスク更新処理
   - カメラサイズ変更イベントを作り、mask.drawRect(0,0,newWidth,newHeight) を再生成
   - worldContainerはtransformを維持したまま

3. 描画入力処理
   - ペンストロークは必ず worldContainer に記録
   - pointermove時はカメラ枠を超えても座標変換して worldContainer に書き込む
   - → マスクが「描画制限」ではなく「表示窓」になる

4. 座標系
   - screenToWorld / worldToScreen を導入して一元管理
   - ペン入力は screenToWorld でワールド座標化してから描画

5. 背景（グリッドや無地バックビュー）
   - worldContainer内に配置
   - カメラサイズ変更時は再描画 or スケール調整
   - 背景はカメラ窓に追従

【検証項目】
- カメラを200x200 → 600x600に変えてもワールド全体はそのまま、窓だけ拡大縮小される
- 画面外から線を引いても worldContainer に描かれる
- UI（ボタンやステータス表示）はマスクされない

参考パッチ案
// カメラサイズ更新
function resizeCamera(newWidth, newHeight) {
  // app.view自体もリサイズ
  app.renderer.resize(newWidth, newHeight);

  // マスク更新
  mask.clear();
  mask.beginFill(0x000000);
  mask.drawRect(0, 0, newWidth, newHeight);
  mask.endFill();

  // カメラ窓の枠線デバッグ
  if (!cameraBorder) {
    cameraBorder = new PIXI.Graphics();
    cameraContainer.addChild(cameraBorder);
  }
  cameraBorder.clear();
  cameraBorder.lineStyle(2, 0xff0000);
  cameraBorder.drawRect(0, 0, newWidth, newHeight);
}


🔑 ポイントは 「マスクは表示窓用」、「描画レイヤーは常に worldContainer に記録」 の2つ。
