ファイルを詳細に分析した結果、キャンバスに線が描画されない問題の根本原因を特定しました。

問題の根本原因

問題の直接的な原因は、PenTool（ペン描画の実処理を行うクラス）のインスタンスがアプリケーションのどこからも生成・初期化されていないことです。

その結果、描画イベントが発生しても、実際の描画処理を持つ PenTool のメソッドが呼び出されず、中身のない「フォールバック処理」が実行されるだけで描画が行われない状態になっています。

詳細な分析

提供いただいたコンソールログとソースコードから、以下の流れで問題が発生していることが判明しました。

    ツールの仮登録: tool-manager.js は初期化の際に registerBasicTools() メソッドを呼び出します。ここでは、pen という名前で、実際の描画機能を持たない「プレースホルダー（仮の器）」のオブジェクトをツールとして登録しています。
    JavaScript

    // tool-manager.js
    this.registeredTools.set(toolName, {
        name: toolName,
        type: 'basic',
        settings: this.globalSettings[toolName] || {}
    });

    PenTool が登録されない: 本来であれば、この後 pen-tool.js に定義されている PenTool のインスタンスが生成され、toolManager.registerTool('pen', penToolInstance) のように呼び出されることで、上記のプレースホルダーが実際の機能を持つ PenTool インスタンスに置き換えられるべきです。しかし、main.js や app-core.js を含め、提供されたどのファイルにも new PenTool() を実行して初期化するコードが存在しませんでした。

    描画処理の空振り:

        ユーザーがキャンバスをクリックすると、app-core.js がイベントを検知し、toolManager.startDrawing() を呼び出します。

        tool-manager.js は、現在選択中のツール（pen）に対応するオブジェクトを registeredTools の中から探します。しかし、見つかるのは描画メソッド（startDrawingなど）を持たないプレースホルダーです。

        そのため、if (currentToolInstance && typeof currentToolInstance.startDrawing === 'function') という条件が false になり、実際の描画処理は呼び出されません。

        代わりに、handleDrawingFallback() という何もしない代替関数が呼び出されます。

    ログとの一致: ユーザーがマウスボタンを離すと、toolManager.stopDrawing() が呼ばれます。この時も同様に PenTool の stopDrawing メソッドが見つからず、フォールバック処理 handleDrawingFallback('stop') が実行されます。これにより、コンソールに 🔧 描画フォールバック: stop - AppCore経由 という、ご提示いただいたログが出力されます。

結論として、描画イベントの連鎖は正しく動いていますが、最終的に描画を担当するべき PenTool が不在のため、線が描画されずに処理が終了してしまっています。

解決策

この問題を解決するには、アプリケーションの初期化プロセスに、PenTool および EraserTool をインスタンス化し、ToolManager に登録する処理を追加する必要があります。

以下に具体的な修正コードを提案します。

1. main.js にツールの初期化処理を追加

main.js ファイルの FutabaDrawingTool クラス内に、以下のメソッドを新たに追加してください。
JavaScript

// main.js の FutabaDrawingTool クラス内に追加

    /**
     * ツール初期化
     */
    async initializeTools() {
        console.log('🛠️ ツール初期化...');

        if (!this.appCore?.toolManager) {
            throw new Error('ToolManagerが初期化されていません。');
        }
        
        try {
            // PenToolのインスタンス化と初期化
            if (window.PenTool) {
                const penTool = new window.PenTool(this.appCore.toolManager);
                await penTool.initialize(); // この中でToolManagerに自身を登録します
                console.log('✅ PenTool 初期化・登録完了');
            } else {
                console.warn('⚠️ PenToolクラスが見つかりません。');
            }

            // EraserToolも同様に初期化
            // EraserToolクラスがグローバルに存在する場合
            if (window.EraserTool) {
                // EraserToolもPenToolと同様のコンストラクタとinitializeメソッドを持つと仮定
                const eraserTool = new window.EraserTool(this.appCore.toolManager);
                await eraserTool.initialize();
                console.log('✅ EraserTool 初期化・登録完了');
            } else {
                console.warn('⚠️ EraserToolクラスが見つかりません。');
            }
            
            this.initializationSteps.push('tools');

        } catch (error) {
            console.error('❌ ツール初期化失敗:', error);
            throw error;
        }
    }

2. main.js の初期化シーケンスに上記処理を組み込む

main.js の initialize() メソッド内、initializeAppCore() の呼び出し直後に、先ほど追加した initializeTools() を呼び出すコードを挿入します。
JavaScript

// main.js の initialize() メソッド内を修正

            // Phase 4: AppCore初期化
            await this.initializeAppCore();
            
            // ===================================
            // ▼▼▼ ここに追記 ▼▼▼
            // Phase 4.5: ツール初期化
            await this.initializeTools();
            // ▲▲▲ ここに追記 ▲▲▲
            // ===================================

            // Phase 5: システム統合
            this.setupSystemIntegration();

修正による効果

上記修正を行うことで、アプリケーション起動時に PenTool と EraserTool が適切にインスタンス化・初期化され、ToolManager に実体として登録されます。
これにより、ToolManager は描画イベントを受け取った際に、フォールバック処理ではなく PenTool に実装されている実際の描画メソッドを呼び出すようになり、キャンバスに線が正しく描画されるようになります。
