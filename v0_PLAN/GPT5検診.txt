🛠 現象と直接原因
main.js:71 ❌ TegakiApplication 初期化エラー: Error: AppCore class not available


main.js の TegakiApplication.initializeAppCore 内で
AppCore クラスを参照しているが、ロードされていない or 定義が存在しない。

app-core.js を見たところ クラス宣言の不足や export/import の齟齬 が疑われる。

また bootstrap.js → main.js → app-core.js の 読み込み順序 が崩れている可能性もある。

👉 このレイヤーの改修で済むなら app-core.js 内で class AppCore { ... } を明示し、グローバル公開 (window.AppCore = AppCore) をする必要あり。


📚 ルールブック & シンボル辞典 点検観点

依頼の趣旨は 実装と設計文書の乖離 を検出すること。以下の観点で精査します：

1.存在チェック

ルールブックに書かれたクラス / メソッド / プロパティが実装に存在するか。

実装にあるがルールブックに無い「野良実装」がないか。

2.重複・誤用

同一メソッド名が異なる責務で使われていないか。

シンボル（イベント名・エラーコードなど）が辞典と一致しているか。

3.架空参照

ルールブックや辞典にしか存在しないメソッドを実装が呼び出していないか。

今回の AppCore class not available がまさに「ルールにはあるが実装が未整備」ケース。

4.進化計画 (phase evolution plan)

現行フェーズ (phase1rev6) と計画書の段階が一致しているか。

未着手のフェーズ要素を呼んでいないか。


🔍 乖離の例（調査サマリ）
一致しない点

AppCore → ルールブックには「初期化中枢クラス」として記載あり。
しかし実装では定義されていないか、window.AppCore に公開されていない。

EventBus / ErrorManager / ConfigManager は utils に存在し、ルールにも記載あり → 一致。

ToolManager / CanvasManager もルールと一致。

PenTool / EraserTool → シンボル辞典で定義済み、実装もある → 一致。

危うい点

ルールブック側の「Application → Core → Managers → Tools」階層の説明にある
LifecycleManager が現状実装に見当たらない。

シンボル辞典内の一部イベントキー（例：app:ready, canvas:updated）が
実装の event-bus.js 内に未定義。

👉 つまり「ルール > 実装」のズレが発生している。

🩹 改修案（2レイヤー）
1. 実装修正パッチ
app-core.js にクラス追加

// js/app-core.js
(function(window) {
  'use strict';

  class AppCore {
    constructor(config) {
      this.config = config || {};
      this.managers = {};
    }

    initialize() {
      // 各種マネージャの初期化
      this.managers.error = new window.ErrorManager();
      this.managers.config = new window.ConfigManager(this.config);
      this.managers.events = new window.EventBus();
      this.managers.canvas = new window.CanvasManager();
      this.managers.tool = new window.ToolManager();
      return this;
    }
  }

  // グローバル公開
  window.AppCore = AppCore;
})(window);

これで main.js から new AppCore() が利用可能になる。


2. ルールブック / 辞典修正パッチ
ルールブック (1_tegaki_rulebook_v1.1.txt)

AppCore クラスを「必須基盤クラス」として明記。

未実装の LifecycleManager は Phase2 移行に移動。

修正例：
[Core Layer]
- AppCore : アプリ中枢。ErrorManager, ConfigManager, EventBus, CanvasManager, ToolManager を統括。
- LifecycleManager : フェーズ2以降に導入予定。Phase1 では未使用。


シンボル辞典 (2_tegaki_symbol_dictionary_v1.1.txt)
app:ready / canvas:updated を Phase2 シンボルに移動。

Phase1 の必須イベントシンボルのみ残す（例：app:init, tool:select, canvas:draw）。

修正例：

[Phase1 イベント]
- app:init
- tool:select
- canvas:draw
- error:occurred

[Phase2 予定イベント]
- app:ready
- canvas:updated

✅ 提案の流れ

実装修正（AppCore クラスを定義し、window に公開）

ルールブック修正（LifecycleManager をPhase2に移す / AppCore を明記）

辞典修正（Phase未着手のシンボルを整理）

進化計画との同期（phase evolution plan で「現在はPhase1rev6」の注記を追加）

GitHub
