# 改修計画 - v8_phase1rev7 初期化エラー対応

## 現況
- Windows 環境で WebGPU `powerPreference` が無視される警告が出ているが致命的エラーではない。
- v8 Tool 初期化時に `coordinate-manager` と `record-manager` が未登録エラーを起こしている。
- v7/v8 両対応コードを残すと「Manager群の二重管理」になり見通しが悪化し、AI コーディング時に誤認識が発生する危険性がある。

## 問題点
1. **AppCore.initializeV8Managers** 内で `CoordinateManager` と `RecordManager` を `this.managers` に登録していない。
2. `tool-manager.js` が要求する Manager 名と、`app-core.js` 側のキー名が一致していない可能性。
3. v7/v8 互換コードが混在しているため、修正箇所が曖昧になりやすい。

## 提案
- `app-core.js` の `initializeV8Managers` に以下を追加する：
  ```js
  import CoordinateManager from "./utils/coordinate-manager.js";
  import RecordManager from "./utils/record-manager.js";

  this.managers.set("coordinate", new CoordinateManager(this));
  this.managers.set("record", new RecordManager(this));
すでに登録済みの tool, canvas, shortcut, config などと同様の形式で統一。

tool-manager.js 側の requires 配列に定義されている Manager 名と必ず一致させること。

v7 互換レイヤーは廃止する。

実装計画
app-core.js を修正し、coordinate-manager と record-manager を登録する。

tool-manager.js の requires リストとキー名を照合・修正。

デバッグログを残し、Manager 群がすべて正常に登録されているか確認。

powerPreference 警告は既知の仕様につき放置。ただし、ログ抑制が必要なら console.warn ラップで対応。

v7 互換コードを段階的に削除し、「v7/v8 両対応で二重管理を行うとコードの見通しが悪化する」 旨を app-core.js および tool-manager.js のファイルヘッダーに明記する。

ヘッダー記述指示
各ファイルのヘッダーに以下を必ず記載する：

⚠️ 注意: v7/v8 両対応による Manager 群の二重管理は禁止。
本ファイルは v8 専用の初期化ロジックを採用している。
