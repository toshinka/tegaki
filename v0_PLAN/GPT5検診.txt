🚨 エラーの要点

PixiJS warning

The powerPreference option is currently ignored when calling requestAdapter() on Windows.


→ これはPixiJS側の仕様で無視してよい（性能選択はWindows環境では反映されない）。

EraserTool 警告

⚠️ RecordManagerInstance not available - v8記録機能が制限されます


→ EraserTool が RecordManager に依存しているが、初期化の順序または Manager 登録キーが不一致。
原因候補:

record-manager.js が先に初期化されていない

window.Tegaki.RecordManagerInstance の登録がない

EraserTool 内での呼び出し名が v7 残骸（RecordManagerInstance 直参照）になっている

ToolManager / PenTool 初期化エラー

💀 v8 Tool初期化エラー: Manager 'event' が見つかりません。利用可能: canvas, coordinate, record, config, error, eventbus, shortcut, navigation, tool


→ abstract-tool.js の getManager("event") 呼び出しが存在するが、実際の登録は eventbus 名義。
不整合:

旧称: event

現行: eventbus

これにより ToolManager が全体的に初期化失敗。

🔎 原因まとめ

Manager 名称の不一致: event → eventbus に変わっているが、ツール側が古い呼び名を維持。

RecordManager 初期化順序・参照不整合: EraserTool が RecordManagerInstance を直接要求しているが、Manager 経由に統一されていない。

二重管理のリスク: v7 と v8 の呼び出しキーが混在（RecordManager の参照が直参照 vs Manager経由）。

🛠️ 改修案
1. Manager呼び出し名統一

abstract-tool.js の getManager("event") を → getManager("eventbus") に修正。

PenTool, EraserTool など Manager 参照箇所を全て確認。

// 修正前
this.eventManager = this.getManager("event");

// 修正後
this.eventManager = this.getManager("eventbus");

2. RecordManager の参照統一

EraserTool 内で RecordManagerInstance を直接参照せず、getManager("record") を使用。

Manager登録キーを統一:

window.Tegaki.RecordManagerInstance // ❌ 廃止
this.getManager("record")           // ✅ 推奨

3. 初期化順序の明文化

app-core.js → TegakiApplication.initializeV8Managers() 内で、
必ず record → tool → 各ツール の順序を保証。

修正イメージ:

await this.initializeV8Manager("record");
await this.initializeV8Manager("tool");
await this.initializeV8Tools();

4. ヘッダー指示追加（再発防止）

各ファイルヘッダーに以下を追記:

⚠️ Manager呼び出しは必ず getManager("xxx") 経由とする
　- event → eventbus に統一
　- RecordManagerInstance 直参照禁止
　- v7/v8 両対応による二重管理禁止

✅ 次のステップ

abstract-tool.js を最優先で修正（event → eventbus）

eraser-tool.js の RecordManagerInstance 呼び出しを Manager経由に統一

app-core.js の初期化順序を見直し

全Manager登録キーをヘッダーに一覧化し、呼び出し名の整合性を確認

👉 ご希望なら、私の方で 修正パッチ（差分形式） を提示できますが、まずは event / eventbus の呼び名修正を一括で行うのが最優先です。
