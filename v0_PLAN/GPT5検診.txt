Q:ペンツールの将来性のある下地について

🎨 選択肢と評価
① PixiJS Graphics ベース（現行）

特徴: 単純な線描画、即時描画（immediate mode）。

長所: 実装が簡単。座標をそのまま描画できる。

短所: 線をベクターデータとして再編集しづらい。長いストロークでパフォーマンス低下。

👉 短期的にはベスト。下地としては「座標を保持し、Graphics はレンダリング専用」と分離しておくのが将来の拡張に効きます。

② PIXI.Graphics を使うが「座標データを必ず保持」

特徴: 全ストロークを this.points に保存し、Graphics は「今のフレームに描画するためのレンダラー」と割り切る。

長所: 将来的に SVG / WebGL Path2D / AI 補正描画 に切り替えやすい。

短所: 再描画の仕組みを別途作る必要がある。

👉 中長期で考えると最も拡張性が高い。AI 補正や「後から線を編集」も対応できる。

③ WebGL シェーダー / Signed Distance Field (SDF) ペン

特徴: 頂点シェーダーやフラグメントシェーダーで「ブラシ」を描画。

長所: 超高速・高品質。ブラシのにじみや物理っぽいペン先表現も可能。

短所: 実装難度が高い。今の段階ではオーバースペック。

👉 研究用・将来的に検討。Phase1 では不要。

④ SVG Path ベース（Canvas にレンダリング）

特徴: 描画は SVG Path のように「命令リスト（M, L, Q, C...）」で管理。

長所: 完全ベクター互換。AI での補正（曲線補間・認識）に相性良い。

短所: PixiJS 直描画との橋渡しが必要。

👉 AI を絡めたいなら 座標を Path 命令に変換して保持 → PixiJS に描画 が理想。

🚀 おすすめ戦略（今すぐやるべきこと）

必ずストロークの「座標配列」を保存する

this.points = [...] を保持し、描画は Graphics に流すだけ。

将来はこの配列を AI 補正 / SVG 出力 / 物理ブラシ再描画に使える。

描画処理を「レンダリング層」に分離

PenTool = 入力（座標収集）＋スタイル（色/太さ）

StrokeRenderer = 実際に Graphics へ線を描く

こうすると、あとで SVGRenderer や ShaderRenderer に差し替え可能。

今は PixiJS Graphics を使う

最小実装はこれで十分。

ただし 「Graphics が主ではなく座標が主」 という設計思想にする。

🌟 結論

現段階: PixiJS Graphics で描画する（軽量・シンプル）。

下地: 「全ストロークをベクターデータ（座標列＋属性）として保持」し、Graphics は「表示用」に限定する。

将来:

SVG 互換形式に変換して編集・AI補正

Shader / SDF ブラシに置き換えて高品質化
