エラーの原因

チェック関数で DOM 操作を先に実行

TegakiV8TrainSystem.checkDependencies


内で document.querySelector(...) を呼んでいますが、この呼び出し時点で該当要素がまだ DOM に存在していない。

列車型の読み込み順序の影響

tegaki-a1-header.html などヘッダーや共通辞典は先に読み込む想定

tegaki-b1-ui.html など UI 関連の DOM はまだ描画されていない

しかし checkDependencies がメインファイルで先に実行されるため、querySelector で取得できずエラーになっている

v7時代と違い、全体を HTML で列車化したため、DOM の準備が従来とは異なるタイミングになった

対策案
1. DOM が完全に描画されるまで待つ

DOMContentLoaded イベントまたは window.onload で起動処理を包むと安全です。

document.addEventListener('DOMContentLoaded', () => {
    TegakiV8TrainSystem.boot();
});


これにより、列車型で分割されても UI 要素が存在する状態でチェックが行われます。

2. checkDependencies の実装を防御的に変更

DOM がない場合はエラーを投げずに、後でチェックするように変更も可能です。

checkDependencies() {
    const element = document.querySelector('#some-tool-button');
    if (!element) {
        console.warn('⚠ DOM要素がまだ準備されていません: #some-tool-button');
        return false; // 後で再チェック
    }
    return true;
}

3. 列車型の読み込み順序の確認

A列車（ヘッダー・コア初期化）は先に読み込む

B列車（UI・ステータス）は DOM 描画後に読み込む

<!-- 列車順序例 -->
<script src="tegaki-a1-header.html"></script>
<script src="tegaki-a2-core.html"></script>
<script src="tegaki-a3-integration.html"></script>
<script src="tegaki-b1-ui.html"></script>
<script src="tegaki-b2-status.html"></script>
<script>
  document.addEventListener('DOMContentLoaded', () => {
      TegakiV8TrainSystem.boot();
  });
</script>


ポイントは DOM 依存部分は必ず描画後に起動 すること。

まとめ

エラーは DOM操作をDOM描画前に実行 しているのが原因

対策は以下のどちらか：

DOMContentLoaded で起動を遅延

checkDependencies を防御的に書き換え、DOMがなくても落ちないようにする

列車型自体は正しいが、DOM依存処理の起動タイミングを調整する必要あり
