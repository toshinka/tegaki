1. 原因分析

ペンポップアップ変更

ペンポップアップ.png → 枠付き表示に変更

サイズ計算ロジックをリニア変換に変更

プリセット保持用の新データ構造追加
→ この段階で「キャンバスのサイズ計算や描画初期化のDOM参照」が遅延していると、Pixi.Applicationの生成前にcanvasサイズが0になり得ます。

ツールバーアイコンの色変更

CSSやSVGのstroke/fillを変更
→ DOM作成順序やclass名指定がPixiやAppCore初期化前だと、描画が正しく反映されない可能性。

初期化フロー破綻

v7rev2では CanvasManager.attach() → CanvasManager.init() の順でcanvas生成

v7rev3では、ペンポップアップとプリセット保持追加により attach/initの順序がずれている、またはDOM参照が失敗
→ 結果として app.view にPixi canvasがアタッチされない。Consoleエラーは出ていないので try/catchで握られているか、Canvasサイズが0のまま になっている。

2. 対策案
A. 初期化順序の明示化（必須）
1. ConfigManager.configure()          // 設定読み込み
2. CanvasManager.attach()             // DOM要素参照
3. CanvasManager.init()               // Pixi.Application生成
4. EventBus.attach/init()             // pointer/touch
5. ToolManager.attach/init()          // ペンポップアップ含む
6. UI/Sidebar 起動                    // アイコン配置


ポイント：ペンポップアップやツールデータは CanvasManager.init() 完了後に参照する

これを守らないと canvasが存在しない状態で描画処理が走り、見えないだけ になる

B. ペンポップアップ変更の安全化

枠表示に変えるなら、初期化時に以下を確認：

// サイズ計算
popupSize = Math.min(preset.size, maxCircleSize);
// DOM作成
const popupCircle = document.createElement('div');
popupCircle.style.border = '2px solid #800000';
popupCircle.style.width = popupSize + 'px';
popupCircle.style.height = popupSize + 'px';
popupCircle.style.borderRadius = '50%';


重要：Pixi canvas生成後にDOMへappendすること

プリセット保持用データは、canvas生成前に宣言するだけでOK

C. アイコンカラー変更

SVGを直接変更する場合：

iconSvg.setAttribute('stroke', '#800000');


CSSの場合：

.futaba-icon {
    stroke: #800000;
    fill: none;
}


注意：Pixi canvas上の描画には影響しない。あくまでDOM側のツールバーアイコン。

3. 確認ポイント

CanvasManager.attach() で app.view が正しく取得できているか

CanvasManager.init() が Promise を返して完了しているか

ペンポップアップやプリセット保持の処理は init完了後に行う

CSS/DOM変更は Pixi canvasより先に行わない
