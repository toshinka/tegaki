🐛 原因分析

CanvasManager.getDrawContainer() が未定義 or 未初期化

エラーログ:

tool-manager.js:73 ⚠️ CanvasManager.getDrawContainer() not available


→ CanvasManager が ToolManager に渡される前に getDrawContainer() を呼んでいる。
→ つまり 初期化順序の問題 か CanvasManager 側にメソッド未定義。

ToolManager.setManagers() が呼ばれていない / 呼んでも canvasManager が無い

app-core.js のログ:

⚠️ ToolManager.setManagers() not available - 代替処理実行


→ AppCore.initializeV8Managers() 内で ToolManager.setManagers() を呼んでいるが、ToolManager が CanvasManager をまだ持っていない。

依存注入の検証で落ちる

app-core.js:519:

throw new Error("ToolManager.canvasManager injection verification failed");


→ ToolManager 側に this.canvasManager が無いため強制エラー。

✅ 改修案
1. CanvasManager.getDrawContainer() の定義確認

managers/canvas-manager.js に以下があるか確認。なければ追加:

getDrawContainer() {
    return this.drawContainer || null;
}


※ drawContainer の初期化が bootstrap.js → main.js → AppCore.initializeCanvasManager() で行われる順序になっているかもチェック。

2. ToolManager 側の初期化を遅延

現在は ToolManager コンストラクタ内で CanvasManager に依存している 可能性が高い。
以下のように修正：

// tool-manager.js
export class ToolManager {
    constructor() {
        this.canvasManager = null; // 後から注入
        this.tools = new Map();
    }

    setManagers(managers) {
        if (managers.canvasManager) {
            this.canvasManager = managers.canvasManager;
        }
        if (!this.canvasManager || !this.canvasManager.getDrawContainer) {
            console.warn("⚠️ ToolManager.canvasManager not properly injected");
        }
    }

    verifyInjection() {
        if (!this.canvasManager || !this.canvasManager.getDrawContainer) {
            throw new Error("ToolManager.canvasManager injection verification failed");
        }
    }
}

3. AppCore.initializeV8Managers() の修正

setManagers() を CanvasManager 初期化後に必ず呼ぶよう保証する。

// app-core.js (initializeV8Managers 内)
this.toolManager.setManagers({
    canvasManager: this.canvasManager,
    // 他のManagerも必要なら追加
});

this.toolManager.verifyInjection();

🛠️ 改修計画

CanvasManager に getDrawContainer() を保証する。

ToolManager は コンストラクタで依存を使わず、setManagers() 後に使う。

AppCore.initializeV8Managers() は

CanvasManager → ToolManager の順で初期化する。

最後に verifyInjection() を呼んで確実に検証。

📌 結論:
エラー原因は 初期化順序の不整合 と CanvasManager に依存する呼び出しをコンストラクタでやっていること。
修正は ToolManager に遅延注入 (DI) + CanvasManager のメソッド保証 で解決できます。

👉 この修正、まずは tool-manager.js に verifyInjection() + setManagers() 対応 を入れて試すのが最小手直しになります。
次に canvas-manager.js を点検して getDrawContainer() を補強するのが安全ルート。

🛑 原因分析

エラーの本質は次の通りです。

Error: ToolManager.canvasManager injection verification failed


これに関連してログでは

tool-manager.js:73 ⚠️ CanvasManager.getDrawContainer() not available

app-core.js:499 ⚠️ ToolManager.setManagers() not available - 代替処理実行

が出ています。

つまり ToolManager が CanvasManager を参照する段階で正しく注入されていない ことが直接原因です。

🔍 詳細な原因

依存関係の注入順序不整合

AppCore.initializeV8Managers() で ToolManager の setManagers() が呼ばれる前に、CanvasManager が初期化されていないか、参照のリンクが切れている。

ToolManager の実装側での堅牢性不足

canvasManager が null / undefined の場合に getDrawContainer() を即座に呼び出してしまいエラー。

例外を潰す代替処理 (app-core.js:499) はあるが、verify check が落ちている。

Claudの改修で tool-manager.js だけ修正

他の関連ファイル（特に canvas-manager.js と app-core.js）との整合性が取れていない。

結果として「注入されるはずの canvasManager が存在しない」状態に。

✅ 改修案
1. tool-manager.js 側の防御的対応
// tool-manager.js
setManagers(managers) {
    if (managers.canvasManager) {
        this.canvasManager = managers.canvasManager;
        if (typeof this.canvasManager.getDrawContainer === "function") {
            this.drawContainer = this.canvasManager.getDrawContainer();
        } else {
            console.warn("⚠️ CanvasManager.getDrawContainer() not available");
            this.drawContainer = null;
        }
    } else {
        console.error("💀 ToolManager: canvasManager injection missing");
        this.canvasManager = null;
    }
}

2. app-core.js 側での注入保証

AppCore.initializeV8Managers() にて CanvasManager → ToolManager の順序保証を明文化する。

// app-core.js
initializeV8Managers() {
    this.canvasManager = new CanvasManager(this); 
    this.toolManager = new ToolManager(this);

    this.toolManager.setManagers({
        canvasManager: this.canvasManager,
        eventBus: this.eventBus,
        recordManager: this.recordManager
    });

    if (!this.toolManager.canvasManager) {
        throw new Error("ToolManager.canvasManager injection verification failed");
    }
}

3. canvas-manager.js 側のAPI保証

getDrawContainer() を最低限の実装で必ず返すよう保証。

// canvas-manager.js
getDrawContainer() {
    return this.drawContainer || null;
}
