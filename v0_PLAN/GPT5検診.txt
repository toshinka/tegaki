🔎 主なエラー内容の整理

Required managers missing for v8 Tool initialization: coordinate, record

ToolManager.initializeV8Tools() 内で、必要な Manager (coordinate, record) が見つからず落ちている

AppCore / TegakiApplication も連鎖的に失敗して初期化未完了になっている

利用可能Managerの一覧には9個出ているが、その中に coordinate / record が含まれていない

つまり Manager 登録の順序 or 存在チェックに不整合 がある

✅ 考えられる原因

Manager 登録漏れ

coordinate-manager.js, record-manager.js はファイルとして存在するが、
AppCore.initializeV8Managers() で this.managers に登録していない可能性。

→ 他の Manager (canvas, tool, config...) は登録されているが、この2つだけ抜けている。

キー名の不一致

ToolManager は "coordinate", "record" を要求しているが、
実際に AppCore 側では "coordinateManager", "recordManager" など別名で登録しているケース。

つまり lookup 名のズレ がある。

初期化順序の問題

ToolManager が Manager 群を要求する時点で coordinate-manager / record-manager がまだ initialize() されていない。

そのため存在チェックで弾かれている。

utils/minimal-deps.js の不具合

Manager 群のロードリストを返す場所（恐らくここか bootstrap.js 経由）が不完全で、
coordinate-manager と record-manager が列挙から漏れている可能性。

🛠 対策案
1. AppCore 側の Manager 登録を確認

app-core.js の Manager 初期化部分（initializeV8Managers 内 or その前）に、以下のような登録があるか確認：

this.managers.set("coordinate", new CoordinateManager(this));
this.managers.set("record", new RecordManager(this));


なければ追加してください。

2. ToolManager が要求するキーと統一

tool-manager.js の initializeV8Tools() 内で

const required = ["coordinate", "record"];


としているなら、AppCore 側も必ず "coordinate" / "record" で登録すること。
もし CoordinateManager 側でクラス名やキーを "coordinateManager" にしていたら修正が必要です。

3. Manager のロード順序を保証

canvas-manager や tool-manager よりも前に、coordinate-manager / record-manager を登録するようにしてください。
（特に record は描画履歴に関わるので、ペンや消しゴムの初期化前に存在している必要があります）

4. minimal-deps.js のリストを点検

このファイルで MANAGERS などの配列があれば、次のように追加確認：

export const MANAGERS = [
  "config",
  "error",
  "event",
  "navigation",
  "shortcut",
  "canvas",
  "tool",
  "coordinate",   // ←抜けてないか？
  "record"        // ←抜けてないか？
];

⚠️ エラーに出ないけど怪しい点

ペンツールと消しゴムツール

Tool 初期化で coordinate / record を必ず参照する設計なら、今のままでは正常に描画履歴が残らないはず。

app-core.js のエラー処理

⚠️ ToolManager.setManagers() not available - 個別Manager確認で継続
→ ここで continue しているのは危険。実際は 初期化中断すべきエラー。

メソッドの順序不整合

TegakiApplication.initialize → AppCore.initializeV8Managers → ToolManager.initializeV8Tools
の順に走っているが、Manager群の整合性が取れる前に ToolManager に処理を渡している可能性あり。

🚑 修正計画（短期）

app-core.js にて coordinate と record を明示的に登録

tool-manager.js の requiredManagers とキー名が一致しているか確認

minimal-deps.js に2つの Manager を追加

初期化順序を整理（ペン・消しゴムが動作する前に2 Manager を登録）

👉 これを直せば「Required managers missing for v8 Tool initialization」エラーは解消するはずです。
