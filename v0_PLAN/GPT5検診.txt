console.txt を確認しました。問題点は大きく 2 点あります。

1. エラー箇所
tool-manager.js:386 ❌ PointerDown処理エラー [pen]: 
TypeError: this.recordManager.startOperation is not a function
    at PenTool.startDrawing (pen-tool.js:167:57)


原因
PenTool.startDrawing() 内で呼ばれている

this.recordManager.startOperation(...)


が存在しない。
console ログにも出ている通り、RecordManager は「Phase1.5スタブ実装」状態で、startOperation がまだ実装されていません
。

対策

RecordManager に最低限のダミーメソッドを追加する（本格 Undo/Redo 実装は後回し）。

startOperation(op) { 
    console.warn("RecordManager.startOperation stub called", op); 
}
endOperation(op) {
    console.warn("RecordManager.endOperation stub called", op);
}


あるいは PenTool 側で if (this.recordManager.startOperation) をチェックして呼ぶようにする。
→ 今の作りなら 1 の方が堅牢。

2. NaN 座標
pen-tool.js:123 ✏️ ペン描画開始: layer=layer1, pos=(NaN, NaN)


原因
AbstractTool で CoordinateManager を通した変換後の座標が NaN になっています。
ログを見ると:

tool-manager.js:370 🖊️ PointerDown [pen]: {x: 191, y: 250.5}
abstract-tool.js:196 🎯 AbstractTool pen マウスダウン: (-95, 154)
pen-tool.js:123 ✏️ ペン描画開始: pos=(NaN, NaN)


つまり PointerEvent座標 → AbstractTool座標 → PenTool座標 のどこかで数値変換が崩れています。
CoordinateManager に canvas要素 は渡されていますが、キャンバス移動やズームをまだ実装していないので、変換関数が未完成の可能性大。

対策

CoordinateManager.toCanvasCoords(event) の戻り値を console.log で直接検証する。

NaN が出ているなら、キャンバス幅やオフセット参照に undefined が混じっている可能性が高い。

とりあえず isNaN(x) || isNaN(y) の場合は fallback で raw 座標を使うようにすれば描画自体は進む。

3. 今のままの仕組みは拡張に耐えられるか？

CanvasManager
すでに「レイヤー分離版」で、layer0=背景, layer1=描画 という PixiJS Container が分離されているので、レイヤー追加・移動には十分対応可能。

ToolManager / AbstractTool
Manager 群と EventBus 経由で疎結合になっており、ツールの追加・差し替えも容易。

RecordManager / NavigationManager
まだスタブ実装なので、Undo/Redo とナビゲーションは将来拡張前提。

👉 まとめると、基盤はレイヤーや非破壊編集を想定した堅牢な設計になっています。
ただし RecordManager のスタブ不足と CoordinateManager の NaN 問題を潰さないとペン描画は安定しません。
