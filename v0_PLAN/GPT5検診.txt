正しい「キャンバス出現フロー」と「ペン描画フロー」（期待する順序）

Bootstrap.start()（DOMContentLoaded 後） → checkPixiJSAvailability() → setupCanvasContainer() が走る。これで #canvas-container が DOM に用意される。
GitHub

Bootstrap.checkPhase15Dependencies() により CanvasManager, ToolManager, PenTool 等が存在かつ実行可能であることを確認する（typeof === 'function' と軽量インスタンス化チェック）。存在しなければここで止まる。
GitHub

AppCore.initializeV8Managers()（想定）で CanvasManager を起動し、内部で PIXI.Application を生成して #canvas-container に app.view を追加、drawContainer（描画用コンテナ）を作る。CanvasManager.getDrawContainer() を公開する。

ToolManager.setManagers({ canvasManager, eventBus, coordinateManager, ... }) → ToolManager.verifyInjection() を実行して依存注入を確定。

ToolManager.initializeV8Tools() で PenTool / EraserTool を登録し、デフォルトツールを選択。ツールは activate() / deactivate() を持つ。

UI（サイドバー）のツール選択で ToolManager.switchTool(toolId) を呼び、現在選択ツールの pointerdown/pointermove/pointerup を CanvasManager の app.view にバインドして実際の描画を行う。

上のどこかでクラスが未定義だと Bootstrap の依存チェックで止まり、キャンバス描画には到達しません（現状と一致）
