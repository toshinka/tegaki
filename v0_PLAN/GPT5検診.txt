📝 Tegaki Phase1.5 状況確認・原因究明・改修計画（キャンバスリサイズ対応版）
1. 現状の確認
実装済み（Phase1ベース）

PixiJS v8 による描画処理の土台あり

ToolManager から Pen / Eraser 切替 UI 一応可能

CoordinateManager による座標取得

RecordManager に履歴保存の雛形あり

発生している問題

⚠️ powerPreference 警告
→ 環境依存で実害なし（PixiJS 側仕様）

⚠️ 描画されない問題
→ PenTool / EraserTool が drawContainer を取得できず、線が出力されない。

⚠️ 初期化未完了フラグ
→ TegakiApplication.initialize() 内で Manager 群の依存順序が乱れている。

⚠️ キャンバスサイズ不一致
→ Pixi.Application がデフォルト 800x600 を生成しており、HTML <canvas> の width="400" height="400" が反映されていない。

2. 原因分析

描画できない件

CanvasManager 側で drawContainer を生成・登録していない。

PenTool / EraserTool は drawContainer 前提で動いており不整合。

初期化未完了件

ToolManager が CanvasManager.getDrawContainer() を要求するが、CanvasManager 側の初期化順序が遅いため失敗。

双方向依存禁止ルールに反して、初期化が循環している。

キャンバスサイズ問題

new PIXI.Application() に width / height を指定していない。

HTML 側の <canvas> 設定は無視されるため、想定外サイズで生成される。

3. 改修方針
(A) CanvasManager

初期サイズを明示的に 400x400 に指定する。

this.app = new PIXI.Application({
  width: options.width || 400,
  height: options.height || 400,
  backgroundColor: 0xffffff
});


this.drawContainer = new PIXI.Container(); を生成し this.app.stage.addChild(this.drawContainer) で登録。

resize(width, height) メソッドを追加し、this.app.renderer.resize(width, height) を呼び出せるようにする。

Manager 登録キーは window.Tegaki.CanvasManagerInstance として統一。

(B) ToolManager

canvasManager.getDrawContainer() を取得してから PenTool / EraserTool に注入。

直接参照禁止、依存逆流防止。

(C) PenTool

pointerdown: 新規 PIXI.Graphics() を生成 → drawContainer.addChild()

pointermove: graphics.lineTo(x, y) でストローク追加

pointerup: RecordManager に保存（将来的に SVG path / JSON TPF 対応）

(D) EraserTool

当面は「未実装警告」に格上げ。

Phase1.5 では「ストローク削除方式」に備える。

(E) 初期化順序

bootstrap.js にて以下の順序を固定化

ConfigManager → EventBus → CanvasManager → ToolManager → Application

4. 拡張方針（リサイズ対応）

CanvasManager.resize(width, height) でサイズ変更可能にする。

変更時は CoordinateManager に通知し、座標系をリセット。

将来的に RecordManager に「リサイズ履歴」を追加できる余地を残す。

デフォルトは 400x400、ユーザー操作で拡張可能。

5. 改修チェックリスト

 CanvasManager に drawContainer を追加し、Pen/Eraser が取得可能にする

 PenTool を drawContainer に描画する方式に改修

 EraserTool は未実装機能として警告表示

 初期化順序を再定義し、循環依存を解消

 CanvasManager.resize(width, height) を導入し、将来のカメラ・リサイズ機能に備える

6. 次アクション

canvas-manager.js を全面改修し、初期化時点で 400x400 のキャンバスと drawContainer を保証する。

pen-tool.js を drawContainer 経由の描画に切替。

bootstrap.js で初期化順序を整理。

eraser-tool.js を「未実装」警告に切替。

将来対応として、resize(width, height) を設計に組み込み。
