# 🔧 描画位置ズレ・高DPI修正 作業手順書（実行中）

## 🚨 問題分析結果

添付画像から確認できる問題：
1. **描画位置が左上にズレる現象** - 座標変換ロジックの問題
2. **0.0から線が引かれる** - boundaryManager との座標連携問題

### 修正2: 座標変換スケール補償調整（coordinate-manager.js）

```javascript
// 📍 screenToCanvas() メソッド内のスケール補償処理
if (this.scaleCompensation) {
    const devicePixelRatio = window.devicePixelRatio || 1;
    // 高DPI無効化時はスケール補償をスキップ
    if (devicePixelRatio !== 1 && this.coordinateConfig.enableHighDPI !== false) {
        // 既存の補償処理は高DPI有効時のみ
        canvasX /= devicePixelRatio;
        canvasY /= devicePixelRatio;
    }
}
```

### 修正3: 境界越え描画対応（pen-tool.js）

```javascript
// 📍 新規メソッド追加: 境界越えからの描画開始
handleBoundaryCrossIn(x, y, options = {}) {
    try {
        console.log(`🎯 境界越え描画開始: (${x.toFixed(2)}, ${y.toFixed(2)})`);
        
        // 座標が有効であることを確認
        if (typeof x !== 'number' || typeof y !== 'number') {
            this.safeError('境界越え描画: 無効な座標', 'warning');
            return;
        }
        
        // 描画セッションを開始（通常の startDrawing と同様）
        return this.startDrawing(x, y, options.pressure || 0.5, options.timestamp || performance.now());
        
    } catch (error) {
        this.safeError(`境界越え描画エラー: ${error.message}`, 'error');
    }
}
```
