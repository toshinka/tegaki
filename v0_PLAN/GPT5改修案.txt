見えている現象（レイヤー切り替え・ドラッグ時に一瞬「▣（透けたサムネイルの枠だけ）」が表示される）の原因は、
main.html 内の サムネイル更新処理 と Sortable.js のドラッグ中スタイル が重なっているためです。

原因ポイント

StableThumbnailManager._updateThumbnailUI
サムネイルを更新するときに <img> の opacity を一瞬 0 にしてから新しい画像を差し替えている。

thumbnailContainer.innerHTML = `<img src="${dataUrl}" alt="" style="opacity: 0;">`;
const newImg = thumbnailContainer.querySelector('img');
setTimeout(() => newImg.style.opacity = '1', 50);


→ この 50ms の間、背景の .layer-thumbnail の枠だけが表示され「▣」に見えている。

Sortable.js のドラッグ中スタイル

.sortable-ghost {
    opacity: 0.3;
    background: rgba(100, 150, 255, 0.15) !important;
    border: 2px dashed #6496ff !important;
    transform: rotate(2deg);
}


→ ドラッグ中はサムネイル全体が半透明や枠線表示になり、さらに目立ってしまう。

改修策（いずれかを選べます）
A. 「▣」を出さない（おすすめ）

サムネイル更新時に 即時置換 する。フェードアニメーションをやめる。

// StableThumbnailManager._updateThumbnailUI の else 部分を修正
if (thumbnailContainer) {
    if (thumbnailImg) {
        thumbnailImg.src = dataUrl;   // 即時差し替え
        thumbnailImg.style.opacity = '1';
    } else {
        thumbnailContainer.innerHTML = `<img src="${dataUrl}" alt="">`;
    }
}

B. 「▣」を塗りつぶし（■にする）

layer-thumbnail に背景色を付けて、画像が消えている間も枠が見えないようにする。

.layer-thumbnail {
    background: var(--futaba-light-medium); /* 枠色と同じ塗りつぶし */
}

C. 切り替え時はサムネイル自体を非表示にする

更新時に一瞬 .layer-thumbnail を display:none にして、差し替え後に表示。

thumbnailContainer.style.visibility = 'hidden';
thumbnailContainer.innerHTML = `<img src="${dataUrl}" alt="">`;
requestAnimationFrame(() => {
    thumbnailContainer.style.visibility = 'visible';
});

まとめ

原因は「サムネイル更新時のフェード処理」と「Sortable.jsのゴーストスタイル」。

対応策は3種類：

フェードをやめ即時差し替え（▣が出ない）

背景を塗りつぶす（▣が■に見える）

更新中は非表示にする（完全に消える）


問題点

resetViewport() は TransformSystem を初期スケール・位置にリセットしますが、
すでに canvas-container は CSS 側で

left: 50%;
top: 50%;
transform: translate(-50%, -50%);


が入っているため、二重で中央寄せが適用されます。

そのため「初回パン開始時に右下へ 150〜200px ほどズレる」という現象が出ます。

2回目以降は firstPanComplete = true になるためリセットが走らず、ズレなくなる。

✅ 対策
方法 1: 初回リセットを無効化する（もっとも簡単）
if (!this.state.firstPanComplete) {
    // window.DrawingEngineAPI?.resetViewport(); ← これを削除/コメントアウト
    console.log('🔄 [HammerJS] First pan - skipped reset');
}


→ これで初回パンでも余計なリセットがかからなくなり、ズレは発生しません。

方法 2: 正しい位置に初期化する

もし「本当に初回だけキャンバスを中央に揃えたい」意図があるなら、
resetViewport() ではなく recordInitialPosition() の値を使って正しく中央補正するように修正します。

例:

if (!this.state.firstPanComplete) {
    const { x, y } = this.initialCanvasPosition;
    this.container.style.transform = `translate(${x}px, ${y}px) translate(-50%, -50%)`;
}

🎯 結論

今回のズレは 「初回パン開始時に resetViewport() を呼んでいる」ことが原因です。

ズレをなくしたいだけなら → resetViewport() を消す。

中央揃えしたいなら → recordInitialPosition() の値を参照して正しい位置に置く。
