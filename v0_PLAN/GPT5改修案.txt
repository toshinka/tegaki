🔍 問題の概要

消しゴムがペンの振る舞いをする: 描画中に消しゴムツールを選択しても、描画が消去ではなく描画として行われてしまう。

描画フローの不整合: drawing-engine.js 内での描画処理がツールの種類に応じて適切に分岐していない可能性がある。

🛠 改善案
1. ツールの切り替えと描画処理の分岐

drawing-engine.js 内で、現在のツール（ペンまたは消しゴム）を明示的に管理し、それに応じて描画処理を分岐させる必要があります。以下のような構造を検討してください。

// 現在のツールを管理する変数
let currentTool = 'pen'; // 初期状態はペン

// 描画処理の関数
function drawStroke(strokePoints) {
  if (currentTool === 'pen') {
    // ペンの描画処理
    drawWithPen(strokePoints);
  } else if (currentTool === 'eraser') {
    // 消しゴムの描画処理
    eraseWithEraser(strokePoints);
  }
}

// ツールの切り替え関数
function setTool(tool) {
  if (tool === 'pen' || tool === 'eraser') {
    currentTool = tool;
  } else {
    console.error('無効なツールが指定されました');
  }
}


このように、currentTool 変数で現在のツールを管理し、drawStroke 関数内でそのツールに応じた描画処理を呼び出すようにします。

2. 描画処理の分離

ペンと消しゴムの描画処理は明確に分けるべきです。例えば、以下のようにそれぞれの描画処理を関数として定義します。

function drawWithPen(strokePoints) {
  // ペンによる描画処理
  const graphics = new PIXI.Graphics();
  graphics.lineStyle(4, 0x000000, 1);
  graphics.moveTo(strokePoints[0].x, strokePoints[0].y);
  strokePoints.forEach(point => {
    graphics.lineTo(point.x, point.y);
  });
  // 描画をステージに追加
  app.stage.addChild(graphics);
}

function eraseWithEraser(strokePoints) {
  // 消しゴムによる描画処理（消去）
  const graphics = new PIXI.Graphics();
  graphics.lineStyle(10, 0xffffff, 1); // 白色で消去
  graphics.moveTo(strokePoints[0].x, strokePoints[0].y);
  strokePoints.forEach(point => {
    graphics.lineTo(point.x, point.y);
  });
  // 描画をステージに追加
  app.stage.addChild(graphics);
}


消しゴムの描画処理では、lineStyle の太さや色を調整することで消去の効果を実現します。

3. 履歴管理と非破壊編集

描画操作を非破壊で行うためには、履歴管理を適切に行う必要があります。history-service.js での履歴管理が適切に行われているか確認し、描画操作ごとに履歴を記録するようにします。

function commitStroke(strokeData) {
  // 履歴に描画データを追加
  historyService.addHistory({
    type: 'draw',
    data: strokeData
  });
  // 描画処理を実行
  drawStroke(strokeData.points);
}


このように、描画処理を実行する前に履歴に記録し、Undo/Redo 操作が可能なようにします。

✅ 確認ポイント

 ツールの切り替え: ペンと消しゴムの切り替えが適切に行われているか。

 描画処理の分岐: 現在のツールに応じて描画処理が分岐しているか。

 履歴管理: 描画操作が履歴として適切に記録されているか。

 非破壊編集: 描画操作が非破壊で行われているか。
