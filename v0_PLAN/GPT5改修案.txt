1. DrawingEngine.js:1628 Uncaught SyntaxError: Unexpected token ')'

これは JavaScript の構文エラーです。
Unexpected token ')' は「余分な ) がある」または「対応する ( が無い」ケースです。

👉 該当ファイルを確認すると、DrawingEngine.js の 1628 行付近に不正な括弧があります。
たとえば以下のようなパターンが典型的です：

function someFunc() {
  doSomething();  // ← ここまではOK
);                // ← 不要な閉じ括弧が残っている


あるいは アロー関数や if 文の閉じ括弧忘れでも出ます：

layers.forEach((layer) => {   // ← { を開いたのに
    console.log(layer);
}); )                         // ← } のあとに余分な )

2. Application startup failed: Error: Library drawingEngine not found

これは main.html 内で

const drawingEngine = window.drawingEngine;
if (!drawingEngine) throw new Error("Library drawingEngine not found");


のようなチェックをしているのに、window.drawingEngine が定義されていないために出ています。

Phase1b のときは DrawingEngine.js が IIFE で window.drawingEngine = { ... } を export していたはずですが、Phase2C で修正時に グローバル登録を削除 / コメントアウトした可能性があります。

改修案
A. SyntaxError の修正

DrawingEngine.js の 1620〜1635 行くらいを開いて、余分な ) を削除してください。

最も安全なのは、その部分を私に貼ってもらうことですが、パッチ例を出します。

修正パッチ例：

// Before (エラーになる)
layers.forEach((layer) => {
   syncLayer(layer);
}); )

// After
layers.forEach((layer) => {
   syncLayer(layer);
});

B. drawingEngine not found の修正

DrawingEngine.js の最後に以下を追加してください：

// Expose DrawingEngine globally
if (typeof window !== "undefined") {
    window.drawingEngine = {
        init: initDrawingEngine,
        layers: layers,
        addLayer: addLayer,
        removeLayer: removeLayer,
        setActiveLayer: setActiveLayer,
        // 必要なAPIをここに列挙
    };
}


これで main.html 側から window.drawingEngine が見えるようになります。

まとめ（修正ポイント）

DrawingEngine.js の 1628行付近の余分な ) を削除。

DrawingEngine.js の末尾に window.drawingEngine = { ... } を追加し、グローバル公開する。


設計意図 vs Phase2C の差分ポイント
1. グローバル公開 (window.drawingEngine) の欠落

意図: main.html から drawingEngine を呼び出せるようにグローバルに登録。

Phase2C: window.drawingEngine が未定義 → 起動失敗。
👉 修正必須（前回答の末尾パッチでOK）。

2. activeLayer の管理方法

意図: インデックスではなく レイヤー固有ID (layer.id) をキーにして追跡。ドラッグで順序が変わっても参照が壊れない。

Phase2C: activeLayer を配列インデックス依存で扱っている部分が残っている。
👉 危険ポイント：ドラッグ移動後にアクティブが勝手に別レイヤーに変わることがある。

3. UIとPixiの並び順の同期

意図: DOM順序 → layers[] → Pixi の stage.children に明示的に反映 (setChildIndex)。

Phase2C: DOM順序を直接使う部分が残っている。Pixi の z-order とズレる可能性あり。特に「背景が前面に来る」症状の再発要因。

4. SortableJS のドラッグ設定

意図: handle, delay / distance を設定し、クリックとドラッグを明確に分離。

Phase2C: handle が未指定または全体クリックでドラッグ可能 → レイヤーをクリックすると入れ替えが誤発火する挙動が残る。
👉 修正推奨。

5. エラー処理とログ

意図: 開発段階なので、console.table(layers) や console.warn でトラブルシュートしやすく。

Phase2C: 多くの場所で無言処理、あるいはエラーを握り潰している。デバッグが難しい。
👉 小さくても良いのでログ追加を推奨。

結論

致命的に意図と違うのは ①グローバル公開と②アクティブレイヤー管理（インデックス依存が残っている）。

再発の原因になりやすいのは ③UIとPixiの同期の不十分さ。

操作感に直結するのは ④Sortableの設定不足（クリックで入れ替わる問題）。

その他は⑤デバッグ性向上のための改善余地。

💡 つまり「大枠の設計方針は守られているけど、実装細部で Phase1b → Phase2C への移行時に意図が一部削られた／戻ってしまった」状態です。
