ChatGPT:

なるほど 👍
いただいたログから原因を整理すると、PixiJS v8 系の イベントシステム API が変わっている ことが主要因です。

🔎 原因

エラー箇所

this.app.renderer.events.pointer.normalizeGlobal


→ PixiJS v7 以前の API で、v8 では削除されています。

v8 では renderer.events.pointer のような低レベル API ではなく、FederatedPointerEvent を通じて event.global が直接使えるようになりました。

そのため、座標変換のフローが旧 API 前提のままになっており、描画ツール・キャンバス移動・サムネイル更新が全部巻き込まれて動かなくなっています。

🛠 改修案
1. CoordinateSystem.domToCanvas の修正

旧:

domToCanvas(event) {
    const global = this.app.renderer.events.pointer.normalizeGlobal(event);
    return this.app.stage.toLocal(global);
}


新 (PixiJS v8 対応):

domToCanvas(event) {
    // FederatedPointerEvent から直接 global を取得
    const global = event.global; 
    // グローバル座標をステージ基準のローカル座標に変換
    return this.app.stage.toLocal(global);
}

2. DrawingTools.onPointerMove / onPointerDown の修正

旧:

onPointerMove(e) {
    const pos = this.coordinateSystem.domToCanvas(e);
    this.drawLine(pos);
}


新:

onPointerMove(e) {
    // e は FederatedPointerEvent
    const pos = this.coordinateSystem.domToCanvas(e);
    this.drawLine(pos);
}


※ 既存の normalizeGlobal 呼び出しを削除するだけで良い。

3. キャンバス移動処理の修正

もし「スクリーン座標からキャンバス座標へ」の変換をやっている箇所が他にあれば、すべて同じ対応が必要です。

例:

// 旧
const global = this.app.renderer.events.pointer.normalizeGlobal(e);

// 新
const global = e.global;

4. レイヤーパネルのサムネイル表示不良

サムネイル生成時に renderer.extract.canvas(layer) のような呼び出しをしている場合、layer が正しくステージに追加されていないと黒塗りになります。

v8 では extract の呼び出しも Promise 化された部分があるため、必要なら await を使う。

例:

// 旧
const thumbnail = this.app.renderer.extract.canvas(layer);

// 新 (v8)
const thumbnail = await this.app.renderer.extract.canvasAsync(layer);

✅ 改修まとめコード（抜粋）
class CoordinateSystem {
    constructor(app) {
        this.app = app;
    }

    domToCanvas(event) {
        // PixiJS v8: event.global を直接利用
        const global = event.global;
        return this.app.stage.toLocal(global);
    }
}

class DrawingTools {
    constructor(app, coordinateSystem) {
        this.app = app;
        this.coordinateSystem = coordinateSystem;
    }

    onPointerDown(e) {
        const pos = this.coordinateSystem.domToCanvas(e);
        this.startDrawing(pos);
    }

    onPointerMove(e) {
        const pos = this.coordinateSystem.domToCanvas(e);
        this.continueDrawing(pos);
    }
}


👉 つまり normalizeGlobal の削除 & event.global への統一置換 が第一段階の修正です。
その後に サムネイル生成を extract.canvasAsync に差し替え することで、描画・移動・レイヤーパネルが復旧するはずです。
