問題の整理
1. 初期キャンバスサイズがやたら大きい（期待値：400×400でない）

表示には400×400と出ているものの、実際にはもっと大きく表示されている様子。

背景としては、canvasContainerやUI初期化時に想定外サイズが使われている可能性あり。

2. レイヤーパネルのサムネイルが機能していない

UIではサムネイル作成が行われているはずだが、表示されない、更新されないなどの不具合と見受けられる。

3. 拡縮（スケール）や反転操作後、画面が真っ白になる

背景色ではなく完全な白表示。canvasやレイヤーが「どこかに飛んでいる」可能性あり。

原因として、カメラ（canvasContainer）のズームや移動と、レイヤーの変形がどこかで干渉していることが考えられる。

Step A: キャンバスサイズの修正

UI「キャンバスリサイズ」からの入力適切に反映。

Step B: サムネイル生成と表示制御の改善

generateTexture()からrenderTexture＋Ticker方式へリファクタリング。

サムネイルへの遅延更新を、needsUpdate フラグ＋ ticker で管理。

Step C: 座標変換と描画領域制御の統一

canvasContainer をカメラ変形用、layer を描画操作用と明確に分離。

toLocal()／toGlobal() による座標一貫管理を全変形処理で徹底。

デバッグツール：操作後に座標／スケール／rotation状態をログ表示。

Step D: 状態検証／テスト

異常時には明示的な Error をthrowして早期検出／解決を図る。

まとめ

現象の主因：キャンバス初期化・UI反映漏れ、サムネイル更新ロジック未導入、座標系統の混在／一貫性欠如。

改善方針：ルールブックに沿ったContainer構造の再整備、描画周りの非同期制御見直し、座標変換の徹底、ステート可視化でのデバッグ性強化。

🔧 パッチコード例
1. 初期キャンバスサイズ（400×400固定化）
// app生成後の初期化部分を修正
const app = new PIXI.Application({
    width: 400,
    height: 400,
    backgroundAlpha: 0, // 背景は透明
    antialias: true
});

// stage にキャンバスコンテナを追加
const canvasContainer = new PIXI.Container();
app.stage.addChild(canvasContainer);

// 念のため resize 処理で常に 400x400 に揃える
function resetCanvasSize() {
    app.renderer.resize(400, 400);
    canvasContainer.width = 400;
    canvasContainer.height = 400;
}
resetCanvasSize();

2. サムネイル生成・更新処理修正
// レイヤーサムネイル用の RenderTexture を用意
function createLayerThumbnail(layer) {
    const rt = PIXI.RenderTexture.create({
        width: 100,   // サムネイルサイズ
        height: 100
    });
    app.renderer.render(layer, { renderTexture: rt });
    return new PIXI.Sprite(rt);
}

// 更新フラグ管理
let needsThumbnailUpdate = false;

// レイヤーが更新されたらフラグを立てる
function markThumbnailDirty() {
    needsThumbnailUpdate = true;
}

// 毎フレーム更新チェック
app.ticker.add(() => {
    if (needsThumbnailUpdate) {
        updateThumbnails();
        needsThumbnailUpdate = false;
    }
});

function updateThumbnails() {
    layerList.forEach((layer, idx) => {
        const rt = PIXI.RenderTexture.create({ width: 100, height: 100 });
        app.renderer.render(layer, { renderTexture: rt });
        layer.thumbnail.texture = rt;
    });
}

3. 座標系の分離と拡縮・反転対策
// キャンバス移動・ズームは canvasContainer にのみ適用
function setCanvasZoom(scale) {
    canvasContainer.scale.set(scale);
}

// レイヤー操作は layer コンテナにのみ適用
function flipLayerX(layer) {
    layer.scale.x *= -1; // 反転
}

// 座標変換は必ず toLocal / toGlobal を使用
function screenToCanvas(point) {
    return canvasContainer.toLocal(point);
}

4. デバッグログで異常検出
function logTransformState() {
    console.log("CanvasContainer", {
        x: canvasContainer.x,
        y: canvasContainer.y,
        scale: canvasContainer.scale.x
    });
    layerList.forEach((layer, i) => {
        console.log("Layer", i, {
            x: layer.x,
            y: layer.y,
            scale: { x: layer.scale.x, y: layer.scale.y }
        });
    });
}


拡縮／反転後に画面が白くなった場合は、このログを確認してください。
飛んでいっているのは scale や x/y が暴走している可能性が高いです。

🔑 改修ポイントまとめ

初期化：app.renderer.resize(400,400) を必ず呼ぶ。

サムネイル：renderTexture＋needsUpdateフラグ方式に。

拡縮／反転：canvasContainer（ズーム・移動担当）とlayer（描画変形担当）を分離。

デバッグ：座標・スケールをログ出力して異常検知。
