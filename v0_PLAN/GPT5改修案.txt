問題と原因

背景の中心が (0,0) になっている

原因：背景レイヤーを通常の表示オブジェクトとして配置してしまい、ワールド座標に対して固定していない。

結果：カメラ移動や初期配置で原点を基準にしてしまい、意図通りの枠内塗りができない。

サムネイルが (0,0)–(200,200) 領域しか表示しない

原因：RenderTexture への描画がビューポート全体ではなく固定矩形領域で切り出されている。

結果：カメラフレームや実際の描画範囲を反映できていない。

カメラフレームがキャンバス移動で動かない

原因：カメラフレームが「ビューポート」ではなく「固定スプライト」として扱われているため。

結果：キャンバス側だけ動いて、フレームとの相対位置がずれる。

拡縮・回転・反転がずれる

原因：変換基準点（pivot）が「カメラの中心」ではなく「キャンバスの原点」基準になっている。

結果：キャンバス移動後に処理すると、中心座標がずれて誤差が出る。

対策案
背景レイヤー（透明＋バケツ塗り対応）

レイヤー0を背景専用に固定

PIXI.RenderTexture を1枚作成し、初期化時に #f0e0d6 で塗りつぶし。

消しゴムは α 値を下げるようにして「透明化」する。

バケツツールは RenderTexture に直接 flood fill 相当の処理を実装する（GPUでは難しいので CPU 側でピクセル操作→更新、または shader を検討）。

サムネイル

RenderTexture.capture の領域をカメラフレーム基準にする

renderer.render(container, {renderTexture, clear, transform}) の引数で transform に camera の worldTransform を適用。

これでサムネイルがカメラフレームに追従。

カメラフレームの動作

@pixi/viewport を使う

PixiJS v8.13 公式互換のカメラライブラリ。

拡縮・移動・回転が「カメラ」座標として自然に適用される。

既存コードの「canvas移動処理」を viewport に置き換えるとズレが解消。

拡縮・回転・反転の基準点

カメラフレームの中心を pivot に設定

viewport.pivot.set(width/2, height/2) としてから拡縮・回転。

反転は scale.x *= -1 のように行えば、中心基準で反転可能。

推奨ライブラリ組み合わせ

PixiJS v8.13 本体

@pixi/viewport

カメラ移動・拡縮・回転を吸収できる。

（必要に応じて）pixi-graphics-smooth

ベクターベースのペン描画を滑らかにする。

（将来的に）pixi-filters

消しゴム・透明処理の補助にカスタムフィルタが使える。

次の改修ステップ（提案）

背景をレイヤー0専用にして RenderTexture で管理
→ 初期化時に #f0e0d6 塗りつぶし。消しゴムは透明化。

viewport を導入
→ キャンバス移動・ズーム・回転・反転のズレを解消。

サムネイル更新処理を cameraTransform 基準に変更
→ (0,0,200,200) 固定をやめ、カメラフレームに一致させる。

ペイント系（バケツ）を CPU or shader 実装
→ 小さな範囲なら CPU flood fill で十分。大規模は shader 検討。

  // Patch file: integrates @pixi/viewport as camera, creates Layer0 background
  // with an offscreen canvas for CPU-side fill + eraser, and provides thumbnail
  // capture based on the current viewport view.

  const appCanvas = document.getElementById('app');
  const app = new PIXI.Application({
    view: appCanvas,
    resizeTo: window,
    backgroundAlpha: 0,
    antialias: true,
  });

  // Stage organization: uiStage is for screen-fixed UI overlays (camera frame)
  const world = new PIXI.Container();    // all drawing layers live here and are transformed by viewport
  const uiStage = new PIXI.Container();  // overlays that should not follow camera (camera frame, HUD)

  app.stage.addChild(world);
  app.stage.addChild(uiStage);

  // Setup viewport (camera) — this will manage pan/zoom/rotate centered on screen
  const viewport = new window.viewport.Viewport({
    screenWidth: app.renderer.width,
    screenHeight: app.renderer.height,
    worldWidth: 10000,
    worldHeight: 10000,
    interaction: app.renderer.plugins.interaction
  });

  // Add viewport to stage and put world content inside it
  app.stage.addChild(viewport);
  viewport.addChild(world);

  // Enable drag, pinch, wheel
  viewport.drag().pinch().wheel().decelerate();

  // Camera frame (UI overlay) - centered and matches visible area
  const cameraFrame = new PIXI.Graphics();
  cameraFrame.lineStyle(2, 0xffffff, 0.6).drawRect(-app.renderer.width/2 + 10, -app.renderer.height/2 + 10, app.renderer.width - 20, app.renderer.height - 20);
  // we put cameraFrame into uiStage and position center of screen so it doesn't move with world
  uiStage.addChild(cameraFrame);
  cameraFrame.x = app.renderer.width/2;
  cameraFrame.y = app.renderer.height/2;

  // Keep cameraFrame updated on resize
  function updateCameraFrame() {
    cameraFrame.clear();
    cameraFrame.lineStyle(2, 0xffffff, 0.6);
    cameraFrame.drawRect(-app.renderer.width/2 + 10, -app.renderer.height/2 + 10, app.renderer.width - 20, app.renderer.height - 20);
    cameraFrame.x = app.renderer.width/2;
    cameraFrame.y = app.renderer.height/2;
  }
  window.addEventListener('resize', () => {
    viewport.resize(app.renderer.width, app.renderer.height, viewport.worldWidth, viewport.worldHeight);
    updateCameraFrame();
  });

  // === Layer 0: Background managed by an offscreen canvas so "eraser" can make transparent ===
  const bgCanvas = document.createElement('canvas');
  // Make this large enough for most use-cases; adapt to your document size
  bgCanvas.width = 5000;
  bgCanvas.height = 5000;
  const bgCtx = bgCanvas.getContext('2d');

  // Fill initial background with futaba-cream (#f0e0d6)
  function initBackground() {
    bgCtx.clearRect(0,0,bgCanvas.width,bgCanvas.height);
    bgCtx.fillStyle = '#f0e0d6';
    bgCtx.fillRect(0,0,bgCanvas.width,bgCanvas.height);
  }
  initBackground();

  // Create PIXI sprite that uses the offscreen canvas as a texture
  let bgTexture = PIXI.Texture.from(bgCanvas);
  const bgSprite = new PIXI.Sprite(bgTexture);
  // Position background so document origin (0,0) is in the center of canvas world coordinates if you prefer
  // Here we place (0,0) at top-left of bgCanvas for simplicity — adapt to your app coordinate conventions
  bgSprite.x = 0; bgSprite.y = 0;
  world.addChildAt(bgSprite, 0); // layer 0

  // Eraser: clear a circle on the bgCanvas where user erases. Accepts world coords.
  function eraseBackgroundAt(worldX, worldY, radius = 20) {
    // Convert world coords to bgCanvas pixel coords
    // If bgSprite is positioned at 0,0 and uses 1:1 scaling then it's straightforward
    const x = Math.round(worldX - bgSprite.x);
    const y = Math.round(worldY - bgSprite.y);
    bgCtx.save();
    bgCtx.globalCompositeOperation = 'destination-out';
    bgCtx.beginPath();
    bgCtx.arc(x, y, radius, 0, Math.PI*2);
    bgCtx.fill();
    bgCtx.restore();
    // Update the PIXI texture to reflect canvas changes
    if (bgTexture.baseTexture) bgTexture.baseTexture.update();
  }

  // === Example drawing layer on top of background ===
  const drawLayer = new PIXI.Container();
  world.addChild(drawLayer);

  // Simple drawing tool for testing — stroke with Graphics
  let drawing = false;
  const pen = new PIXI.Graphics();
  drawLayer.addChild(pen);

  function startStroke(worldX, worldY) {
    drawing = true;
    pen.lineStyle(4, 0x000000, 1);
    pen.moveTo(worldX, worldY);
  }
  function continueStroke(worldX, worldY) {
    if (!drawing) return;
    pen.lineTo(worldX, worldY);
    pen.moveTo(worldX, worldY);
  }
  function endStroke() {
    drawing = false;
  }

  // Map pointer events on the viewport to world coords
  app.view.addEventListener('pointerdown', (e) => {
    const rect = app.view.getBoundingClientRect();
    const px = e.clientX - rect.left;
    const py = e.clientY - rect.top;
    const worldPos = viewport.toWorld(px, py);
    if (e.shiftKey) {
      // Use shift+pointer as eraser on background to demonstrate erasing
      eraseBackgroundAt(worldPos.x, worldPos.y, 30);
    } else {
      startStroke(worldPos.x, worldPos.y);
    }
  });
  app.view.addEventListener('pointermove', (e) => {
    if (e.buttons === 0) return;
    const rect = app.view.getBoundingClientRect();
    const px = e.clientX - rect.left;
    const py = e.clientY - rect.top;
    const worldPos = viewport.toWorld(px, py);
    if (e.shiftKey) {
      eraseBackgroundAt(worldPos.x, worldPos.y, 30);
    } else {
      continueStroke(worldPos.x, worldPos.y);
    }
  });
  app.view.addEventListener('pointerup', (e) => { endStroke(); });

  // === Thumbnail rendering: capture current visible area of the stage according to viewport ===
  // Approach: use renderer.extract.canvas(stage) to get full canvas, then crop to viewport visible rect.
  // This is simple and robust for a developer patch. If performance matters, later switch to partial renderTexture rendering.

  function getViewportVisibleRect() {
    // Get top-left world point and size of the viewport visible area
    const topLeft = viewport.toWorld(0, 0);
    const bottomRight = viewport.toWorld(app.renderer.width, app.renderer.height);
    const w = Math.abs(bottomRight.x - topLeft.x);
    const h = Math.abs(bottomRight.y - topLeft.y);
    return { x: Math.min(topLeft.x, bottomRight.x), y: Math.min(topLeft.y, bottomRight.y), width: w, height: h };
  }

  const thumbCanvas = document.createElement('canvas');
  thumbCanvas.width = 200; thumbCanvas.height = 200; // thumbnail size
  const thumbCtx = thumbCanvas.getContext('2d');

  function updateThumbnail() {
    // Extract a canvas snapshot of the entire stage (this returns a canvas element)
    const fullCanvas = app.renderer.extract.canvas(app.stage);
    const visible = getViewportVisibleRect();

    // Compute scale between world units and pixels — assume bgSprite at 1:1 for simplicity
    // Convert world visible rect to pixel coordinates on fullCanvas
    // Note: In simple setups world coords == pixels. If you use scaling on bgSprite, adapt accordingly.
    const sx = visible.x;
    const sy = visible.y;
    const sw = visible.width;
    const sh = visible.height;

    // Crop and scale to thumbnail size
    thumbCtx.clearRect(0,0,thumbCanvas.width,thumbCanvas.height);
    // drawImage(image, sx, sy, sw, sh, dx, dy, dw, dh)
    thumbCtx.drawImage(fullCanvas, sx, sy, sw, sh, 0, 0, thumbCanvas.width, thumbCanvas.height);

    // For demo show thumbnail in document body (developer tool)
    let existing = document.getElementById('thumbPreview');
    if (!existing) {
      existing = document.createElement('img');
      existing.id = 'thumbPreview';
      existing.style.position = 'fixed';
      existing.style.right = '8px';
      existing.style.bottom = '8px';
      existing.style.border = '2px solid #fff';
      existing.style.width = '160px';
      existing.style.height = '160px';
      document.body.appendChild(existing);
    }
    existing.src = thumbCanvas.toDataURL();
  }

  // Update thumbnail periodically or on demand
  setInterval(updateThumbnail, 500);

  // === Example controls: center, zoom, flip horizontally/vertically anchored to viewport center ===
  function centerOn(x, y) {
    viewport.moveCenter(x, y);
  }
  function zoomTo(factor) {
    // zoom toward center of screen
    viewport.zoomPercent(factor, true);
  }
  function flipHorizontally() {
    // Flip the world container around viewport center
    // Implementation: translate world so center is at origin, flip scale.x, translate back
    const worldCenter = viewport.toWorld(app.renderer.width/2, app.renderer.height/2);
    world.pivot.set(worldCenter.x, worldCenter.y);
    world.position.set(worldCenter.x, worldCenter.y);
    world.scale.x *= -1;
  }
  function flipVertically() {
    const worldCenter = viewport.toWorld(app.renderer.width/2, app.renderer.height/2);
    world.pivot.set(worldCenter.x, worldCenter.y);
    world.position.set(worldCenter.x, worldCenter.y);
    world.scale.y *= -1;
  }

  // Demo buttons for controls (simple)
  const ctrl = document.createElement('div');
  ctrl.style.position='fixed'; ctrl.style.left='8px'; ctrl.style.bottom='8px'; ctrl.style.zIndex=9999;
  ctrl.innerHTML = '<button id="btnCenter">Center (0,0)</button> <button id="btnZoomIn">ZoomIn</button> <button id="btnZoomOut">ZoomOut</button> <button id="btnFlipH">FlipH</button>';
  document.body.appendChild(ctrl);
  document.getElementById('btnCenter').onclick = ()=> centerOn(200,200);
  document.getElementById('btnZoomIn').onclick = ()=> zoomTo(1.2);
  document.getElementById('btnZoomOut').onclick = ()=> zoomTo(0.8);
  document.getElementById('btnFlipH').onclick = ()=> flipHorizontally();

  // === Initialization demo content ===
  // Draw a test rectangle and some content on drawLayer
  const rect = new PIXI.Graphics();
  rect.beginFill(0xff0000, 0.2);
  rect.drawRect(100,100,400,300);
  rect.endFill();
  drawLayer.addChild(rect);

  // Move viewport to a sensible place
  viewport.moveCenter(250, 250);

  // Main ticker
  app.ticker.add(() => {
    // nothing heavy here — UI frame kept separate from world so it won't move when viewport pans
  });

  // Export small helper to console for further manual testing
  window.__tegaki_patch = { app, viewport, world, uiStage, bgCanvas, updateThumbnail };

主な変更点（短く）：

@pixi/viewport を導入して「カメラ（viewport）」を明確化。パン／ズーム／回転／慣性等を viewport に任せることで、移動とカメラ中心に起因するズレを解消します。

Layer 0（背景）をオフスクリーン canvas で管理し、初期塗り #f0e0d6（futaba-cream）で塗り、Shift+ドラッグで消しゴム（透明化）できるデモを実装。背景はそのまま透過状態に戻せます。

サムネイルは renderer.extract.canvas(stage) で全体を取得してから viewport の可視領域を切り出す方式で実装（堅牢だが最適化の余地あり）。

カメラフレーム（UI）は screen 固定レイヤーに置き、viewport の移動に追従しないようにして見た目と動作を分離。

中心基準の拡縮・反転ボタン（簡易）を追加し、反転が viewport の中心基準で動くようにしています。
