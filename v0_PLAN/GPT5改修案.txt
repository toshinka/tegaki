まとめ（結論）

現象原因：消しゴムが「背景色で上書きしている」挙動 → レイヤーのピクセルのアルファ（透明度）を消していないため。

対策（推奨）：各レイヤーをアルファ付きの RenderTexture（透明背景）で保持し、消しゴム操作はその RenderTexture に対して 「透明ピクセルを書き込む（＝アルファを落とす）」 方式で行う。Pixi の blendMode = ERASE（または同等のブレンド）を使って、描画内容のアルファを“消す”のが自然で高速。

重要ポイント：RenderTexture は作成直後に透明（alpha=0）で初期化しておくこと。古いRT・不透明RTを使うと「背景色で上書き」になる。

Pixi バージョン：最近の v8 系（例：v8.13.x）では erase 関連の不具合修正が入っているため、少なくとも v8.12+/8.13.x へ更新を強く推奨。（erase に関する既知 issue が過去にあり、最新版で改善されています）。

改修方針（ルールブックに沿って）

MainController の責務は「イベント仲介・状態管理・初期化・非破壊記録（undo/redo 用の差分ログ）」のみに限定する。消しゴムのペン状態やレイヤーの RenderTexture は「衛星（layer モジュール）」に実装。

衛星（layer）同士は直接参照せず、MainController を介して操作要求（例：eraseAt(x,y)）を行う。

透明消しゴム実装はレイヤー単位で完結させる（各レイヤーが自分の RenderTexture と合成ルールを持つ）。

命名・APIを厳密に：例 Layer.createRenderTexture(width,height), Layer.drawPen(path, options), Layer.eraseAt(path, width) 等。

console.log は最小化し、DEBUG フラグで詳細出力を許す。エラーは error-service を通して表示。

MainController はレイヤー生成を行い、pointer イベント（pointerdown/move/up）を受け、現在のツール（pen / eraser）に従って該当レイヤーにコマンドを送る。

Layer の公開メソッド（例）

layer.init(app, width, height) — RenderTexture 作成、Sprite 生成、透明初期化

layer.drawPen(graphics) — app.renderer.render({ container: graphics, target: layer.rt, clear: false })

layer.erase(graphics) — graphics.blendMode = PIXI.BLEND_MODES.ERASE（要求が満たされない場合、エラーを返す）

layer.clear() 等

Layer（レイヤー）: RenderTexture 作成（重要）
// inside layer initialization (engine-position.js or layer-tool-ui.js depending)
const rt = PIXI.RenderTexture.create({ width: W, height: H, alpha: true });
// Ensure the RT is cleared to fully transparent once:
app.renderer.render({ container: new PIXI.Graphics(), target: rt, clear: true });

消しゴム（eraser）描画ハンドラ（例）

ユーザーの pointer move を受けて eraserGraphics を毎フレーム描く（短命オブジェクト）。描画後に renderer.render(...) で 該当レイヤーの RT に対して描画する。

// create temporary eraser graphics per stroke update
function strokeEraseToLayer(app, layer, pathPoints, width) {
  const g = new PIXI.Graphics();
  // draw stroke (白や何色でも良い — 色はアルファに影響しない)
  g.lineStyle({ width: width, cap: 'round', join: 'round' });
  g.moveTo(pathPoints[0].x, pathPoints[0].y);
  for (let i = 1; i < pathPoints.length; i++) {
    g.lineTo(pathPoints[i].x, pathPoints[i].y);
  }
  // set erase blend mode. Use constant to avoid ambiguous fallback.
  if (PIXI.BLEND_MODES && PIXI.BLEND_MODES.ERASE !== undefined) {
    g.blendMode = PIXI.BLEND_MODES.ERASE;
  } else {
    // ルールに従い暗黙のフォールバックは避ける -> 明示的にエラーで扱う
    throw new Error('PIXI.BLEND_MODES.ERASE is not available. Pixi version incompatible.');
  }

  // Render this graphics into the layer's renderTexture (do NOT clear)
  app.renderer.render({ container: g, target: layer.rt, clear: false });

  // cleanup
  g.destroy({ children: true, texture: true, baseTexture: true });
}


注意点：明示的にフォールバックしない（ルール）ため、ERASE 定義が無い場合は更新を要求（error-service に通す）。

（参考：erase を RT にレンダリングする手順や sample。過去には erase が RT に効かないバグが報告されていたため、RT の透明初期化と clear:false が重要）。

実装例（最小の差分パッチ） — 消しゴムコア関数（そのまま使える単体）
<script>
  // 省略：app = new PIXI.Application({...});
  function initLayer(app, W, H, layerName) {
    const container = new PIXI.Container();
    const rt = PIXI.RenderTexture.create({ width: W, height: H, alpha: true });
    // clear to transparent once
    app.renderer.render({ container: new PIXI.Graphics(), target: rt, clear: true });
    const sprite = new PIXI.Sprite(rt);
    sprite.name = layerName;
    app.stage.addChild(sprite);
    return { rt, sprite, container };
  }

  function eraseStroke(app, layer, pathPoints, width) {
    const g = new PIXI.Graphics();
    g.lineStyle({ width: width, cap: 'round', join: 'round' });
    g.moveTo(pathPoints[0].x, pathPoints[0].y);
    for (let i=1;i<pathPoints.length;i++) g.lineTo(pathPoints[i].x, pathPoints[i].y);

    if (!PIXI.BLEND_MODES || PIXI.BLEND_MODES.ERASE === undefined) {
      throw new Error('ERASE blend mode not available in this Pixi build. Please update to a compatible v8.x.');
    }
    g.blendMode = PIXI.BLEND_MODES.ERASE;

    app.renderer.render({ container: g, target: layer.rt, clear: false });
    g.destroy();
  }
</script>
