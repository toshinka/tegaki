エラーは出ないがキャンバスもツールポップアップも表示されない というものです。これは描画や UI がそもそも初期化されていない、または DOM にマウントされていないことが主な原因です。仕組みとして確認すべき箇所を整理します。

1. DrawingEngine の初期化と canvas DOM マウント
症状

canvas が DOM に追加されない → 描画されない

PIXI.Application は作成されているが表示されない可能性

原因候補

document.getElementById('drawing-canvas') が存在しない（main.html 側で canvas コンテナ未作成）

DrawingEngine.register(mainApi) が呼ばれる前に DOM が読み込まれていない

appendChild が実行されていない（this.app.view が正しくない）

対策

main.html に <div id="drawing-canvas"></div> を確実に配置

DrawingEngine.register(mainApi) を DOMContentLoaded または main 初期化後に呼ぶ

IIFE 内で this.app.view が正しく container に追加されているか確認

2. UIService / ToolManager の初期化順序
症状

ツールポップアップが表示されない

ツール選択やイベントが無効

原因候補

UIService が DOM にツールバーを作成する前に DrawingEngine が必要な container を参照している

ToolManager が UIService に依存している場合、register / attach イベントが順序不正

MainController.notify を通してイベントが伝わっていない

対策

MainController の init で

1. DrawingEngine.register(mainApi)
2. LayerService.register(mainApi)
3. PositionService.register(mainApi)
4. ToolManager.register(mainApi)
5. UIService.register(mainApi)


の順序で初期化

UIService のツールバー作成は DrawingEngine.app や #drawing-canvas が存在することを前提とする

ポップアップや sidebar は UIService.render() で DOM に確実に追加

3. Layer / Position / Transform の依存関係
症状

Space キーで手のマークになるがキャンバス移動できない

レイヤーや Transform 操作が効かない

原因候補

PositionService が DrawingEngine.app.stage を参照していない

LayerService が container を取得できていない

TransformTool が engineBridge を経由していない

対策

PositionService は必ず DrawingEngine.register 以降に初期化

LayerService は ensureLayerContainer(layerId) で確実に container を生成

TransformTool は mainApi.getEngineBridge() から描画用 container にアクセス

EventListener は MainController.notify → ToolManager → Tool に届くことを確認

4. まとめ・チェックリスト
[DOM]
- #drawing-canvas が main.html に存在する
- ツールバー、レイヤーサイドバー用 container も存在

[初期化順序]
- MainController.init:
  1. DrawingEngine.register(mainApi)
  2. LayerService.register(mainApi)
  3. PositionService.register(mainApi)
  4. ToolManager.register(mainApi)
  5. UIService.register(mainApi)

[DrawingEngine]
- this.app = PIXI.Application(...) が正常
- container.appendChild(this.app.view) が DOM に追加される
- engineBridge が mainApi に渡される

[ToolManager / UIService]
- ツールボタンクリックで event が MainController.notify を通る
- UIService.render() で toolbar, layer sidebar が DOM に追加
- ToolManager.getActiveTool() が常に取得可能

[Layer / Position / Transform]
- ensureLayerContainer(layerId) で container が作成
- TransformTool / BrushTool / EraserTool が engineBridge を通じて描画
- Space + ドラッグで PositionService が stage を translate

[イベント]
- MainController.notify(event) → 各衛星が受信可能
- Event payload が正しく構造化されている


💡 結論

キャンバス・ツール表示されないのは、ほぼ DOM マウントと初期化順序 の問題です。

DrawingEngine と UIService の初期化タイミングがずれると canvas が表示されず、ポップアップも無効になります。

まず main.html 側で #drawing-canvas を確実に作ることと、MainController.init の初期化順序を上記の通りに揃えることが最優先です。]]

※キャンバスが確実に表示され、ツールポップアップも機能する初期化順序のサンプル
// star_architecture_main.js (rev9 安全化サンプル)
class MainController {
  constructor() {
    this.mainApi = {
      notify: this.notify.bind(this),
      requestConfirm: this.requestConfirm.bind(this),
      getEngineBridge: () => this.engineBridge,
      getLayerList: () => this.layerService?.getLayerList() || [],
      debugMode: false
    };

    // 衛星のインスタンス保持
    this.drawingEngine = null;
    this.layerService = null;
    this.positionService = null;
    this.toolManager = null;
    this.uiService = null;
    this.historyService = null;
    this.errorService = null;

    document.addEventListener("DOMContentLoaded", () => {
      this.init();
    });
  }

  init() {
    // ------------------------------
    // 1. エラー・履歴系のサービス初期化
    // ------------------------------
    this.errorService = new ErrorService();
    this.errorService.register(this.mainApi);

    this.historyService = new HistoryService();
    this.historyService.register(this.mainApi);

    // ------------------------------
    // 2. 描画エンジン初期化（canvas DOM追加含む）
    // ------------------------------
    this.drawingEngine = new DrawingEngine();
    this.drawingEngine.register(this.mainApi);
    this.engineBridge = this.drawingEngine.getEngineBridge();

    // ------------------------------
    // 3. レイヤー / 位置管理
    // ------------------------------
    this.layerService = new LayerService();
    this.layerService.register(this.mainApi);

    this.positionService = new PositionService();
    this.positionService.register(this.mainApi);

    // ------------------------------
    // 4. ツール系
    // ------------------------------
    this.toolManager = new ToolManager();
    this.toolManager.register(this.mainApi);

    // 必要ツール登録
    this.toolManager.registerTool(new BrushTool());
    this.toolManager.registerTool(new EraserTool());
    this.toolManager.registerTool(new TransformTool());

    // ------------------------------
    // 5. UI系
    // ------------------------------
    this.uiService = new UIService();
    this.uiService.register(this.mainApi);

    // ツールバー / レイヤーサイドバーを描画
    this.uiService.render();

    // ------------------------------
    // 初期化完了ログ
    // ------------------------------
    if (this.mainApi.debugMode) {
      console.log("[MainController] init complete.");
    }
  }

  // ------------------------------
  // MainController API
  // ------------------------------
  notify(event) {
    // 各衛星への通知やログ集約
    if (this.mainApi.debugMode) {
      console.log("[MainController.notify]", event.type, event.payload);
    }
    // 例：ツール操作イベントを layerService へ伝達
    if (event.type.startsWith("tools.")) {
      this.layerService?.handleToolEvent(event);
    }
  }

  requestConfirm(request) {
    // 例：レイヤー順序変更の確定など
    this.historyService?.record(request);
  }
}

    this.app = new PIXI.Application({

    this.container.appendChild(this.app.view);

    this.engineBridge = {

      clearLayer: (layerId) => { /* 実装 */ },
      takeSnapshot: (layerId) => { /* 実装 */ }

  register(mainApi) {
    mainApi.engineBridge = this.engineBridge;

}
