要点（短く）

症状（あなたの説明）

パネル内でドラッグしているのに場所が入れ替わったり、レイヤーを押すと入れ替わる（クリックで入れ替わる）。

背景レイヤーが勝手に一番上に行くことがある。

主要な原因候補（優先度高→低）
A. Sortable のイベント（click と drag）の衝突：クリックイベントがドラッグ中/直後に誤発火して順序更新を起こしている。
B. UI（DOM）順序と内部データモデル（layers 配列）／Pixi の z-order（stage.children の順序）が逆転して同期ミスしている（DOM上の上⇄下 と PIXI 上の上⇄下 の対応が逆のことが多い）。
C. 順序入れ替え処理で index を oldIndex/newIndex として扱う箇所の誤計算（特に配列 splice 後のインデックス参照ミス）。
D. レイヤー識別を配列インデックスでしか扱っておらず、クリック・移動でインデックスが変わると「アクティブレイヤー」が別のレイヤーを指してしまう（IDベースで追跡していない）。
E. Sortable 初期化時のオプション不足（handle 指定・distance/delay が無い）で、短いクリックでドラッグと判定される、またはその逆。

改修方針（設計原則）

DOM順序と Pixi の z-order を**単一のソースオブトゥルース（layers 配列）**で決める。DOMは表示、Pixiは描画、両方を layers 配列で再構築する。直接 DOM だけを信じない。

レイヤーは固有 id を持たせ、アクティブ状態は activeLayerId で管理する（インデックスではなく ID で追跡）。

Sortable の設定は「ハンドル指定」「drag 判定の閾値（distance または delay）」「onStart/onEnd で model を更新」する。

小さなパッチで追加できるよう、syncPixiLayerOrder() と onSortableEnd() を独立ファイル・関数化する。

具体的パッチ（貼るだけで動く小関数群）

以下は layer-sortable-patch.js というファイルを作って <script src="layer-sortable-patch.js"></script> を main.html のレイヤーパネルを描画するスクリプトの直後（か DrawingEngine の前）に読み込む想定のパッチです。Claude にコピペしてもらいやすいよう、コメント多めで単独で読めるようにしています。

/* layer-sortable-patch.js
   - 目的: Sortable のドラッグ/クリック競合を防ぎ、
     DOM順序 -> layers[] -> PIXI順序 の同期を単純化する。
   - 使い方: main.html で layersListEl (レイヤーのUL/CONTAINER) と
     global の `layers` 配列・`app` (PIXI.Application)・`renderLayerPanel` が存在する想定。
*/

(function(){
  // --- 設定: 必要に応じて変えてください ---
  const HANDLE_SELECTOR = '.layer-handle'; // レイヤー行のドラッグ用ハンドル要素
  const DRAG_DELAY_MS = 100;               // クリックとドラッグ判定の分離 (delay 方式)
  // ----------------------------------------

  if(typeof Sortable === 'undefined'){
    console.warn('Sortable is not loaded; layer-sortable-patch.js skipped.');
    return;
  }

  // layers: [{id, name, displayObject, ...}, ...] を前提とする
  // activeLayerId: 現在アクティブなレイヤーの id を保持する想定
  // layersListEl: DOM のレイヤーリスト（Sortable を適用する要素）

  // ---- ヘルパー: Pixi と DOM の順序を layers[] に従わせる ----
  function syncPixiLayerOrder(layers) {
    // NOTE: UI 上で上に表示される要素を layers[0] とした時の変換例。
    // PIXI の stage.children は 0 が背面、最後が最前面のことが多いので
    // layers の並び順を逆順にして setChildIndex する。
    if(!window.app || !app.stage) return;
    const stage = app.stage;
    const n = layers.length;
    for (let i = 0; i < n; i++) {
      const layer = layers[i];
      const desiredIndex = n - 1 - i; // reverse mapping
      try {
        if (layer && layer.displayObject && layer.displayObject.parent === stage) {
          stage.setChildIndex(layer.displayObject, desiredIndex);
        }
      } catch (e) {
        // setChildIndex は例外出るので安全に
        // （存在しなければ addChildAt 等で挿入する実装にする）
        console.warn('syncPixiLayerOrder: setChildIndex failed', e);
      }
    }
  }

  // ---- ヘルパー: レイヤーIDで active を復元 ----
  function restoreActiveLayerById(layers, activeId) {
    const found = layers.find(l=>l.id === activeId);
    if(found){
      window.activeLayer = found;
    } else if(layers.length){
      window.activeLayer = layers[0]; // フォールバック
      window.activeLayerId = layers[0].id;
    } else {
      window.activeLayer = null;
      window.activeLayerId = null;
    }
    // UI 側の選択表示更新関数があれば呼ぶ（renderLayerPanel 等）
    if(typeof renderLayerPanel === 'function') renderLayerPanel();
  }

  // ---- Sortable 初期化 ----
  function initLayerSortable(layersListEl) {
    if(!layersListEl) {
      console.warn('initLayerSortable: layersListEl not found');
      return;
    }

    const sortable = Sortable.create(layersListEl, {
      handle: HANDLE_SELECTOR,
      animation: 150,
      ghostClass: 'sortable-ghost',
      delay: DRAG_DELAY_MS,   // click と drag の分離（短いクリックで並び替わらない）
      delayOnTouchOnly: false,
      preventOnFilter: true,
      onStart(evt){
        // クリックとドラッグの競合を防ぐためのフラグ
        sortable._isDragging = true;
        // optional: visual state
      },
      onEnd(evt){
        sortable._isDragging = false;
        // oldIndex/newIndex は DOM のインデックス（0 ベース）
        const oldIndex = evt.oldIndex;
        const newIndex = evt.newIndex;
        if(typeof oldIndex !== 'number' || typeof newIndex !== 'number') return;

        // 安全に layers 配列を移動する（immutable に近い意識）
        const moved = window.layers.splice(oldIndex, 1)[0];
        window.layers.splice(newIndex, 0, moved);

        // 同期: Pixi の z-order を layers[] に合わせる
        syncPixiLayerOrder(window.layers);

        // アクティブレイヤーを ID で復元（移動で参照がぶれない）
        if(window.activeLayerId) {
          restoreActiveLayerById(window.layers, window.activeLayerId);
        } else {
          // ないなら新しいインデックスを active にする（必要なら）
          window.activeLayer = window.layers[newIndex];
          window.activeLayerId = window.activeLayer && window.activeLayer.id;
          if(typeof renderLayerPanel === 'function') renderLayerPanel();
        }
      },
      onChoose(evt){
        // drag 開始前にクリックで active が切り替わるのを防ぐならここで一時記録
        // evt.item.dataset.layerId などを参照できる想定
        sortable._chosenId = evt.item && evt.item.dataset && evt.item.dataset.layerId;
      }
    });

    // クリックとドラッグの競合を完全に避けたい場合
    // レイヤー行の click ハンドラ内で sortable._isDragging をチェックすること。
    // 例:
    layersListEl.addEventListener('click', function(e){
      // もしドラッグ直後の click イベントなら無視する
      if(sortable._isDragging) {
        e.stopImmediatePropagation();
        return;
      }
      // それ以外は既存のクリック処理に任せる
    }, true);

    return sortable;
  }

  // --- エクスポート (global に公開して差し替えやすく) ---
  window.layerSortablePatch = {
    init: initLayerSortable,
    syncPixiLayerOrder,
    restoreActiveLayerById
  };

})();


設置手順（簡潔）

上の内容を layer-sortable-patch.js として保存。

main.html のレイヤーパネルを描画するスクリプト（既存の layer-* スクリプトの直後）に <script src="layer-sortable-patch.js"></script> を追加。

ページ読み込み後に layerSortablePatch.init(document.querySelector('#layersListElement')) を呼ぶ。※#layersListElement は実際のレイヤーUL/CONTAINER の selector に置き換える。

DrawingEngine.js 側で layers 配列・app(PIXI.Application)・renderLayerPanel・activeLayerId を使っている想定で互換性を保つ（必要なら small adapter を追加）。

よくあるミスと簡単チェックリスト（実行前に確認）

 レイヤー各行に data-layer-id（ユニークID）が付与されているか？（無ければ付ける）

 Sortable の handle が指定されているか？(引っ張る場所を限定して誤操作を減らす)

 DOM の「上→下」と Pixi の z-order の「上→下」が逆と思われる場合は syncPixiLayerOrder() 内の reverse ロジックを反転して試す。

 アクティブレイヤーは ID で記憶しているか（インデックスで覚えていると移動で変わる）。

 onEnd 内で layers を更新した直後に UI を再描画しているか（renderLayerPanel 等）。

テスト手順（再現と確認）

レイヤーをドラッグして順序が入れ替わるか確認（パネル内表示が期待通りか）。

ドラッグ直後にクリックしてアクティブを切り替えても順序が狂わないか。

背景（最背面）のレイヤーをドラッグしても勝手に最前面に飛ばされないか。

ペンで描画 → レイヤー重ね替え → 表示が崩れないか（PIXI の表示確認）。

ブラウザのコンソールで layers 配列・activeLayerId をログして期待通りか確認。

追加の小さな改善（推奨）

onEnd で変更前後の layers 配列を console.table() して差分デバッグできるようにしておく（開発時のみ）。

Sortable の fallbackOnBody: true を入れるとドラッグ中の ghost の挙動が安定するケースがあります。

もし DrawingEngine.js がモジュール的に大きくまとまっているなら、上の syncPixiLayerOrder をそこに移植して layers 更新の single source にする（ただし Claude に差し替えしやすい形で）。

なぜこの案で直る可能性が高いのか（短く根拠）

「クリックで入れ替わる」という症状はほぼ常に「ドラッグとクリックの判定競合」か「onClick ハンドラが DOM 再構築を行っている」ことが原因です。Sortable の delay（または distance）を入れることで誤トリガーをほぼ排除できます。

「背景が上に行く」問題は DOM と Pixi の順序概念（どちらが前面を示すか）が混同されているケースが典型で、明示的に layers[] -> stage.setChildIndex(...) の同期を行えば解消します。
