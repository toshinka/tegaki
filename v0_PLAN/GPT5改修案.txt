原因は UI 側 (SortableJS のインデックス) とエンジン側 (ImprovedLayerManager) のレイヤー配列順の解釈がズレていることです。

問題点

1.エラーメッセージ

console.error('[LayerManager] Layer ID mismatch in reorder');

ImprovedLayerManager.reorderLayer() のチェックで

if (!layer || layer.id !== layerId) { ... }


に引っかかっています。これは fromIndex にあるレイヤーの id が、UI から渡された layerId と一致していないため。


2.インデックス不一致

SortableJS (onEnd の evt.oldIndex / evt.newIndex) は DOM 上の順序を返します。

DOM では通常「一番上に見えるレイヤー」が index=0 ですが、

ImprovedLayerManager.layers.items では「背景(下層)が index=0」「上に行くほど大きい index」になるように管理されています。

つまり UI と内部配列の並びが逆転しているため、layerId と fromIndex が食い違い、Layer ID mismatch が出ています。結果、並びも反映されず「上に置いたのに下にある」状態になる。


改修策

A. インデックスを反転して渡す

onEnd: (evt) => {
    this.dragInProgress = false;
    layerList.classList.remove('drag-disabled');

    if (evt.oldIndex !== evt.newIndex) {
        const layerId = parseInt(evt.item.dataset.layerId);

        const total = layerList.children.length;
        // DOMインデックスを内部インデックスに変換
        const fromIndex = total - 1 - evt.oldIndex;
        const toIndex   = total - 1 - evt.newIndex;

        setTimeout(() => {
            window.DrawingEngineAPI?.reorderLayer(layerId, fromIndex, toIndex);
        }, 50);
    }
}


B. または updateLayerUI() で DOM を逆順に構築する

内部配列を reverse() してから描画すれば、Sortable のインデックスと一致する。

ただし、この場合 reorderLayer の引数はそのままでよい。

おすすめ

**A の「インデックス反転」**の方が影響範囲が小さく安全。

UI はそのまま「上から下へ並んでいる」見た目を維持できる。


✅ まとめ:
エラーの原因は UI (DOM) と内部配列の並び方向の不一致。
対策は Sortable の oldIndex/newIndex を内部配列用に反転して渡すこと。

この修正を UI 側 initializeLayerSortable の onEnd に入れれば改善できそうです。
