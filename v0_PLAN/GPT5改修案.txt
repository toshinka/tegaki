🔎 問題の整理

レイヤーの順序が逆転する

上にドラッグしたのに最下層へ行ったり、その逆がある。

ただし毎回ではなく、時々は正しく動く。

→ PixiJS の描画スタック（子要素配列）と SortableJS の DOM 順序が逆方向で管理されている可能性が高い。
（PixiJS は zIndex / children の昇順で描画、DOM は上にあるほどリストの上に表示、これが食い違いやすい）

破線の色が茶色・青に変わる

これは SortableJS の ghostClass / chosenClass / dragClass / fallbackClass のスタイル切り替えによるもの。

「どの要素の上に入るか」判定が失敗しているときに違う色になっている可能性がある（DOM の上下判定、invertSwap オプションの影響）。

ペンでは選択できるがドラッグできない

タッチ／ペン入力が pointerdown → dragstart に変換されていない。

SortableJS は touchstart は扱うが、ペン入力（pointerType=pen） を明示的に許可していないケースがある。

→ touchStartThreshold や forceFallback: true を設定しないとペン入力でドラッグできない。

🛠 原因の詳細
レイヤーの順序逆転

DOM: 上にある要素ほど「前に表示される」ように見える

PixiJS: stage.addChild(layer) → children 配列の後ろが「前面」になる

つまり DOM の順序と PixiJS の描画順序が逆。

ソート後に PixiJS 側で zIndex を再設定してやらないとズレる。

破線の色

SortableJS が insertBefore と insertAfter を内部で切り替える時に、候補位置を示すスタイルが変わる。

これは仕様上問題ないが、DOM と Pixi の対応が狂っていると「青のとき正しく動いて、茶色のとき逆に行く」という風に見える。

ペン操作

SortableJS のデフォルトは touch を mouse に変換して扱うが、pointerType: pen は考慮されない。

Windows + Chrome のペン入力だと pointerdown が発火するが dragstart に繋がらないため動かない。

✅ 改修方法
1. PixiJS のレイヤー順を DOM と同期させる

SortableJS の onEnd に以下を入れる：

onEnd: function (evt) {
  const layerElements = Array.from(document.querySelectorAll(".layer-item"));
  // 上にあるDOMほど前面になるように逆順にPixiへ適用
  layerElements.reverse().forEach((el, index) => {
    const layerId = el.dataset.layerId;
    const layer = layerMap[layerId];
    layer.zIndex = index;
  });
  app.stage.sortChildren();
}


こうすることで DOM と Pixi の順序が常に一致する。

2. 破線の色問題

色自体は ghostClass / chosenClass の CSS で制御されているので 無害。

「色によって法則が変わる」と感じたのは、Pixi と DOM の順序が一致していないため。

上記の修正で自然に解決する。

3. ペン入力を有効化

SortableJS 初期化時に以下を追加：

Sortable.create(layerPanel, {
  animation: 150,
  forceFallback: true,       // ペンやタッチを強制的に扱う
  fallbackTolerance: 3,      // 小さな動きでもドラッグ開始
  touchStartThreshold: 1,    // ペンでもドラッグ扱いに
  onEnd: function(evt) { /* 上記のzIndex処理 */ }
});


これでマウス・タッチ・ペン全対応になる。

📌 まとめ

順序逆転の原因 → DOM と Pixi のレイヤー管理方向が逆。
→ onEnd で DOM 順序を Pixi の zIndex に正しく反映させれば解決。

破線の色 → SortableJS の内部状態。Pixi とのズレが原因で「挙動が逆に見える」。

ペン非対応 → forceFallback: true と touchStartThreshold の設定で対応可能。
