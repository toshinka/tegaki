■原因究明
エラーログ：
futaba_main.html:1326 💥 Application startup failed: ReferenceError: fromIndex is not defined
    at LayerManager.createLayer (futaba_main.html:667:44)

調査結果：
- `LayerManager.createLayer()` 内で `fromIndex` という変数を参照しているが、事前に定義されていない。
- 以前の一体型ファイルではスコープ上に存在していたか、もしくは引数で受け取っていた可能性がある。
- 分割作業の過程で `fromIndex` が未定義になり、`createLayer` 実行時に落ちている。

■改修方針
1. ルールブック（V8_rulebook）に従い、冗長化を避けつつ「レイヤー生成の入口関数」でのパラメータ明示を徹底する。
2. `createLayer(fromIndex, options)` のように引数として受け取り、内部で使用する。
3. 呼び出し元（`LayerManager.initialize` や `AppController.initialize`）で、必要に応じてデフォルト値（例：`0`）を渡す。
4. `fromIndex` の役割が「基準レイヤー位置」であれば、座標系ガイドに従って `0` 始まりのインデックス管理を保証する。

■改修計画
- futaba_tools.js 側（LayerManager 定義部）
  - `createLayer` のシグネチャを修正：`createLayer(fromIndex = 0, options = {})`
  - 内部で未定義アクセスしないように `fromIndex` のデフォルト値を保証。
- futaba_main.html 側（initialize 呼び出し部）
  - `this.layerManager.createLayer();` を `this.layerManager.createLayer(0);` のように修正。
- 全体検証
  - レイヤー追加／削除／初期化が正しく動作するかを通しテスト。
  - キャンバス移動・座標系処理と競合がないか確認。

■最小改修例
futaba_tools.js (LayerManager 内部)
```js
createLayer(fromIndex = 0, options = {}) {
    // fromIndex を基準に新規レイヤーを作成
    const layer = {
        id: this.layers.length,
        index: fromIndex,
        ...options
    };
    this.layers.push(layer);
    return layer;
}


futaba_main.html (AppController.initialize 内呼び出し)

this.layerManager.createLayer(0, { name: "base" });


■まとめ
原因は「分割時に fromIndex が未定義化」したこと。
対応として「createLayer の引数化＋デフォルト値保証」を行い、呼び出し側で初期値を明示することで解決。
