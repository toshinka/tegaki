「コンソールにエラーは出ないが、UI が反応しない／描けない／パンできない」系は イベントの伝達経路が切れている（または期待する要素やフラグが未設定） ことがほとんどです。

結論（要点まとめ）

発生している問題と主要原因（確度高 → 低）

1.ツールボタンを押してもポップアップしない
→ UI 作成 と ToolManager の「DOM セレクタ／イベントの期待」が噛み合っていない。
対策：UI 側に適切な id/class を付ける／または ToolManager を DOM 依存から外して mainApi 経由で受け取る（推奨）。

2.レイヤーサイドが表示されない
→ 初期レイヤーが作られていない、または UI がレイヤー一覧を再描画していない。
対策：Main の初期化で デフォルトレイヤー作成＋UI に refreshLayers() 呼び出しを行う。

3.Space のハンドルでカーソルだけ替わるがパンしない
→ Space 状態フラグはセットされているが、DrawingEngine が canvas pointer イベントでパン処理をしていない（もしくはパンイベントが main に伝搬していない）。
対策：DrawingEngine に pointerdown/move/up を実装し、Space 時は isPanning=true にして containers.world を移動する。

4.ペンで描けない
→ pointer イベントが ToolManager → activeTool へ届いていない（DrawingEngine が pointer イベントを mainApi.notify に流していない、もしくは ToolManager が pointer イベントを start/move/end に変換していない）。
対策：DrawingEngine が pointer イベントを mainApi.notify({type:'pointer.down'|'pointer.move'|'pointer.up', payload}) で流し、ToolManager がそれを受けて active tool の start/move/end を呼ぶ。

優先適用順（短時間で動くように）

MainController：mainApi に getAppState()（space 状態）と requestConfirm を実装。起動時にデフォルトレイヤーを作成。

DrawingEngine：キャンバスに pointer イベントを追加 → ポインタを main に通知。Space時はパン処理を実装。

ToolManager：handleEvent を拡張して pointer.down/move/up をツールの start/move/end にマッピングする（DOM クエリではなく mainApi 経由ルートを確実にする）。

UIManager/UIService：ツールボタンに id と class='tool-button' をつける／refreshLayers() を公開して Main が呼べるようにする。

BrushTool：_isSpacePressed() を mainApi.getAppState() 参照にする（requestConfirm 依存をやめる）。また一時描画は engineBridge を通しているので、pointer経路が直れば描画が動くはず。

差分コード（貼り付け／置換用）

下にファイル別に最小差分コードを出します。各コードは 既存ファイルの該当箇所に置き換えるか、関数追加 してください。行番号が違っても、関数の中身を差し替え／追加すればOK。

// -------  MainController._initGlobals or constructor area -------

2) drawing-engine.js — _createPixiApp() の .then(...) 内に以下を追加してください

（目的：canvas pointer handlers を追加し、space 時は panning を行い、それ以外は mainApi に pointer イベントを通知）

// after container.appendChild(this.app.view) or after app created & appended...
// ensure canvas receives pointer events and no default touch action
this.app.view.style.touchAction = 'none';
this._isPanning = false;
this._lastPan = null;

// helper: screen -> local canvas coords
const getCanvasPos = (e) => {
  const rect = this.app.view.getBoundingClientRect();
  return { x: e.clientX - rect.left, y: e.clientY - rect.top };
};

// pointerdown
this.app.view.addEventListener('pointerdown', (e) => {
  const pos = getCanvasPos(e);
  // if main says space pressed -> start panning
  const spaceState = this.mainApi && this.mainApi.getAppState ? this.mainApi.getAppState().spacePressed : false;
  if(spaceState) {
    this._isPanning = true;
    this._lastPan = { x: e.clientX, y: e.clientY };
    this.app.view.setPointerCapture(e.pointerId);
    return;
  }
  // otherwise forward to main as pointer.down
  if(this.mainApi && this.mainApi.notify) {
    this.mainApi.notify({ type: 'pointer.down', payload: { x: pos.x, y: pos.y, pointerType: e.pointerType, pressure: e.pressure } });
  }
});

// pointermove
this.app.view.addEventListener('pointermove', (e) => {
  const pos = getCanvasPos(e);
  if(this._isPanning && this._lastPan) {
    const dx = e.clientX - this._lastPan.x;
    const dy = e.clientY - this._lastPan.y;
    // move world container (UI container stays fixed)
    if(this.containers && this.containers.world) {
      this.containers.world.x += dx;
      this.containers.world.y += dy;
    }
    this._lastPan = { x: e.clientX, y: e.clientY };
    return;
  }
  if(this.mainApi && this.mainApi.notify) {
    this.mainApi.notify({ type: 'pointer.move', payload: { x: pos.x, y: pos.y, pointerType: e.pointerType, pressure: e.pressure } });
  }
});

// pointerup / cancel
this.app.view.addEventListener('pointerup', (e) => {
  if(this._isPanning) {
    this._isPanning = false;
    this._lastPan = null;
    this.app.view.releasePointerCapture(e.pointerId);
    return;
  }
  const pos = getCanvasPos(e);
  if(this.mainApi && this.mainApi.notify) {
    this.mainApi.notify({ type: 'pointer.up', payload: { x: pos.x, y: pos.y, pointerType: e.pointerType, pressure: e.pressure } });
  }
});
this.app.view.addEventListener('pointercancel', (e) => {
  if(this._isPanning) {
    this._isPanning = false;
    this._lastPan = null;
  }
  if(this.mainApi && this.mainApi.notify) {
    this.mainApi.notify({ type: 'pointer.up', payload: { x: 0, y: 0 } });
  }
});


注意：この実装は “world コンテナのみパンさせる” 方針です（UI コンテナを動かさない）。もし UI も一緒にパンしたいなら this.app.stage を移動してください。

3) tool-manager.js — handleEvent を pointer 系に対応させる（既存の handleEvent と置換）

（目的：main → ToolManager → active tool の start/move/end を確実に実行）

handleEvent(event) {
  if(!event || !event.type) return;
  const tool = this.getActiveTool();
  switch(event.type) {
    case 'pointer.down':
      if(tool && tool.start) tool.start(event.payload);
      break;
    case 'pointer.move':
      if(tool && tool.move) tool.move(event.payload);
      break;
    case 'pointer.up':
      if(tool && tool.end) tool.end(event.payload);
      break;
    default:
      // fallback: if tool implements handleEvent, forward
      if(tool && tool.handleEvent) tool.handleEvent(event);
      else if(global.MyApp.debug) console.log('[ToolManager] unhandled event', event.type);
  }
}


既存の DOM クエリ _setupUIHandlers() は残しても良いですが「DOM 依存は弱める」方が安定します。UI 側は mainApi.notify を呼べば良い設計にするとさらに堅牢です。

_isSpacePressed() {
