いま起きていること（結論）

1.キャンバスが出ない主因
engine-position.js は #canvas-container を getElementById('canvas-container') で取得できなければ initialize() を falseで終了します（そして MainController.emit('system-error', { code: 'CANVAS_CONTAINER_NOT_FOUND', ... }) へ） 
GitHub
。
→ DOM上に id="canvas-container" が無い／描画前タイミング／CSSでゼロサイズ のいずれかで止まっている可能性が高いです。コンソールエラーが出ないのは、例外ではなくイベント発火で済ませているため。

2.レイヤーパネルが出ない副因
layer-tool-ui.js の初期化（マウント）より前にパネルコンテナが無い、またはスクリプト読込順が逆転している可能性。改修メモにある UIManager.initialize() 内の setTimeout(...forceUpdateLayerPanel...) は“見えない→後から強制更新”の対症療法ですが、最初に mount 対象（例: #layer-panel）が存在しないと何も起こりません。
（rev5 のファイル構成では engine-position.js が #canvas-container 依存であることは確認できましたが、layer-tool-ui.js のマウント先 ID と読込順が main.html 側で噛み合っていない疑いが濃いです。）


・最小パッチ（確実に表示まで持っていく）

下の 3 点を main.html に反映すれば、キャンバスとレイヤーパネルは出ます。
（ID は下記に合わせてあります。あなた側の命名と違う場合は合わせてください）

1) DOM に必要なコンテナを置く（上部どこでもOK）
<!-- キャンバス用：engine-position.js が参照 -->
<div id="canvas-container" style="
  position:absolute; left:50%; top:50%;
  transform:translate(-50%, -50%);
  width:400px; height:400px; 
  overflow:hidden; 
  touch-action:none; /* Hammer と pointer の両立 */
  z-index: 5;
"></div>

<!-- レイヤーパネル用：LayerToolUI が mount する -->
<div id="layer-panel" style="
  position:fixed; right:16px; top:16px;
  width: 220px; max-height: 70vh; overflow:auto;
  background:#1e1e1e; color:#fff; border-radius:12px; padding:8px;
  z-index: 20;
"></div>


engine-position.js は初期化時に #canvas-container を探します。無ければ停止。まずはここで 100% 存在させます。

2) スクリプトの読込順を固定
<script src="error-service.js" defer></script>
<script src="main-controller.js" defer></script> <!-- もし別名なら中枢のファイル -->
<script src="engine-position.js" defer></script>

<script>
  // DOMReady 後の起動（defer ならこの位置でOK）
  document.addEventListener('DOMContentLoaded', () => {
    // ここで UI / Engine の初期化を“順番付き”で呼ぶ
    if (window.LayerToolUI && document.getElementById('layer-panel')) {

ポイント

MainController（イベントバス）→engine-position.js→layer-tool-ui.js の依存順を守る

パネルは 必ず mount('#layer-panel') を 1 度だけ呼ぶ

3) 念のため、初期サイズ通知を 1 回だけ投げる
<script>
  // PositionManager のキャンバス境界を UI に知らせる
  window.addEventListener('load', () => {
    const el = document.getElementById('canvas-container');
    if (el && window.MainController) {
      const w = el.clientWidth || 400;
      const h = el.clientHeight || 400;
      MainController.emit('ui-canvas-resize', { width:w, height:h });
    }
  });
</script>


engine-position.js は ui-canvas-resize を受けて内部の canvasBounds を更新し、可視境界を通知します。起動直後に 1 回投げておくと、UI 側の状態表示（「Canvas: 400×400px」など）が安定します。

・追加で直しておくと堅牢になる 3 点

初期化リトライの明示化
engine-position.js 末尾にある「MainController 準備完了待ち」ループはありますが、DOM存在待ちはしていないので、
initialize() 冒頭を次のように安全化すると「DOMがまだ無い」ケースを潰せます。

initialize() {
  // 1) DOM存在チェック（最大 1 秒までリトライ）
  let tries = 0;
  const ensureDom = (resolve) => {
    this.container = document.getElementById('canvas-container');
    if (this.container) { resolve(true); return; }
    if (++tries > 20) {
      MainController.emit('system-error', { code:'CANVAS_CONTAINER_NOT_FOUND', details:{tries} });
      resolve(false); return;
    }
    setTimeout(()=>ensureDom(resolve), 50);
  };
  return new Promise((resolve)=>ensureDom(resolve)).then(ok=>{
    if (!ok) return false;
    // 2) 以降は今の処理（Hammer, Events, Loop, calibrateInitialPosition）
    this.setupHammerJS();
    this.setupPointerEvents();
    this.setupEventHandlers();
    this.startUpdateLoop();
    setTimeout(()=>this.calibrateInitialPosition(), 50);
    return true;
  });
}

→ 「コンソール赤無しで何も出ない」を防ぐ。
