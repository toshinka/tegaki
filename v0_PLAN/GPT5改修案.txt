根本原因（高確率）

1.DrawingEngine のコンストラクタが重い初期化（this.thumbnailManager.clearCache() など）を行っており、thumbnailManager がまだ生成されていないタイミングで呼ばれている。ファイル内の初期化順序の誤りです（参照ファイル確認済み）。 
GitHub

2.DrawingEngine.initialize() が途中で途切れている／非同期処理を待たずに次の処理へ進んでいるため、他のモジュール（サムネイル、レイヤー、ショートカット登録等）が未初期化のまま使われる。

3.API の初期化フロー（コンストラクタで重い処理 → すぐに外部に公開）は設計ミス。コンストラクタは「軽く」して、実際の作業は async initialize() で順序良く行うべきです（ルールブックの方針とも整合）。 


以下、具体的な改修案とそのまま差し替え可能なコード（最小限）を提示します。これで 1〜3 を同時に解消できます。

1) 原則（設計ルール）

コンストラクタではフィールドの初期化（null や bind）だけにする。重い処理（PIXI.Application 作成、アセット読み込み、レンダリング／サムネイル生成等）は async initialize() で行う。

サムネイル生成は PIXI.RenderTexture を使って Ticker で更新する（ルールブックの推奨）。 
GitHub

2) 変更ポイント（要修正）

DrawingEngine.constructor から this.thumbnailManager.clearCache() を削除（またはガード + 遅延実行へ移動）。

DrawingEngine.initialize() を完全な async フローに実装： renderer/app → containers → layerSystem → thumbnailManager（ここで clearCache()）→ イベント/ショートカット登録 → ready フラグ/resolve Promise。

外部から new DrawingEngine() を作るだけで即使用するコードがあれば、await engine.initialize() または engine.readyPromise を使うよう呼びかける。

3) 差し替えサンプルコード（抜粋 — tegaki_canvas_fixed.html 内の該当クラスをこれに差し替えてください）
// --- DrawingEngine (修正版) ---
class DrawingEngine {
  constructor(options = {}) {
    // 軽い初期化のみ。重い処理は initialize() で。
    this.options = options;
    this.app = null;
    this.canvasContainer = null;
    this.layerSystem = null;
    this.thumbnailManager = null;
    this.history = null;
    this.ready = false;

    // キーボードハンドラを later に解除できるように束縛しておく
    this._boundOnKeyDown = this._onKeyDown.bind(this);

    // Promise ベースで初期化の完了を通知できるようにしておく
    this._readyResolve = null;
    this.readyPromise = new Promise((resolve) => { this._readyResolve = resolve; });
  }

  // 外部から await engine.initialize() する想定
  async initialize({ mountEl = document.body, width = 800, height = 600 } = {}) {
    // 1) PIXI アプリ生成（非同期なアセットはここで await する）
    this.app = new PIXI.Application({
      width, height, backgroundAlpha: 0, antialias: true
    });
    // mountEl が DOM にあることを保証する（呼び出し側は DOMContentLoaded 後に）
    mountEl.appendChild(this.app.view);

    // 2) containers の作成（canvasContainer を中心に）
    this.canvasContainer = new PIXI.Container();
    this.app.stage.addChild(this.canvasContainer);

    // 3) layerSystem / managers の初期化（必要なら await）
    this.layerSystem = new LayerSystem(this.app, this.canvasContainer);

    // 4) サムネイルマネージャの初期化（ここで clearCache を安全に呼べる）
    this.thumbnailManager = new ThumbnailManager(this.app, this.layerSystem, { thumbSize: 64 });
    // safe: thumbnailManager が確実に存在するので clearCache を行う
    this.thumbnailManager.clearCache();

    // 5) ショートカット等の UI イベント登録
    this._initShortcuts();
    this._attachPointerEvents();

    // 6) ループや Ticker 登録（必要なら）
    this.app.ticker.add(() => { /* tick-based updates */ });

    // 7) ready フラグと Promise の解決
    this.ready = true;
    if (this._readyResolve) this._readyResolve(this);
    return this;
  }

  _initShortcuts() {
    window.addEventListener('keydown', this._boundOnKeyDown, { passive: false });
  }

  _onKeyDown(e) {
    const ctrl = e.ctrlKey || e.metaKey;
    if (ctrl && e.key.toLowerCase() === 'z') { e.preventDefault(); this._undo(); return; }
    if (ctrl && (e.key.toLowerCase() === 'y' || (e.shiftKey && e.key.toLowerCase() === 'z'))) {
      e.preventDefault(); this._redo(); return;
    }
    // ここにツール切替など
  }

  destroy() {
    window.removeEventListener('keydown', this._boundOnKeyDown, { passive: false });
    this.app?.destroy(true);
    this.ready = false;
  }

  // undo/redo 等のダミー
  _undo() { /* ... */ }
  _redo() { /* ... */ }

  // pointer イベントの attach（例）
  _attachPointerEvents() {
    // 例: this.app.view の pointerdown を登録する等
  }
}

// --- ThumbnailManager（最小実装例）---
class ThumbnailManager {
  constructor(app, layerSystem, opts = {}) {
    this.app = app;
    this.layerSystem = layerSystem;
    this.thumbSize = opts.thumbSize || 64;
    this.cache = new Map();
    this.needsUpdate = new Set();
  }

  clearCache() {
    this.cache.forEach((tex) => {
      try { tex.destroy(true); } catch(e) { /* ignore */ }
    });
    this.cache.clear();
  }

  scheduleUpdate(layerId) {
    this.needsUpdate.add(layerId);
    // Ticker 内でバッチ更新を行う想定
    this.app.ticker.addOnce(() => this._updateScheduled());
  }

  _updateScheduled() {
    for (const layerId of this.needsUpdate) {
      this.updateLayerThumb(layerId);
    }
    this.needsUpdate.clear();
  }

  updateLayerThumb(layerId) {
    const layer = this.layerSystem.getLayer(layerId);
    if (!layer) return;
    const rt = PIXI.RenderTexture.create({ width: this.thumbSize, height: this.thumbSize });
    // app.renderer.render を使って高速にサムネイルを作る（ルールブック推奨）
    this.app.renderer.render(layer, { renderTexture: rt, clear: true });
    // 既存テクスチャは破棄して置き換え
    const prev = this.cache.get(layerId);
    if (prev && prev !== rt) { try { prev.destroy(true); } catch(e){ } }
    this.cache.set(layerId, rt);
  }
}

4) API 初期化フローの改善（呼出し側）

従来（NG）

const engine = new DrawingEngine(); // constructor が即座に重い処理をする
// すぐに engine を使おうとするコードがエラーになる -> thumbnailManager undefined


推奨（OK）

const engine = new DrawingEngine();
await engine.initialize({ mountEl: document.getElementById('canvasRoot'), width: 1024, height: 768 });
// ここから engine を安全に使える


もし既存コードが const engine = new DrawingEngine(); のみで他コードが即アクセスする設計なら、window.drawingEngineReady = engine.readyPromise; のように Promise を公開して await window.drawingEngineReady を呼ぶようにする移行が安全です。

5) 追加のポイント（見落としがちな原因）

スクリプトの実行タイミング：<script> が head にあって DOM（mountEl）がまだ無い場合にエラーになる → DOMContentLoaded 後に initialize() を呼ぶか、スクリプトを </body> の直前に置く。

thumbnailManager クラス定義がファイルの後半にあり、コンストラクタで new ThumbnailManager() を呼ぶと定義前の参照になるケース（クラス宣言は巻き上げられない）。必ず new より前にクラス定義が存在するかモジュール化する。

非同期なアセット読み込み（PIXI.Assets.load）を initialize() 内で await する。
（上記は設計ルール／サムネイル最適化とも整合します）。 


6) デバッグチェックリスト（素早く確認できる順）

tegaki_canvas_fixed.html 内で clearCache( を grep して呼び出し箇所を特定。

DrawingEngine の constructor 内にある処理を軽くする（console.log を置いて何が undefined か確認）。

initialize() の最終行まで到達しているか console.log('init done') を入れて確認。

DOM のマウント要素が存在するか確認（document.getElementById('...') が null になっていないか）。

コンソールの最初のエラー行番号をたどる（最初の例外が本質原因であることが多い）。

7) 最後に（まとめ）

まずは this.thumbnailManager.clearCache() をコンストラクタから取り除き、initialize() の中で this.thumbnailManager = new ThumbnailManager(...) の直後に呼ぶ（上のサンプル通り）。

initialize() を async にして、必要な非同期処理を await する。呼び出し側は await engine.initialize() を使うようにする。

サムネイルは PIXI.RenderTexture ベースでバッチ更新（Ticker）にするのがルールブック推奨。
