B. 座標系を一元化する（推奨：PIXI 側へ統一） — これが最も効果が大きい

問題の本質：PositionManager が DOM/CSS トランスフォームでキャンバスを移動する実装で、DrawingEngine（PIXI）のコンテナ座標とは同期していない。これが「描けない」「座標が 0 のまま」「最初にズレる」などの原因になる。

対策：PositionManager.updatePosition() で DOM の style.transform を直接操作する代わりに、DrawingEngine（PIXI）の containers.camera か containers.world に対して座標・スケールを設定する。つまり PositionManager は PIXI の容器を操作する API を使う。

具体的パッチ（抜粋）

position-manager.js の updatePosition() を以下のように差し替え（コメント付き）：

// 変更点: DOM transform をやめ、PIXI コンテナ（camera/world）を操作する
updatePosition() {
  if (!this.updateScheduled) return;
  try {
    const { targetX, targetY } = this.position;
    const { current: currentZoom, target: targetZoom } = this.zoom;

    // 補間（必要なら）
    this.position.x = Math.round(targetX);
    this.position.y = Math.round(targetY);
    if (Math.abs(this.zoom.current - targetZoom) > 0.001) {
      this.zoom.current += (targetZoom - this.zoom.current) * 0.1;
      if (Math.abs(this.zoom.current - targetZoom) < 0.001) this.zoom.current = targetZoom;
    }

    // DrawingEngine のコンテナに適用する（mainApi 経由で drawing engine を取得）
    const drawingEngine = this.mainApi?.getSatellite?.('drawing') || this.mainApi?.getSatellite?.('drawingEngine') || window.DrawingEngineInstance;
    if (drawingEngine && drawingEngine.getApi) {
      const api = drawingEngine.getApi();
      const containers = api.getContainers(); // { camera, world, ui }
      if (containers && containers.camera) {
        // camera 側の transform を更新（ワールドの中心合わせなどは適宜調整）
        containers.camera.x = this.position.x;
        containers.camera.y = this.position.y;
        containers.camera.scale.set(this.zoom.current);
      }
    } else {
      // フォールバック：従来の DOM transform（暫定）
      const viewportCenter = { x: (window.innerWidth - 310) / 2, y: window.innerHeight / 2 };
      const offset = { x: viewportCenter.x + this.position.x, y: viewportCenter.y + this.position.y };
      this.container.style.transform = `translate3d(${offset.x}px, ${offset.y}px, 0) translate(-50%, -50%) scale(${this.zoom.current})`;
    }

    this.updateScheduled = false;
  } catch (error) {
    this.reportError('UPDATE_POSITION_ERROR', 'Failed to update position', error);
    this.updateScheduled = false;
  }
}


ポイント：

mainApi.getSatellite('drawing') 等で DrawingEngine インスタンスを取得できるインタフェースが必要（なければ DrawingEngine を window に置く等で参照可能にする）。

PIXI 側に座標操作を移すことで、pointer の event.global（PIXI が返すグローバル座標）と canvas 内のワールド座標が一致します。

（注）この変更をすると PositionManager.screenToWorld() / worldToScreen() のロジックも PIXI の transform を使うように見直す必要があります（app.renderer.plugins.interaction.mapPositionToPoint や container.toLocal を使う）。

参照：PositionManager, DrawingEngine の実装箇所。

C. pointer / space 判定の見直し（ペン描画不可の可能性）

DrawingEngine.onPointerDown では const spacePressed = window.futabaApp?.getSpacePressed?.() || false; if (spacePressed) { event.stopPropagation(); return; } としている（スペースでパン抑止）。一方 PositionManager.onPointerDown は if (!spacePressed) return; としており、逆にパンが動かないときにしか動作しないようにも見えます。

対応：両方の条件を見直し、スペースが押されているときにパン、押されていないときに描画 という意図なら PositionManager.onPointerDown は if (!spacePressed) return; で良いが、DrawingEngine と整合しているか確認してください。実際の UI 期待動作に合わせて boolean を揃えてください。

D. PIXI.Graphics の正しい使い方（マスク/背景が効かない問題）

drawing-engine.js、layer-manager.js のマスク・背景作成は以下のように書き換えを推奨します。

例（mask と background）：

// mask
const maskGraphics = new PIXI.Graphics();
maskGraphics.beginFill(0x000000, 1);
maskGraphics.drawRect(0, 0, config.canvas?.width || 400, config.canvas?.height || 400);
maskGraphics.endFill();
this.app.stage.addChild(maskGraphics);
this.containers.camera.mask = maskGraphics;

// background
const backgroundGraphics = new PIXI.Graphics();
backgroundGraphics.beginFill(0xf0e0d6, 1);
backgroundGraphics.drawRect(0, 0, config.canvas?.width || 400, config.canvas?.height || 400);
backgroundGraphics.endFill();
container.addChild(backgroundGraphics);


rect() / fill() の非標準的な使い方は互換問題を起こすことがあります。

LayerManager.register(mainApi) 内で mainApi に自分を登録して受信できるようにする（もし mainApi が衛星登録 API を持つなら）。

async register(mainApi) {
  this.mainApi = mainApi;
  if (this.mainApi?.registerSatellite) {

UI 側（main.html）の「レイヤー＋」ボタンが mainApi.dispatch('layers', { type:'create-layer' }) のような形で呼ぶこと。受け手（mainApi）が layers チャネルに対して create-layer を方法として呼び出すルーティングを持っているか確認してください。layer-manager は getApi() を返すので、mainApi 側ルーティングで satellite.getApi().createLayer() を呼ぶ実装が必要です。

F. イベント/ログの改善（デバッグを簡単にする）

mainApi 側で受け取ったイベントを console.log('EVENT from', channel, payload) と全体出力する一時パッチを入れて、type が欠落する送信元を特定する。

さらに Unhandled event type: undefined の出力箇所を改修して、受け取ったオブジェクト全体をログするようにする（原因追跡が速くなります）。
