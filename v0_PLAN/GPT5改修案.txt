問題点の解析と原因
1. キャンバス左上に赤い四角がある

原因:
これは PixiJS が初期化時に Graphics のテスト描画を残しているか、デバッグ用に残した矩形(beginFill(0xFF0000))がシーンルートに残っているもの。

確認方法:
app.stage.addChild(debugRect) のような処理があれば削除。

2. ペンが右下に x200 y100 くらいズレる

原因:

pointermove で得た event.clientX / clientY をそのまま使っている。

ところが Pixi v8 では renderer.events.pointermove の座標は「スクリーン座標」で、描画は worldTransform（カメラ・ズーム・パン適用後のワールド座標）で行う必要がある。

改修指針:
e.global を使って Pixi座標に変換する。

const pos = e.global; // PIXI.Point
drawLineTo(pos.x, pos.y);

3. キャンバス移動すると「カメラ窓」を残して他が動く。初回は x400 y400 くらい動く

原因:

stage.pivot や stage.position の初期値設定が不完全。

初回ドラッグで一気にズレるのは「直前の保存位置が (0,0) 固定」のため。

2回目から安定するのは lastPointerPosition が更新されるから。

改修指針:

初回 mousedown で 現在の stage.position を基準にドラッグ開始座標を保存。

移動は newPos = savedStagePos + (pointerNow - pointerStart) で計算。

4. レイヤーパネルのサムネイルが描画を反映せず「▣」のまま

原因:

サムネイル更新処理で renderer.extract.image(layer) を呼んでいない。

代わりに background CSS が残っているだけなので枠線と塗りつぶしだけが表示される。

改修指針:
レイヤー更新時に以下を実行して、<img> に反映する:

const thumb = app.renderer.extract.image(layer.container);
layerThumbnail.src = thumb.src;

改修案（まとめ）

赤い四角削除
→ デバッグ矩形を削除。

ペンのズレ修正
→ pointermove の座標取得を e.global に統一。

app.stage.eventMode = 'static';
app.stage.on('pointerdown', e => {
    const pos = e.global;
    startDrawing(pos.x, pos.y);
});
app.stage.on('pointermove', e => {
    if (!drawing) return;
    const pos = e.global;
    continueDrawing(pos.x, pos.y);
});


カメラ移動の初回ジャンプ防止
→ ドラッグ開始時に stage.position を保存。

let dragging = false;
let dragStart = null;
let stageStart = null;

app.stage.on('pointerdown', e => {
    dragging = true;
    dragStart = { x: e.global.x, y: e.global.y };
    stageStart = { x: app.stage.x, y: app.stage.y };
});

app.stage.on('pointermove', e => {
    if (!dragging) return;
    app.stage.x = stageStart.x + (e.global.x - dragStart.x);
    app.stage.y = stageStart.y + (e.global.y - dragStart.y);
});

app.stage.on('pointerup', () => dragging = false);


サムネイル反映
→ レイヤー描画完了後に extract.image() を呼んで <img> 更新。

function updateThumbnail(layer) {
    const image = app.renderer.extract.image(layer.container);
    layer.thumbnail.src = image.src;
}

提案する次ステップ

今回は 4箇所修正点が分散しているので「パッチ差し込み案内」よりも、
一括で差し替えやすい関数単位の改修ファイルを用意した方が良いです。

特に pointer座標処理 と カメラ移動処理 は一緒に直さないと不具合が続きます。

以下は参考パッチ
=====
🔧 パッチ内容
1. 赤い四角削除

探す:

const graphics = new PIXI.Graphics();
graphics.beginFill(0xFF0000);
graphics.drawRect(0, 0, 100, 100);
graphics.endFill();
app.stage.addChild(graphics);


削除してください（完全に不要なデバッグ描画です）。

2. ペン座標ズレ修正

探す: （例: pointerイベントで event.clientX / clientY を使用している箇所）

const x = e.client.x;  
const y = e.client.y;  
drawLineTo(x, y);


置換:

const pos = e.global;  
drawLineTo(pos.x, pos.y);


※ すべての pointerdown / pointermove で同様に e.global を使ってください。

3. カメラ移動の初回ジャンプ防止

探す: （stageを直接 += している箇所）

app.stage.x += e.movementX;
app.stage.y += e.movementY;


置換:

let dragging = false;
let dragStart = null;
let stageStart = null;

app.stage.eventMode = 'static';
app.stage.on('pointerdown', e => {
    dragging = true;
    dragStart = { x: e.global.x, y: e.global.y };
    stageStart = { x: app.stage.x, y: app.stage.y };
});

app.stage.on('pointermove', e => {
    if (!dragging) return;
    app.stage.x = stageStart.x + (e.global.x - dragStart.x);
    app.stage.y = stageStart.y + (e.global.y - dragStart.y);
});

app.stage.on('pointerup', () => dragging = false);

4. レイヤーパネルのサムネイル更新

探す: （サムネイル更新が ▣ で止まっている箇所）

追加:

function updateThumbnail(layer) {
    const image = app.renderer.extract.image(layer.container);
    layer.thumbnail.src = image.src;
}


呼び出し例:
レイヤーに描画が発生した直後に

updateThumbnail(currentLayer);


を追加してください。
