🛠️ Tegaki 改修手順書（Phase1 → Phase2 移行準備）
📌 基本ルール

禁止: Vite / TypeScript / import/export (ESM) / ES5構文へのダウングレード

必須: ES2015+ 構文 (class, let/const, async/await, ES2023 まで使用可)

依存: ConfigManager / ErrorManager / StateManager / EventBus に統一

公開: 全クラス・関数は window.* に明示的に公開すること

原則: DRY / SOLID を遵守し、架空のメソッドは生成しない

1. AppCore 修正

現状問題: initialize() 内で非同期処理が混在し、await の扱いが不統一

改修手順:

initialize() を async initialize() に変更

内部の非同期処理（fetch, manager初期化など）は await で統一

呼び出し元も await appCore.initialize() とする

✅ class AppCore {} 構文を維持し、ES5 の function 形式に戻さないこと。

2. ToolManager 修正

現状問題: await が非 async 関数内で使われているケースあり

改修手順:

initializeCoordinateManagerIntegration() を async 化

内部で必要な await を有効化

CanvasManager 初期化時に呼び出しを統一

3. BoundaryManager 修正

現状問題: 初期化時に canvasElement が未渡しのまま動く場合あり

改修手順:

initialize(canvasElement, coordinateManager) の引数を必須にする

AppCore.initialize() → CanvasManager.initialize() → BoundaryManager.initialize() の順序で呼び出す

canvasElement と CoordinateManager を常にセットで渡す

4. Coordinate 系統の統一

現状問題: coordinates.js が残存し、重複処理が存在

改修手順:

すべての座標処理を CoordinateManager に統一

window.CoordinateUtils 等の旧ユーティリティ呼び出しを削除/置換

Phase2 で coordinates.js を完全削除

5. PenTool / 描画処理

現状問題: ログは出力されるが Pixi.Graphics に描画されない

改修手順:

CanvasManager に this.graphics = new PIXI.Graphics(); this.app.stage.addChild(this.graphics); を追加

PenTool に attachGraphics(graphics) を実装

startStroke/updateStroke/endStroke に描画処理を追加（筆圧で線幅変化させる）

ToolManager から PenTool.attachGraphics(CanvasManager.graphics) を呼ぶ

6. GPU / Memory モニタリング

現状: 値を取得しているが、実装依存で実作業には未活用

改修手順:

GPU → WebGL2 のコンテキスト情報を取得して StateManager に記録

Memory → performance.memory API が利用可能ならブラウザヒープを監視

実際の描画処理には未反映だが、開発用モニターとして利用

✅ 最終確認

app-core.js, tool-manager.js, boundary-manager.js, canvas-manager.js それぞれの修正後、
initializeAppCore() → 各 Manager 初期化のフローを確認

Console 警告「coordinates.js 削除予告」「統合未完了」が出ない状態を目標にする

ペンツールで 実際に線が描け、筆圧で線幅が変化することを確認
