# Phase 3b: クイックアクセスポップアップ（quick-access-popup.js） UI大改修とプリセットスロット実装

## 【GPTへの作業依頼概要】
このドキュメントは、お絵かきアプリの「クイックアクセスポップアップ（QuickAccessPopup）」のUIを最新化・省スペース化し、ツールごとに独立したサイズ/不透明度プリセットスロットを実装するための指示書です。

**対象ファイル:** `ui/quick-access-popup.js`
（この指示書と共に、現在の `quick-access-popup.js` のコードを読み込ませて以下の改修を行ってください）

---

## 【改修の要件】

### 1. UIの縦長・スリム化とモダン化
- **横幅の縮小**: 現在のポップアップは横幅が広すぎるため、現在の60%〜70%程度の幅にスリム化してください。縦方向には伸びて構いません。
- **デザインの刷新**: リサイズポップアップのような、半透明の背景（glassmorphism的）と洗練された境界線を持つモダンなUI構造に書き換えてください。
- **アイコンの密集化**: ツールボタンアイコンの間隔を詰め、スペースを無駄なく使用してください。

### 2. ツール完全独立の「サイズ＆不透明度スロット（5枠）」の実装
- **スロットの仕様**: 
  - 1つのツールにつき「5つのプリセットスロット」を持たせます。
  - 各スロットは「ブラシサイズ」と「不透明度」のペアを保存・復元できるようにしてください。
  - **ツール間の完全独立**: `pen`, `eraser`, `airbrush` でそれぞれ独立した5スロットの状態を管理します。（例：ペンのスロット3がサイズ5でも、消しゴムに切り替えた際は消しゴムのスロット状態が復元される）。
- **視覚的フィードバック（●のスケール）**:
  - スロットのUIは「◯」の中に「●」があるデザインとしてください。
  - 記憶されているサイズ値（1〜100）に応じて、中の「●」の大きさが変化するようにCSS/インラインスタイルを制御してください。
  - ただし、サイズ20〜50の段階で「●」は枠内最大の大きさ（カンスト）に達するものとし、それ以上のサイズは数値等で判断させる仕様としてください。
- **永続化（localStorage）**:
  - 各ツールのプリセット状態は `localStorage` に保存し、ブラウザ再起動時にも前回の状態を読み込んで復元するロジックをクラス内に組み込んでください。

### 3. カラーパレットの拡張と高密度化
- カラーパレットに登録できる色数を現在より増やし、2段組みなどにしてスロットを密集させて配置してください。
- （※カラーサークルやスポイト機能の実装は今回は不要です。パレットのUI構造のみ最適化してください。）

### 4. 既存処理（Phase 3a）の維持
- 現在の `quick-access-popup.js` は、`airbrush` と `airbrush-erase`（透明スプレー）のトグル機能や、動的な `MAX_SIZE` 取得機能を持ち合わせています。これらの既存ロジックは破壊せずに維持してください。
