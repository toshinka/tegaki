PixiJS v7/v8 描画関連拡張ライブラリ導入提案書
============================================

この提案書は、PixiJS v7を基盤としたお絵かきツールの既存実装（ペン機能、消しゴムなど、お絵描き関連）に対して、導入が勧められる描画関連ライブラリをまとめています。
提案はPixiJSエコシステム内のライブラリを中心にし、車輪の再発明を防ぎつつ、描画品質の向上や追加機能を目的とします。
導入によりAI実装のしやすさを維持し、互換性失敗リスクを低減します。
本文はPixiJS v7を中心に記述し、v8に関する情報は最後にまとめます。
既存のペン機能（Graphicsを使った線描画）を強化するものとして、特にスムーズ描画と追加形状に焦点を当てています。

npmインストールガイド (PixiJS v7対応版)
----------------------------------------
以下のコマンドで提案ライブラリをインストールしてください。
これらはPixiJS v7と互換性があり、既存のdrawing-tools.jsやpen-tool.jsなどに追加可能です。

npm install @pixi/graphics-extras@^7.4.0 @pixi/graphics-smooth@^0.0.31

インストール後、JavaScriptファイルでインポート例:
import { Graphics } from '@pixi/graphics';
import '@pixi/graphics-extras';
import { SmoothGraphics } from '@pixi/graphics-smooth';

PixiJS v7 vs v8の選択とアップグレードの検討
--------------------------------------------
PixiJS v7は安定した基盤を提供し、既存コードとの互換性が高いです。
v8はv7からの進化版ですが、v7で追加機能を実装し、後で移行することを推奨します。
詳細は最後のv8セクションを参照。

描画関連ライブラリ組み合わせ提案
----------------------------------
PixiJSエコシステム内のライブラリを中心に提案。
これらはv7とシームレスに統合され、導入による複雑化が少なく、AIコード生成がしやすいです。
ペン機能のスムーズ化や追加形状描画に特化。

ライブラリ                  | 説明と利点（実装のしやすさ/メンテナンス）                                                                 | AI実装のしやすさと失敗リスク                 | PixiJS v7互換性      | 純粋PixiJSでの代替（車輪再発明度）
----------------------------|-----------------------------------------------------------------------------------------------------------|---------------------------------------------|----------------------|-----------------------------------
@pixi/graphics-extras      | Graphicsクラスに追加描画メソッド（drawTorus, drawChamferRect, drawFilletRect, drawRegularPolygon）を追加。お絵描きで多様な形状（ドーナツ形、面取り/丸角矩形、正多角形）を簡単に描画。メンテ: 拡張性が高く、既存Graphicsと統合。 | シンプルなメソッド追加でAIが扱いやすい。PixiJSネイティブなので失敗なし。 | v7: フル対応 (v7.4.0以降) | Graphicsで自作可能だが、複雑形状で再発明多め（カスタムパス計算）。ライブラリ推奨。
@pixi/graphics-smooth      | HHAAシェーダーでスムーズな線/形状描画。ペン/ブラシのジャギーを減らし、アンチエイリアス効果。ドロップイン置き換え（SmoothGraphics使用）。線幅スケール制御やピクセルパーフェクト設定可能。メンテ: パフォーマンス考慮で拡張しやすい。 | APIがGraphics互換でAI生成向き。PixiJS拡張なので互換性高。 | v7: フル対応 (v0.0.31以降) | 標準Graphicsでアンチエイリアスなしだが、再発明多（カスタムシェーダー作成）。ライブラリ強く推奨。

導入方法の例
------------
npmインストール後、既存のpen-tool.jsなどに追加。
例: スムーズペン描画 (drawing-tools/tools/pen-tool.jsなどに)
import { SmoothGraphics } from '@pixi/graphics-smooth';
const graphics = new SmoothGraphics();
graphics.lineStyle({ width: 2, color: 0x000000, scaleMode: 'none' }); // 固定幅スムーズ線
graphics.moveTo(startX, startY);
graphics.lineTo(endX, endY);

例: 追加形状 (お絵描きツール拡張で)
import { Graphics } from '@pixi/graphics';
import '@pixi/graphics-extras';
const shapes = new Graphics().beginFill(0xffffff).drawTorus(100, 100, 20, 50); // ドーナツ形描画

全体の組み合わせの利点
------------------------
@pixi/graphics-extras + @pixi/graphics-smoothの組み合わせで、ペン機能の品質向上と形状拡張が可能。
実装時間が短縮（例: スムーズ線をシェーダー自作せず活用）。
メンテナンス性向上（バグ修正はライブラリ側で共有）。
PixiJSエコシステム内なので、v7互換性が高く、AIがドキュメントを基にコードを生成しやすい（失敗例なしのシンプルAPI）。

リスク回避
----------
外部大規模ライブラリは避け、PixiJS専用拡張に限定。
導入でバイブコーディングが滞る心配は低く、むしろAIが提案しやすくなる。
@pixi/graphics-smoothは頂点数増加でパフォーマンス影響あり（ドキュメント参照）。

堅実なアプローチのまとめ
------------------------
ライブラリ組み合わせをおすすめ: ペン機能のスムーズ化で描画品質を楽に向上。
純粋PixiJSで進める場合: 見通しが良く、依存ゼロ。スムーズ線はカスタムフィルターで、形状はGraphicsパスで実現可能。ただし、再発明が増え、実装時間が増す。
次ステップ: ライブラリを試すなら、サンプルプロジェクトでSmoothGraphicsをペンに適用してみて、AIにコードを生成させて互換性を確認。提供されたファイルURLを基に詳細を尋ねてください。

PixiJS v8に関する情報
----------------------
利点:
- グラフィックスAPIの改善（スムーズ描画のネイティブ強化）。
- パフォーマンス向上（WebGPUで高速シェーダー）。
- @pixi/graphics-extras/@pixi/graphics-smoothのv8対応版で、さらに効率的。
- 描画ツールで線/形状のレンダリングがより滑らかになる。

リスク:
- v7からのAPI変更（例: Graphicsのビルド方法変更）で既存コードの修正が発生。
- 移行コストがかかる可能性。

導入手引:
- 移行ガイドを参照: https://pixijs.com/8.x/guides/migrations/v8
- npmでv8インストール: npm install pixi.js@^8.0.0
- ライブラリ更新: @pixi/graphics-extras@^8.x, @pixi/graphics-smooth (v8対応版を確認)。
- ステップ: v7で機能追加後、v8へ移行。API変更を一つずつ修正（例: Graphicsのフィル/ストローク順序）。
- テスト: サンプルコードでv8互換性を確認。
