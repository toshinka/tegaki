# JavaScript + PixiJS v7 Rulebook v7（ライブラリ統合指針版）

## 1. 基本理念

### 1.1 核心方針
- **安定動作・可読性・拡張性を優先**し、将来的なTypeScript + Vite + ES6移行を意識する
- **PixiJSエコシステム活用**により、車輪の再発明を避け、標準実装を採用する
- **段階的品質向上**と既存システムとの互換性維持を両立する
- **AI協働最適化**を考慮した標準パターンの採用を推奨する

### 1.2 実装原則
- 現在はJavaScript+fetch API分割でありグローバル変数形式で実装する
- ファイル間の依存は明確化し、循環参照を作らない
- **SOLID・DRY原則準拠**: 単一責任・責務分離・重複コード排除を徹底する
- **モジュール分割設計**: 機能ごとの専門ファイル分離による保守性向上
- PixiJS v7の公式API仕様に沿い、非推奨機能は使用しない
- ES2023構文使用OK、OffscreenCanvasや最新Canvas APIも使用OK
- 最新Chrome+デスクトップPC環境を想定、後方互換は意識しない
- **ファイルサイズ制限**: 単一ファイルは800行以下を目安とし、それを超える場合は責務に応じて分割する
- 必ずdebug/またはmonitoring/ディレクトリの既存モジュールを確認し、重複を避ける事

## 3. ライブラリ統合設計原則

### 3.1 ライブラリ選定基準
- **PixiJSエコシステム内優先**: 公式・準公式ライブラリを最優先
- **fetch API分割対応**: グローバル変数形式での使用が可能
- **AI生成親和性**: 標準的なAPIパターンを持つ
- **メンテナンス継続性**: アクティブに開発・保守されている
- **互換性保証**: PixiJS v7との互換性が確認されている

### 3.2 統合実装パターン

#### 3.2.1 ライブラリ読み込みパターン
```javascript
// libs/pixi-extensions.js での統合パターン
window.PixiExtensions = {
    // @pixi/ui
    Button: null,
    Slider: null,
    // @pixi/layers  
    Layer: null,
    // @pixi/gif
    AnimatedGIF: null,
    // @pixi/graphics-smooth
    SmoothGraphics: null,
    // 読み込み状況
    loaded: {
        ui: false,
        layers: false,
        gif: false,
        graphicsSmooth: false,
        graphicsExtras: false
    }
};
```

#### 3.2.2 段階的有効化パターン
設定による機能の段階的有効化を支援
```javascript
// config.js でのライブラリ制御
LIBRARY_FLAGS: {
    ENABLE_PIXI_UI: true,
    ENABLE_PIXI_LAYERS: true,
    ENABLE_PIXI_GIF: false,  // Phase別有効化
    ENABLE_SMOOTH_GRAPHICS: true,
    ENABLE_GRAPHICS_EXTRAS: true
}
```

### 3.3 互換性保持原則
- **既存API維持**: 既存の関数・メソッド署名は維持
- **段階的移行**: 内部実装のみライブラリ使用に変更
- **フォールバック実装**: ライブラリ読み込み失敗時の代替処理
- **設定による制御**: ライブラリ使用の有効・無効切り替え対応

## 4. コードスタイル規約（ライブラリ統合対応）

### 4.1 命名規則（拡張版）
- **変数 / 関数**: `camelCase`（例：`drawLine`, `pointerPosition`）
- **定数**: `SCREAMING_SNAKE_CASE`（例：`CANVAS_WIDTH`, `BG_COLOR`）
- **クラス**: `PascalCase`（例：`VectorPenTool`, `PresetManager`）
- **ファイル名**: `kebab-case`（例：`pen-tool-ui.js`, `preview-sync.js`）
- **ライブラリ統合関数**: `lib{Feature}`（例：`libSmoothGraphics`, `libCreateButton`）
- **ライブラリ設定**: `{LIBRARY}_CONFIG`（例：`PIXI_UI_CONFIG`）

### 4.2 ライブラリ使用パターン
```javascript
// 推奨: ライブラリ機能の標準使用
const button = new window.PixiExtensions.Button({ text: 'OK' });

// 推奨: フォールバック付き使用
const graphics = window.PixiExtensions.SmoothGraphics ? 
    new window.PixiExtensions.SmoothGraphics() : 
    new PIXI.Graphics();

// 禁止: ライブラリ機能の独自実装
// const customButton = this.createCustomButton(); // NG
```

        // 内部状態更新

## 10. 禁止事項（ライブラリ統合対応）

### 10.1 基本禁止事項（継続）
- DOM直接操作（Pixi外の`querySelector`等）
- Pixi内部プロパティへの直接アクセス
- 複数責務を1関数にまとめる
- **単一ファイル800行超過**（分割必須）

### 10.2 ライブラリ統合固有禁止事項
- **利用可能ライブラリの独自実装**: 標準機能の再実装禁止
- **ライブラリ機能の部分的複製**: 必ずライブラリ経由で使用
- **非標準APIパターンの新規作成**: AI生成困難パターンの回避
- **ライブラリ設定の直接変更**: 必ずCONFIG経由で制御
- **段階的導入の無視**: 一度に複数ライブラリ変更禁止

### 10.3 品質劣化防止事項
- **エラーハンドリング省略**: ライブラリ読み込み失敗対応必須
- **パフォーマンス監視省略**: 導入前後の測定必須
- **互換性テスト省略**: 既存機能の非回帰テスト必須
- **責務の重複実装**: debug/, monitoring/ディレクトリ確認必須
