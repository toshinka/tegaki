📌「ドラッグで線を描けないバグ」の修正と、構造明確化パッチ
✅ 背景と目的
現在のツールは 「点は描けるがドラッグしても線が描けない」 状態にあります。
この修正では以下を実現してください：

ポインターのドラッグ動作で「連続線（stroke）」を描画できるようにする。

点（クリック）と線（ドラッグ）のどちらも機能するようにする。

初期化や表示が壊れないよう注意する。

⚠️ 前回までの失敗からの知見
this.viewport.drawLineSegment() は存在しない関数です。使用禁止。
　→ 正しくは this.engine.drawLineSegment(...) を使うべきです。

WebGLRenderer に StrokeRenderer をimportしようとして壊れました。
　→ WebGLRenderer は DrawingEngine に注入されるので、それ単体で StrokeRenderer を import する必要はありません。

ファイルの内容が痩せすぎるとキャンバスそのものが表示されなくなります。
　→ 必要な初期化処理・コンテキスト取得・バッファ設定などは削除せず保持すること。

🧩 修正対象ファイルと責務
ファイルパス	役割	修正内容
PointerInteractionHandler.js	UI層のポインター操作受け取り	連続するポインター座標をエンジンに渡す。this.engine.drawLineSegment(...) で呼び出すこと。
DrawingEngine.js	描画APIの中継層	drawLineSegment(x1, y1, x2, y2) メソッドを実装し、rendererに転送する。
WebGLRenderer.js	WebGL描画の最下層	drawLineSegment(x1, y1, x2, y2) メソッドを新設。gl.LINESを用いて直線描画を行う。Shaderの初期化なども壊さないこと。

🧪 テスト要件
ペンツールでキャンバス上をドラッグすると、連続した赤い線が描画されること。

クリック（点）でも描画できること。

エラーが出ないこと（特に「hasPointerCapture」「未定義関数」など）。

Canvasが表示されること（WebGLコンテキストが正しく初期化されること）。

✍️ 実装時の注意点（よくあるミス）
🔴 viewport.drawLineSegment() のような未定義関数を使わないこと。

🔴 WebGLRenderer の中で export されていない StrokeRenderer を import しようとしないこと。

🔴 gl描画前に bufferData を使ってバッファを更新しないと何も描かれません。

🔴 シェーダーや初期化処理を削除しないでください。何も描かれなくなります。

// PointerInteractionHandler.js（抜粋）
handlePointerMove(e) {
  if (!this.isDrawing) return;
  const pos = getLocalPosition(this.canvas, e);
  if (this.prevPos) {
    this.engine.drawLineSegment(this.prevPos.x, this.prevPos.y, pos.x, pos.y);
  }
  this.prevPos = pos;
}

}
js
コピーする
編集する
// WebGLRenderer.js（抜粋）
drawLineSegment(x1, y1, x2, y2) {
  const positions = new Float32Array([x1, y1, x2, y2]);
  this.gl.bufferData(this.gl.ARRAY_BUFFER, positions, this.gl.STATIC_DRAW);
  this.gl.drawArrays(this.gl.LINES, 0, 2);
}
