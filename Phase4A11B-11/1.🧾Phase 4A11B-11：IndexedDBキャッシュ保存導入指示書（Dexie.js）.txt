🎯 目的
レイヤーの描画状態をIndexedDBに保存・復元できるようにする

初期レイヤー破壊・描画不能・転写時のアンチエイリアスなどの重大バグを防ぐ構造で構築する

💾 Step 2: IndexedDBからの復元

コピーする

  console.log("💾 IndexedDBからレイヤーデータの復元を試みます...");
  const layers = await loadLayersFromIndexedDB();

  for (const layer of layers) {
    layerManager.createLayer(layer.id, layer.name);

    // ✅ 描画前に補間を明示的にOFFにする
    const ctx = canvasManager.getLayerContext(layer.id);
    if (ctx) {
      ctx.imageSmoothingEnabled = false;
      const img = new Image();
      img.onload = () => ctx.drawImage(img, 0, 0);
      img.src = layer.imageData;
    }
  }

  layerManager.setCurrentLayer(layers.at(-1)?.id ?? 0); // ✅ 最終レイヤーを描画対象に
  layerUIManager.renderLayerList?.(); // ✅ レイヤーUIが未定義でも安全に
  console.log(`✅ ${layers.length}件のレイヤーをIndexedDBから復元しました。`);

canvasManager.onDrawEnd = async () => {
  const current = layerManager.getCurrentLayer();
  const canvas = canvasManager.getLayerCanvas(current.id);
  await saveLayerToIndexedDB(current.id, current.name, canvas.toDataURL());
};

🚨 Step 5: バグになりやすいパターンに対する対策一覧
問題	対策
復元後にキャンバスが出ない	createLayer() → drawImage() の順守
描けない／ツール不明	toolManager.setTool("pen") を初期化時に実行
転写でジャギー／ぼやけ	imageSmoothingEnabled = false または putImageData 使用
renderLayerList()でクラッシュ	layerUIManager.renderLayerList?.() で安全呼び出し
setCurrentTool is not a function	canvasManager.setCurrentTool(tool) を定義すること
