✅ 今回の目的
描画レイヤーの状態を IndexedDB（Dexie.js経由）に保存し、ページリロード後に復元できるようにします。キャンバス（WebGL）が初期化されている前提で復元処理を行います。

2. core/db/db-indexed.js を作成
js
コピーする
編集する
const db = new Dexie("TegakiProjectDB");
db.version(1).stores({
  layers: "++id, name, imageData"
});

export async function saveLayerToIndexedDB(layerId, layerName, imageDataUrl) {
  await db.layers.put({ id: layerId, name: layerName, imageData: imageDataUrl });
}

export async function loadLayersFromIndexedDB() {
  return await db.layers.toArray();
}
3. core-engine.js に保存・復元処理＋WebGL初期化を必ず追加
js
コピーする
編集する
// 必要なモジュールの読み込み
import { RenderingBridge } from './core/rendering/rendering-bridge.js';
import { saveLayerToIndexedDB, loadLayersFromIndexedDB } from './core/db/db-indexed.js';

window.addEventListener('load', async () => {
  // ✅ WebGLキャンバスを生成・初期化（必須！！）
  RenderingBridge.init();

  // ✅ IndexedDBからのレイヤー復元は、WebGLキャンバスが生成された後に行うこと！！
  const layers = await loadLayersFromIndexedDB();
  for (const layer of layers) {
    // 適宜 layerManager などに渡す
    console.log("復元レイヤー:", layer.name);
    // 例: createLayer(layer.id, layer.name); drawImageFromDataURL(layer.imageData);
  }
});
🛑 注意：RenderingBridge.init() を絶対に忘れないこと！
キャンバスが生成されないと、何も表示されず、復元も機能しません。

🔧 よくあるエラーと対策
エラー	原因	対策
Dexie is not defined	dexie.min.js の読み込み忘れ	HTMLに <script src=...> を入れる
mat4 is undefined	gl-matrixの import 忘れ	webgl-engine.js に import { mat4 } from 'gl-matrix'
Canvasが出ない	RenderingBridge.init() を呼び忘れている	core-engine.js に必ず記述すること！
