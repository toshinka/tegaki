🎯目的
レイヤー情報や描画状態を、IndexedDB（Dexie.js使用）に保存・復元できるように改修してください。
この対応により、ユーザーが描いたレイヤーの状態をブラウザにキャッシュし、ページリロード後でも復元できるようにします。

📁使用対象ファイル（Phase 4A11B-10 版）
core-engine.js（改修対象）

core/db/db-indexed.js（新規作成）

🧱ステップ構成
✅ Step 1. Dexie.js を導入してください
js
コピーする
編集する
// core/db/db-indexed.js の先頭に記述
import Dexie from 'dexie';
※ブラウザ環境で使う場合はESModuleを前提にします。Geminiで生成したコードが動かない場合は<script>版を検討します。

✅ Step 2. データベースの定義と初期化
js
コピーする
編集する
// core/db/db-indexed.js
const db = new Dexie('TegakiProjectDB');
db.version(1).stores({
  layers: '++id, name, imageData'
});
✅ Step 3. 保存関数の作成
js
コピーする
編集する
export async function saveLayerToIndexedDB(layerId, layerName, imageDataUrl) {
  await db.layers.put({
    id: layerId,
    name: layerName,
    imageData: imageDataUrl
  });
}
✅ Step 4. 復元関数の作成
js
コピーする
編集する
export async function loadLayersFromIndexedDB() {
  const layers = await db.layers.toArray();
  return layers;
}

🧪 エラー対策（Consoleエラー回避）
❌ mat4 が undefined になる
webgl-engine.js に直接以下を追加してください：

js
コピーする
編集する
import { mat4 } from 'gl-matrix';
core-engine.js ではなく、WebGL内部で使用しているwebgl-engine.jsにインポートする方が妥当です。

❌ Dexie is not defined
原因：Dexieライブラリが読み込まれていない

対応：

モジュールで読み込んでいない → import Dexie from 'dexie' を確認

CDNの場合 → HTMLに以下を追加すること（今回は import を推奨）

html
コピーする
編集する
<script src="https://unpkg.com/dexie@3/dist/dexie.min.js"></script>
📝注意
Dexieの put() は id が同じなら上書きします。

toDataURL() の保存は画像が重くなりがちなので、将来的には ImageBitmap や Blob ベースに変更検討余地あり。

🧠補足：新人Gemini向けTips
saveLayerToIndexedDB() の呼び出しタイミングは「レイヤー移動終了時」「描画完了時」など、変更が確定するタイミングに設置。

loadLayersFromIndexedDB() は window.onload または initializeEngine() の完了後に呼び出すのが安全。
