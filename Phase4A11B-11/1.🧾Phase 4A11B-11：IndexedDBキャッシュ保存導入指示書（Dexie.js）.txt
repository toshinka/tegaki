🎯 目的
レイヤー画像（ImageData）を永続保存できる「キャッシュ保存機能」を実装する。

使用ライブラリは Dexie.js。IndexedDB操作をシンプル・安定化するため。

将来的な「レイヤーの自動バックアップ」や「セッション復元」の基盤とする。

📁 対象ファイル
ファイル名	目的
core-engine.js	レイヤー保存・読込の呼び出しロジック追加
libs/Dexie.js	IndexedDB用ライブラリ（CDN不要、自前設置）

✅ ステップ3：ImageDataを保存する関数を追加
js
コピーする
編集する
async function saveCurrentLayerToCache(layer) {
  const ctx = layer.canvas.getContext("2d");
  const imageData = ctx.getImageData(0, 0, layer.canvas.width, layer.canvas.height);
  const blob = new Blob([imageData.data], { type: "application/octet-stream" });

  await TegakiDB.layers.add({
    name: layer.name || "unnamed",
    timestamp: Date.now(),
    width: imageData.width,
    height: imageData.height,
    buffer: blob
  });

  console.log("✅ レイヤーをIndexedDBに保存しました");
}
✅ ステップ4：保存したレイヤーを復元する関数を追加
js
コピーする
編集する
async function loadLatestLayerFromCache(layer) {
  const latest = await TegakiDB.layers.orderBy("timestamp").last();
  if (!latest) {
    console.warn("⚠️ 保存されたレイヤーが見つかりませんでした");
    return;
  }

  const buffer = await latest.buffer.arrayBuffer();
  const uint8 = new Uint8ClampedArray(buffer);
  const imageData = new ImageData(uint8, latest.width, latest.height);

  const ctx = layer.canvas.getContext("2d");
  ctx.putImageData(imageData, 0, 0);
  console.log("📥 レイヤーを復元しました");
}

  }

💡 注意点・備考
imageData.data は Uint8ClampedArray であり、IndexedDBにはそのまま保存できないため、Blobとして保存します。

複数レイヤーやセッション管理は今後のPhaseで拡張予定です（例：レイヤーID, layerId, transform など）。

Dexie.js のバージョンは 3.x を想定。CDN使用時でも動作しますが、ローカル ./libs/ に設置済なのでそちらを優先。
