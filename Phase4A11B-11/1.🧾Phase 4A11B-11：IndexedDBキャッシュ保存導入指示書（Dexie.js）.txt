🎯目的
WebGLベースのレイヤー描画状態をブラウザ内にキャッシュ保存し、ページをリロードしても復元できるようにする。保存には Dexie.js を使用し、IndexedDBを扱いやすくする。

✅ 対象ファイル
ファイル	用途
/core/db/db-indexed.js	IndexedDB保存・復元ロジック（新規作成）
core-engine.js	初期化と保存/復元呼び出し処理を追加（改修）
ToshinkaTegakiTool.html	dexie.min.js を読み込むよう <script> タグ追加（済）

✅ やること（ステップ一覧）
1. dexie.min.js を HTML から読み込む（済）
html
コピーする
編集する
<script src="./libs/dexie.min.js"></script>
2. core/db/db-indexed.js を新規作成
js
コピーする
編集する
// IndexedDB用のデータベースを初期化
const db = new Dexie("TegakiProjectDB");
db.version(1).stores({
  layers: "++id, name, imageData"
});

// レイヤーを保存する関数
export async function saveLayerToIndexedDB(layerId, layerName, imageDataUrl) {
  await db.layers.put({
    id: layerId,
    name: layerName,
    imageData: imageDataUrl
  });
}

// レイヤーを復元する関数
export async function loadLayersFromIndexedDB() {
  const layers = await db.layers.toArray();
  return layers;
}

🔹 起動時のレイヤー復元
js
コピーする
編集する
loadLayersFromIndexedDB().then(layers => {
  for (const layer of layers) {
    createLayer(layer.id, layer.name);
    drawImageFromDataURL(layer.imageData);
  }
});
createLayer() や drawImageFromDataURL() はプロジェクト内の関数に置き換えてください

🔧 注意点とエラー対策
❗ Dexie is not defined
原因：CDN未読み込み

対応：dexie.min.js の <script> タグが正しく挿入されているか確認

❗ mat4 undefined
原因：gl-matrix がインポートされていない

対応：WebGL使用ファイル（webgl-engine.js）の冒頭で明示的に

js
コピーする
編集する
import { mat4 } from 'gl-matrix';
✅ 呼び出しタイミングの目安
処理	呼び出し場所の目安
saveLayerToIndexedDB()	レイヤー移動後／描画完了後のイベントフックに追加
loadLayersFromIndexedDB()	起動後の初期化処理内（描画エンジンが準備完了してから）

🧠 補足
toDataURL() は画像が重くなりやすいため、Phase 4D 以降で ImageBitmap や Blob 保存への移行を検討。

Undo/Redoや履歴保存との組み合わせは Phase 4B〜4C で調整予定。
