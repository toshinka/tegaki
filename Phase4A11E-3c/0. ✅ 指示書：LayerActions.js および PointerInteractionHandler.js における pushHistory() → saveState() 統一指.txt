✅ 現在の主な問題
Undo（アンドゥ）が一切効かない

バケツツールがUIボタンから反応しないが、ショートカットGでは動作する

🔍【1】Undo が効かない原因候補
Undo（Ctrl+Zなど）に関わるのは以下の構成です：

モジュール	役割
HistoryStore.js	状態履歴の保存・復元本体
ShortcutHandler.js	キー入力の捕捉（Ctrl+Z等）
PointerInteractionHandler.js	描画後に saveState() 呼び出し
LayerActions.js	レイヤー操作後の saveState() 呼び出し
AppController.js	初期化時に ShortcutHandler に依存関係を注入

✅ チェックポイント（可能性の高い原因順）
 ShortcutHandler.js に Ctrl+Z（または Command+Z）を検出するコードがあるか

 ShortcutHandler が AppBootstrap 経由で HistoryStore にアクセスできているか

 HistoryStore.saveState() が何も実際には記録していない可能性（履歴配列が機能していない）

 saveState() の中身に push 処理（履歴追加）が無い

✅ 確認・修正内容
【A】HistoryStore.js の saveState() 実装確認
js
コピーする
編集する
// 以下のような履歴配列が必要
this.historyStack = [];

saveState() {
  const snapshot = this._createSnapshot(); // レイヤーなどの状態をコピー
  this.historyStack.push(snapshot);        // pushして履歴として保存
}
_createSnapshot() や push 処理が未実装、または中身が空だと記録されません。

undo() で this.historyStack.pop() のような巻き戻し処理が必要です。

【B】ShortcutHandler.js に Ctrl+Z 処理があるか
js
コピーする
編集する
document.addEventListener('keydown', (e) => {
  if ((e.ctrlKey || e.metaKey) && e.key === 'z') {
    this.historyStore.undo(); // ← 実行されているか？
  }
});
存在しない or this.historyStore が null の場合、反応しません。

【C】AppController.js（または AppBootstrap）の初期化処理で ShortcutHandler に historyStore を渡しているか？
js
コピーする
編集する
this.shortcutHandler = new ShortcutHandler({
  historyStore: this.historyStore,
});
もし渡していなければ、this.historyStore.undo() が発火しても undefined になります。

🔧【2】バケツツールがUIから反応しない
UI（Toolbar.jsなど）でバケツツールのボタンが "fill" などのIDで ToolStore.setTool("fill") を呼んでいない可能性

または、**ボタン押下時にイベントが止まっている（preventDefaultされている）**可能性もあります

G では動く → ToolStore の中では "fill" 自体は正しく動作しているということ
