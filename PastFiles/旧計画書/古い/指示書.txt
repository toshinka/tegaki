ブラウザお絵かきツール - ガイドライン（ラスター版 v3.0 WebGL2対応）

【実行環境・技術スタック】
・環境: Chrome最新、file:// 直開き可能、CSPはブックマークレット対応予定
・フレームワーク: PixiJS v8.14(CDN)、ES2023(モジュール不使用)
・GPU API: WebGL2 / fallback 不要
・禁止: Canvas2D, TypeScript, Vite, ESM, bundler, Babel
・file://プロトコルでは外部ファイル読み込みが不可能なのでGLSLコードをJavaScript内にインライン化する。
・描画方式: **ラスター（テクスチャ書き込み）方式**

【座標変換パイプライン - 重要】
・PointerEvent.clientX/Y
・→ screenClientToCanvas() [DPI補正]
・→ canvasToWorld() [worldContainer逆行列]
・→ worldToLocal() [手動逆算・親チェーン遡査]
・→ Local座標確定
・→ strokeRecorder記録

・PointerEvent は常に WebGL2 キャンバスの生の DOM イベントで受け取り、
  screenClientToCanvas()→canvasToWorld()→worldToLocal() を drawing-engine が単一で実行する。
・Pixi の toLocal()/toGlobal() は使用禁止。
・Pixi は UI 表示のために DOM イベント登録をしてもよいが、
  座標変換のオーナーシップは drawing-engine に一本化する。
・PixiJSの内部関数に依存せず、純粋な数学計算のみで座標変換を行う

【ファイル責務原則】
・coordinate-system.js : Screen/Canvas/World/Local 変換のみ
・camera-system.js : worldContainer transform, ズーム・パン
・drawing-engine.js : PointerEvent → Local座標変換、ストローク開始
・stroke-recorder.js : Local座標ポイント直接記録(座標変換なし)
・raster-brush-core.js : ラスターブラシの実装 - テクスチャへの直接描画
・raster-layer.js : WebGL2フレームバッファベースのレイヤー管理
・brush-stamp.js : ブラシスタンプ生成 - アンチエイリアス円形/テクスチャ
・webgl2-drawing-layer.js : 合成・出力統合（Pixi背景上への描画）

【命名ルール - 必須】
・パラメータ名に座標系を明示:
  localX, localY / worldX, worldY / screenX, screenY
・メソッド名で座標系を区別:
  screenClientToCanvas() / canvasToWorld() / worldToLocal()
・イベント名: component:action フォーマット統一
・WebGL2系は "gl" 接頭: glDevice, glProgram, glTexture など
・レンダーループは WebGL2 が唯一のマスターとする。
  Pixi の ticker を停止し、必要な場合は WebGL2 の renderLoop 内で
  pixiApp.renderer.render(stage) を手動で呼ぶ
  （順序：glRender() → pixiRender()）。

・pointer イベントの一次取得は WebGL2 キャンバスが担う（setPointerCapture を用いる）。
  Pixi のインタラクティブ管理機能は無効化し、
  UI が pointer を必要とする場合は EventBus を経由して同期すること。

【二重変換防止 - 最重要】
・Local座標変換は drawing-engine で完結。
・stroke-recorder では受け取った座標をそのまま保存（追加変換禁止）。
・同一ポイントに対する複数座標変換を明示的に禁止。
・Pixi 側座標変換は UI 取得用に限定。

【グローバルオブジェクト統一】
✓ window.CoordinateSystem
✓ window.cameraSystem
✓ window.layerManager
・WebGL2 関連は window.WebGLContext（gl, programs, shaders, format 等）に集約する。
・Pixi 用の global は window.PixiUI 等に分離する。グローバル重複を厳禁。

【worldToLocal() 実装 - 注意】
❌ PIXI v8 toLocal() は使用厳禁
理由: 親チェーン全体の worldTransform を使用し、
      worldContainer.position が含まれて負数が混ざるため。

✓ 手動逆算で実装
  親コンテナをスキャンし、
  各 position/scale/rotation を逆順に適用する。

■ ラスター描画アーキテクチャ【v3.0】

【基本方針】
・各レイヤーは WebGL2 フレームバッファ + テクスチャで管理
・ストロークはテクスチャに直接ブラシスタンプを描画
・筆圧・傾き・回転データは stroke-recorder に記録し、ブラシ描画時に参照

【描画フロー】
1. PointerEvent → Local座標 + 筆圧データ取得
2. stroke-recorder に記録
3. raster-brush-core が前回点との間を補間
4. 各補間点でブラシスタンプをテクスチャに描画（glDrawArrays）
5. 合成時に webgl2-drawing-layer が全レイヤーをブレンド

【ブラシスタンプ仕様】
・円形スタンプ: フラグメントシェーダーでアンチエイリアス円を生成
・テクスチャスタンプ: カスタムブラシ画像を使用可能
・筆圧対応: スタンプサイズ・不透明度を動的調整
・回転対応: ペンの twist/tilt データから回転行列を適用

【レイヤーテクスチャ管理】
・各レイヤー = 1 framebuffer + 1 RGBA texture
・描画時は該当レイヤーの framebuffer にバインド
・合成時は全テクスチャを順番に描画
・サムネイル生成: gl.readPixels() でテクスチャデータを取得

■ 実装品質基準
・多数ある競合ツールとの差別化のためモダン実装を心がけ、後方互換は気にしない。
・目標：120Hz/ひねり回転・傾き検出ラスターペンの実装。
・DRY/SOLID原則に基づく明確な責務分担を行い、Claudeからの可視性を高める。
・APIの不一致・断片化・EventBus統合の不完全性・設定参照の混在・二重実装などを排除する。
・命名規則・初期化順・イベント・管理構造の統一を行い保守性を高める。
・ラスター描画はすべて GPU（WebGL2 フレームバッファ）で実施【v3.0】
・PixiJSはUIホスト・レイヤー管理のみ。描画責務は一切持たせない。
・DPRは一倍で固定化。Retina対応による品質のブレを避ける。

■ 禁止事項【ラスター版 v3.0】

🚫 二重変換（pointer → view → world → geometry の座標変換を複数箇所で実施）
    ※ 座標変換は drawing-engine の単一責務に集約する。
🚫 PIXI v8 の toLocal() / toGlobal() を worldToLocal 実装に使用
    ※ Pixi の座標系計算は使用しない。描画座標は WebGL2 側で一元管理。
🚫 グローバルオブジェクトの重複定義
    ※ WebGLContext / StrokeRecorder / DrawingEngine などは単一インスタンスを厳守。
🚫 Canvas2D の使用（CPU ラスタライズが発生するため）
    ※ debug 可視化目的での一時使用のみ可（本番ロジックへの混入は禁止）。
🚫 ベクターメッシュ生成（Perfect-Freehand、ポリゴン生成等）【v3.0】
    ※ ラスターブラシで直接描画する方式に統一。
🚫 CPU側でのブラシ描画・ラスタライズ処理
    ※ 補間ロジックは CPU で実行してもよいが、実際の描画は必ず GPU で行う。
🚫 SDF/MSDF/JFA 関連処理【v3.0】
    ※ ラスター方式では Distance Field 計算は不要。
🚫 フォールバック処理・曖昧な暗黙修復動作
    ※ エラーは例外として明示的に扱い、黙って内部状態を書き換えない。
🚫 バージョン混在（PixiJS v7 / v8 の同時対応）
    ※ v8 固定。依存は最小限、UI レイヤー管理以外の責務を与えない。
🚫 二実装による肥大化、過度なエラー演出
    ※ 同じ機能を Pixi / WebGL / custom-class の 3 カ所に書かない。
🚫 循環依存
    ※ DrawingEngine → StrokeRecorder → CoordinateTransformer → DrawingEngine のループ禁止。
🚫 ペン/消しゴムの分離実装
    ※ strokeType: 'draw' | 'erase' を統合パスで扱い、別エンジンは持たせない。
🚫 Pixi.Graphics を本番描画に使用（CPUラスタライズのため）
    ★例外：座標デバッグ・パス可視化のための一時的使用は許可
    ※ 本番ロジック（ペン本体・消しゴム・水彩など）には混在させない。
🚫 Pixi が WebGL2 の gl コンテキストを直接操作すること
    ※ WebGL の全管理は WebGLContext クラスのみが行い、Pixi は UI レイヤーに限定する。
