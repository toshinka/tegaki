## バケツツール：囲い塗り

### 要望

線で囲った領域の中をバケツでクリックしたとき、内側だけが塗られるようにしたい。

例：

```text
○ を描く
内側でバケツを使う
● になる
```

### Phase 1c での扱い

実装しない。Phase 1c では「バケツツールが起動し、現在レイヤーへ基本塗りができ、サムネイルへ反映される」ことを復旧範囲とする。

### 後続実装時に決めること

- 参照対象を現在レイヤーのみにするか、表示中レイヤー合成にするか。
- 半透明線やアンチエイリアスの薄い縁を境界として扱うしきい値。
- 小さな隙間を閉じる gap close の有無と強さ。
- 塗りを同一レイヤーへ入れるか、線画の下の別レイヤーへ入れるか。
- 大きいキャンバスでの CPU flood fill の速度対策。
- クリック地点が透明領域・背景色領域・既存塗り領域の場合の挙動。

### 推奨タイミング

Phase 1d の基盤整理後、Phase 2 以降で「高機能バケツ」として設計する。
Phase 1d 中に扱う場合も、実装ではなく設計メモと検証方針までに留める。

## Vキー変形：キャンバス外クリップ

### 症状

Vキーのレイヤー編集モードで拡大後に縮小すると、一度キャンバス外へ出た描画が bake 時に失われる可能性がある。

### 推定

固定サイズの `RenderTexture` へ destructive bake しているため、キャンバス外のピクセルがクリップされる可能性が高い。

### 扱い

Phase 1c では緊急修正しない。Phase 1d 以降で、変形中の仮想領域・bake 範囲・履歴との関係を確認してから対応する。

## ログ整理

Phase 1c の調査で追加された診断ログが残っている。

Phase 1d では、以下へ分類する。

- 恒久ログ: 起動時の重要設定、重大エラー、復旧不能な状態。
- debugログ: `TEGAKI_CONFIG.debug` が true のときだけ出す診断。
- 削除ログ: ペン移動やサムネイル生成など高頻度で、確認完了後は不要なもの。

## アンドゥ・リサイズ後の描画座標ズレ

### 症状

時々、アンドゥで描画座標がズレる。アンドゥ単体で起きるのか、リサイズ後のアンドゥで起きるのかは未確定。

Vキーモードでアクティブレイヤーを少し移動すると、描画可能座標がキャンバスと一致して元に戻ることがある。

### 推定

履歴復元後に、RenderTexture、layerSprite、キャンバスサイズ、coordinate cache、layer-transform のいずれかが古い状態のまま残っている可能性がある。

リサイズ undo/redo が絡む場合、旧サイズへ戻すときの alignment offset と各レイヤーの RenderTexture 復元が特に疑わしい。

### 扱い

Phase 1f の最優先。保存・出力・レイヤー編集拡張へ進む前に、履歴・リサイズ・座標整合の土台として調査する。

## オーナーメモ整理：Phase 1d 後の優先順位

### 優先度A：次に触る価値が高い基礎体験

- 遅く描くと丸の集合に見える問題を、線分接続・補間・半透明時の重なり方の観点で改善する。
- 筆圧を実装する。最初はスライダー設定でよい。0荷重付近は 0.01 から 0.1 程度の細い点として始まり、リニアに立ち上がる挙動を目指す。
- 筆圧 ON/OFF を `Shift+P` で切り替えられるようにする。`Ctrl+P` は印刷と衝突するため使わない。単体 `0` は仮置き候補だったが正式初期案にはしない。
- ステータスバーの `ベクターペン` 表示は現行のライブラスター方針とズレるため、通常時は `ペン（固定幅）`、筆圧 ON 時は `ペン（筆圧ON）` を第一候補にする。
- 余裕があればサイドバーのペンアイコンに小さい筆圧状態バッジを追加する。アイコン全体の反転はアクティブツール表示と混ざるため避ける。
- ペン補正を設定値から反映する。最初はスライダー、将来的には 2D グラフ編集を検討する。
- 四角ペンを追加する。
- 設定ポップアップをタブ化し、増える項目を整理する。
- V キー変形パネルをスリム化し、操作部品以外を掴んで移動できるようにする。
- 左サイドバーと Q クイックパネルの視覚圧迫を下げる。透過、細分化、色プリセット、ペン/消しゴムサイズプリセットを検討する。

### 優先度B：制作を成立させる保存・出力・編集基盤

- エクスポート機能を復旧する。ラスター方針なので倍率指定は一旦削除してよい。
- 保存機能を復旧する。
- レイヤー複製、下レイヤーとの結合、クリッピングを実装する。
- レイヤーフォルダを実装する。
- レイヤーパネルをコンパクト化する。常時表示はサムネイル中心にし、詳細操作はクリックで出す案を検討する。
- 範囲選択、切り取りなどの編集操作を実装する。

### 優先度C：便利な描画補助

- 囲い塗り、隙間閉じ、表示レイヤー参照塗りを設計して実装する。
- エアブラシ、ぼかしツールを追加する。
- 直線、円、四角矩形ツールを追加する。直線は Shift 押下中の制約描画も検討する。
- 定規、自在定規を追加する。
- クイックパネルにスポイト、カラーサークルタブ、複数カラー置き場を入れる。
- 設定タブにショートカット表、変更、初期化ボタンを置く。

### 優先度D：後回しでよいアニメ・漫画向け機能

- GIF アニメパネル表示の復旧。
- Photoshop 的なフレーム管理から、ToonSquid 参考のトラック/キーフレーム型動画ツールへ再設計する。
- 物理演算で XY 移動、回転、拡縮、グラフによるイージング管理を行う。
- スクリーントーン、網点、グラデーション、縦線ストライプ。
- 集中線、ウニフラッシュ。

### 推奨フェーズ分け

- Phase 1e: ペン品質、筆圧、補正、四角ペン、設定タブ、Vキー/クイックパネルの軽量UI改善。
- Phase 1f: アンドゥ/リドゥ、リサイズ、Vキー変形後の描画座標整合を安定化する。実機確認で良好。
- Phase 1g: 保存、エクスポート、レイヤー複製/結合/クリッピング/フォルダ、範囲選択の基盤。
- Phase 2: 高機能バケツ、エアブラシ、ぼかし、図形、定規、クイックパネル拡張。
- Phase 3以降: アニメ、動画タイムライン、物理演算、漫画向け効果。
