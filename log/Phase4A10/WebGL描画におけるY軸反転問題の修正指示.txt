WebGL描画におけるY軸反転問題の修正指示
プロジェクト名: ブラウザお絵描きツール WebGL化プロジェクト
フェーズ: Phase 4A10後のY軸反転問題の修正
目的: WebGL描画におけるY軸の上下反転問題を完全に解消し、描画ロジックを統一する。特に、「Y軸の反転は、頂点シェーダーの一箇所でのみ行う」というルールを厳守する。

1. 現状の認識と問題点
現在、WebGLによるレイヤー描画において、Y軸の上下反転に関する不整合が発生していると推測されます。具体的には、以下の点が問題として特定されています。

ブラシストローク描画時のY軸二重反転:

webgl-engine.js 内のブラシ描画関連メソッド（_drawPoint など）で、JavaScript側でY軸の反転処理 (this.superHeight - superY) が行われている。

これにより、頂点シェーダーでのY軸反転と合わせて、二重にY軸が反転されている可能性が高い。

関連ファイル: webgl-engine.js

テクスチャアップロード時のY軸フリップ設定の不足:

webgl-engine.js で、Canvas2DのImageDataをWebGLテクスチャにアップロードする際に、gl.pixelStorei(gl.UNPACK_FLIP_Y_WEBGL, true); の設定が明示的に行われていない。

この設定がない場合、テクスチャデータはY軸下向きのままGPUに送られ、その後のシェーダーでのY軸反転処理と整合性が取れなくなる可能性がある。

関連ファイル: webgl-engine.js

Core Engineに残るY軸関連の古いプロパティ:

core-engine.js の Layer クラスや CanvasManager クラスに、過去のY軸反転処理に関連すると思われる flipY プロパティが残存している。

これらのプロパティが現在もY軸反転のロジックに影響を与えている場合、意図しない挙動やデバッグの困難さを招く可能性がある。

関連ファイル: core-engine.js

2. 修正ルール（再確認）
以下のルールを厳守して修正を行ってください。

Y軸反転は、頂点シェーダーの gl_Position 設定時の一箇所でのみ行う。

JavaScript側で行列にY軸反転スケール（mat4.scale(..., [1, -1, 1])）を適用することは避ける。

JavaScript側で直接Y座標を反転させるような計算（例: canvas.height - y）は、テクスチャの読み込み時（gl.UNPACK_FLIP_Y_WEBGL）を除き行わない。

ファイルの内容を丸ごと置き換えるのではなく、「元の構造や責務を維持したまま、新しいWebGL対応のコードに刷新する」 ことを意識する。

提供された「WebGL化バグ防止ガイド.txt」および「WebGLレイヤー移動対応 指示書 ver.5.0.txt」の記述を最優先する。

3. 具体的な修正指示
以下の手順でコードの修正を行ってください。

対象ファイル: webgl-engine.js
ブラシストローク描画時のY軸反転処理の削除:

WebGLEngine クラス内の _drawPoint メソッド（またはブラシ描画を担当する他のメソッド）を見つけ、const webglSuperY = this.superHeight - superY; という行を特定してください。

この行を削除し、代わりに const webglSuperY = superY; と記述してください。

理由: この処理はJavaScript側での二重反転の原因となるため削除します。ブラシストロークのY軸反転は、対応する頂点シェーダー（vsBrushなど）で統一して行われるべきです。

テクスチャアップロード時のY軸フリップ設定の追加:

WebGLEngine クラス内で、レイヤーの imageData をWebGLテクスチャとしてGPUにアップロードしている箇所（例: updateLayerTexture メソッド内や、描画ループ内でテクスチャを更新している箇所）を特定してください。

具体的には、gl.texImage2D または gl.texSubImage2D の呼び出し直前に、以下の行を追加してください。

JavaScript

gl.pixelStorei(gl.UNPACK_FLIP_Y_WEBGL, true);
理由: HTML CanvasのImageDataはY軸が下向きの座標系であるため、WebGLテクスチャ（Y軸上向き）に正しくアップロードするためにこの設定が必要です。これにより、頂点シェーダーでのY軸反転処理が期待通りに機能するようになります。

対象ファイル: core-engine.js
残存するY軸関連プロパティの評価と対応:

Layer クラス内の this.transform = { ..., flipY: 1 }; というプロパティ、および CanvasManager クラス内の this.viewTransform = { ..., flipY: 1, ... }; というプロパティを特定してください。

これらの flipY プロパティが、現在の modelMatrix と頂点シェーダーによるY軸反転のロジックと競合していないか、コード全体を慎重に確認してください。

方針:

もしこれらの flipY プロパティが、現在もJavaScript側でY軸反転の計算に利用されている箇所があれば、その計算ロジックを削除し、modelMatrix および頂点シェーダーでの反転に統一してください。 （「JavaScript側で行列にY軸反転スケールを適用することは避けてください」のルールを優先）。

もし、これらの flipY プロパティが全く使われていない残骸であると明確に判断できる場合は、コードのクリーンアップとして削除を検討してください。ただし、削除する場合は「ファイル量を20%以上削除する場合は、必ず理由を明文化し、事前に確認の上でコードを作成してください。」というルールに従い、なぜ不要であるかを明確にしてください。

注意: まずは webgl-engine.js の修正を優先し、この core-engine.js の flipY プロパティについては、現在のところ「問題を引き起こしているとは断定できないが、潜在的な懸念がある」として残しても構いません。ただし、その場合はコメント等で「Y軸反転の統一化を妨げないか、将来的に再評価が必要」と明記しておくと良いでしょう。今回の最優先は webgl-engine.js の具体的なバグ解消です。

動作確認:

上記の修正後、以下の動作を必ず確認してください。

レイヤーの移動・回転・拡大縮小が、dat.gui およびマウス操作（既存の移動ツール）で正しく行われること。

描画されたブラシストローク（円、線）が、上下反転せずに正しく表示されること。

新しいレイヤーや読み込まれた画像が、上下反転せずに正しく表示されること。
