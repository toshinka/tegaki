# 📄 PixiJS v8 Migration Design Notes for Futaba Drawing Tool (Supplement to Rulebook)

This document is an updated supplement to the **JavaScript + PixiJS v7 Rulebook** for the Futaba Drawing Tool project (Phase0).  
Its purpose is to ensure smooth migration from PixiJS v7 to v8, tailored to the current drawing application focusing on pen/eraser tools, history management, UI, and performance monitoring. It prepares for future additions like layers, while incorporating preparations for TypeScript migration. Based on official PixiJS v8 migration guide (as of 2025).

---

## 1. Core Principles for v8 Readiness

1. **Centralize PixiJS API Calls**  
   - All PixiJS rendering, graphics drawing, texture handling, and event setup should be confined to the `app-core.js` (PixiDrawingApp).  
   - Avoid direct `PIXI.*` calls in `drawing-tools/`, `ui/`, or other modules like `history-manager.js`.

2. **Abstract Rendering and Event Logic**  
   - Create wrapper methods in `app-core.js` for drawing operations (e.g., line drawing with Graphics), adding/removing objects, and handling pointer events.  
   - Example:
     ```js
     // In app-core.js
     drawLine(graphics, fromX, fromY, toX, toY) {
       graphics.moveTo(fromX, fromY).lineTo(toX, toY);
     }
     ```

3. **Separate Tool Logic from Rendering**  
   - `drawing-tools/core/` (e.g., `drawing-tools-system.js`, `tool-manager.js`) should handle abstract data (points, states) and pass to core wrappers. No direct Pixi dependencies.  
   - This eases v8 Graphics API changes and TypeScript typing.

4. **Define Interfaces for Key Components**  
   - Prepare for TypeScript by defining loose interfaces for tools, history snapshots, and UI events.  
   - Keep geometry (e.g., paths in PenTool) independent of Pixi Graphics for easy swapping to v8's new GraphicsContext.

5. **Performance and Debugging Abstraction**  
   - Wrap ticker, renderer options, and performance monitors (`performance-logger.js`) to handle v8 changes like ticker callbacks.

## 2. Known API Changes (v7 → v8)

Based on official PixiJS v8 migration guide. Focused on relevant areas for this drawing tool (Graphics for pen/eraser, Application, Events, Textures for history snapshots, Rendering).

| Area                  | v7 Approach                           | v8 Approach                                | Migration Tip |
|-----------------------|---------------------------------------|--------------------------------------------|--------------|
| Application Init      | `new PIXI.Application({ ... })`       | Async: `await app.init({ options })`      | Wrap init in async function in `main.js`. Update for WebGPU if needed. |
| Graphics API          | `graphics.beginFill().drawRect()` etc.| New syntax: `graphics.rect().fill()`      | Refactor all drawing in `pen-tool.js` and `eraser-tool.js` via wrappers. Test lineTo/moveTo. |
| Textures              | `PIXI.Texture.from(canvas)`           | Pre-load with `Assets.load`, use `Texture.from(source)` | Update `history-manager.js` snapshots; centralize in new `TextureManager`. |
| Events                | Pointer events with `interaction`     | Default `passive` mode; minor handler changes | Check `drawing-tools-system.js` for pointerdown/move/up. Use wrappers. |
| Rendering Settings    | `PIXI.settings.RENDER_OPTIONS`        | Removed; use `app.renderer.options`       | Move all configs from `config.js` to app init options. |
| Ticker Callback       | Callback gets deltaTime               | Gets `Ticker` instance; access `deltaTime`| Update any animation/ticker in `app-core.js` or performance monitors. |
| Filters (if added)    | `new PIXI.Filter()`                   | New shader structure with resources       | Prepare for future eraser/filters; abstract creation. |
| Meshes (future)       | `PIXI.Mesh`                           | Renamed classes like `MeshSimple`         | Not used yet, but prepare wrappers for layer deformations if planned. |

New Features Relevant:
- WebGPU support: Improves rendering performance for high-res drawing; enable in app init.
- Simplified Graphics: Better for vector paths in pen tool.
- TypeScript Improvements: Better typing for Application/Renderer; use for safer code.

## 5. Preparing for TypeScript Migration

To align with v8's improved TypeScript support:

1. **Setup tsconfig.json**  
   - Use strict mode: `"strict": true`, target ES2023.
   - Include PixiJS types: Install `@pixi/*` or use `pixi.js` types.

2. **Define Interfaces**  
   - For tools: `interface IBaseTool { activate(): void; onPointerDown(event: PointerEvent): void; }`
   - For core: `interface IDrawingApp { drawLine(graphics: PIXI.Graphics, ...): void; }`
   - Use Symbol Dictionary to name types consistently.

3. **Gradual Typing**  
   - Start with `main.ts`, add types to classes (e.g., extend PIXI.Application).
   - Avoid `any`; use unions for flexible params.
   - Test: Compile to JS, ensure no runtime changes initially.

4. **Integration with v8**  
   - Leverage v8's better renderer typing (e.g., `Application<WebGLRenderer>`).
   - Type wrappers: Make abstract methods typed for safety.

Migrate TypeScript first, then v8, as per previous analysis.

## 6. When to Migrate to v8

Recommended timing (aligned with project phases):
1. **After Phase0 stabilization** (current basic tools).  
2. **Before adding layers** (Phase1) to leverage v8 performance for multi-layer rendering.  
3. After TypeScript migration for type-safe updates.  
4. Use this document + Rulebook + Symbol Dictionary for AI-assisted code conversion.
