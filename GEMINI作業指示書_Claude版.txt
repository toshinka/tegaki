【修正1】extract.pixels() の引数を修正
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 原因
PixiJS v8 の extract.pixels() は RenderTexture を直接受け取れない。
Sprite でラップしてから渡す必要がある。
現在のコードはRenderTextureを直接渡しているため、空データが返っている。

■ 変更箇所
import 行に Sprite を追加する。

【変更前】
import { RenderTexture } from 'pixi.js';

【変更後】
import { RenderTexture, Sprite } from 'pixi.js';

■ 変更箇所2
generateLayerThumbnail() 内の pixels 取得部分。

【変更前】
const sourceRT = layer.layerData?.renderTexture;
if (!sourceRT) {
    console.warn('[ThumbnailSystem] No renderTexture on layer', layerId);
    return null;
}
pixels = this.app.renderer.extract.pixels(sourceRT);

【変更後】
const sourceRT = layer.layerData?.renderTexture;
if (!sourceRT) {
    console.warn('[ThumbnailSystem] No renderTexture on layer', layerId);
    return null;
}
// PixiJS v8 では extract.pixels() に RenderTexture を直接渡せない
// Sprite でラップして渡す
const tempSprite = new Sprite(sourceRT);
pixels = this.app.renderer.extract.pixels(tempSprite);
tempSprite.destroy();  // texture は破棄しない（sourceRT はレイヤーが所有）
