🧾 改修目的・背景・指示文（冒頭に貼るべき前提まとめ）
🧭 【目的】
本お絵かきアプリでは、複数のレイヤーに対し、回転・拡大縮小・移動・反転といった「見た目上の変形」を自由に行いたい。
これにより、花びらを複数回転させて複製・反転するなど、対称性を利用した描画を効率化することが目的である。

🧩 【現状の問題】
現在は CSS transform によってアクティブレイヤーの見た目の変形（回転・拡縮など）を行っているが、

Canvas API (ctx) 側の描画には影響せず、視覚と実際の描画位置がずれる

また、transformの状態は CanvasManager 全体に対してしか管理されておらず、レイヤーごとの個別変形ができない

🎯 【今回の改修で実現したいこと】
各レイヤーに独立した transform 情報（位置・回転・スケール・反転）を持たせる

アクティブなレイヤーを個別に変形できるようにする

ショートカットや UI 操作を通じてアクティブレイヤーにのみ transform を適用できるようにする

🔧 【そのために行うこと】
LayerManager クラスを改修し、以下の要素を追加・実装する：

各レイヤーに transform 情報を持たせる

js
コピーする
編集する
transform: {
    left: 0,
    top: 0,
    scale: 1,
    rotation: 0,
    flipX: 1,
    flipY: 1
}
アクティブレイヤーの移動・拡縮・回転・反転を行うメソッドを追加する

moveActiveLayer(dx, dy)

scaleActiveLayer(factor)

rotateActiveLayer(degrees)

flipActiveLayerHorizontal()

flipActiveLayerVertical()

その変形を DOM/CSS に適用する applyLayerTransform() を定義し、canvas にスタイルで反映する

📌 【改修の条件・制約】
今回の改修は既存機能を壊さず、安定動作している状態を保つことが最優先

他のファイル・モジュールに影響を与えるような整理・リファクタ・変数名変更は禁止

LayerManager クラス全体は App クラスの一部として new LayerManager(this) の形式で使用されることを前提とする

setupInitialLayers() の呼び出しは App 側で既に行われるため、内部からは呼ばないこと

✅ 【目的の最終到達点】
ペンで描いた花びらを、30度ずつ回転させて複製し、反転・結合してお花のような模様を作る

変形中の起点（キャンバス中央）も十字で可視化済み

今後は transform を ctx.setTransform() に統合する本格対応も視野に入れている
