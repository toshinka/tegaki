主な問題点の分析
1. ポイント描画の重複問題
現在のコードでは、onPointerDownで初期点を描画し、onPointerMoveでも線を描画しているため、描画が重複して意図しない結果になっている可能性があります。
2. 座標変換の不整合
getCanvasCoordinatesでの座標変換が複雑で、ペンの移動中に微妙な座標ずれが生じている可能性があります。
3. ポインターイベントの競合
activePointerIdの管理で、複数のポインターイベントが競合している可能性があります。
具体的な対策

対策1: 描画フローの整理

onPointerDown(e) {
    // 初期点では「点」を描かず、線の開始点として記録のみ
    const coords = this.getCanvasCoordinates(e);
    if (!coords) return;
    
    this.points = [{ x: coords.x, y: coords.y, pressure: e.pressure || 1.0 }];
    
    // 描画準備のみ行い、実際の描画はmoveで開始
    activeLayer.drawingCtx.beginPath();
    activeLayer.drawingCtx.moveTo(coords.x, coords.y);
}

onPointerMove(e) {
    if (!this.isDrawing) return;
    
    const coords = this.getCanvasCoordinates(e);
    if (!coords) return;
    
    // 前の点から現在の点への線を描画
    activeLayer.drawingCtx.lineTo(coords.x, coords.y);
    activeLayer.drawingCtx.stroke();
    
    // 点を記録
    this.points.push({ x: coords.x, y: coords.y, pressure: e.pressure || 1.0 });
}


対策2: 座標変換の簡素化


javascriptgetCanvasCoordinates(e) {
    const rect = this.displayCanvas.getBoundingClientRect();
    return {
        x: e.clientX - rect.left,
        y: e.clientY - rect.top
    };
}


対策3: イベント処理の安定化

onPointerDown(e) {
    if (this.activePointerId !== null) return; // 既に描画中なら無視
    
    this.activePointerId = e.pointerId;
    this.canvasArea.setPointerCapture(e.pointerId);
    
    // 描画開始処理...
}

onPointerUp(e) {
    if (this.activePointerId !== e.pointerId) return;
    
    // 描画終了処理...
    
    this.canvasArea.releasePointerCapture(e.pointerId);
    this.activePointerId = null;
}
